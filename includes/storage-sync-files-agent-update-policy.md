Azure 文件同步代理将定期更新，以便添加新功能和解决问题。 建议配置 Microsoft 更新，以便在 Azure 文件同步代理更新发布时获得这些更新。 我们明白某些组织希望严格控制和测试更新。

对于使用早期版 Azure 文件同步代理进行的部署：

- 存储同步服务将在新的主版本第一次发布后的 3 个月内允许使用以前的主版本。 例如，在发布版本 2.\* 后 3 个月内，存储同步服务支持版本 1.\*。
- 3 个月后，存储同步服务会阻止使用过期版本的已注册服务器与同步组进行同步。
- 在允许使用以前的主版本的这 3 个月内，所有 Bug 修复程序都仅针对当前的（新的）主版本。

> [!Note]  
> 如果你的 Azure 文件同步版本在三个月内过期，我们将通过 Azure 门户中的 toast 通知来通知你。
