| 每个批处理帐户的低优先级核心数（批处理服务模式）<sup>3</sup> | 50 | N/A<sup>4</sup> |

<sup>1</sup> 显示的专用核心配额仅针对池分配模式已设置为“批处理服务”的帐户。 对于其模式已设置为“用户订阅”的帐户来说，核心配额基于订阅中区域级别的或单个 VM 系列的 VM 核心配额。

<sup>2</sup> 可以增加每个批处理帐户的专用核心数，但不指定其最大数目。 请联系 Azure 支持以讨论增加选项。

<sup>3</sup> 显示的低优先级核心配额仅针对池分配模式已设置为“批处理服务”的帐户。 低优先级核心不可用于池分配模式设置为“用户订阅”的帐户。

<sup>4</sup> 可以增加每个批处理帐户的低优先级核心数，但不指定其最大数目。 请联系 Azure 支持以讨论增加选项。

<sup>5</sup> 已完成的作业和作业计划不受限制。

<sup>6</sup> 如果你请求增加的配额超出此限制，请与 Azure 支持部门联系。
