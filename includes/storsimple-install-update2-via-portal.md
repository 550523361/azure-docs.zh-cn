#### <a name="to-install-an-update-from-the-azure-portal"></a>从 Azure 门户安装更新

1. 在“StorSimple 服务”页上，选择你的设备。 导航到“设备” > “维护”。
2. 在页面底部，单击“扫描更新”。 已创建一个用于扫描可用更新的作业。 在作业成功完成时，系统会发出通知。
3. 在同一页的“软件更新”部分，提供新的软件更新。 我们建议先查看版本信息，然后在设备上应用更新。
4. 在页面底部，依次单击“安装更新”、“确定”。
5. 在“安装更新”对话框中，确保根据建议操作，选择“我已了解上面所述的要求并已准备好升级我的设备”，然后单击勾选按钮。
   
    ![确认消息](./media/storsimple-install-update2-via-portal/InstallUpdate12_2M.png)
6. 此时将开始执行一系列先决条件检查。 这些检查包括：
   
   * **控制器健康状况检查**：验证两个设备控制器是否状况良好且联机。
   * **硬件组件运行状况检查**：验证 StorSimple 设备上的所有硬件组件是否状况良好。
   * **DATA 0 检查**：验证设备上是否已启用 DATA 0。 如果未启用此接口，必须将它启用，然后重试。
   * **DATA 2 和 DATA 3 检查**：验证 DATA 2 和 DATA 3 网络接口是否未启用。 如果已启用这些接口，必须将其禁用，然后再尝试更新设备。 仅当要从运行 GA 软件的设备更新时，才需要执行此项检查。 运行 0.1、0.2 或 0.3 版本的设备不需要此项检查。
   * **网关检查**：在运行 Update 1 之前版本的任何设备上执行。 此项检查只能在运行 Update 1 之前版本软件的所有设备上运行。在为除 DATA 0 以外的网络接口配置了网关的设备上，此项检查将会失败。
     
     如果所有检查都已成功完成，则会应用更新。 在进行检查的同时，系统会通知你。
     
     ![预先检查通知](./media/storsimple-install-update2-via-portal/InstallUpdate12_3M.png)
     
     下面是检查失败的示例。 必须验证两个设备控制器是否状况良好且联机。 还需要检查硬件组件的运行状况。 在此示例中，控制器 0 和控制器 1 组件需要引以关注。 如果无法自行解决这些问题，可能需要联系 Microsoft 支持人员。
     
       ![检查失败](./media/storsimple-install-update2-via-portal/HCS_PreUpgradeChecksFailed-include.png)
7. 检查成功完成后，将创建更新作业。 成功创建更新作业后，系统会发出通知。
   
    ![创建更新作业](./media/storsimple-install-update2-via-portal/InstallUpdate12_44M.png)
   
    然后，在设备上应用更新。
    
8. 若要监视更新作业的进度，请单击“查看作业”。 在“作业”页上可以看到更新进度。
9. 更新需要数小时才能完成。 随时可以选择更新作业并单击“详细信息”来查看作业详细信息。
10. 作业完成后，请导航到“维护”页，然后向下滚动到“软件更新”。
