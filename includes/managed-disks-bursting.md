高级 SSD 支持磁盘爆发。 <= 512 GiB（P20 或以下）的任何高级 SSD 磁盘大小都支持爆破。 这些磁盘大小支持尽最大努力的爆破，并利用信用系统来管理爆破。 每当磁盘流量低于其磁盘大小的预配性能目标时，积分都会在突发存储桶中累积，当流量超出目标时，将消耗积分。 磁盘流量根据预配目标中的 IOPS 和带宽进行跟踪。 磁盘爆爆不会绕过虚拟机 （VM） 对 IOPS 或吞吐量的大小限制。

默认情况下，在支持它的磁盘大小的新部署上启用磁盘爆发。 现有磁盘大小（如果支持磁盘爆发）可通过以下任一方法启用突发：

- 分离并重新连接磁盘。
- 停止并启动 VM。

## <a name="burst-states"></a>突发状态

当磁盘连接到虚拟机时，所有突发适用的磁盘大小都将从完整的突发信用存储桶开始。 爆破的最大持续时间由突发信用存储桶的大小决定。 您只能累积未使用的积分，直到信用存储桶的大小。 在任何时候，磁盘突发信用存储桶可以处于以下三种状态之一： 

- 应计时，当磁盘流量使用小于预配性能目标时。 如果磁盘流量超出 IOPS 或带宽目标或两者，则可以累积积分。 当您使用全磁盘带宽时，您仍然可以累积 IO 积分，反之亦然。  

- 当磁盘流量使用超过预配性能目标时，拒绝。 突发流量将独立使用来自 IOPS 或带宽的配额。 

- 保持恒定，当磁盘流量正好达到预配的性能目标时。 

下表汇总了提供突发支持以及突发规范的磁盘大小。

## <a name="regional-availability"></a>区域可用性

磁盘爆发在公共云中的所有区域都可用。

## <a name="example-scenarios"></a>示例方案

为了让您更好地了解其工作原理，下面是一些示例方案：

- 从磁盘爆发中获益的一个常见方案是在 OS 磁盘上更快地启动 VM 和应用程序启动。 以带有 8 GiB 操作系统映像的 Linux VM 为例。 如果我们使用 P2 磁盘作为 OS 磁盘，则预配的目标是 120 IOPS 和 25 MiB。 当 VM 启动时，操作系统磁盘将加载启动文件。 引入爆破后，您可以读取最大突发速度为 3500 IOPS 和 170 MiB，将负载时间至少提高 6 倍。 VM 启动后，OS 磁盘上的流量级别通常较低，因为应用程序的大多数数据操作将针对附加的数据磁盘。 如果流量低于预配目标，您将累积积分。

- 如果要托管远程虚拟桌面环境，每当活动用户启动 AutoCAD 等应用程序时，操作系统磁盘的读取流量就会显著增加。 在这种情况下，突发流量将消耗累积的积分，使您能够超出预配目标，并更快地启动应用程序。

- P1 磁盘的预配目标为 120 IOPS 和 25 MiB。 如果磁盘上的实际流量在过去 1 秒间隔中为 100 IOPS 和 20 MiB，则未使用的 20 个 IO 和 5 MB 将记入磁盘的突发存储桶。 稍后，当流量超过预配目标时，可以使用突发存储桶中的配额，最高为最大突发限制。 最大突发限制定义磁盘流量的上限，即使您有突发积分需要使用。 在这种情况下，即使您在信用存储桶中有 10，000 个 IO，P1 磁盘的发货量也不能超过每秒 3，500 IO 的最大突发数。
