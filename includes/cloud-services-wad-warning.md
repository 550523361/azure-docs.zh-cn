> [!WARNING]
> 当启用了对现有角色的诊断时，在部署包时，将禁用已启用的任何扩展。 这些地方包括：
>
> * Microsoft 监视代理诊断
> * Microsoft Azure 安全监视
> * Microsoft 反恶意软件                 
> * Microsoft Monitoring Agent
> * Microsoft 服务探查器代理      
> * Windows Azure 的域扩展名        
> * Windows Azure 诊断扩展   
> * Windows Azure 的远程桌面扩展
> * Windows Azure 日志收集器
>
> 部署更新的角色之后，可以重新启用你的扩展通过 Azure 门户或 PowerShell。
>
