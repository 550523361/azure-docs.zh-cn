# <a name="what-is-video-indexer"></a>什么是视频索引器？

Azure 视频索引器是基于 Azure 媒体分析、Azure 搜索、认知服务（例如人脸 API、Microsoft Translator、计算机视觉 API 和自定义语音服务）生成的云应用程序。 借助视频索引器可以使用下面所述的视频索引器视频和音频模型从视频中提取见解：
  
## <a name="video-insights"></a>视频见解

- **人脸检测**：检测和分组视频中显示的人脸。
- **名人识别**：视频索引器会自动在全球范围内标识超过 1 百万次名人 – 等领域的全球领导者、 参与者、 actresses、 运动员、 研究人员、 业务和技术主管。 有关这些名人的数据也可以在各种著名网站上找到，例如 IMDB 和维基百科。
- **基于帐户的人脸识别**：视频索引器针对特定帐户训练模型。 然后可以识别视频基于训练的模型中的人脸。 有关详细信息，请参阅[来自定义人员模型从视频索引器网站](customize-person-model-with-website.md)和[自定义具有视频索引器 API 的人模型](customize-person-model-with-api.md)。
- **人脸缩略图提取**（“最佳人脸”）：在每组人脸中自动识别捕获的最佳人脸（基于质量、大小和正面位置），并将其提取为图像资产。
- **视觉文本识别** (OCR)：提取视频中显示的可视文本。
- **视觉内容审核**：检测成人和/或挑逗性视觉对象。
- **标签识别**：识别显示的视觉对象和动作。
- **场景分段**： 确定当场景更改在视频中基于视觉提示。场景描述了单个事件，并由一系列连续快照，在语义上是相关。 
- **快照检测**： 确定当截图更改在视频中基于视觉提示。截图是从相同的运动图片相机中提取的帧的一系列。 有关详细信息，请参阅[场景、 快照和关键帧](scenes-shots-keyframes.md)。
- **黑帧检测**：识别视频中的黑帧。
- **关键帧提取**：检测视频中稳定的关键帧。
- **滚动信用额度**： 标识的开头和结尾的电视节目和电影端中的滚动信用额度。

## <a name="audio-insights"></a>音频见解

- **自动语言检测**：自动识别主导性的讲述语言。 支持的语言包括英语、西班牙语、法语、德语、意大利语、中文（简体）、日语、俄语和葡萄牙语（巴西）。 检测不到语言时会回退到英语。
- **音频听录**：将语音转换为 12 种语言的文本并允许扩展。 支持的语言包括英语、西班牙语、法语、德语、意大利语、中文（简体）、日语、阿拉伯语、俄语、葡萄牙语（巴西）、印地语和韩语。
- **隐藏式字幕**：以三种格式创建隐藏式字幕：VTT、TTML、SRT。
- **双通道处理**：自动检测、隔离脚本及合并到单个时间线。
- **噪声消减**：清理电话音频或有噪音的录制内容（基于 Skype 滤波器）。
- **脚本自定义** (CRIS)：定型自定义语音转文本模型，创建特定于行业的学习记录。 有关详细信息，请参阅[自定义语言模型从视频索引器网站](customize-language-model-with-website.md)并[自定义具有视频索引器 Api 的语言模型](customize-language-model-with-api.md)。
- **说话人枚举**：映射和了解哪个说话人在何时说了哪些话。
- **说话人统计信息**：提供说话人发言比率的统计数据。
- **文本内容审核**：检测音频脚本中的显式文本。
- **音效**：识别击掌、讲话和静音等音效。
- **情感检测**：基于语音和音频提示识别情感。情感可能是：快乐、悲伤、愤怒或恐惧。
- **翻译**：将音频脚本翻译成 54 种不同的语言。

## <a name="audio-and-video-insights-multi-channels"></a>音频和视频见解（多通道）

通过一个通道编制索引时，这些模型的部分结果将可用

- **关键字提取**：从语音和视觉文本中提取关键字。
- **品牌提取**：从语音和视觉文本中提取品牌。
- **主题推理**：根据脚本推理主要主题。 包括第一级 IPTC 分类。
- **项目**：提取每个模型的丰富的“下一种详细程度”项目。
- **情绪分析**：在语音和视觉文本中识别积极、消极和中性情绪。
 
等到视频索引器处理和分析完毕，你就可以查看、策展、搜索和发布视频见解。

不管你的角色是内容管理员还是开发人员，视频索引器服务都可以满足你的需求。 内容管理员可以通过视频索引器 Web 门户使用此服务，无需编写任何代码，详见[视频索引器网站入门](video-indexer-get-started.md)。 开发人员可以利用 API 来大规模处理内容，详见[使用视频索引器 REST API](video-indexer-use-apis.md)。 客户还可以通过此服务使用小组件来发布视频流以及在其自己的应用程序中提取的见解，详见[在应用程序中嵌入视觉小组件](video-indexer-embed-widgets.md)。

可以使用现有的 AAD、LinkedIn、Facebook、Google 或 MSA 帐户注册此服务。 有关详细信息，请参阅[入门](video-indexer-get-started.md)。

## <a name="scenarios"></a>方案

下面是一些可以使用视频索引器的方案

- 搜索 - 从视频中提取的见解可以用来增强整个视频库的搜索体验。 例如，对所说内容和人脸进行索引可以启用在视频中查找特定瞬间的搜索体验，视频中的特定的人可能说出了特定的内容，或者两个人出现在一起。 根据视频中的此类见解进行的搜索适用于新闻机构、教育机构、广播公司、娱乐内容所有者、企业 LOB 应用。推而广之，它适用于任何行业，前提是该行业有一个需要用户对其进行搜索的视频库。
- 内容创建 – insights 从视频中提取和帮助有效地创建内容如尾部、 社交媒体内容、 新闻剪辑等从组织存档中的现有内容 
- 盈利 - 视频索引器有助于提高视频的价值。 例如，依赖广告收入的行业（如新闻媒体、社交媒体等）可以利用提取出来的见解作为广告服务器的附加信号（在足球比赛中展示运动鞋广告比在游泳比赛中展示更具针对性），从而提供更相关的广告。
- 用户参与度 - 可以通过视频见解来提高用户参与度，方法是为用户提供相关的视频瞬间。 以某个教育视频为例，该视频在头 30 分钟讲述球体，在接下来的 30 分钟讲述锥体。 如果某个学生想要了解锥体，则该视频从 30 分钟处开始播放更合适。
