# <a name="what-is-video-indexer"></a>什么是视频索引器？

视频索引器 （VI） 是 Azure 媒体服务 AI 解决方案，也是 Azure 认知服务品牌的一部分。 视频索引器提供基于多个渠道（语音、人声、视觉）的机器学习模型提取深入见解（无需数据分析或编码技能）的能力。 您可以进一步自定义和训练模型。 该服务支持深度搜索，降低运营成本，实现新的获利机会，并在大型视频存档上创建新的用户体验（进入障碍较低）。

要开始使用视频索引器提取见解，您需要创建帐户并上传视频。 当您将视频上传到视频索引器时，它会通过运行不同的 AI 模型来分析视觉对象和音频。 当视频索引器分析您的视频时，AI 模型提取的见解。

## <a name="compliance-privacy-and-security"></a>符合性、隐私和安全性

作为重要提醒，在使用视频索引器时必须遵守所有适用法律，并且不得以侵犯他人权利或可能有害他人的方式使用视频索引器或任何 Azure 服务。

在将任何视频/图像上载到视频索引器之前，您必须拥有使用视频/图像的所有适当权利，包括在法律要求的情况下，在视频/图像中使用、处理和存储其数据的所有必要同意（如果有）。 有些法域可能对收集、在线处理和存储某些类别的数据（如生物识别数据）规定了特殊的法律要求。 在使用视频索引器和 Azure 处理和存储受特殊法律要求约束的任何数据之前，必须确保符合可能适用于您的任何此类法律要求。

要了解视频索引器中的合规性、隐私和安全，请访问 Microsoft[信任中心](https://www.microsoft.com/TrustCenter/CloudServices/Azure/default.aspx)。 有关 Microsoft 的隐私义务、数据处理和保留做法，包括如何删除您的数据，请查看 Microsoft[的隐私声明](https://privacy.microsoft.com/PrivacyStatement)、[在线服务条款](https://www.microsoft.com/licensing/product-licensing/products?rtc=1)（"OST"） 和[数据处理附录](https://www.microsoftvolumelicensing.com/DocumentSearch.aspx?Mode=3&DocumentTypeId=67)（"DPA"）。 通过使用视频索引器，您同意受 OST、DPA 和隐私声明的约束。

## <a name="what-can-i-do-with-video-indexer"></a>我可以使用视频索引器进行哪些操作？

视频索引器的见解可应用于许多方案，其中包括：

* *深度搜索*：使用从视频中提取的见解来增强整个视频库的搜索体验。 例如，索引口语和面孔可以搜索视频中一个人说出某些单词或两个人一起被看见的瞬间的搜索体验。 基于视频此类见解的搜索适用于新闻机构、教育机构、广播公司、娱乐内容所有者、企业 LOB 应用，以及一般适用于具有用户需要搜索的视频库的任何行业。
* *内容创建*：根据视频索引器从内容中提取的见解创建预告片、突出显示卷轴、社交媒体内容或新闻剪辑。 关键帧、场景标记和用户和标签外观的时间戳使创建过程更加顺畅和容易，并允许您访问所创建内容所需的视频部分。
* *辅助功能*：无论您是希望使内容可供残障人士使用，还是希望内容使用不同语言分发到不同区域，您都可以使用视频索引器提供的多种语言的转录和翻译。
* *货币化*：视频索引器可以帮助增加视频的价值。 例如，依赖广告收入的行业（新闻媒体、社交媒体等）可以使用提取的见解作为广告服务器的附加信号来投放相关广告。
* *内容审核*：使用文本和可视内容审核模型来保护用户免受不当内容的遵守，并验证发布的内容是否与组织的价值观相符。 您可以自动阻止某些视频或提醒用户有关内容。
* *建议*：视频见解可用于通过向用户突出显示相关的视频时刻来提高用户参与度。 通过标记每个视频附加元数据，您可以向用户推荐最相关的视频，并突出显示视频中符合其需求的部分。

## <a name="features"></a>功能

以下列表显示了您可以使用视频索引器视频和音频模型从视频中检索的见解：

### <a name="video-insights"></a>视频见解

* **人脸检测**：检测和分组视频中显示的人脸。
* **名人识别**：视频索引器会自动识别超过 100 万名人，如全球的世界领导人、演员、女演员、运动员、研究人员、商业和技术领导者。 有关这些名人的数据也可以在各种网站（IMDB、维基百科等）中找到。
* **基于帐户的人脸识别**：视频索引器针对特定帐户训练模型。 然后，它根据训练的模型识别视频中的人脸。 有关详细信息，请参阅[从视频索引器网站自定义人员模型](customize-person-model-with-website.md)，以及[使用视频索引器 API 自定义人员模型](customize-person-model-with-api.md)。
* **人脸的缩略图提取**（"最佳面"）：自动识别每组人脸中捕获的最佳面数（基于质量、大小和正面位置），并将其提取为图像资产。
* **可视文本识别**（OCR）：提取视频中视觉显示的文本。
* **可视内容审核**：检测成人和/或猥亵的视觉内容。
* **标签识别**：识别显示的视觉对象和动作。
* **场景分割**：根据视觉提示确定场景在视频中何时更改。场景描述单个事件，由一系列连续镜头组成，这些镜头在语义上是相关的。
* **拍摄检测**：根据视觉提示确定拍摄的视频何时更改。拍摄是从同一张电影相机拍摄的一系列帧。 有关详细信息，请参阅[场景、镜头和关键帧](scenes-shots-keyframes.md)。
* **黑帧检测**：识别视频中的黑帧。
* **关键帧提取**：在视频中检测稳定的关键帧。
* **滚动学分**：在电视节目和电影的结尾标识滚动积分的开始和结束。
* **动画人物检测**（预览）：通过与[认知服务自定义视觉](https://azure.microsoft.com/services/cognitive-services/custom-vision-service/)的集成，检测、分组和识别动画内容中的人物。 有关详细信息，请参阅[动画角色检测](animated-characters-recognition.md)。
* **编辑拍摄类型检测**：根据自己的类型标记拍摄（如宽拍摄，中射，特写，极端特写，两枪，多人，室外和室内，等等）。 有关详细信息，请参阅[编辑拍摄类型检测](scenes-shots-keyframes.md#editorial-shot-type-detection)。

### <a name="audio-insights"></a>音频见解

* **自动语言检测**：自动识别主导性的讲述语言。 支持的语言包括英语、西班牙语、法语、德语、意大利语、中文（简体）、日语、俄语和巴西葡萄牙语。 如果无法自信地识别语言，视频索引器假定口语为英语。 有关详细信息，请参阅[语言标识模型](language-identification-model.md)。
* **多语言语音识别和转录**（预览）：自动识别音频不同段中的口语。 它将要转录的每个介质文件段，然后将转录合并回一个统一转录。 有关详细信息，请参阅[自动识别和转录多语言内容](multi-language-identification-transcription.md)。
* **音频转录**：以 12 种语言将语音转换为文本并允许扩展。 支持的语言包括英语、西班牙语、法语、德语、意大利语、中文（简体）、日语、阿拉伯语、俄语、巴西葡萄牙语、印地语和韩语。
* **隐藏式字幕**：以三种格式创建隐藏式字幕：VTT、TTML 和 SRT。
* **两个通道处理**：自动检测单独的脚本并合并到单个时间线。
* **降噪**：清除电话音频或嘈杂的录音（基于 Skype 滤波器）。
* **脚本自定义**（CRIS）：将自定义语音训练到文本模型以创建特定于行业的脚本。 有关详细信息，请参阅[从视频索引器网站自定义语言模型](customize-language-model-with-website.md)，以及[使用视频索引器 API 自定义语言模型](customize-language-model-with-api.md)。
* **演讲者列举**：地图，了解哪个发言者说什么词和什么时候。
* **发言人统计**：提供发言人演讲比例的统计数据。
* **文本内容审核**：在音频脚本中检测确切的文本。
* **音频效果**：识别音频效果，如手拍、语音和静音。
* **情绪检测**：根据言语（正在说的内容）和声音音调（如何说）来识别情绪。 情绪可以是喜悦、悲伤、愤怒或恐惧。
* **翻译**：将音频脚本翻译成 54 种不同的语言。

### <a name="audio-and-video-insights-multi-channels"></a>音频和视频见解（多通道）

当按一个通道编制索引时，这些模型的部分结果将可用。

* **关键字提取**：从语音和视觉文本中提取关键字。
* **命名实体提取**：通过自然语言处理 （NLP） 从语音和视觉文本中提取品牌、位置和人员。
* **主题推理**：从脚本中推理主要主题。 包括二级 IPTC 分类。
* **项目**：提取每个模型的丰富“下一种详细程度”项目。
* **情绪分析**：在语音和视觉文本中识别积极、消极和中性情绪。

## <a name="how-can-i-get-started-with-video-indexer"></a>如何开始使用视频索引器？

您可以通过三种方式访问视频索引器功能：

* 视频索引器门户：一个易于使用的解决方案，允许您评估产品、管理帐户和自定义模型。

    有关门户的详细信息，请参阅[使用视频索引器网站](video-indexer-get-started.md)入门。  

* API 集成：通过 REST API 提供视频索引器的所有功能，这使您可以将解决方案集成到应用和基础架构中。

    要开始作为开发人员，请参阅 [使用视频索引器 REST API](video-indexer-use-apis.md)。

* 可嵌入小部件：允许您将视频索引器见解、播放器和编辑器体验嵌入到应用中。

    有关详细信息，请参阅 [在应用程序中嵌入可视小部件](video-indexer-embed-widgets.md)。

如果您正在使用该网站，则见解将作为元数据添加，并在门户中可见。 如果您使用的是 API，则见解可作为 JSON 文件提供。
