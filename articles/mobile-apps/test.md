# <a name="perform-automated-testing-on-thousands-of-mobile-devices-simultaneously"></a>同时在成千上万个移动设备上执行自动测试
由于设备碎片、应用程序的复杂性、快速发布周期、会话和更高的用户期望，测试可能具有挑战性和复杂性。 你需要定期确保提供经过全面测试的应用程序，这些应用程序可提供卓越的客户体验。 测试不是一项简单的任务，开发人员无需手动测试每个版本。

可以有效地处理测试挑战。 你可以通过使用自动测试来测试各种设备上的移动应用程序。 您可以使用云服务加速移动应用程序的发布，并克服为开发和部署应用程序带来瓶颈的测试挑战。

## <a name="importance-of-automated-testing"></a>自动测试的重要性
- 通过确定不同的外形规格、连接类型和操作系统版本，提高测试范围。
- 通过在不同的设备和操作系统范围内进行测试来节省时间、资金和资源。
- 通过避免重复的手动测试来提高准确性，并有更多时间来创建新的自动化软件测试并处理复杂的功能。
- 连续测试，分析应用程序性能，查找并修复问题，然后用户将其放入其中并快速发布。
- 通过确保各种设备上的客户获得最佳体验，提供高质量的应用程序。

## <a name="visual-studio-app-center"></a>Visual Studio App Center
[App Center 测试](/appcenter/test-cloud/)是一项基于云的服务，可用于在云中跨 Android 和 iOS 跨云和混合移动应用运行自动 UI 测试。 对于超过2000台设备，您可以编写可自动执行的测试脚本，使测试变得轻松。

**主要功能**
   - 在实际设备上运行的、不是 root 或越狱的测试，以提供用户环境的最佳近似值。
   - 屏幕截图自动捕获并附加到每个测试步骤。 可以在测试运行中并行查看所有目标设备。
   - 测试运行中所有目标设备的设备日志以及失败的测试步骤的堆栈跟踪。
   - 并发测试同时在多个设备上运行，以加快产生结果的时间。
   - 适用于 Android 和 iOS 的受欢迎的测试框架支持，例如适用于 Android 和 iOS 的 Appium （Java，适用于 Espresso （Java），适用于 android 和 ios 的 Calabash，适用于 Android 和 iOS 的，以及适用于 iOS 的 XCUITest。
   - 适用于 iOS、Android Xamarin 和响应本机的平台支持。
