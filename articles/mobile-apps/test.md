# <a name="perform-automated-testing-on-thousands-of-mobile-devices-simultaneously"></a>同时在成千上万个移动设备上执行自动测试
由于各种原因（例如，设备碎片、应用程序复杂性、快速发布周期、会话和高用户期望），测试可能会很困难且越来越复杂。 你需要不断地确保提供经过全面测试的应用程序，以提供卓越的客户体验。 测试不是一项简单的任务，开发人员无需手动测试每个版本。

通过在各种设备（可通过**自动测试**来实现）上测试移动应用程序，可以有效地处理这些挑战。 你可以利用云服务加速移动应用程序的发布，并克服向开发和部署应用程序带来瓶颈的测试挑战。

## <a name="importance-of-automated-testing"></a>自动测试的重要性
- 通过确定不同的外形规格、连接类型和操作系统版本，**提高测试范围**。
- 通过测试不同的设备和操作系统范围来**节省时间、资金和资源**。
- 通过避免重复的手动测试来**提高准确性**，并有更多时间来创建新的自动化软件测试并处理复杂的功能。
- **连续测试**、分析**应用程序性能**、**查找并修复问题**，然后让用户进入这些问题并**快速发布**。
- 通过确保各种设备上的客户获得最佳体验，**提供高质量的应用程序**。

## <a name="visual-studio-app-center"></a>Visual Studio App Center
[App Center 测试](/appcenter/test-cloud/)是一项基于云的服务，可让你跨 Android 和 iOS 跨本机和混合移动应用在云中的数千个实际设备上运行自动 UI 测试。 对于超过2000台设备，您可以轻松编写可自动执行的测试脚本，使测试变得轻松。

**关键功能**
   - 在非根或已越狱的**真实设备上运行测试**，从而最大程度地提高用户环境的性能。
   - **屏幕截图**自动捕获并附加到每个测试步骤，并且可以在测试运行中并行查看所有目标设备。
   - 在测试运行中查看所有目标设备**的设备日志**，并为失败的测试步骤查看堆栈跟踪。
   - **并发测试**同时在多个设备上运行，以加快产生结果的时间。
   - 适用于 Android 和 iOS 的**受欢迎的测试框架支持**，例如适用于 Android 和 Ios 的 Appium （java，适用于 Espresso （java），适用于 android 和 Ios 的 Calabash，适用于 Android 和 Ios 的，以及适用于 Ios 的 XCUITest。
   - **平台支持**-IOS、Android Xamarin、响应本机。
