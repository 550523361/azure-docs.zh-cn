# <a name="perform-automated-testing-on-thousands-of-mobile-devices-simultaneously"></a>同时在数千台移动设备上执行自动测试
由于设备碎片、应用程序复杂性、快速发布周期、短会话和较高的用户期望，测试可能具有挑战性且越来越复杂。 您需要定期确保您提供经过全面测试的应用程序，从而提供卓越的客户体验。 测试不是一项简单的任务，而且使用开发人员时间手动测试每个版本也不是很好。

测试挑战可以有效应对。 您可以使用自动测试在各种设备上测试移动应用程序。 您可以使用云服务加快移动应用的发布，并克服为应用程序开发和部署添加瓶颈的测试挑战。

## <a name="importance-of-automated-testing"></a>自动测试的重要性
- 通过定位不同的外形规格、连接类型和操作系统版本，提高测试覆盖率。
- 通过在不同的设备和操作系统覆盖范围上进行测试，节省时间、资金和资源。
- 通过避免重复的手动测试来提高准确性，并有更多时间创建新的自动化软件测试并处理复杂的功能。
- 持续测试，分析应用程序性能，并在用户遇到问题并快速发布之前查找和修复问题。
- 通过确保客户在各种设备上获得最佳体验，从而提供高质量的应用程序。

## <a name="visual-studio-app-center"></a>Visual Studio App Center
[App Center 测试](/appcenter/test-cloud/)是一种基于云的服务，可用于在云中的数千个真实设备上运行针对本机和混合移动应用的自动 UI 测试。 使用 2，000 多个设备，您可以编写可自动执行的测试脚本，使测试变得简单。

**主要功能**
   - 在真实设备上运行的测试不是根植或越狱，以提供用户环境的最佳近似值。
   - 屏幕截图会自动捕获并附加到每个测试步骤。 可在试运行中对所有目标设备并排查看它们。
   - 测试运行中所有目标设备的设备日志和失败测试步骤的堆栈跟踪。
   - 并发测试同时在多个设备上运行，以加快结果的时间。
   - 流行的测试框架支持，如适用于Android和iOS的Appium（Java与JUnit），适用于Android的Espresso（Java），适用于Android和iOS的卡拉巴什，适用于Android和iOS的Xamarin.UITest，以及适用于iOS的XCUITest。
   - 对 iOS、Android Xamarin 和反应本机的平台支持。
