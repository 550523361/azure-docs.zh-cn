# <a name="what-is-devops"></a>什么是 DevOps？
"DevOps" 一词来自 "开发" 和 "操作" 的组合。 DevOps 强调软件开发人员与生产 IT 专业人员之间的沟通和协作的重要性，同时自动部署软件和基础结构更改。 DevOps 侧重于采用敏捷和精益方法，并自动执行重复任务。 DevOps 的目标是及时、成功的应用程序版本。

Microsoft 通过[Azure DevOps](https://azure.microsoft.com/services/devops/) （云）和[Visual Studio App Center](https://azure.microsoft.com/services/app-center/)提供了一套全面的[DevOps 解决方案](https://azure.microsoft.com/solutions/devops/)。 这些产品创建了一个工作环境，团队成员可在其中快速、频繁、可靠地生成、测试和部署软件。 由于部署新功能、安全修补程序和 bug 修补程序，组织可以更快地实现其目标。

**DevOps 的重要性**
- 从代码提交到生产部署，使整个软件生命周期更快。
- 允许组织在内部和向客户提供更快的软件和安全更新。
- 旨在减少开发和运营团队之间发生的冲突。 开发人员通常注重满足业务需求，以添加功能和提高应用程序的可用性。 通常，运营团队更侧重于计算环境的稳定性和安全性。
- 提供更大的灵活性、提高客户响应能力并缩短上市时间。
- 通过将人员、流程和产品组合在一起，实现向用户持续交付价值，从而扩展了灵活性。
- 当组织遵循 DevOps 实践时，会导致成本降低。 无论组织的任务如何，组织都可以更好地实现其目标。
