# <a name="what-is-devops"></a>什么是 DevOps？
DevOps （"开发" 和 "操作" 的组合）强调软件开发人员与生产 IT 专业人员之间的沟通和协作的重要性，同时自动部署软件和基础结构更改。 它侧重于采用敏捷和精益方法，并自动执行重复的任务，并具有一种及时、成功的应用程序版本。

Microsoft 通过[Azure DevOps](https://azure.microsoft.com/services/devops/) （云）和[Visual Studio App Center](https://azure.microsoft.com/services/app-center/)提供了一套全面的[DevOps 解决方案](https://azure.microsoft.com/solutions/devops/)。 这些产品创建了一个工作环境，在该环境中构建、测试和部署软件可以快速、频繁地进行。 它们还使组织能够更快地实现其目标，从而使部署新功能、安全修补程序和 bug 修复程序的时间更短。

**DevOps 的重要性**
- 从代码提交到生产部署，使整个软件生命周期更快。
- 允许组织在内部和向客户提供更快的软件和安全更新。
- 旨在减少当开发人员重点满足业务需求以添加功能并提高应用程序的可用性时出现的冲突，但操作团队通常更侧重于计算的稳定性和安全性情形.
- 提供更大的灵活性、提高客户响应能力并缩短上市时间。
- 通过将人员、流程和产品组合在一起，实现向最终用户持续交付价值，从而扩展了灵活性。
- DevOps 实践降低了成本，并且所有组织都可以更好地实现其目标，无论其任务是什么。
