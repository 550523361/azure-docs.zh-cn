# <a name="what-is-devops"></a>什么是 DevOps？
"DevOps"一词来自"开发"和"操作"的组合。 DevOps 强调了软件开发人员和生产 IT 专业人员之间通信和协作的重要性，同时自动部署软件和基础架构更改。 DevOps 专注于采用敏捷和精益方法以及自动化重复性任务。 DevOps 的目标是及时、成功的应用程序版本。

微软通过[Azure DevOps（](https://azure.microsoft.com/services/devops/)云）和[可视化工作室应用中心](https://azure.microsoft.com/services/app-center/)，拥有一套全面的[DevOps 解决方案](https://azure.microsoft.com/solutions/devops/)。 这些产品创造了一个工作环境，团队成员可以在其中快速、频繁、可靠地构建、测试和部署软件。 由于部署新功能、安全修补程序和 Bug 修复的周转时间更快，组织可以更快地实现其目标。

**DevOps 的重要性**
- 从代码提交到生产部署，使整个软件生命周期更快。
- 允许组织在内部和客户更快地交付软件和安全更新。
- 旨在减少开发和运营团队之间的冲突。 开发人员通常专注于满足业务需求，以添加功能并提高应用程序的可用性。 通常，运营团队更注重计算环境的稳定性和安全性。
- 提供更高的敏捷性，提高客户的响应能力，并缩短上市时间。
- 通过将人员、流程和产品汇集在一起，从而持续向用户交付价值，从而扩展敏捷性。
- 当组织遵循 DevOps 实践时，降低成本。 无论组织的使命是什么，它们都能更好地实现其目标。
