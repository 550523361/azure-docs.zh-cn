# <a name="monitor-failures-and-errors-in-real-time-for-your-mobile-application"></a>实时监控移动应用程序的故障和错误
应用程序被数千个用户使用后，它很可能出现 Bug。 您的用户可能会遇到意外的应用程序失败。 从用户的角度来看，他们只想是一个工作和可靠的应用程序。 错误和失败的应用程序不能提供良好的用户体验。 失望的用户将卸载应用程序，留下不好的评论，或公开批评和抱怨他们的经验。

了解实际故障行为对于移动应用程序至关重要。 发布到应用商店后，开发人员的工作不会立即完成。 持续监控和衡量性能以确保应用稳定性并保持用户满意度非常重要。 您需要的是一种监视应用程序的方法。 您需要了解出现问题时发生的情况，以便确定优先级并修复对业务需求至关重要的问题。

## <a name="importance-of-failure-monitoring"></a>故障监控的重要性
作为开发人员，您需要：
- 了解有关应用程序运行状况的详细信息，以及应用程序如何为用户执行。
- 查找用户设备上发生故障的根本原因和原因以及发生位置。
- 主动地对问题进行会审，解决故障，使应用程序稳定。
- 构建更好的应用程序并更快地迭代，以提供出色的用户体验，以便用户继续使用您的应用程序。

利用以下服务来监视应用程序故障、诊断问题并快速修复它们，从而提供稳定和高质量的移动应用程序体验。

## <a name="visual-studio-app-center"></a>Visual Studio App Center
[应用中心诊断](/appcenter/diagnostics/)服务可帮助您监视应用程序的运行状况。 它提供了详细的见解，以便您可以了解发生了什么以及应用程序失败的原因。 应用中心诊断还可帮助您确定故障报告的优先级和管理。

**主要功能**
- 实时、自动故障和错误，巧妙分组，以突出您需要的最严重问题和相关信息。
- 完整的堆栈跟踪和设备数据，提供上下文和清晰度，以帮助您确定根本原因。
- 强大的搜索功能，帮助您找到您最关心的问题。
- 分析集成，以便您可以了解用户的行为并查看导致故障的事件。
- 自定义属性、用户 ID 和附件，以便您可以添加更多上下文以帮助诊断问题。
- 警报、通知和错误跟踪器集成，以便您可以随时了解影响用户的新故障和意外故障。
- 单行式 SDK 集成，可在几分钟内开始使用此服务。
- 对 iOS、Android、macOS、tvOS、Xamarin、反应原生、统一、科尔多瓦、WPF 和 WinForms 的平台支持。
