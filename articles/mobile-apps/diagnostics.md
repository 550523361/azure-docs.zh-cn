# <a name="monitor-crashes-and-errors-in-real-time-for-your-mobile-application"></a>实时监视移动应用程序的故障和错误
一旦你的应用程序处于无数用户的使用中，就很可能会有 bug，用户将会遇到意外的应用程序崩溃。 从用户的角度来看，他们所需的一切都是有效且可靠的应用程序。 出现错误和故障的应用程序并不能提供良好的用户体验。 失望的用户将卸载该应用程序，留下错误的评论，或者公开 criticize 和抱怨其体验。

因此，了解实际的崩溃行为对于移动应用程序而言是至关重要的。 在发布到应用商店后，开发人员不会立即完成作业。 持续监视和衡量性能，以确保应用稳定性并维持用户满意度，这一点也同样重要。 你真正需要的是监视应用程序并了解发生了什么问题的方法，这样你就可以根据你的业务需求对你的时间进行优先级调整和修复问题。

## <a name="importance-of-crashes-monitoring"></a>崩溃监视的重要性
作为开发人员，您需要：
- 了解有关**应用程序运行状况的**详细信息以及应用程序**为最终用户执行**的操作。
- 查找用户设备上发生崩溃的**根本原因和原因**。
- **主动诊断问题并解决故障**，使你的应用程序稳定。
- 构建更好的应用程序并**更快地循环访问**，以提供极佳的用户体验，使用户能够返回到使用您的应用程序。

利用下面的服务监视应用程序的故障、诊断问题，并快速修复问题，以提供稳定且优质的移动应用程序体验。

## <a name="visual-studio-app-center"></a>Visual Studio App Center
[App Center 诊断](/appcenter/diagnostics/)服务可帮助你监视应用程序的运行状况，并提供详细的见解来了解应用程序崩溃的情况和原因，并允许你确定崩溃报告的优先级和管理故障报告。

**关键功能**
- 实时、自动**崩溃和错误**智能分组，以突出显示所需的最严重问题和相关信息。
- **完整的堆栈跟踪和设备**数据，提供上下文和清晰度，以帮助你确定根本原因。
- **强大的搜索功能**，可帮助你查找你最关注的问题。
- 分析集成，以便你可以了解用户的行为，并查看导致崩溃的事件。
- 自定义属性、用户 ID 和附件，以便可以添加其他上下文来帮助诊断问题。
- **警报、通知和 Bug 跟踪器集成**，因此你可以在影响用户的新崩溃和意外崩溃上停留。
- **单一的单一项集成了 SDK 集成**，可在几分钟内开始使用此服务。
- **平台支持**-IOS、Android、MacOS、TvOS、Xamarin、响应本机、Unity、CORDOVA、WPF、WinForms。
