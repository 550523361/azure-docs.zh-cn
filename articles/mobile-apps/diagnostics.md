# <a name="monitor-failures-and-errors-in-real-time-for-your-mobile-application"></a>实时监视移动应用程序的故障和错误
当应用程序被数千个用户使用后，就有可能存在 bug。 用户可能会遇到意外的应用程序故障。 从用户的角度来看，他们所需的一切都是有效且可靠的应用程序。 出现错误和失败的应用程序不能提供良好的用户体验。 失望的用户将卸载该应用程序，留下错误的评论，或者公开 criticize 和抱怨其体验。

了解实际的故障行为对于移动应用程序而言是至关重要的。 当你发布到应用商店后，你的作业已作为开发人员完成。 需要持续监视和度量性能，以确保应用稳定性并维持用户满意度，这一点很重要。 您需要的是一种监视应用程序的方法。 您需要了解发生问题时所发生的情况，以便您能够确定对您的业务需求至关重要的问题并解决这些问题。

## <a name="importance-of-failure-monitoring"></a>失败监视的重要性
作为开发人员，您需要：
- 了解有关应用程序运行状况的详细信息以及如何为用户执行此操作。
- 查找用户设备上出现故障的根本原因和原因。
- 主动诊断问题并解决失败，使应用程序保持稳定。
- 构建更好的应用程序并更快地循环访问，以便用户可以继续使用你的应用程序。

利用以下服务监视应用程序的故障、诊断问题并快速修复问题，以提供稳定且优质的移动应用程序体验。

## <a name="visual-studio-app-center"></a>Visual Studio App Center
[App Center 诊断](/appcenter/diagnostics/)服务可帮助你监视应用程序的运行状况。 它提供详细的见解，以便你可以了解发生了什么情况以及应用程序失败的原因。 App Center 诊断还可帮助你排定故障报告的优先级和管理故障报告。

**主要功能**
- 实时、自动失败和错误智能分组，可突出显示所需的最严重问题和相关信息。
- 完整的堆栈跟踪和设备数据，提供上下文和清晰度，以帮助你确定根本原因。
- 强大的搜索功能，可帮助你查找你最关注的问题。
- 分析集成，以便你可以了解用户的行为，并查看导致故障的事件。
- 自定义属性、用户 ID 和附件，以便您可以添加更多上下文来帮助诊断您的问题。
- 警报、通知和 bug 跟踪器集成，以便你可以在影响用户的新的和意外的故障上保持最新。
- 单项单行 SDK 集成，可在几分钟内开始使用此服务。
- 适用于 iOS、Android、macOS、tvOS、Xamarin、响应本机、Unity、Cordova、WPF 和 WinForms 的平台支持。
