# <a name="mobile-application-development-in-microsoft"></a>Microsoft 中的移动应用程序开发
决策者、开发团队和 IT 部门需要制定可靠且符合以下条件的移动应用程序开发策略：
- 确定与业务目标匹配的应用程序类型，如“企业对中大型企业”(B2E)、“企业对企业”(B2B) 或“企业对消费者”(B2C)。
- 确定是使用单平台语言投资并构建适用于 iOS、Android 或 Windows 的应用程序，还是采用跨平台方法。
- 提供可缩放的后端服务。
- 与后端数据无缝集成，无论是联机还是脱机。
- 构建带有身份验证和设备管理功能的安全应用程序。
- 持续快速地交付。
- 在云中或本地连接到后端系统。
- 与现有的工具和流程集成。
- 随着企业的发展改进应用程序。
- 缩短投放市场的时间，让开发人员专注于构建高质量的应用程序。

本文档系列探讨 Microsoft 提供的服务，这些服务可用于构建安全、可靠、可缩放且基于云的移动应用程序。

**目标受众**

本文档系列对于这样的开发人员非常有用：作为构建移动应用程序方面的新手或专家，希望了解适用于应用程序开发的云服务。 本系列概述了你需要了解的所有 Microsoft 服务、工具和技术。 此文档中未添加任何代码示例或演示。 它旨在让你大致了解要使用的应用程序开发过程和相关服务。 每篇文章都有其他资源的链接，为你提供更深入的信息。 该系列的每个部分均可独立阅读，因此你可以只选择自己感兴趣的文章。
