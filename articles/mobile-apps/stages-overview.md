# <a name="different-stages-in-mobile-application-development"></a>移动应用程序开发中的不同阶段
构建移动应用程序涉及多个阶段。 您可以轻松利用 Microsoft 服务、工具和技术来构建高品质的移动应用程序。

## <a name="app-development"></a>应用开发
可以用客观-C 和 Java 等语言构建本机单平台客户端应用程序。 可以通过使用 Xamarin、响应本机和 Unity 来生成跨平台应用。 你还可以使用 Cordova 构建混合应用。 若要构造客户端应用程序，请使用 Ide 和代码编辑器（如 Visual Studio、Visual Studio for Mac 或电脑和 Mac 上的 Visual Studio Code）。

## <a name="plan-and-design-with-back-end-services"></a>通过后端服务进行规划和设计
 Microsoft 提供了多种服务，用于为移动应用程序设置安全、可缩放且可靠的后端。 无需维护自己的服务器即可创建无服务器 Api 和触发器。 使用推送通知、脱机数据同步、身份验证和数据服务，形成一个全面且大规模的移动后端，以支持各种应用程序构造方案。 
 
 构建后端服务有许多要求。 你需要：
   - 限制设备上的数据存储。
   - 跨多个设备同步数据。
   - 请确保应用程序在没有网络连接的情况下运行。
   - 向用户发送通知。
   - 降低电池消耗。
   - 构建可伸缩、可靠且安全的后端。
   - 自动执行服务器维护。

## <a name="devops-and-continuous-monitoring"></a>DevOps 和持续监视
若要创建移动和以云为中心的应用程序，请利用持续集成（CI）、持续交付（CD）和持续监视服务，为所有平台生成本机生成。 你可以在数千台物理设备上运行 UI 自动化测试，通过 beta 测试通道自动执行发布，或直接部署到应用商店。 你还可以为其应用程序收集使用情况分析、失败和异常信息。

## <a name="additional-services"></a>其他服务
许多 Azure 服务都可以根据要生成的应用程序的类型和性质使用：
  - Azure 认知搜索搜索各种类型的内部数据，并进行快速查询。
  - Azure 认知服务，可在应用程序中使用人工智能。
  - Azure SignalR 在应用程序中启用实时通信。
