# <a name="different-stages-in-mobile-application-development"></a>移动应用程序开发的不同阶段
构建移动应用程序涉及多个阶段。 您可以轻松地利用 Microsoft 服务、工具和技术构建高级移动应用程序。

## <a name="app-development"></a>应用开发
您可以使用 Objective-C 和 Java 等语言构建本机单平台客户端应用程序。 您可以使用 Xamarin、反应本机和 Unity 构建跨平台应用。 您还可以使用科尔多瓦构建混合应用程序。 要构建客户端应用程序，请使用 ID 和代码编辑器，例如 Visual Studio、适用于 Mac 的可视化工作室或 PC 和 Mac 上的可视化工作室代码。

## <a name="plan-and-design-with-back-end-services"></a>使用后端服务进行规划和设计
 Microsoft 提供各种服务，用于为您的移动应用程序设置安全、可扩展和可靠的后端。 您可以创建无服务器 API 和触发器，而无需维护自己的服务器。 使用推送通知、脱机数据同步、身份验证和数据服务来形成一个全面而广泛的移动后端，支持各种应用构建方案。 
 
 构建后端服务有许多要求。 你需要：
   - 限制设备上的数据存储。
   - 跨多个设备同步数据。
   - 确保应用程序在没有网络连接的情况下运行。
   - 向用户发送通知。
   - 减少电池消耗。
   - 构建可扩展、可靠且安全的后端。
   - 自动执行服务器维护。

## <a name="devops-and-continuous-monitoring"></a>DevOps 和持续监控
要创建以移动和面向云为中心的应用程序，请利用持续集成 （CI）、持续交付 （CD） 和持续监视服务为所有平台生成本机生成。 您可以在数千个物理设备上运行 UI 自动化测试，通过 beta 测试渠道自动执行其发布，或者直接部署到应用商店。 您还可以为其应用程序收集使用情况分析、故障和异常信息。

## <a name="additional-services"></a>其他服务
根据正在构建的应用程序的类型和性质，可以使用许多 Azure 服务：
  - Azure 认知搜索以快速查询搜索所有类型的内部数据。
  - Azure 认知服务在应用程序中使用人工智能。
  - Azure SignalR 在应用程序中启用实时通信。
