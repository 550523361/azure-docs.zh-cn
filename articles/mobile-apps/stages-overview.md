# <a name="different-stages-in-mobile-application-development"></a>移动应用程序开发中的不同阶段
构建移动应用程序涉及多个阶段，开发人员可以轻松利用 Microsoft 服务、工具和技术来构建高品质的移动应用程序。

## <a name="app-development"></a>应用开发
开发人员可以使用 Ansi-c 等语言构建本机单平台客户端应用程序、使用 Xamarin 的跨平台应用、使用 Cordova 响应本机、Unity 和混合。 它们可以利用 Ide 和代码编辑器（如 Visual Studio、Visual Studio for Mac 或电脑和 Mac 上的 Visual Studio Code）来构造其客户端应用程序。

## <a name="plan-and-design-with-back-end-services"></a>通过后端服务进行规划和设计
 Microsoft 提供了多种服务，用于为移动应用程序设置安全、可缩放且可靠的后端。 开发人员无需维护自己的服务器即可创建无服务器 Api 和触发器。 他们可以利用推送通知、脱机数据同步、身份验证和数据等服务，它们共同构成了一个全面且广泛的移动后端即服务，可支持多种应用程序构建方案。 构建后端服务有很多要求：
   - 限制在设备上存储数据。
   - 跨多个设备同步数据。
   - 确保应用程序在没有网络连接的情况下运行。
   - 向用户发送通知。
   - 降低电池消耗。
   - 构建可缩放、可靠和安全的后端。
   - 服务器自动维护。

## <a name="devops-and-continuous-monitoring"></a>DevOps 和持续监视
创建移动和以云为中心的应用程序的开发人员可以利用持续集成（CI）、持续交付（CD）和持续监视服务，为所有平台生成本机生成物理设备，通过 beta 测试通道自动执行发布，或直接部署到 app store。 此外，他们还可以为其应用程序收集使用情况分析、崩溃和异常信息。

## <a name="additional-services"></a>其他服务
根据所生成应用程序的类型和性质，可以使用许多 Azure 服务。
  - Azure 搜索，用于搜索各种类型的内部数据，并进行快速查询。
  - Azure 认知服务，可在应用程序中使用人工智能。
  - Azure SignalR 在应用程序中启用实时通信。
