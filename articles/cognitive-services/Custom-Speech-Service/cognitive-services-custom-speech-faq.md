# <a name="custom-speech-service-frequently-asked-questions"></a>自定义语音服务常见问题解答

[!INCLUDE [Deprecation note](../../../includes/cognitive-services-custom-speech-deprecation-note.md)] 

如果未在本常见问题解答中找到答案，请尝试在 [StackOverflow](https://stackoverflow.com/questions/tagged/project-oxford+or+microsoft-cognitive) 和 [UserVoice](https://cognitive.uservoice.com/) 的自定义语音服务社区中提问

## <a name="general"></a>常规

**问**：如何判断数据集或模型处理已完成？

**答**：目前，只需知道表中的模型或数据集状态即可。
处理完成后，状态将为“就绪”。
我们正在研究改善通信处理状态的方法，例如电子邮件通知。

**问**：能否一次创建多个模型？

**答**：集合中可以有任意多个模型，但一次只能在一个页面上创建一个模型。
例如，如果当前正在处理一个语言模型，则无法启动另一语言模型创建过程。
但是，可以同时处理一个声学模型和一个语言模型。 

**问**：我意识到自己犯了一个错误。 如何取消正在进行的数据导入或模型创建？ 

**答**：当前无法回退声学适应或语言适应过程。
可在导入完成后删除导入的数据

**问**：搜索和听写模型与对话模型之间有何区别？

**答**：自定义语音服务中有两种基本声学和语言模型可供选择。
搜索查询或听写。 Microsoft 对话 AM 适用于识别以对话形式说出的语音。
此类语音通常是面对另一个人发出，例如呼叫中心或会议等情况。

**问**：能否更新现有模型（模型堆叠）？

**答**：不支持使用新数据更新现有模型。
如果你有新数据集且想要自定义现有模型，则必须使用新旧数据集重新适应。
新旧数据集必须合并为单个 .zip（如果是声音数据）或 .txt 文件（如果是语言数据），适应完成后，需要取消部署新更新的模型，获取新的终结点

**问**：如果所需并发高于默认值，该怎么办？ 

**答**：以 5 个并发请求（称为“缩放单元”）为增量纵向扩展模型。 每个缩放单元可确保模型可同时处理 5 个音频流。 最多可购买 100 个缩放单元（或 500 个并发请求）。

如需更大的规模，请与我们联系。

**问**：能否下载模型并在本地运行？

**答**：禁止下载模型并在本地执行。

**问**：是否记录了我的请求？

**答**：在部署创建期间，可选择切换禁用跟踪，此时不会记录音频或听录。 否则，通常会将请求记录在 Azure 中并安全存储。 如果还有禁止使用自定义语音服务的进一步隐私问题，请与我们联系。

## <a name="importing-data"></a>导入数据

**问**：数据集的大小限制是多少？ 为什么？ 

**答**：由于供 HTTP 上传的文件大小受限，因此数据集当前的大小限制为 2GB。 

**问**：能否压缩文本文件，以便上传更大的文本文件？ 

**答**：不能，目前只允许使用解压缩的文本文件。

**问**：数据报告表明，有言语导入失败。 这有问题吗？

**答**：如果只有少数言语无法成功导入，就不成问题。
如果成功导入了声学或语言数据集中的绝大多数话语（如 95% 以上的话语），则该数据集可用。 但是，建议尝试了解话语失败的原因并修复问题。
大多数常见问题（如格式设置错误）很容易修复。 

## <a name="creating-am"></a>创建 AM

**问**：需要多少声学数据？

**答**：建议从 30 分钟到 1 小时的声学数据入手

**问**：应收集什么类型的数据？

**答**：应收集尽可能接近于应用程序方案和用例的数据。
这是指数据收集应在设备、环境和说话人类型方面与目标应用程序和用户匹配。 一般而言，应从尽可能广泛的说话人中收集数据。 

**问**：应如何收集数据？ 

**答**：可以创建独立的数据收集应用程序，也可以使用现成的录音软件。
还可以创建记录音频数据并使用该数据的应用程序版本。 

**问**：是否需要自行转录适应数据？ 

**答**：必须转录数据。 可以自行转录或使用专业听录服务进行转录。 一些服务使用专业转录器，其他使用众包。 我们还可根据要求推荐听录服务。

**问**：创建自定义声学模型需要多长时间？

**答**：创建自定义声学模型的处理时间大约与声学数据集的长度相同。
因此，使用时长为五个小时的数据集创建的自定义声学模型大约需要五个小时才能处理完毕。 

## <a name="offline-testing"></a>离线测试

**问**：能否使用自定义语言模型执行自定义声学模型的脱机测试？

**答**：能，只需在设置脱机测试时，选择下拉菜单中的自定义语言模型即可

**问**：能否使用自定义声学模型执行自定义语言模型的脱机测试？

**答**：能，只需在设置脱机测试时，选择下拉菜单中的自定义声学模型即可。

**问**：什么是误字率？如何计算误字率？

**答**：误字率是语音识别的评估指标。 它由错误总数（包括插入、删除和替换）除以引用听录中的总字数得出。

**问**：我现在知道自定义模型的测试结果，如何确定结果数字是好是坏？

**答**：测试结果展示了对基线模型和自定义模型的比较。
应以超越基线模型为目标，使自定义模型变得有价值

**问**：为了了解是否有所改进，如何计算基础模型的 WER？ 

**答**：脱机测试结果展示了基线模型的准确度、自定义模型的准确度，以及与基线相比的改进情况

## <a name="creating-lm"></a>创建 LM

**问**：需要上传多少文本数据？

**答**：对于这个问题，很难给出准确的答案，因为这取决于应用程序中使用的词汇和短语与初始语言模型的差异有多大。 对于所有新字词，请尽可能多地提供这些字的使用示例，这一点非常有用。 对于应用程序中使用的常用短语，在语言数据中包括短语也非常有用，因为这也会通知系统侦听这些术语。
语言数据集中具有至少 100 句话语（通常几百句）或更多话语是很常见的。
另外，如果某些类型 * 的查询比其他类型更加常用，则可以在数据集中插入常用查询的多个副本。

**问**：能否只上传字词列表？

**答**：上传字词列表会将字词添加到词汇中，但不会告知系统通常如何使用这些字词。
通过提供完整或部分话语（用户很可能会说的句子或短语），语言模型可以学习这些新字词及其用法。 自定义语言模型不仅适用于获取系统中的新字词，而且还可用于调整应用程序的已知字词概率。 提供完整话语可帮助系统进行学习。
