# <a name="disclosure-design-guidelines"></a>披露设计准则
了解如何通过对语音体验的综合特性保持透明，与客户建立信任。

## <a name="what-is-disclosure"></a>什么是披露？

披露是让人们知道他们&#39;与合成产生的声音重新互动或倾听声音的一种手段。

## <a name="why-is-disclosure-necessary"></a>为什么需要披露？

需要披露计算机生成的语音的合成来源是相对较新的。 过去，计算机产生的声音显然——没有人会把他们误认为是真人。 然而，每天合成声音的现实主义都会提高，它们越来越不能与人类的声音区别开来。

## <a name="goals"></a>目标
在设计合成语音体验时，需要牢记以下原则：

**增强信任**
<br>设计的目的是在不降低经验的情况下使图灵测试失败。 让用户认识到他们正与合成语音交互，同时允许他们无缝地参与体验。

**适应使用环境**
<br>了解用户何时、何地以及如何与合成语音交互，以便在正确的时间提供正确的披露类型。

**设定明确的期望**
<br>允许用户轻松发现和理解代理的功能。 提供可应要求了解有关合成语音技术的更多信息的机会。

**拥抱失败**
<br>使用失败时刻加强代理的功能。

## <a name="how-to-use-this-guide"></a>如何使用本指南

本指南可帮助您确定哪些公开模式最适合您的合成语音体验。 然后，我们举例说明如何使用它们以及何时使用它们。 每种模式都旨在最大限度地提高用户对合成语音的透明度，同时保持以人为中心的设计。

考虑到关于语音体验的大量设计指导，我们在此特别关注：

1. [**披露评估**](#disclosure-assessment)：确定为您的合成语音体验推荐的披露类型的过程

2. [**如何披露**](concepts-disclosure-patterns.md)：可应用于合成语音体验的披露模式示例

3. [**何时披露**](concepts-disclosure-patterns.md#when-to-disclose)：在整个用户旅程中披露的最佳时刻

## <a name="disclosure-assessment"></a>披露评估
考虑您的用户&#39;对交互以及他们将体验语音的上下文的期望。 如果上下文清楚表明，综合的声音正在&quot;说话，&quot;披露可能是最小的，短暂的，甚至不必要的。 影响披露的主要上下文类型包括角色类型、方案类型和曝光级别。 它还有助于考虑谁可能正在倾听。

### <a name="understand-context"></a>了解上下文

使用此工作表确定合成语音体验的上下文。 您将在下一步中应用此应用，以确定您的披露级别。

|                                    | 使用上下文                                                                                                                                                                                                                                                                                                                                                       | 潜在风险&挑战                                                                                                                                                                                                                                                                                                                                                                       |
|------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **1. 人物类型**               | **如果适用以下任何一项，您的角色将属于"类似人角色"类别：**<br><br><ul><li> 角色体现了一个真实的人类，不管它是否是虚构的表现。 （例如，照片或计算机生成的真实人物渲染）<br><br><li> 综合声音基于一个广为人知的真人（例如名人、政治人物）的声音 | 您给角色的陈述越像人一样，用户就越有可能将其与真人关联，或让他们相信内容是由真实的人而不是计算机生成的。 </ul>                                                                                                                                                                      |
| **2. 方案类型**            | **如果适用以下任何条件，您的语音体验将适合"敏感"类别：**<br><br><ul><li> 从用户获取或显示个人信息 <br><br> <li> 广播时间敏感新闻/信息（例如紧急警报）<br><br><li> 旨在帮助真实的人相互交流（例如，阅读个人电子邮件/文本）<br><br> <li> 提供医疗/健康援助 </ul>            | 当主题与敏感、个人或紧急事项相关时，使用合成语音可能感觉不适合或使用合成语音的人感到不适当或值得信任。 他们也可能期望与真正的人类一样，得到同样的同情和背景意识。 |
| **3. 暴露程度** |**如果出现以下情况，您的语音体验最有可能属于"高"类别：** <br><br><ul><li>用户将经常听到或长时间听到合成语音或与合成语音交互 </ul>                                                                                                                                                                             | 建立长期关系时，保持透明和与用户建立信任的重要性更高。                                                                                                                                                                                                                                                                      |
