# <a name="what-is-multi-device-conversation-preview"></a>什么是多设备对话（预览）？

**多设备对话**便于在多个客户端之间创建语音或文本对话，并协调它们之间发送的消息。

使用**多设备对话**，您可以：

- 将多个客户端连接到同一会话中，并管理它们之间的消息的发送和接收。
- 通过可选翻译，轻松转录每个客户端的音频并将转录发送给其他人。
- 通过可选翻译，在客户端之间轻松发送文本消息。

可以构建可在一系列设备中工作的功能或解决方案。 每个设备可以独立地将消息（音频或即时消息的转录）发送到所有其他设备。

对话[**转录**](conversation-transcription.md)在具有多通道麦克风阵列的单个设备上工作，**而多设备对话**适用于具有多个设备（每个设备具有单个麦克风）的方案。

>[!IMPORTANT]
> 多设备**对话不支持在**客户端之间发送音频文件：仅转录和/或翻译。

## <a name="key-features"></a>主要功能

- **实时转录**– 每个人都会收到对话的脚本，以便他们可以实时跟踪文本或将其保存以供以后使用。
- **实时翻译**– 使用超过 60 种[支持的文本翻译语言](language-support.md#text-languages)，用户可以将对话翻译成他们的首选语言。
- **可读的抄本**– 转录和翻译易于理解，带有标点符号和句子中断。
- **语音或文本输入**– 每个用户可以在自己的设备上说话或键入内容，具体取决于为参与者所选语言启用的语言支持功能。 请参阅[语言支持](language-support.md#speech-to-text)。
- **消息中继**- 多设备会话服务将以他们选择的语言将一个客户端发送的消息分发给所有其他客户端。
- **消息标识**– 用户在对话中收到的每条消息都将用发送该消息的用户的昵称进行标记。

## <a name="use-cases"></a>用例

### <a name="lightweight-conversations"></a>轻量级对话

创建和加入对话非常简单。 一个用户将充当"主机"并创建一个对话，该对话生成随机的五个字母对话代码和 QR 码。 所有其他用户可以通过键入对话代码或扫描 QR 码加入对话。 

由于用户通过对话代码加入，并且不需要共享联系信息，因此可以轻松创建快速的现场对话。

### <a name="inclusive-meetings"></a>包容性会议

实时转录和翻译可以帮助说不同语言和/或失聪或听力障碍的人访问对话。 每个人还可以通过说出自己的首选语言或发送即时消息来积极参与对话。

### <a name="presentations"></a>演示文稿

您还可以为屏幕上和听众自己的设备上的演示和讲座提供字幕。 在访问群体加入对话代码后，他们可以在自己的设备上以首选语言查看成绩单。

> [!NOTE]
> 要查看示例，请查看[演示文稿转换器](https://www.microsoft.com/translator/apps/presentation-translator/)，一个使用多设备会话服务的 PowerPoint 外接程序。 可以在[此处](https://www.microsoft.com/download/details.aspx?id=55024)下载。

## <a name="how-it-works"></a>工作原理

所有客户端都将使用语音 SDK 创建或加入对话。 语音 SDK 与多设备会话服务进行交互，该服务管理会话的生存期，包括参与者列表、每个客户端选择的语言和发送的消息。  

每个客户端都可以发送音频或即时消息。 该服务将使用语音识别将音频转换为文本，并根据需要发送即时消息。 如果客户端选择不同的语言，则服务将将所有消息转换为每个客户端的指定语言。

## <a name="overview-of-conversation-host-and-participant"></a>对话、主持人和参与者概述

**会话**是一个用户为其他参与用户加入而启动的会话。 所有客户端都使用五个字母**的对话代码**连接到对话。

每个对话创建元数据，其中包括：
-    对话开始和结束时间戳
-    对话中所有参与者的列表，其中包括每个用户为语音或文本输入选择的昵称和主要语言。


对话中有两种类型的用户：**主机**和**参与者**。

**主机**是启动对话的用户，充当该对话的管理员。
- 每个对话只能有一个主机
- 在会话期间，主机必须连接到会话。 如果主持人离开对话，所有其他参与者的对话将结束。
- 主机有几个额外的控件来管理对话： 
    - 锁定对话 - 防止其他参与者加入
    - 使所有参与者静音 - 阻止其他参与者向对话发送任何消息，无论是从语音或即时消息中转录的
    - 将单个参与者静音
    - 取消隐藏所有参与者
    - 取消取消单个参与者的静音

**参与者**是加入对话的用户。
- 参与者可以随时离开并重新加入同一对话，而不会结束其他参与者的对话。
- 参与者无法锁定对话或静音/取消静音他人

> [!NOTE]
> 每次对话最多可以有 100 名参与者，其中 10 个可以在任意给定时间同时发言。

## <a name="language-support"></a>语言支持

创建或加入对话时，每个用户必须选择**主要语言**：他们将使用的语言和发送即时消息的语言，以及他们将看到其他用户的消息的语言。

有两种语言：**语音到文本**和**纯文本**：
- 如果用户选择**语音到文本**语言作为主要语言，那么他们将能够在对话中使用语音和文本输入。

- 如果用户选择**纯文本**语言，则他们只能使用文本输入并在对话中发送即时消息。 纯文本语言是文本翻译支持的语言，但不支持语音到文本的语言。 您可以在[语言支持](supported-languages.md)页面上查看可用语言。

除了主要语言之外，每个参与者还可以指定用于翻译会话的其他语言。

以下是用户根据所选主要语言在多设备对话中能够执行的操作的摘要。


| 用户可以在对话中执行哪些操作 | 语音转文本 | 纯文本 |
|-----------------------------------|----------------|------|
| 使用语音输入 | ✔️ | ❌ |
| 发送即时消息 | ✔️ | ✔️ |
| 翻译对话 | ✔️ | ✔️ |

> [!NOTE]
> 有关可用语音到文本和文本翻译语言的列表，请参阅[支持的语言](supported-languages.md)。
