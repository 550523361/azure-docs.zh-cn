# <a name="custom-neural-voice-gating-overview"></a>自定义神经语音门控概述

详细了解开始使用自定义神经语音的过程。

## <a name="commitment-to-responsible-innovation"></a>致力于负责任的创新

作为 Microsoft 设计负责任 AI 承诺的一部分，我们收集了一组材料，以指导客户使用自定义神经语音。 此处找到的指南和见解基于 Microsoft 在[AI 中负责任创新的原则。](https://www.microsoft.com/AI/our-approach-to-ai)

### <a name="guidance-for-deploying-custom-neural-voice"></a>部署自定义神经语音的指南

- [负责任部署指南](concepts-guidelines-responsible-deployment-synthetic.md)：我们基于我们研究的顶级建议
- [语音人才的披露](https://aka.ms/disclosure-voice-talent)：您需要了解什么，并告知语音人才如何负责任地使用它
- [披露设计](concepts-disclosure-guidelines.md)：如何设计体验，以便用户知道何时使用合成语音并信任您的服务

### <a name="why-custom-neural-voice-is-a-gated-technology"></a>为什么自定义神经语音是门控技术

我们的目的是保护个人和社会的权利，促进透明的人机交互，并抵制有害深层虚假和误导性内容的扩散。 因此，我们已经对自定义神经语音的使用进行了封闭。 客户只有在对其应用程序进行审查并承诺根据我们的道德原则使用该技术后，才能访问该技术。

### <a name="our-gating-process"></a>我们的浇注过程

要访问自定义神经语音，您需要首先填写我们的在线入学表单。 [在此处](https://aka.ms/custom-neural-intake-form)开始您的应用程序。

根据我们的资格标准、审核流程以及在此门控预览期间支持有限数量的客户的可用性，访问自定义神经语音服务需由 Microsoft 自行决定。

作为申请过程的一部分，您需要承诺在创建语音字体之前从语音人才处获得明确的书面许可，其中包括共享[语音人才的披露](https://aka.ms/disclosure-voice-talent)。 您还必须同意，在部署语音字体时，您的实现将向用户[披露服务的综合性质](concepts-disclosure-guidelines.md)，在服务条款中提供 Microsoft 综合语音服务的归因，并支持一个反馈渠道，允许服务用户报告问题并与 Microsoft 共享详细信息。 在此处了解有关我们的使用条款[的更多详细信息](https://aka.ms/custom-neural-code-of-conduct)。
