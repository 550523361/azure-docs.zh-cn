# <a name="select-a-domain-for-a-custom-vision-project"></a>为自定义愿景项目选择域

从自定义视觉项目的设置边栏选项卡中，可以为项目选择域。 选择最接近您的方案的域。

## <a name="image-classification"></a>图像分类

|域|目的|
|---|---|
|__泛型__| 针对各种图像分类任务进行优化。 如果其他域都不合适，或者不确定要选择哪个域，请选择“通用”域。|
|__食品__|针对餐厅菜肴的照片进行优化。 如果要对各种水果或蔬菜的照片进行分类，请使用“食品”域。|
|__特征点__|针对可识别的自然和人造地标进行优化。 在照片中的地标清晰可见的情况下，该域效果最佳。 即使照片中的人物稍微遮挡了地标，该域仍然有效。|
|__零售__|针对购物目录或购物网站中的图像进行优化。 若想对连衣裙、裤子和衬衫进行精准分类，请使用此域。|
|__压缩域__| 针对边缘设备上的实时分类约束进行了优化。|

## <a name="object-detection"></a>对象检测

|域|目的|
|---|---|
|__常规__| 已针对各种对象检测任务进行优化。 如果其他域都不合适，或者不确定要选择哪个域，请选择“通用”域。|
|__标志__|已针对在图像中查找品牌徽标进行优化。|
|__货架上的产品__|针对检测和分类货架上的产品进行了优化。|
|__压缩域__| 针对边缘设备上的实时对象检测约束进行了优化。|

## <a name="compact-domains"></a>压缩域

可导出压缩域生成的模型在本地运行。 模型性能因所选域而异。 在下表中，我们报告英特尔台式机 CPU 和 NVidia GPU \[1\]上的型号大小和推理时间。 

> [!NOTE]
> 这些数字不包括预处理和后处理时间。

|任务|域|模型大小|CPU 推理时间|GPU 推理时间|
|---|---|---|---|---|
|分类|常规(精简)|5 MB|13 毫秒|5 ms|
|对象检测|常规(精简)|45 MB|35 毫秒|5 ms|
|对象检测|常规（压缩） [S1]|14 MB|27 毫秒|7 毫秒|

## <a name="vaidk-vision-ai-dev-kit"></a>VAIDK（视觉 AI 开发套件）

选择紧凑域时，会提供一个额外的选项"导出功能"，以便区分"基本平台"和"视觉 AI 开发套件"。

在 _"导出功能"_ 下，两个选项是：

- 基本平台（天流、核心ML、ONNX等）
- 愿景 AI 开发套件。

当_选择视觉 AI 开发套件_时，_通用_、_地标_和_零售_域，但_食品_紧凑域不可用于图像分类，而_通用（压缩）_ 和_常规（压缩）[S1]_ 可用于对象检测。

>[!NOTE]
>对象检测__的一般（紧凑）__ 域需要特殊的后处理逻辑。 有关详细信息，请参阅导出的 zip 包中的示例脚本。 如果需要没有后处理逻辑的模型，请使用__常规（压缩）[S1]__。

>[!IMPORTANT]
>不能保证导出的模型给出的结果与云上的预测 API 完全相同。 运行平台或预处理实现的细微差异可能会导致模型输出中更大的差异。 有关预处理逻辑的详细信息，请参阅[本文档](python-tutorial.md)。

\[1\]英特尔至强 E5-2690 CPU 和 NVIDIA 特斯拉 M60
