# <a name="machine-learning-features"></a>机器学习功能

在机器学习中，一 *项功能*   是您的系统观察和学习的数据的特征或属性。

计算机学习功能为 LUIS 提供了有关在何处查找区分概念的事项的重要提示。 它们是 LUIS 可以使用的提示，但并不是硬规则。 LUIS 将这些提示与标签结合使用来查找数据。

可以将功能描述为类似于 f （x） = y 的函数。 在示例查询文本中，该功能告诉您查找区分特征的位置。 使用此信息来帮助创建你的架构。

## <a name="types-of-features"></a>功能的类型

LUIS 支持词组列表和模型作为功能：

* 短语列表功能 
* 模型（意向或实体）作为功能

应将功能视为架构设计的必需部分。

## <a name="find-features-in-your-example-utterances"></a>查找示例中的功能最谈话

由于 LUIS 是基于语言的应用程序，因此这些功能是基于文本的。 选择指示要区分的特征的文本。 对于 LUIS，最小单位是*标记*。 对于英语，标记是不包含空格或标点的字母和数字的连续范围。

由于空格和标点不是标记，因此请专注于可用作功能的文本线索。 请记住包括单词的变体，例如：

* 复数形式
* verb 时态
* 缩写
* 拼写和拼写错误

确定文本是否区分特征，因为它具有以下特点：

* 匹配精确的词或短语：考虑将正则表达式实体或列表实体作为功能添加到实体或意向。
* 匹配众所周知的概念（如日期、时间或人员的姓名）：使用预生成的实体作为实体或意向的功能。
* 在一段时间内学习新示例：使用概念的短语列表作为实体或意向的功能。

## <a name="combine-features"></a>组合功能

您可以使用多个功能来描述特征或概念。 常见的配对是使用短语列表功能和通常用作功能的实体类型：

 * 预生成实体
 * 正则表达式实体
 * 列表实体

### <a name="ticket-booking-entity-example"></a>票据预订实体示例

作为第一个示例，请考虑使用航班预订意向和票据预订实体来预订航班的应用。

票据预订实体是航班目标的机器学习实体。 若要帮助提取位置，请使用以下两个功能来帮助：

* 相关词的短语列表，如，**平面**，**航班**，预订，**预订**或**票据**
* 预生成的**geographyV2**实体作为实体的功能

### <a name="pizza-entity-example"></a>比萨饼实体示例

作为另一个示例，请考虑使用一个应用来对具有 "创建比萨饼" 和 "比萨饼" 实体的比萨饼进行排序。

比萨饼实体是用于比萨饼详细信息的机器学习实体。 若要帮助提取详细信息，请使用两个功能来帮助：

* 相关词的短语列表，例如，**奶酪**、 **crust**、**意大利辣肠**或**pineapple**
* 预生成的**数字**实体作为实体的功能

## <a name="create-a-phrase-list-for-a-concept"></a>为概念创建短语列表

短语列表是描述概念的单词或短语的列表。 短语列表在标记级别应用为不区分大小写的匹配项。

添加短语列表时，可以将功能设置为 "**[全局](#global-features)**"。 全局功能适用于整个应用。

### <a name="when-to-use-a-phrase-list"></a>何时使用短语列表

需要 LUIS 应用来通用化和确定概念的新项目时，请使用短语列表。 短语列表类似于域特定词汇。 它们增强了意向和实体的理解质量。

### <a name="how-to-use-a-phrase-list"></a>如何使用短语列表

使用短语列表，LUIS 将上下文和通用化视为类似的项，但不是精确的文本匹配项。 请按照以下步骤使用短语列表：

1. 从机器学习实体开始：
    1. 添加示例最谈话。
    1. 使用机器学习实体进行标记。
1. 添加短语列表：
    1. 添加具有相似含义的单词。 不要添加每个可能的单词或短语。 只需一次添加几个字词或短语。 然后重新训练并发布。
    1. 查看并添加建议的单词。

### <a name="a-typical-scenario-for-a-phrase-list"></a>短语列表的典型方案

短语列表的典型方案是增加与特定想法相关的单词。

医疗术语是可能需要词组列表才能提高其重要性的词的很好示例。 这些术语可以具有特定的物理、化学、therapeutic 或抽象含义。 LUIS 不知道这些条款对你的 subject 域没有短语列表很重要。

若要提取医学术语：

1. 创建示例最谈话并在这些最谈话中标记医学术语。
2. 使用 subject 域中的条款的示例创建短语列表。 此短语列表应包含标记的实际字词以及描述相同概念的其他术语。
3. 向实体或子实体添加短语列表，用于提取短语列表中使用的概念。 最常见的方案是机器学习实体的组件（子级）。 如果短语列表应应用于所有方法或实体，请将短语列表标记为全局短语列表。 **EnabledForAllModels**标志控制 API 中的此模型作用域。

## <a name="a-model-as-a-feature-helps-another-model"></a>作为功能的模型可帮助另一模型

您可以将模型（意向或实体）作为功能添加到另一个模型（意向或实体）。 通过将现有意向或实体添加为功能，您将添加一个具有标记示例的明确定义的概念。

添加模型作为功能时，可以将该功能设置为：
* **[必需](#required-features)**。 若要从预测终结点返回模型，必须找到必需的功能。
* **[全局](#global-features)**。 全局功能适用于整个应用。

### <a name="when-to-use-an-entity-as-a-feature-to-an-intent"></a>何时使用实体作为意向的功能

当该实体的检测对于意向非常重要时，将实体作为功能添加到意向。

例如，如果目的是预订航班（如**BookFlight**），并且实体是票据信息（如座位数量、来源和目标），则查找票据信息实体应为**BookFlight**意图的预测添加巨大的权重。

### <a name="when-to-use-an-entity-as-a-feature-to-another-entity"></a>何时使用实体作为另一个实体的功能

当实体（A）的检测对于实体（B）的预测很重要时，应将实体（A）作为功能添加到另一个实体（B）。

例如，如果邮寄地址实体包含在街道地址子实体中，则查找街道地址子实体会将重要权重添加到 "发货地址" 实体的预测。

* 寄送地址（机器学习实体）：

    * 街道编号（子实体）
    * 街道地址（子实体）
    * City （子实体）
    * 省/直辖市/自治区（子实体）
    * 国家/地区（子实体）
    * 邮政编码（子实体）

## <a name="nested-subentities-with-features"></a>带功能的嵌套子实体

计算机学习子实体表示存在对父实体的概念。 父代可以是另一个子实体或顶级实体。 子实体的值充当其父项的功能。

子实体可以同时包含短语列表和模型（另一实体）作为功能。

当子实体包含短语列表时，它会提升概念的词汇，但不会将任何信息添加到预测的 JSON 响应。

当子实体具有另一实体的功能时，JSON 响应将包含该其他实体的提取数据。


## <a name="required-features"></a>所需功能

若要从预测终结点返回模型，必须找到必需的功能。 如果你知道传入数据必须与功能匹配，请使用所需功能。

如果查询文本文本不匹配所需的功能，则不会提取该文本。

必需的功能使用非计算机学习实体：

* 正则表达式实体
* 列表实体
* 预生成实体

如果确信将在数据中找到模型，请根据需要设置该功能。 如果未找到所需的功能，则不会返回任何内容。

继续执行发货地址的示例：

寄送地址（机器学习的实体）

 * 街道编号（子实体） 
 * 街道地址（子实体） 
 * 街道名称（子实体） 
 * City （子实体） 
 * 省/直辖市/自治区（子实体） 
 * 国家/地区（子实体） 
 * 邮政编码（子实体）

### <a name="required-feature-using-prebuilt-entities"></a>使用预先构建的实体所需的功能

城市、省/市/自治区和国家/地区通常是一组封闭列表，这意味着它们不会随着时间的推移而变化。 这些实体可能具有相关的推荐功能，可以将这些功能标记为必需。 这意味着，如果找不到具有所需功能的实体，则不会返回整个发货地址。

如果城市、省/市/自治区或国家/地区位于查询文本中，但它们处于某个位置或俚语 LUIS 不会出现什么情况？ 如果要提供一些 post 处理来帮助解析实体，因为 LUIS 中的置信度较低，请不要将该功能标记为 "必需"。

寄送地址所需功能的另一个示例是，使街道号成为必需的[预](luis-reference-prebuilt-entities.md)生成编号。 这允许用户输入 "1 Microsoft 方式" 或 "一种 Microsoft 方式"。 对于街道号子实体，两者都解析为数字 "1"。

### <a name="required-feature-using-list-entities"></a>使用列表实体所需的功能

[列表实体](reference-entity-list.md)用作规范名称及其同义词的列表。 作为必需功能，如果查询文本不包括规范名称或同义词，则不会作为预测终结点的一部分返回该实体。

假设你的公司仅交付到一组有限的国家/地区。 你可以创建一个包含多个方式的列表实体，让客户引用该国家/地区。 如果 LUIS 在查询文本的文本中找不到完全匹配项，则不会在预测中返回实体（具有列表实体所需的功能）。

|规范名称|同义词|
|--|--|
|美国|美国<br>手杖. A<br>美国<br>USA<br>0|

例如，一个客户端应用程序（如聊天机器人）可以提出问题来帮助。 这可帮助客户了解国家/地区选择是否受到限制和*要求*。

### <a name="required-feature-using-regular-expression-entities"></a>使用正则表达式实体的所需功能

用作必需功能的[正则表达式实体](reference-entity-regular-expression.md)提供丰富的文本匹配功能。

在送货地址示例中，可以创建一个正则表达式来捕获国家/地区邮政编码的语法规则。

## <a name="global-features"></a>全局功能

尽管最常见的用途是将功能应用到特定模型，但你可以将此功能配置为**全局功能**，以将其应用到整个应用程序。

全局功能最常见的用途是向应用添加其他词汇。 例如，如果您的客户使用的是主要语言，但希望能够在同一查询文本中使用其他语言，则可以添加一项包含次要语言的单词的功能。

由于用户希望跨任何意向或实体使用辅助语言，因此请将次要语言中的单词添加到短语列表。 将短语列表配置为全局功能。
