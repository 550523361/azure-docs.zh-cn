# 概述：Azure 认知服务中的说话人识别 API

了解说话人识别功能如何增强应用程序。说话人识别 API 是基于云的 API，提供最高级的算法进行说话人验证和说话人识别。说话人识别可分为两类：

  * 说话人验证
  * 说话人识别

## 说话人验证

语音具有唯一的特征，像指纹一样可以用于识别个人。将语音用作访问控制和身份验证方案的信号已成为新的创新工具 – 给安全性带来质的提高，并简化客户的身份验证体验。

说话人验证 API 可以使用用户的语音或言谈，自动确认和验证用户的身份。

### 说话人验证注册

说话人验证注册与文本相关，也就是说，在注册和验证阶段，说话人需要选择所用的特定通行短语。

在注册时，说话人需要讲出特定的短语，系统会对其录音，然后提取多个特征并识别所选的短语。提取的特征和所选短语共同构成了唯一的语音签名。

### 验证输入语音和短语

在验证时，会将输入语音和短语与注册的语音签名和短语进行比较，以验证它们是否来自同一个人，以及说出的短语是否正确。

有关说话人验证的详细信息，请参阅[说话人识别 API 文档的“验证”部分](https://dev.projectoxford.ai/docs/services/563309b6778daf02acc0a508/operations/563309b7778daf06340c9652)。

## 说话人识别

在可能有一群人讲话的情况下，说话人识别 API 可自动识别音频文件中讲话的人。系统会将输入音频与提供的说话人群进行比较，如果找到匹配项，则返回说话人的标识。

所有说话人应事先完成注册过程，以便在系统中注册其语音并创建声纹。

### 说话人识别注册

说话人识别注册与文本无关，也就是说，说话人在音频中的讲话内容不受限制。系统将为说话人录音，并提取一些特征来构成唯一的语音签名。

### 说话人识别：匹配输入语音

未知说话人以及一群可能的说话人的音频将在识别期间提供。系统会将输入语音与所有说话人进行比较，以确定这是谁的语音。如果找到匹配项，则返回说话人的标识。

有关说话人识别的详细信息，请参阅[说话人识别 API 文档的“识别”部分](https://dev.projectoxford.ai/docs/services/563309b6778daf02acc0a508/operations/5645c068e597ed22ec38f42e)。
