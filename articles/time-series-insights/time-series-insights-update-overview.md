# <a name="azure-time-series-insights-overview"></a>Azure 时序见解概述

Azure 时序见解 (TSI) 是端到端平台即服务，用于引入、处理、存储和查询高度情景化、针对时序优化的 IoT 规模的数据。 因此，Azure TSI 适用于即席数据浏览和运营分析。 TSI 是唯一可扩展的自定义服务套餐，满足行业 IoT 部署的广泛需求。

## <a name="what-is-iot-data"></a>什么是 IoT 数据？

IoT 数据是在资产密集型组织中提供的“行业”数据。 IoT 数据通常高度非结构化，因为它是从各种资产发送的，记录噪声很强的度量，例如温度、移动、湿度。 另外，这些数据流通常带有下述特征：显著性差异、损坏的消息和错误读数。 在开始分析之前，必须对这些流中的数据进行清理。 通常只有在第一方数据源（例如 CRM 或 ERP）或第三方数据源（例如天气或位置）提供其他数据输入的情况下，IoT 数据才有意义。

因此，此类数据只有很少的一部分用于运营和业务目的。 此类数据提供一致、广泛、最新且正确的信息，适用于业务报告和分析。 因此，将收集的 IoT 数据转换成可行的见解需要一些关键功能：

* 数据处理，用于数据的清理、筛选、内插、转换和准备操作，以便进行分析
* 结构，适用于导航和了解数据（目的是规范化和情景化数据）
* 经济有效的存储，适用于长期/无限期保留数十年时间的已处理（派生）数据和原始数据

## <a name="azure-time-series-insights-for-industrial-iot"></a>适用于行业 IoT 的 Azure 时序见解

当前的 IoT 环境很多样。 它包括的客户涵盖制造、汽车、石油和天然气、电力和公共设施、智能建筑以及咨询等领域。 方案包括即席数据浏览（其中的数据形状未知），以及针对架构化（显式建模）数据进行运营分析，目的是提高运营效率。 这些方案通常并行存在，并且支持不同的用例。 平台功能，例如多层存储（热和冷）、存储数十年时序数据的功能以及针对基于资产的运营智能显式建模和优化查询的功能正变得越来越重要，是行业 IoT 企业及其数字变革能否成功的关键。

Azure TSI 是全面的端到端平台即服务套餐，适用于 IoT 数据浏览和获取运营见解。 时序见解提供完全托管的云服务，适用于分析 IoT 规模的时序数据。

客户可以在无架构内存中存储中存储原始数据，并通过分布式查询引擎和 API 执行交互式即席查询，同时利用丰富的用户体验在数秒钟内将数十亿事件可视化。 详细了解[数据浏览功能](./time-series-insights-overview.md)。

TSI 还提供目前为公共预览版的运营见解功能。 时序见解将交互式数据浏览和运营智能结合在一起，可以让客户充分利用从 IoT 资产收集的数据。 具体说来，公共预览版套餐支持以下重要功能：

* 可缩放且经过性能和成本优化的时序数据存储，使基于云的 IoT 解决方案能够在数秒内呈现多年来的时间序列数据趋势。
* 对于来自资产和设备的派生和非派生信号，语义模型支持描述与其相关联的域和元数据。
* 用户体验增强，将基于资产的数据见解与丰富的临时数据分析相结合，可推动业务和运营智能。
* 与先进的机器学习和分析工具（如 Azure Databricks、Apache Spark、Azure 机器学习、Jupyter Notebooks、Power BI 等）集成，有助于客户应对时序数据挑战并提高运营效率。

提供运营见解和数据浏览功能，而且即用即付定价模型很简单，适用于数据处理、存储和查询，因此为客户提供的模型可缩放性更强，让客户能够根据不断变化的业务需求对模型进行调整。

由于引入了这些关键的行业 IoT 功能，Azure TSI 具有下述主要优势。

* 多层存储，适用于 IoT 规模的时序数据

  * 通过使用常用数据处理管道来引入数据，客户可以将数据存储在热存储中进行交互式查询，以及/或者将数据存储在冷存储中，以便存储大量数据并利用高性能的基于资产的查询。

  * 很快会推出跨存储层进行动态路由的功能。

* 时序模型，适用于情景化原始遥测数据并派生基于资产的见解

  * 客户可以通过描述性时序模型来情景化原始遥测数据，并通过高性能的、成本优化型的、基于设备的查询来派生丰富的运营智能。

* 与其他数据解决方案无缝集成
  
  * 由于 Azure 时序见解中的数据存储在开源 Apache Parquet 文件中，因此客户可以轻松地将其与其他数据解决方案（不管是第一方还是第三方）集成，这适用于端到端方案，其中包括业务智能、高级机器学习、预测分析等。

* 近实时数据浏览

  * Azure 时序见解资源管理器用户体验提供的可视化功能适用于通过引入管道流式传输的所有数据。 在连接事件源之后不久，客户就可以查看、浏览和查询事件数据，以便验证设备是否按预期方式送出数据并监视 IoT 资产的运行状况、生产效率和总体效率。

* 根本原因分析和异常情况检测

  * Azure 时序见解资源管理器支持的模式和透视视图可以执行和保存多步骤根本原因分析。 客户可以将 Azure 流分析与时序见解配合使用，以便通过近实时方式检测警报和异常。

* 在时序见解平台上生成自定义应用程序

  * Azure 时序见解支持 JavaScript SDK 并提供丰富的控制和简化的查询访问功能，可以让客户在时序见解平台之上生成自定义 IoT 应用程序，满足各项业务的需求。
  * 客户还可以直接使用时序见解查询 API 将数据推送到自定义 IoT 应用程序中。
