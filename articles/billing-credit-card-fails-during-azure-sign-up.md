# 当我尝试注册 Azure 时，我的信用卡不被接受
本文讨论使用信用卡或借记卡注册时出现的问题的常见原因。

[AZURE.INCLUDE [support-disclaimer](../includes/support-disclaimer.md)]

## 除了信用卡或借记卡以外，是否还有其他支付选项？
Azure 付款选项包括：

- 信用卡或借记卡（虚拟卡和预付卡除外）
- 发票。选择发票选项之后，便无法对同一个 Azure 订阅改用信用卡选项。如果你以公司名义注册，并想要了解可开票的其他付款方式，请阅读 [Azure Billing - How to invoice（Azure 计费 - 如何开票）](https://azure.microsoft.com/pricing/invoicing/)。发票付款方式包括支票、隔夜支票和电汇。

##  用于激活或购买 Azure 的信用卡是否受支持？
- 目前我们仅接受非虚拟或非预付的信用卡/借记卡。
- 根据你所在的国家/地区，接受的信用卡和借记卡提供商会有所不同。

建议：

- 不要使用虚拟或预付型借记卡/信用卡。
- 如果付款或激活失败，请尝试使用其他有效的且受支持的信用卡。
- 确保选择的付款方式处于活动状态，并且你的银行不会出于安全政策而阻止任何在线交易。

## 信用卡信息是否正确且完整？
- 信用卡详细信息必须正确，并且与信用卡银行帐单信息相符。
- 输入的姓名必须完全符合信用卡上压印的姓名，并且不包含任何符号/字符。
- 输入的地址必须完全符合信用卡对帐单中的地址，包括街道地址、城市、州/省、国家/地区和邮政编码。
- 插入的 CVV 代码必须完全符合信用卡背面或正面的代码。
- 与你的银行沟通，以更正信用卡上的任何拼写错误或其他问题。

建议：

- 注册时始终提供准确的信息，因为个人资料详细信息还将用于生成对帐单。

## 帐户个人资料信息是否正确且完整？

- 帐户所有者在注册期间必须插入请求的所有相关信息，以便确认注册者拥有该帐户，以及在需要时生成正确的对帐单。
- 字段填写不正确或不完整的示例包括：缺少名字或姓氏、电话号码错误、不标准的（特殊）字符。

建议：

- 注册时反复检查个人资料中的每个字段；如果你创建了一个 Azure 帐户，请在[你的个人资料设置](https://account.windowsazure.com/Profile/)中检查字段。
- 提供准确的信息，因为个人资料详细信息还将用于生成对帐单。

## 你是否正在尝试激活 MSDN、BizSpark 或 MPN 等 Azure 权益计划？  
通过权益计划渠道验证你是否符合所选计划的资格：

- MSDN：

  - 在 [MSDN 帐户页](https://msdn.microsoft.com/subscriptions/manage/deult.aspx)中验证你的资格状态。
  - 如果你无法验证资格状态，请联系 [MSDN 订阅客服中心](https://msdn.microsoft.com/subscriptions/contactus.aspx)
- MPN

  - 登录到 [MPN 门户](https://mspartner.microsoft.com/en/us/Pages/Locale.aspx)并验证你的资格状态。如果你有相应的[云平台能力](https://mspartner.microsoft.com/en/us/pages/membership/cloud-platform-competency.aspx)，可能符合其他权益的资格。
  - 如果你无法验证自己的资格，请联系 [MPN 支持](https://mspartner.microsoft.com/en/us/Pages/Support/Premium/contact-support.aspx)。
- Bizpark：

  - 登录到 [BizSpark 门户](https://www.microsoft.com/bizspark/default.aspx#start-two)，然后验证 BizSpark 和 BizSpark Plus 的资格状态。如果你无法验证资格状态，请向 [BizSpark 团队](mailto:bizspark@microsoft.com?subject=BizSpark%20Support&body=Thank%20you%20for%20contacting%20BizSpark.%20Please%20provide%20as%20much%20of%20the%20following%20information%20as%20possible,%20as%20it%20will%20help%20expedite%20our%20response%20to%20you.%0aContact%20name:%0aStartup%20name:%0aMicrosoft%20Account/Live%20ID:%0aSpecific%20description%20of%20issue%20experienced%20or%20question:%0a%0aThank%20you,%0a%0aThe%20BizSpark%20Team)发送电子邮件，以联系 BizSpark 支持

建议：

如果在注册新权益订阅时发生错误，可以转到 [Azure 订阅页](http://account.windowsazure.com/Subscriptions)，以确定是否已创建该订阅。某些类型的订阅可能显示为等待处理状态，几分钟后即变为活动状态。如果几分钟后状态未改变，你可以联系 [Azure 支持](http://go.microsoft.com/fwlink/?linkid=544831&clcid=0x409)，以获取免费帮助。

## 你以前是否用过 Azure 订阅，而现在要尝试激活 Azure 免费试用版？
Azure 免费试用版优惠产品仅限于免费试用激活一次。

建议：

如果由于你以前激活过 Azure 订阅而导致免费试用版激活失败，请使用 [Microsoft Azure 优惠产品详细信息](https://azure.microsoft.com/support/legal/offer-details/)中所列的另一个可用 Azure 计划，而不要使用 Azure 免费试用计划。
