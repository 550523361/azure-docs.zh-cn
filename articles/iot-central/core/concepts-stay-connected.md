# <a name="stay-connected-with-azure-iot-central"></a>保持与 Azure IoT Central 的连接

本文概述了 Azure IoT Central 如何帮助保持设备连接和正常运行。

使用任何旨在大规模操作的 IoT 解决方案，设备管理的结构化方法非常重要。 只需将设备 "连接" 到 "已连接"，您需要一种方法来使设备保持连接状态，并使其在您的解决方案发展、发展和老化时保持正常运行。 Azure IoT Central 提供所需的功能，以确保 IoT 解决方案中的设备在整个应用程序生命周期中都关心。

## <a name="dashboards"></a>仪表板 
内置[仪表板](howto-manage-devices.md#import-devices)提供了一个可自定义的外围应用，用于监视设备运行状况和遥测数据。 从[应用程序模板](howto-use-app-templates-pnp.md)预构建的仪表板开始，或者创建自己的仪表板，以满足应用程序的需求。 可以与应用程序中的所有用户共享仪表板，或将其保留为私有。

## <a name="rules-and-actions"></a>规则和操作 
基于设备状态和遥测构建[自定义规则](howto-create-event-rules.md)，快速确定需要注意的设备。 将[操作](howto-create-event-rules.md#configure-actions)配置为及时通知正确的人员，确保纠正措施及时发生。

## <a name="jobs"></a>工作 
[作业](howto-run-a-job.md)允许对设备属性、设置和命令执行单个或批量更新。 

## <a name="user-roles-and-permissions"></a>用户角色和权限
[角色和权限](howto-manage-users-roles-pnp.md)使你能够定制每个用户的体验。 直接通过 UI 创建角色并轻松分配适当的权限。
