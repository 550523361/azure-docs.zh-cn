# <a name="troubleshoot-azure-netapp-files-resource-provider-errors"></a>排查 Azure NetApp 文件资源提供程序错误
本文介绍常见 Azure NetApp 文件资源提供程序错误、 其原因、 解决方案和解决方法。 

<a name="error_01"></a>***未配置 azure 密钥保管库。***   
Azure 密钥保管库存储所需的凭据用于访问基础 API。 此错误表示 Azure 密钥保管库未收到用于访问基础 API 的完整凭据。

* 原因  
Azure 密钥保管库未收到正确的凭据或凭据不完整。  

* 解决方案   
Azure NetApp 文件服务使用 Azure 密钥保管库。 Azure 密钥保管库使用来自 Azure Active Directory 的令牌进行身份验证。 因此，应用程序的所有者必须在 Azure Active Directory 中注册应用程序。

* 解决方法   
无。  Azure 密钥保管库必须正确设置为使用 Azure NetApp 文件。  

<a name="error_02"></a>***创建令牌不能更改。***   
当你尝试创建卷后更改创建令牌时，将发生此错误。
创建令牌必须设置的卷创建，并且以后无法更改。

* 原因   
想要更改创建令牌，该卷创建后，即不支持该操作。

* 解决方案   
创建卷后，请考虑从请求后，若要消除错误消息中删除参数。

* 解决方法   
如果需要更改创建令牌，可以创建具有新创建令牌后，新卷，然后将数据迁移到新卷。


<a name="error_03"></a>***创建令牌必须至少为 16 个字符。***   
当创建令牌不满足长度要求时，将发生此错误。 创建令牌的长度必须至少 16 个字符。

* 原因   
创建令牌不满足长度要求。  使用 API 创建卷时创建令牌是必需的。 如果使用门户，可以自动生成令牌。

* 解决方案   
增加创建标记的长度。 例如，您可以添加另一个词开头或末尾创建令牌。

* 解决方法   
创建标记的长度不能绕过所需的最低。  可以使用的前缀或后缀来提高创建标记长度。


<a name="error_04"></a>***删除的卷，找不到在 Azure 的 NetApp 文件时出错。***   
发生此错误的资源内部的注册表是不同步。

* 原因   
该卷可能保持状态在门户中显示一段时间后已被删除。 如果使用 API 删除该卷，就可以，该卷未正确指定。 此错误还可能引起过期浏览器缓存。

* 解决方案   
如果使用的门户，则清除浏览器缓存。 此外，还有每 10 分钟刷新一次的内部缓存。  您可以尝试再次清除缓存。  如果问题仍然存在 10 分钟后，可以创建支持票证。

* 解决方法   
在此期间使用的其他卷，并忽略现有。


<a name="error_05"></a>***插入在 Azure 的 NetApp 文件中找到的新卷时出错。***   
此错误是因为内部资源的注册表不同步。

* 原因   
该卷可能会保持在门户中显示一段时间后已被删除。 如果使用 API 删除该卷，就可以，该卷未正确指定。

* 解决方案   
如果使用门户时，已创建该卷。  卷应会自动显示。 如果问题仍然存在，则可以创建支持票证。

* 解决方法   
可以使用不同的名称和不同的创建令牌来创建卷。


<a name="error_06"></a>***文件路径名称可以包含字母、 数字和连字符 (""-"") 仅。***   
此错误发生时的文件路径包含不受支持的字符，例如，句点 ("。")，以逗号 （"，"），下划线 ("\_")，或美元符号 （"$"）。

* 原因   
文件路径包含不受支持的字符，例如，句点 ("。")，以逗号 （"，"），下划线 ("\_")，或美元符号 （"$"）。

* 解决方案   
删除不是字母、 数字或连字符的字符 ("-") 从输入文件路径。

* 解决方法   
可以下划线替换为连字符，也可以使用大小写而不是空格，以指示新单词 （例如，使用"NewVolume"而不"新建卷"） 开头。


<a name="error_07"></a>***无法更改卷 ID。***   
此错误发生时尝试更改卷 id。  更改卷 ID 不支持该操作。

* 原因   
创建卷时设置的文件系统的 ID。 以后无法更改卷 ID。

* 解决方案   
无。

* 解决方法   
无。  该卷创建并随后不能更改时，将生成以下卷 ID。


<a name="error_08"></a>***无效值{0}为接收{1}。***   
此消息指示 RuleIndex、 AllowedClients、 UnixReadOnly、 UnixReadWrite、 Nfsv3 和 Nfsv4 字段中的错误。

* 原因   
输入的验证请求失败的至少一个以下字段：RuleIndex、 AllowedClients、 UnixReadOnly、 UnixReadWrite、 Nfsv3 和 Nfsv4。

* 解决方案   
请确保命令行上设置所有必需的和不冲突的参数。 例如，不能同时设置 UnixReadOnly 和 UnixReadWrite 参数。

* 解决方法   
请参阅的解决方案部分。  


<a name="error_09"></a>***缺少值{0}。***   
此错误表示至少一个以下参数请求中缺少所需的属性：RuleIndex、 AllowedClients、 UnixReadOnly、 UnixReadWrite、 Nfsv3 和 Nfsv4。

* 原因   
输入的验证请求失败的至少一个以下字段：RuleIndex、 AllowedClients、 UnixReadOnly、 UnixReadWrite、 Nfsv3 和 Nfsv4。

* 解决方案   
请确保命令行上设置所有必需的和不冲突的参数。 例如，你不能同时设置 UnixReadOnly 和 UnixReadWrite 参数

* 解决方法   
请参阅的解决方案部分。  


<a name="error_10"></a> ***{0} 已在使用中。***   
此错误表示已使用资源的名称。

* 原因   
要创建具有与现有卷相同的名称的卷。

* 解决方案

* 解决方法

<a name="error_11"></a> ***{0} 太短。***   
此错误表示卷名称不符合最小长度要求。

* 解决方案   
此问题很可能是暂时的。  请求一段时间后应该会成功。

无。 基础 API 是必需的管理卷。

<a name="error_17"></a>***接收到只读的属性的值{0}。***

<a name="error_20"></a>***未知 Azure NetApp 文件时出错。***

此问题很可能是暂时性的并请求一段时间后应该会成功。 如果问题仍然存在，请创建支持票证以调查问题。

无。  基础 API 是必需的管理卷。

<a name="error_21"></a>***收到的未知属性的值{0}。***   
当为某个资源例如卷、 快照或装载目标提供了不存在的属性时，将发生此错误。

* 原因   
请求具有一组可用于每个资源的属性。  不能在请求中包含不存在的任何属性。

* 解决方案   
请确保所有属性名称都拼写正确，并且属性是可用于订阅和资源。

* 解决方法   
减少请求以消除导致错误的属性中定义的属性的数量。


<a name="error_22"></a>***此资源类型不支持更新操作。***   
可以更新仅卷。 发生此错误时尝试执行一个不支持的更新操作，例如，更新快照。

* 原因   
尝试更新的资源不支持更新操作。  仅卷可以修改其属性。

* 解决方案   
无。  要更新的资源不支持更新操作。 因此，不能更改它。

* 解决方法   
对于卷，就地更新创建新的资源并迁移数据。


<a name="error_23"></a>***项目数：{0}对象：{1}不在最小最大范围。***   
导出策略规则不能满足最小值或最大范围要求时，将发生此错误。  如果你定义的导出策略，它必须具有至少一个导出策略规则和五个导出策略规则达到最大值。

* 原因   
您定义的导出策略不符合要求的区域。  

* 解决方案   
请确保尚未使用索引并且所在的范围从 1 到 5。

* 解决方法   
不一定要使用的卷上的导出策略。 因此，可以省略的导出策略完全如果不需要具有导出策略规则。


<a name="error_24"></a>***重复的对象的值错误{0}。***   
导出策略未定义具有唯一索引时，将发生此错误。  在定义导出策略时，导出的所有策略规则都必须介于 1 和 5 之间的唯一索引。

* 原因   
定义的导出策略不符合导出策略规则的要求。 必须具有至少一个导出策略规则和五个导出策略规则达到最大值。  

* 解决方案   
请确保尚未使用索引，它位于范围从 1 到 5。

* 解决方法   
要设置的规则使用不同的索引。
