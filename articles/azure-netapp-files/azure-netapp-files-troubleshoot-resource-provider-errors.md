# <a name="troubleshoot-azure-netapp-files-resource-provider-errors"></a>排查 Azure NetApp 文件资源提供程序错误 

本文介绍了常见的 Azure NetApp 文件资源提供程序错误、其原因、解决方案和解决方法（如果可用）。

## <a name="common-azure-netapp-files-resource-provider-errors"></a>常见 Azure NetApp 文件资源提供程序错误

***无法更改裸机租户Id。***  

当您尝试更新或修补卷且属性具有更改的值时，`BaremetalTenantId`将发生此错误。

* 原因：   
您正在尝试更新卷，`BaremetalTenantId`并且该属性的值与存储在 Azure 中的值不同。
* 解决方案：   
不要在修补程序中包含`BaremetalTenantId`更新（放置）请求。 或者，请确保`BaremetalTenantId`请求中相同。

***无法更改服务级别。***  

当您尝试更新或修补具有不同服务级别的容量池时，当容量池中已具有卷时，将发生此错误。

* 原因：   
当您尝试更新容量池服务级别时，池包含卷。
* 解决方案：   
从容量池中删除所有卷，然后更改服务级别。
* 解决方法：   
创建另一个容量池，然后在新的容量池中再次创建卷。

***无法更改池 Id***  

当您尝试使用已更改`PoolId`的属性更新或修补容量池时，将发生此错误。

* 原因：   
您正在尝试更新容量池`PoolId`属性。 属性`PoolId`是只读属性，无法更改。
* 解决方案：   
不要在修补程序中包含`PoolId`更新（放置）请求。  或者，请确保`PoolId`请求中相同。

***无法更改创建令牌。***

创建卷后尝试更改文件路径 （`CreationToken`） 时，将发生此错误。 创建卷时`CreationToken`必须设置文件路径 （ ， 以后不能更改它。

* 原因：   
您正在尝试在创建卷后更改文件`CreationToken`路径 （ ）， 这不是受支持的操作。 
* 解决方案：   
如果不需要更改文件路径，请考虑从请求中删除参数以关闭错误消息。
* 解决方法：   
如果需要更改文件路径 （），`CreationToken`可以使用新文件路径创建新卷，然后将数据迁移到新卷。

***创建令牌必须至少为 16 个字符长。***

当文件路径 （`CreationToken`） 不符合长度要求时，将发生此错误。 文件路径的长度必须至少为一个字符的长度。

* 原因：   
文件路径为空。  使用 API 创建卷时，需要创建令牌。 如果使用 Azure 门户，将自动生成文件路径。
* 解决方案：   
输入至少一个字符作为文件路径 （`CreationToken`。

***无法更改域名。***

当您尝试更改活动目录中的域名时，将发生此错误。

* 原因：   
您正在尝试更新域名属性。
* 解决方案：    
无。 您不能更改域名。
* 解决方法：   
使用活动目录配置删除所有卷。 然后删除活动目录配置并重新创建卷。

***对象导出策略的重复值错误。***

当导出策略未使用唯一索引定义时，将发生此错误。 定义导出策略时，所有导出策略规则必须具有介于 1 和 5 之间的唯一索引。

* 原因：   
定义的导出策略不符合出口策略规则的要求。 您必须至少有一个导出策略规则，最多有五个导出策略规则。
* 解决方案：   
确保索引尚未使用，并且索引在 1 到 5 之间。
* 解决方法：   
对尝试设置的规则使用不同的索引。

***错误 [操作] [资源类型名称]***

当其他错误处理在资源执行操作时未能处理错误时，将显示此错误。   它包括文本"错误"。 `{action}`可以是 的任意 （`getting` `creating`、 `updating`、 `deleting`、 或 。  `{resourceTypeName}`是`resourceTypeName`（例如 ， `netAppAccount`、 `capacityPool` `volume`、 、 等）。

* 原因：   
此错误是未处理的异常，原因不知道。
* 解决方案：   
请与 Azure 支持中心联系，在日志中报告详细原因。
* 解决方法：   
无。

***文件路径名称只能包含字母、数字和连字符（"-"）。***

当文件路径包含不受支持的字符（例如，句点（"."）、逗号（""）、下划线（"*"）或美元符号 （"$"）时，将发生此错误。

* 原因：   
文件路径包含不支持的字符，例如句点（"."）、逗号（""）、下划线（"*"），或美元符号（"$"）。
* 解决方案：   
从输入的文件路径中删除不是字母字母、数字或连字符 （"-"） 的字符。
* 解决方法：   
您可以将下划线替换为连字符，或使用大写而不是空格来指示新单词的开头。  例如，使用"新卷"而不是"新卷"。

***无法更改文件系统 Id。***

当您尝试更改`FileSystemId`时，将发生此错误。  更改`FileSystemdId`不是受支持的操作。 

* 原因：   
创建卷时设置文件系统的 ID。 `FileSystemId`以后无法更改。
* 解决方案：   
不要在修补程序中包含`FileSystemId`更新（放置）请求。  或者，请确保请求`FileSystemId`中相同。

* 原因：

* 解决方案：   
不要在创建（放置`ActiveDirectoryId`）请求中包含。

***无效的 api 版本。***

API 版本未提交或包含无效值。

* 原因：

* 解决方案：

***不允许混合协议类型 CIFS 和 NFS***

当您尝试创建卷时，会出现此错误，并且卷属性中同时存在 CIFS （SMB） 和 NFS 协议类型。

* 原因：

* 解决方案：

***项数：对象的 [值]：导出策略.规则[规则索引]超出最小最大值范围。***

当导出策略规则不符合最小或最大范围要求时，将发生此错误。 如果定义导出策略，它必须至少有一个导出策略规则，最多有五个导出策略规则。

* 原因：

* 解决方案：

* 解决方法：   
对卷使用导出策略不是强制性的。 如果不需要使用导出策略规则，则可以完全省略导出策略。

***池大小太小，无法占总卷大小。***

* 解决方案：   
将容量池大小设置为较大的值，或为卷创建较小的卷。
* 解决方法：   
删除足够的卷，以便容量池大小可以更新到此大小。

* 解决方法：   
无。 请参阅上面的解决方案。 

***子网必须具有"Microsoft.NetApp/卷"委派。***

创建卷且所选子网未委派给`Microsoft.NetApp/volumes`时，将发生此错误。

* 原因：   
您尝试创建卷，并选择了未委派到`Microsoft.NetApp/volumes`的子网。
* 解决方案：   
选择委派给`Microsoft.NetApp/volumes`的另一个子网。
* 解决方法：   
向子网添加正确的委派。

***指定的资源类型未知/不适用。***

当对不适用的资源类型或未知资源类型请求名称检查时，将发生此错误。

* 原因：   
已请求对未知或不受支持的资源类型进行名称检查。
* 解决方案：   
检查您执行请求的资源是否受支持或不包含拼写错误。
* 解决方法：   
请参阅上面的解决方案。

***未知 Azure NetApp 文件错误。***

Azure API 依赖于 Azure NetApp 文件 API 来管理卷。 该错误指示与 API 的通信中出现问题。

* 原因：   
基础 API 发送未知错误。 此错误可能是临时的。
* 解决方案：   
问题可能是暂时的，请求应在一段时间后成功。 如果问题仍然存在，请创建一个支持票证以调查问题。
* 解决方法：   
无。 基础 API 对于管理卷至关重要。

***为未知属性"[属性名称]"接收的值。***

当为资源（如卷、快照或装载目标）提供不存在的属性时，将发生此错误。

* 原因：   
请求具有一组可用于每个资源的属性。 不能在请求中包含任何不存在的属性。
* 解决方案：   
确保所有属性名称拼写正确，并且属性可用于订阅和资源。
* 解决方法：   
减少请求中定义的属性数，以消除导致错误的属性。

***此资源类型不支持更新操作。***

只能更新卷。 当您尝试执行不受支持的更新操作（例如，更新快照）时，将发生此错误。

* 原因：   
您尝试更新的资源不支持更新操作。 只有卷可以修改其属性。
* 解决方案：   
无。 您尝试更新的资源不支持更新操作。 因此，无法更改它。
* 解决方法：   
对于卷，创建具有更新的新资源并迁移数据。

***无法在未成功处于状态的池中创建卷。***

当您尝试在未处于成功状态的池中创建卷时，将发生此错误。 最有可能的是，容量池的创建操作由于某种原因失败。

* 原因：   
包含新卷的容量池处于失败状态。
* 解决方案：   
检查容量池是否已成功创建，以及容量池未处于失败状态。
* 解决方法：   
创建新容量池并在新池中创建卷。

***正在创建卷，目前无法删除。***

当您尝试删除仍在创建的卷时，将发生此错误。

* 原因：   
当您尝试删除该卷时，仍在创建卷。
* 解决方案：   
等待卷创建完成，然后重试删除。
* 解决方法：   
请参阅上面的解决方案。

***卷正在删除，目前无法删除。***

当您尝试删除卷时，该卷已被删除时，将发生此错误。

* 原因：   
当您尝试删除该卷时，卷已被删除。
* 解决方案：   
等待，直到当前删除操作完成。
* 解决方法：   
请参阅上面的解决方案。

***卷正在更新，目前无法删除。***

当您尝试删除正在更新的卷时，将发生此错误。

* 原因：   
当您尝试删除卷时，将更新卷。
* 解决方案：   
等待更新操作完成，然后重试删除。
* 解决方法：   
请参阅上面的解决方案。

***未找到或未成功创建卷。***

当卷创建失败，并且您尝试更改卷或为卷创建快照时，将发生此错误。

* 原因：   
卷不存在，或者创建失败。
* 解决方案：   
检查更改的卷是否正确，以及卷的创建是否成功。 或者，检查要为其创建快照的卷是否确实存在。
* 解决方法：   
无。  请参阅上面的解决方案。 

***指定的创建令牌已存在***

当您尝试创建卷，并指定已存在的卷的创建令牌（导出路径）时，将发生此错误。

* 原因：   
在卷创建期间指定的创建令牌（导出路径）已与另一个卷关联。 
* 解决方案：   
选择其他创建令牌。  或者，删除其他卷。

***保留指定的创建令牌***

当您尝试创建卷，并将"默认"或"无"指定为文件路径（创建令牌）时，将发生此错误。

* 原因：    
您正在尝试创建卷，并将"默认"或"无"指定为文件路径（创建令牌）。
* 解决方案：   
选择其他文件路径（创建令牌）。
 
***活动目录凭据正在使用中***

当您尝试从至少一个 SMB 卷存在的帐户中删除 Active Directory 配置时，将发生此错误。  SMB 卷是使用您尝试删除的活动目录配置创建的。

* 原因：   
您尝试从帐户中删除 Active Directory 配置，但至少仍然存在一个最初使用活动目录配置创建的 SMB 卷。 
* 解决方案：   
首先，删除使用活动目录配置创建的所有 SMB 卷。  然后重试配置删除。

***如果凭据正在使用中，则无法修改组织单位分配***

当您尝试更改活动目录配置的组织单位时，将发生此错误，但至少仍然存在一个 SMB 卷。  SMB 卷是通过使用您尝试删除的活动目录配置创建的。

* 原因：   
您正在尝试更改活动目录配置的组织单位。  但是，至少一个 SMB 卷仍然存在，这些卷最初是使用活动目录配置创建的。
* 解决方案：   
 首先，删除使用活动目录配置创建的所有 SMB 卷。  然后重试配置删除。 

***活动目录更新已在进行中***

当您尝试编辑已为其进行编辑操作的活动目录配置时，将发生此错误。

* 原因：   
您正在尝试编辑活动目录配置，但另一个编辑操作已在进行中。
* 解决方案：   
等待当前正在运行的编辑操作完成。

***首先使用所选凭据删除所有卷***

当您尝试删除活动目录配置时，将发生此错误，但至少仍然存在一个 SMB 卷。  SMB 卷是使用您尝试删除的活动目录配置创建的。

* 原因：   
您正在尝试删除活动目录配置，但至少仍然存在一个最初使用活动目录配置创建的 SMB 卷。
* 解决方案：   
首先，删除使用活动目录配置创建的所有 SMB 卷。  然后重试配置删除。 

***在区域中找不到活动目录凭据***

当您尝试创建 SMB 卷时，将发生此错误，但尚未将活动目录配置添加到该区域的帐户中。

* 原因：   
您正在尝试创建 SMB 卷，但帐户中未添加活动目录配置。 
* 解决方案：   
在创建 SMB 卷之前，将活动目录配置添加到帐户。

***无法查询 DNS 服务器。验证网络配置是否正确，以及 DNS 服务器是否可用。***

当您尝试创建 SMB 卷时，将发生此错误，但无法访问 DNS 服务器（在 Active Directory 配置中指定）。 

* 原因：   
您正在尝试创建 SMB 卷，但无法访问 DNS 服务器（在 Active Directory 配置中指定）。
* 解决方案：   
查看活动目录配置，并确保 DNS 服务器 IP 地址正确且可到达。
如果 DNS 服务器 IP 地址没有问题，请验证没有防火墙阻止访问。

***并发作业太多***

当您尝试在为订阅创建其他三个快照创建操作时，将发生此错误。

* 原因：   
当您正在为订阅创建其他三个快照创建操作时，您正在尝试创建快照。 
* 解决方案：   
快照创建作业最多需要几秒钟才能完成。  等待几秒钟，然后重试快照创建操作。

***无法生成其他作业。请等待正在进行的作业完成，然后重试***

当您尝试在特定情况下创建或删除卷时，可能会发生此错误。

* 原因：   
您正在尝试在特定情况下创建或删除卷。
* 解决方案：   
等待一分钟左右，然后重试该操作。

***卷已在状态之间转换***

当您尝试删除当前处于转换状态的卷（即当前处于创建、更新或删除状态）时，可能会发生此错误。

* 原因：   
您正在尝试删除当前处于转换状态的卷。
* 解决方案：   
等待当前正在运行的（状态转换）操作完成，然后重试该操作。

***无法从源卷快照拆分新卷***

 当您尝试从快照创建卷时，可能会发生此错误。  

* 原因：   
您尝试从快照创建卷，卷以错误状态结束。
* 解决方案：   
删除卷，然后从快照重试卷创建操作。
