# <a name="overview-of-azure-security-baselines"></a>Azure 安全基线概述

Azure 安全基线通过改进的工具、跟踪和安全功能帮助您增强产品的安全性，并在保护环境时为您提供一致的体验。

Azure 服务安全基线侧重于以云为中心的控制区域。 这些控制措施与众所周知的安全基准（如互联网安全中心 （CIS） 描述的安全基准一致。 我们的基线为[Azure 安全基准](overview.md)中列出的控制区域提供了指导。

每个建议包括以下信息：
- **Azure ID**：对应于建议的 Azure 安全基准 ID。
- **建议**：直接遵循 Azure ID 后，该建议提供控件的高级说明。
- **指导**：建议的理由和如何实施该建议的指导的链接。 如果 Azure 安全中心支持该建议，该信息也将列出。
- **责任**：谁负责实施控制。 可能的情况是客户责任、微软责任或分担责任。
- **Azure 安全中心监视**：控件是否由 Azure 安全中心监视，并带有指向引用的链接。

所有建议（包括不适用于此特定服务的建议）都包含在基线中，以便全面了解 Azure 安全基准与每个服务的关系。 有时可能有一些控件由于各种原因不适用，例如，IaaS/以计算为中心的控件（例如特定于操作系统配置管理的控件）可能不适用于 PaaS 服务。
