# <a name="azure-security-benchmark-introduction"></a>Azure 安全基准简介

您可能拥有数年甚至几十年的本地计算经验。 您知道如何保护这些部署;但是，您可以保护这些部署。但云是不同的。 您如何知道云部署是否安全？ 本地系统的安全实践和云部署的安全实践有何区别？

白皮书、最佳实践、参考体系结构、Web 指南、开源工具、商业解决方案、智能源等大量可用于帮助保护云的安全。 您应该使用哪个选项？ 您可以做些什么来获得云中可接受的安全级别？ 

保护云部署的最佳方法之一是关注云安全基准建议。 保护任何服务的基准建议首先从对网络安全风险以及如何管理网络安全风险的基本理解开始。 然后，您可以通过从云服务提供商处采用基准安全建议来帮助选择环境中的特定安全配置设置来使用此理解。 

Azure 安全基准测试包括一系列高影响安全建议，可用于帮助保护在 Azure 中使用的大多数服务。 您可以将这些建议视为"常规"或"组织"，因为它们适用于大多数 Azure 服务。 然后为每个 Azure 服务自定义 Azure 安全基准建议，并且此自定义指南包含在服务建议文章中。 

Azure 安全基准文档指定安全控制和服务建议。

- **安全控制**：Azure 安全基准建议按安全控件分类。 安全控制表示与供应商无关的高级安全要求，如网络安全和数据保护。 每个安全控制都有一组安全建议和说明，可帮助您实现这些建议。 
- **服务建议**：如果可用，Azure 服务的基准建议将包括专为该服务量身定制的 Azure 安全基准建议。 

Azure 安全基准文档中经常使用术语"控制"、"基准"和"基准"，了解 Azure 如何使用这些术语非常重要。 

| 术语 | 说明 | 示例 |
|--|--|--|
| 控制 | **控件**是需要处理且不特定于技术或实现的功能或活动的高级描述。 | 数据保护是安全控制之一。 此控件包含需要处理的特定操作，以帮助确保数据受到保护。 |
| 基准 | **基准测试**包含特定技术（如 Azure）的安全建议。 建议按它们所属的控件进行分类。 | Azure 安全基准包括特定于 Azure 平台的安全建议  |
| 基线 | **基线**是组织的安全要求。 安全要求基于基准建议。 每个组织决定在其基线中包括哪些基准建议。 | Contoso 公司通过在 Azure 安全基准中选择要求特定建议来创建其安全基准。 |
