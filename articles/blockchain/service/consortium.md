# <a name="azure-blockchain-service-consortium"></a>Azure 区块链服务联盟

使用 Azure 区块链服务，您可以创建专用联盟区块链网络，其中每个区块链网络可以限制为网络中的特定参与者。 只有私人联盟区块链网络中的参与者才能查看区块链并与之交互。 Azure 区块链服务中的联盟网络可以包含两种类型的成员参与者角色：

* **管理员**- 特权参与者，可以采取联合体管理行动，并可以参与区块链交易。

* **用户**- 参与者不能采取任何联合管理行动，但可以参与区块链交易。

联盟网络可以是参与者角色的组合，并且可以具有每个角色类型的任意数量。 必须至少有一个管理员。

下图显示了具有多个参与者的联营网络：

![私人财团网络图](./media/consortium/network-diagram.png)

借助 Azure 区块链服务中的联合管理，您可以管理联合体网络中的参与者。 联合体的管理基于网络的共识模型。 在当前预览版本中，Azure 区块链服务为联合管理提供了集中式共识模型。 具有管理角色的任何特权参与者都可以采取联合管理操作，例如从网络添加或删除参与者。

## <a name="roles"></a>角色

联合体的参与者可以是个人或组织，也可以为用户角色或管理员角色分配。 下表列出了这两个角色之间的高级差异：

| 操作 | 用户角色 | 管理员角色
|--------|:----:|:------------:|
| 创建新成员 | 是 | 是 |
| 邀请新成员 | 否 | 是 |
| 设置或更改成员参与者角色 | 否 | 是 |
| 更改成员显示名称 | 仅适用于自己的成员 | 仅适用于自己的成员 |
| 删除成员 | 仅适用于自己的成员 | 是 |
| 参与区块链交易 | 是 | 是 |

### <a name="user-role"></a>用户角色

用户是没有管理员功能的联盟参与者。 他们不能参与管理与联合体相关的成员。 用户可以更改其成员显示名称，并可以从联合体中删除自己。

### <a name="administrator"></a>管理员

管理员可以管理联合体内的成员。 管理员可以邀请成员、删除成员或更新联合体内的成员角色。
联合体中必须始终至少有一个管理员。 最后一个管理员必须在离开联合体之前指定另一个参与者作为管理员角色。

## <a name="managing-members"></a>管理成员

只有管理员才能邀请其他参与者加入联合体。 管理员使用其 Azure 订阅 ID 邀请参与者。

邀请后，参与者可以通过在 Azure 区块链服务中部署新成员加入区块链联盟。 要查看并加入受邀的联合体，必须指定网络管理员在邀请中使用的相同 Azure 订阅 ID。

管理员可以从联合体中删除任何参与者，包括其他管理员。 成员只能将自己从联合体中除名。

## <a name="consortium-management-smart-contract"></a>联盟管理智能合同

Azure 区块链服务的联盟管理是通过联合体管理智能合约完成的。 当您部署新的区块链成员时，智能合约会自动部署到您的节点。

可以在 Azure 门户中查看根联合体管理智能协定的地址。 **根合同地址**位于区块链成员概述部分。

![根合同地址](./media/consortium/rootcontract-address.png)

您可以使用联合体管理[PowerShell 模块](manage-consortium-powershell.md)、Azure 门户，或者直接使用 Azure 区块链服务生成的以地伦姆帐户与联合体管理智能合约进行交互。

## <a name="ethereum-account"></a>以里瑟姆帐户

创建成员时，将创建以地图姆帐户密钥。 Azure 区块链服务使用密钥创建与联合管理相关的事务。 以非庭帐户密钥由 Azure 区块链服务自动管理。

可以在 Azure 门户中查看成员帐户。 会员账户位于区块链成员概述部分。

![会员帐户](./media/consortium/member-account.png)

您可以通过单击会员帐户并输入新密码来重置您的以地图姆帐户。 将重置以地庭帐户地址和密码。
