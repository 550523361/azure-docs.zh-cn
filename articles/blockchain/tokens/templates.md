# <a name="azure-blockchain-tokens-templates"></a>Azure 区块链令牌模板

[!INCLUDE [Preview note](./includes/preview.md)]

Azure 区块链令牌模板是一个标准化且可重用的模板，可简化基于分类帐的令牌的创建和部署。 模板由基于[令牌分类框架 （TTF）](overview.md#token-taxonomy-framework)语法的公式组成。 语法包括基令牌类型和令牌的行为集。  

例如 **，\d，m，b，r]** 令牌模板描述了一个可替换的基础令牌，该基令牌是子 div、可小、可刻录且具有角色支持。
  
## <a name="base-token-types"></a>基本令牌类型

为特定资产定义和创建基于分类帐的令牌时，必须考虑要使用的基本令牌。

### <a name="fungible"></a>可互换

可互换令牌 （*F） 具有相互互换的值，只要它们位于同一类或系列中。 一个令牌具有与另一个令牌相同的值，或者给定数量的令牌与另一个等量具有相同的值。 例如，美元是可互换的令牌。 如果两个人每人拿着一张美元钞票，他们可以兑换这些美元钞票，而不会造成后果。 美元钞票具有等值。 

### <a name="non-fungible"></a>不可防

不可替换的令牌 （*N） 不能与相同类型的其他令牌互换，因为它们通常具有不同的值。 例如，属性标题是不可替换的令牌。 公寓大楼内两栋不同公寓的产权不一定具有同等价值，因为公寓的位置或公寓位于哪个楼层。 两个属性标题令牌的感知值不相等。

### <a name="hybrid"></a>混合

混合令牌是具有可替换令牌和不可替换令牌组件的令牌。 混合令牌是具有其他令牌类型的类的基本令牌类型。

#### <a name="hybrid-non-fungible-base-with-fungible-segments"></a>带可互换段的混合不可替代底座

具有可替换段令牌的混合不可替换基具有不可替换的基段和可替换令牌段。
例如，音乐会门票是混合令牌，其中音乐会的日期和时间是不可替换的基础令牌。 特定音乐会各座位部分的门票是带有可互换代币的片段。 门票可在各自的座位部分兑换，但不能跨部分兑换。

#### <a name="hybrid-fungible-base-with-non-fungible-segments"></a>具有不可替代段的混合可互换底座

具有不可替代段令牌的混合可替换基具有不可替换的令牌段的可替换基。 例如，抵押支持的安全性是混合令牌，其中多个所有者是跨多个所有者拆分的可替换基础。 安全性是可互换的。 个人抵押贷款是代表特定抵押贷款支持证券的非可互换部分。

## <a name="token-behaviors"></a>令牌行为

令牌行为定义令牌的功能或限制。 该行为包括作为令牌定义一部分的支持属性。 行为可以应用于所有令牌类型，也可以只应用于一种。 行为可以是内部行为，也可以是外部行为，具体取决于行为的影响。 内部行为启用或限制令牌本身的属性。 外部行为启用或限制从外部执行组件调用行为。

有关 Azure 区块链令牌支持的令牌分类框架 （TTF） 令牌行为的详细信息，请参阅[令牌可合成性](composability.md)。

## <a name="pre-built-token-templates"></a>预构建令牌模板

Azure 区块链令牌提供了四个预构建的令牌模板，无需修改即可使用。 您可以调用这些预构建的模板，以便大多数用例快速开始创建、部署和管理令牌。

### <a name="commodity-tokens"></a>商品代币

商品代币具有一致的价值，并且可转让。 例如，一桶油或一单位的能量。

**[F_d，t，m，b，r]** - 可替换，整体，可转移，可挖掘，可刻录，并且具有角色支持

许多区块链方案需要整个供应链或多个组织的透明度和可见性。 商品代币基于这些常见的用例。 令牌是可互换的，并且一致。 商品令牌模板是灵活且可自定义的元数据。

### <a name="qualified-tokens"></a>合格令牌

限定令牌表示赚取的内容，通常与一个实体关联，不能传输。 例如，文凭或违章停车。

**[N]，[t]** - 不可替换、单笔和不可转让

各种审核和证明方案要求更改令牌的所有权。 有一组用例，无论关联是好还是坏，都需要提供限定令牌。

### <a name="asset-tokens"></a>资产代币

资产令牌具有依赖于物料的唯一价值，并且不商品化。 例如，博物馆文物或属性所有权。

**[N]，t]** - 不可替换、单一和可转让

资产代币可能与商品代币混淆。 这两个令牌之间的主要区别是，资产令牌本质上是唯一的，并且值与它所值的令牌类型无关。 例如，像一幅由老牌艺术家创作的油画这样的艺术品是一种资产象征。 然而，蒙娜丽莎的艺术版画被认为是商品的象征。 同样，属性所有权是资产令牌，因为该值存在于属性的主观品质中。

### <a name="ticket-tokens"></a>票证令牌

票证令牌具有一致值，但通常过期。 例如，平面机票。

**[N]，b，r]** - 不可替换、可小、可刻录且具有角色支持。

票证令牌通常具有到期日期，使其不同于常规商品代币。 例如，飞机票、音乐会门票或体育票都有指定座位的选项，并带有特定的使用日期。 您不能轻松地在日期或休息区之间交换门票。
