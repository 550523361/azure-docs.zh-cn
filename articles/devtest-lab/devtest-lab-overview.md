# <a name="about-azure-devtest-labs"></a>有关 Azure DevTest 实验室
## <a name="overview"></a>概述
开发人员和测试正在寻求解决在创建和管理其环境，通过转到云的延迟问题。  Azure 解决了环境延迟的问题，并允许自助服务在新的成本高效结构。  但是，开发人员和测试仍需要花费相当长时间来配置其提供自助服务的环境。 此外，决策制定者不确定应如何利用云，以使其节省成本而无需添加太多进程开销最大化。

Azure DevTest 实验室是一个服务，可帮助开发人员和测试人员环境快速创建在 Azure 中时最大程度减少浪费和控制成本。 可以通过快速设置使用可重用的模板和项目的 Windows 和 Linux 环境来测试你的应用程序的最新版本。 轻松地与 DevTest 实验室设置按需环境集成部署管道。 负载测试设置多个测试代理，通过向上扩展并创建用于培训和演示的预设置的环境。

DevTest 实验室中创建、 配置和管理云中的开发人员和测试环境具有以下优点

## <a name="worry-free-self-service"></a>尽情自助服务
DevTest 实验室允许你设置策略，在你的实验室-如虚拟机 (VM) 每个用户数和每个实验室的 Vm 数，从而使更轻松地控制成本。 DevTest 实验室还可以创建策略，自动关闭和启动 Vm。

## <a name="quickly-get-to-ready-to-test"></a>快速访问准备就绪，可以测试
DevTest 实验室可以使用你的团队需要启动开发和测试应用程序的所有内容创建预设置的环境。 只需声明你的应用程序上一个好生成的安装位置的环境，并获取立即使用。 或者，将容器用于甚至更快、 更精简环境的创建。

## <a name="create-once-use-everywhere"></a>创建一次，请使用 everywhere
捕获和共享环境模板和在你的团队或组织的所有源代码管理中的项目创建开发人员和轻松测试环境。

## <a name="integrates-with-your-existing-toolchain"></a>与你现有的工具链集成
利用预制的插件或我们的 API 来设置开发/测试环境，直接从您首选的持续集成 (CI) 工具，集成开发环境 (IDE)，或自动发行管道。 你还可以使用我们全面的命令行工具。
