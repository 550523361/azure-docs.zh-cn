# <a name="about-azure-devtest-labs"></a>关于 Azure 开发测试实验室
## <a name="overview"></a>概述
开发人员和测试人员力求通过云解决创建和管理其环境出现的延迟问题。  Azure 解决了环境的延迟问题，并允许在经济有效的新结构中采用自助服务。  但是，开发人员和测试人员仍需要花费相当长的时间来配置其自助环境。 此外，决策制定者不确定应如何利用云，以最大化节省成本，而无需添加太多进程开销。

Azure 开发测试实验室是一项可帮助开发人员和测试人员在 Azure 中快速创建环境，同时尽量减少浪费并控制成本的服务。 可使用可重用模板和项目通过快速预配 Windows 和 Linux 环境来测试应用程序的最新版本。 轻松地将部署管道与开发测试实验室集成，以预配按需环境。 通过预配多个测试代理，增加负载测试，并为培训和演示创建预先预配的环境。

开发测试实验室为创建、配置和管理开发人员和云中的测试环境提供了以下优势

## <a name="worry-free-self-service"></a>无忧的自助服务
开发测试实验室通过允许设置实验室策略（如每个用户的虚拟机 (VM) 数量和每个实验室的 VM 数量）可更加轻松地控制成本。 通过开发测试实验室还可以创建自动关闭和启动 VM 的策略。

## <a name="quickly-get-to-ready-to-test"></a>快速转至测试就绪
通过开发测试实验室可以创建预配置的环境，其中包含团队需要开始开发和测试应用程序的所有内容。 只需声明安装了应用程序最近一次的良好生成的环境，就可立即使用。 或者，使用环境容器来创建更加快速和高效的环境。

## <a name="create-once-use-everywhere"></a>一次创建，随处可用
在团队或组织内部捕获并共享环境模板和相关资源（一切尽在源代码控制下），从而轻松创建开发和测试环境。

## <a name="integrates-with-your-existing-toolchain"></a>与现有工具链进行集成
利用预先做好的插件或 API 直接从喜欢的持续集成 (CI) 工具、集成开发环境 (IDE) 或自动发布管道设置开发/测试环境。 还可以使用综合命令行工具。
