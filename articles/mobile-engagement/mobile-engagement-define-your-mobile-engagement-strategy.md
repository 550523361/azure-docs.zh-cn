# <a name="define-your-mobile-engagement-strategy"></a>定义你的 Mobile Engagement 策略
*你编写你的应用程序的原因： 让用户使用它 ！*

我们认为可以肯定将大量精力放在尝试更加出色的应用，用户将喜爱。 你还可能投入大量相当大的市场营销预算获取用户。 但之后的用户的初始 exhilarating 峰值，可能会看到它们渐变停止使用你的应用程序。 *这是有关哪些 Azure Mobile Engagement ！*： 让用户逗留并且使您能够以增量方式改进你的应用程序通过测试并了解。

我们用于提高保持和使用情况方法基于具有吸引力的应用程序用户到推送通知和应用内消息，但在非常特殊的方式，具有消息和定制，每个根据在你的应用程序中的行为的通信。 我们的目标是让你在正确的时间和适当的位置与正确的受众进行通信。

但为此，你必须首先*了解你的用户*，然后创建基于在相同或 （我们称之为段） 及其特性的组，然后创建相关的通信，向每个段。

## <a name="mobile-engagement-serves-your-objectives"></a>Mobile Engagement 提供你的目标
*我们讨论有关保留和使用情况，但对于？*

构建你的 Mobile Engagement 策略需要查找第一个在你的应用的目标和关键绩效指标 (Kpi)。

首先定义的目标和帮助定义你参与用例使用正确的 prism 的 Kpi。

你用例是你想要使与你的用户，范围从简单的欢迎，到由你的 IT 系统触发高级非常实用程序通知进行通信的营销活动的简单列表。 正确构造的用例必须包含至少三个*模拟的用户-在*:

1. 一个很短名称 （例如，"欢迎使用市场活动"）。
2. **什么**: （例如，"高兴载入让您消息示例 ！ 请记住登录以获取可用后第一个月 ！"）。 此消息并不意味着最终，你将能够将其更改您喜欢，任何时间，但它通常有助于开始考虑我们想要显示的内容。
3. **谁**： 将接收此消息 （例如，"3 天前，在第一次启动应用程序的所有用户访问了登录页但不是登录后"） 的段。
   * 是，你可以这样做很容易与 Azure Mobile Engagement:)
   * 同样，这不必是最终的因为你可以在任何时候，定义你段，但务必要提前定义分段条件，确保你收集适当的数据。
4. **当**： 活动的计时。 它可能在给定的日期，或在特定的操作后，基于触发器。 Mobile Engagement 提供了重要的可能性来将您的通信的时间量。

用例和段定义后，它提供指导，定义必须在应用程序收集的数据。 这是角色的*"标记计划"*。 标记计划，可确保到开发人员指定数据集合。 因此，开发人员可以使用你可以将适当的数据与你的活动的正确设置嵌入 Mobile Engagement。 它也将运行测试，以确保集成正确，以及收集你需要非常重要。

根据相集成，发布应用程序后，你的市场营销人员将能够查看实时分析，请细分受众，然后开始发送智能针对推送通知，若要与中或从应用程序的最终用户沟通。

## <a name="why-do-you-need-analytics-to-engage"></a>为什么需要分析与合作？
如你此时可能会发现，进行广播的推送通知仅是不够的。 Mobile Engagement 的核心概念是帮助营销人员和开发人员与正确的最终用户在正确的时间和在正确的位置。 若要了解这些三个主要概念，这至关重要能够从你的应用程序中收集分析并使用它来细分受众。 此外，当行为段进行了补充数据从其他数据库或 CRM 或交叉通道对这也是更加强大。 Mobile Engagement 允许从任何位置检索数据并使用它以面向正确的受众。

要最上下文时具有吸引力的受众，可能至关重要掌握的最终用户行为，知道他们在实时状态。 数据收集允许营销人员真正关注可以播放用例并实现其移动用户参与策略策略目标很重要。 达到目标集之前也是的原因，最佳做法实际上不以分析但仅显示那些允许你专注于你想要了解和用例收集的任何内容和所有内容。 这是应用的好的方法，以启动、 try、 测试和了解如何使用解决方案和解决智能推送通知以及增加程序，以使其在成功情景级别保留期。

### <a name="how-to-create-a-great-tag-plan"></a>如何创建出色的标记计划？
标记计划必须类似于用户路径或一种提供所有必要的标记 （数据），必须收集能够了解用户行为和正确段用户群足够分析的应用程序的工作流的说明。 这不是技术的过程。 因此，营销人员都能够指定他们想要收集的数据基于其 Mobile Engagement 策略。

最小值是进行标记至少所有屏幕 (调用*活动*Mobile Engagement 中) 的应用程序。 这有助于确定用户路径。

活动可以嵌入*事件*收集操作信息，如单击的按钮。 这样的应用程序中的交互的集合。 因此，营销人员都能够知道屏幕用户访问的新增功能和他们执行的操作。

`Jobs`是一个持续时间的操作。 这是非常有用的市场营销人员以了解需要多少时间为用户创建帐户，或对实例登录。 这还可能要监视它所需的时间调用 web 服务的开发人员有用。

`Errors`也可以监视要知道用户在应用程序中遇到问题。 例如，获取频繁连接问题。

所有这种类型的数据可以使用参数来进行扩充 (*额外信息*Mobile Engagement 中) 允许您从应用程序中收集动态数据。 这是重要，才允许细化分段。 例如，营销人员可以分段用户根据它们已用完的内容类型。 内容的类型将是活动或事件的动态信息。

*应用程序信息*是数据，您可以确认应用程序或实时的用户的状态。 这也有助于进行分类受众基和快速将其作为目标。 例如，它可以使用 true/false 状态是否将用户登录，或其订阅到期日期。
