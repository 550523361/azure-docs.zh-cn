# <a name="service-request-sr-information"></a>服务请求 (SR) 信息
打开任意 Mobile Engagement 服务的服务请求时，提供以下信息：

## <a name="ids-applicable-identifiers-related-to-your-issue"></a>ID：问题相关的相应标识符：
* 应用 ID
* 市场活动 ID
* 设备 ID
* 用户 ID
* 用户名
* 应用信息标记

## <a name="errors-applicable-error-information"></a>错误：相应的错误信息
* API 的名称或发生问题的 UI 部分
* 收到的任何错误消息的文本
* [故障排除指南](http://go.microsoft.com/fwlink/?LinkId=524382)中已经执行的任何测试的结果

## <a name="code-applicable-coding-information"></a>代码：相应的编码信息
* SDK 版本和应用的平台（Android SDK 2.4.1 和 iOS 1.16.2 等）
* 生产应用的下载位置（或开发应用的 APK/TGZ 文件）
* “AndroidManifest.xml”和/或 Azure Mobile Engagement 相关的应用中的任何代码片段（适用于高级故障排除）
