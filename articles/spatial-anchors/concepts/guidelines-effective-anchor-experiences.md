# <a name="create-an-effective-anchor-experience-by-using-azure-spatial-anchors"></a>使用 Azure 空间锚点创建有效的锚点体验

本文提供了指南和注意事项，以帮助您使用空间锚点有效地创建和定位锚点。

## <a name="good-anchors"></a>良好的锚点

空间锚点可帮助您创建良好的锚点。 花时间教育或指导用户体验 （UX） 中的用户以创建好锚点非常重要。 通过投资预先创建良好的锚点，您可以帮助最终用户可靠地找到锚点：

- 跨不同的设备。
- 在不同的时候。
- 在不同的照明条件下。
- 从空间内所需的角度。

## <a name="static-and-dynamic-locations"></a>静态和动态位置

设计锚点体验的一部分是选择位置。 位置是静态的，由空间的管理员定义吗？ 或者它们是动态的，由用户定义？

零售商店经理可能希望有静态店内体验来吸引用户访问。 但是，混合现实棋盘游戏的开发者可能希望让用户选择在哪里玩。

对于静态位置，您可以教管理员花时间使用良好的锚点来整理空间。

对于动态位置，应考虑如何在 UX 中教导或指导用户创建良好的锚点。

## <a name="stable-visual-features"></a>稳定的视觉功能

混合现实和增强现实设备中使用的视觉跟踪系统依赖于环境的视觉功能。 要获得最可靠的体验：

- *一些在*具有稳定视觉特征的位置（即不经常更改的要素）中创建锚点。

- *不要在*没有显著特征的大空白曲面上创建锚点。

- *不要在*高反射材料上创建锚。

- *不要在*图案重复的表面上创建锚点，例如地毯或墙纸。

## <a name="various-viewing-perspectives"></a>各种视角

创建锚点时，考虑稍后会尝试定位锚点的人员。

例如，考虑在有两扇门的房间中间的锚。 您可能希望允许用户从任一门进入房间。 创建锚点时，您需要从两个门口扫描其位置。 更改透视以捕获锚点周围的环境数据，以便用户可以从任一门找到锚点。

通常，在创建锚点时，从尝试定位锚点的人的角度进行扫描。 因此，如果您将虚拟内容放在户外雕塑上，那么在创作锚时，在雕塑周围走动是有意义的。 如果你的锚在房间的角落里，只有一个方向接近它。 创建此锚点时，只需从此角度进行扫描即可。

## <a name="multiple-anchors"></a>多个锚点

照明可以改变应用检测到的视觉功能。 在强自然光下形成的锚可能很难在人造光中定位，反之亦然。

如果发生此问题，则创建两个锚点会有所帮助。 在同一地点，在白天创建一个锚，在人造光下创建另一个锚。 然后，你的应用可以查询两个锚点。 当任一锚点位于时，应用将具有锚点的姿势。

同样，在视觉特征由于大多数对象移动而发生变化的环境中，多个锚点可以提供帮助。 当锚点由于环境发生重大变化而变得难以找到时，可以使用新锚点替换锚点。 例如，在零售商店中，布局每隔几个月刷新一次，则可以执行此操作。

## <a name="targets-and-rooms"></a>目标和房间

在许多情况下，锚点是应用体验的入口点。 您需要快速可靠地完成此步骤，以便用户可以输入您的体验。 花时间在用户如何找到锚点上是一个重要的设计步骤。 从两个广泛的场景中考虑寻找锚点是很有帮助的：*目标和**房间*。

### <a name="targets"></a>目标

在目标方案中，锚点的位置是众所周知的。 例如，在虚构的混合现实绘画应用中，一个用户将虚拟画布放在墙上。 她指示房间中的其他用户将设备指向墙上的同一位置，以定位锚点并开始体验。

目标场景的另一个示例可能是咖啡店中的一个标志，上面写着"扫描交易"。 咖啡店在这里放了一个锚。 当用户扫描标志时，他们找到锚并进入增强现实体验，以查找咖啡上的优惠。

在目标方案中，照片可以提供帮助。 如果向用户显示其设备上的预期目标的照片，他们可以快速识别要在现实世界中扫描的内容。 例如，您可以使用 GPS 帮助用户到达目标一般区域。 当用户到达时，你的应用会显示目标的照片。 用户环顾空间，查找目标并扫描锚点。

![锚的插图，在用户的移动设备上显示目标的照片](./media/start-here-edit.png)

### <a name="rooms"></a>聊天室

在房间场景中，用户只需知道某处有锚点即可进入空间。 用户使用设备扫描空间并快速找到锚点。

这种体验通常需要创建精心绘制的锚点，如各种查看角度中所述。 如果在创建锚点时从多个角度扫描房间，则用户在尝试定位该区域时几乎可以在任何地方进行扫描。

![用户如何扫描房间以查找锚点的说明](./media/scan-room.png)

从本质上讲，在创建锚点时，您花费更多时间扫描空间，以便以后用户可以快速扫描和定位锚点。 在创建体验时，您需要考虑这一重要的权衡。

我们前面讨论的混合现实绘画应用程序的示例不能很好地作为房间场景。 在这里，放置锚点的用户希望其他人快速加入体验。 用户不想等到房间扫描良好之前才能开始体验。 由于所有用户都确切知道将设备指向何处以定位锚点，因此此示例作为目标方案效果更好。

## <a name="anchor-location"></a>锚点位置

可视化跟踪系统依赖于环境中的可视功能。 扫描包含的可视功能越多，找到锚点的可能性就越大。

按照本节中的一般准则构建一个 UX，鼓励对环境进行有用的扫描。

首先，如果用户在几秒钟内找不到锚点，应用应鼓励用户移动设备以捕获更多透视图。 该应用程序还可以鼓励用户在环境中移动自己，从更多角度扫描锚点。 设备看到的要素透视越多越好。

对于目标方案，请要求用户在目标周围移动，从不同的角度查看目标。 换句话说，要求用户从新的角度捕获目标，直到定位锚点。

对于房间方案，请要求用户慢慢扫描房间。 例如，要求用户转向捕捉 180 度甚至 360 度的房间。 或者要求用户从新的角度查看房间。

最有意义的方法是扫描整个房间。 例如，整个房间的扫描可以捕获比附近墙壁扫描更多的环境视觉特征。 对附近墙壁的扫描无法捕获环境的许多有用的视觉特征。

在查找锚点时，反复移动设备是没有帮助的。 这只需从相同的角度捕获相同的点。

## <a name="experience-tests"></a>体验测试

在本文中，我们讨论了一般准则。 使用空间锚点，您正在编写与现实世界交互的应用。 因此，您应该花时间在实际环境中测试应用的锚定方案。 对于表示希望用户使用应用的环境尤其如此。
