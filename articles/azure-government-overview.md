# <a name="welcome-to-azure-government"></a>欢迎使用 Azure 政府版
## <a name="overview"></a>概述
Microsoft Azure 政府版是一个基于[安全性、隐私性与控制性、合规性和透明性基本原则](http://azure.com/gov)而构建的云平台。 公共部门实体将会收到物理隔离的 Microsoft Azure 实例，其中包含对于美国政府构建在其体系结构基础之上的所有系统和应用程序至关重要的世界级安全性与[合规性服务](https://azure.microsoft.com/support/trust-center/compliance/)。 这些服务包括 FedRAMP 和 DoD 合规性认证、CJIS 国家级协议、颁发 HIPAA 业务合作协议的能力，且支持 IRS 1075。 Azure 政府版由甄选出来的美国人运作，支持在本地或云中构建和部署解决方案的多个混合方案。 公共部门实体还可利用超大规模云服务的即时可伸缩性和保证可用性。

Azure 政府版包括服务架构 (IaaS)、平台即服务 (PaaS) 和服务型软件 (SaaS) 的核心组件。  这包括基础结构、网络、存储、数据管理、身份管理和其他许多服务。

Azure 政府版支持的功能与公共 Azure 客户利用的大多数功能相同，如地区同步数据复制和自动调整。 

* 请参阅[区域页](https://azure.microsoft.com/regions/#services)，了解正式发布版的最新服务。
* [注册试用版](https://azuregov.microsoft.com/trial/azuregovtrial)

希望使用云服务的美国政府组织可以确信的是，Azure 政府版能严格执行大规模安全操作，满足其不断发展变化的需求。

## <a name="azure-government-documentation"></a>Azure Government 文档
此网站描述了 [Microsoft Azure Government](https://azure.microsoft.com/features/gov/) 服务的功能，并提供了适用于所有客户的一般性指南。 在 Azure Government 订阅中包含特定管制的数据之前，应熟悉 Azure Government 功能，并在有任何疑问时咨询你的客户团队。

你应参考 [Microsoft Azure 信任中心法规遵从页](http://www.microsoft.com/en-us/TrustCenter/Compliance/default.aspx)，了解在特定合格证和法规涵盖下关于 Azure Government 服务的当前信息。 虽然可能也提供了其他 Microsoft 服务，但不在 Azure Government 涵盖服务的范围内，也不在本文讨论范围内。 通过 Azure 政府版服务还可使用其他不同的资源、应用程序或由第三方提供的服务，或由 Microsoft 根据单独的使用条款和隐私策略提供的服务，但这些不在本文讨论范围内。 你有责任审阅所有此类“外接”产品（如应用商店中的产品/服务）的条款，以确保它们满足你在法规遵从方面的需要。

Azure 政府版适用于受特定政府法规和要求（如 NIST 800.171 (DIB)、ITAR、IRS 1075、DoD L4 和 CJIS）约束处理数据的实体。这些实体在使用 Azure 政府版时需要遵守这些法规。 Azure Government 客户必须接受资格验证。

对 Azure Government 的资格验证有疑问的实体应咨询其客户团队。

## <a name="general-guidance-for-customers"></a>客户一般指南
由于大部分目前可用的技术内容假定应用程序正在为全局服务而开发，而不是 Microsoft Azure Government，务必要确保开发人员意识到开发应用程序并将其托管在 Azure Government 中的主要区别。

* 首先，存在服务和功能上的差异，这意味着全局服务的特定区域中的某些功能可能在 Azure Government 中不可用。
* 其次，Azure Government 中提供的功能，与全局服务的配置不同。  因此，应检查你的示例代码、配置和步骤，确保是在 Azure 政府版云服务环境中进行构建和执行。
* 第三，应参考从该站点提供的 Azure Government 技术服务文档，了解有关标识 Azure Government 边界和客户管制/控制的数据指导和最佳实践的信息。
