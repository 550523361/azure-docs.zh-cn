# <a name="azure-data-catalog-common-scenarios"></a>Azure 数据目录常见方案
本文将介绍常见方案，通过常见方案 Azure 数据目录可帮助组织从现有数据源获取更多价值。

## <a name="scenario-1---registration-of-central-data-sources"></a>方案 1 – 中央数据源的注册
组织通常具有多个高价值数据源。 这些数据源包括业务 OLTP 系统线、数据仓库和商业智能/分析数据库。 通常随着业务需求的发展和通过合并和收购企业本身的发展，系统的数量和系统之间的重叠部分会增长。

通常用户很难知道如何在这些数据源内查找数据。 以下问题非常常见：

* 我应该使用公司中三个 HR 系统的哪一个系统来创建此类型的报表？
* 在何处获取刚结束的会计年度的已认证销售数据？
* 我应该询问谁或我应该使用什么流程访问数据仓库？
* 我不知道这些数字是否正确 – 在我将此仪表板共享给我的团队之前应该向谁询问如何使用该数据？

在此方案中，Azure 数据目录可提供帮助。 在组织中使用的中央、高价值和 IT 管理的数据源通常为填充目录的逻辑起点。 虽然所有用户都可注册数据源，但使用非常可能为最大数量的用户提供价值的数据源将目录启动会帮助采用和使用该系统。 对于开始使用 Azure 数据目录的客户，标识和注册由许多不同的数据使用者团队使用的关键数据源是成功的第一步。

此方案还会提供机会批注高价值数据源，让其更易理解和访问。 这项工作的一个关键方面是包括有关用户如何请求访问数据源的信息。 Azure 数据目录可让用户提供负责控制数据源访问的用户或团队的电子邮件地址、现有工作或文档的链接或说明访问请求流程的自定义文本。 将此信息包含在目录中可帮助发现已注册数据源但没有访问权限的用户使用数据源所有者定义和控制的流程请求访问权限。

## <a name="scenario-2---self-service-business-intelligence"></a>方案 2 - 自助式商业智能
尽管传统公司商业智能解决方案将继续作为许多组织数据布局的重要组成部分，但商业节奏的改变让自助式 BI 越来越重要。 自助式 BI 允许信息辅助角色和分析人员创建自己的报表、工作簿和仪表板，而不依赖于中央 IT 团队，也不会因 IT 团队的时间安排而受到限制。

在自助式 BI 方案中，用户经常将多个数据源中的数据进行结合，其中很多数据可能之前未用于 BI 和分析。 虽然可能已知某些数据源。但通常仍存在发现和评估给定任务的潜在数据源的流程。

根据惯例，此发现流程为手动流程：分析人员会使用其对等网络连接来标识处理已找到数据源的其他人员。 找到数据源后即可使用它，但该流程会对每个后续自助式 BI 工作自动重复一次，多个用户执行同一手动发现流程。

使用 Azure 数据目录，用户可打破此冗余的循环工作。 通过传统方式发现数据源后，分析人员可注册数据源，让其以后更易发现。 虽然用户可通过批注已注册数据资产添加更多价值，但不需要在注册时进行批注。 用户可进行长期贡献，因为他们的计划允许逐渐将值添加到注册在目录中的数据源。

目录内容的自然增长是对中央数据源的预先注册进行的自然补充。 使用许多用户将需要的数据预填充目录可称为初次使用和发现的促进因素。 允许用户注册和批注其他数据源可能会成为留住用户及其对等方的方法。

此外值得注意的是，虽然此方案特别注重自助式 BI，但相同的模式和难题也适用于大规模企业的 BI 项目。 数据源发现的手动流程可通过使用 Azure 数据目录将值添加到组织。

## <a name="scenario-3---capturing-tribal-knowledge"></a>方案 3-捕获部落知识
如何知道完成作业所需的数据以及在哪里找到该数据？

进行一段时间作业后，可能就会知道。 你已经历了逐步学习过程，并了解对你的日常工作重要的数据源。

新员工加入你的团队时，如何知道完成作业所需的数据以及在哪里找到该数据？

他可能会询问你。

部落知识的传递也是组织（或大或小）中数据源发现流程的一部分。 有经验的团队成员已建立起多年的知识，新来的团队成员在有问题的时候就会询问他们。 通常仅少数关键人员掌握最重要的信息，那么当他们度假或离开团队时，则组织会受到影响。

有时这些数据专家会努力记录他们的知识、将其通过电子邮件或在团队 SharePoint 站点上通过 Word 文档进行分享。 尽管这可能有价值，但它带入了新的发现问题 – 大家如何知道存在什么文档以及在哪里找到它们…

Azure 数据目录提供了用于共享部落知识的位置，让其易于发现。 数据专家可直接批注数据资产，还可包括现有文档的链接。 不仅会捕获知识本身，还会将知识放入用于数据源发现的同一经验中。 某人使用目录发现数据源时，不仅会找到数据源本身，还会找到以前只存在于专家头脑里的专家知识。
