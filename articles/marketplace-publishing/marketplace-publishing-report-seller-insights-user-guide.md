# <a name="how-to-use-the-azure-marketplace-seller-insights-portal"></a>如何使用 Microsoft Azure Marketplace 卖方 Insights 门户

本文介绍管理员如何访问卖方 Insights 门户，添加或删除用户以及设置用户的权限。

## <a name="onboard-users"></a>载入用户

本部分讨论管理员如何首次访问卖方 Insights 门户。 此部分中的任务适用于管理员。 


### <a name="overview-of-roles"></a>角色概述

卖方 Insights 门户管理员可以添加、编辑和删除用户。

卖方 Insights 门户用户可以查看提供给组织的所有数据。

管理员和用户需要使用有效的 [Microsoft 帐户](http://signup.live.com)或组织的工作或学校帐户进行身份验证。 请确保新帐户 ID 与受监视的邮箱相关联。

![帐户的类型][5]



### <a name="invitation-to-activate-your-account"></a>邀请激活帐户

在登录到卖方 Insights 门户之前，管理员应识别他们要设置为用户的个人的帐户。 通过工作或学校帐户或者 Microsoft 帐户设置用户。

为了激活服务，管理员应转到邀请电子邮件中所显示的 [URL 地址](https://reports.azure.com)。

>[!NOTE]
>如果管理员电子邮件列为 Microsoft 帐户，并且你尚未创建与从中接收邀请的电子邮件地址关联的 Microsoft 帐户，则需要在登录之前[创建该帐户](https://signup.live.com)。


### <a name="administrator-sign-in-to-the-seller-insights-portal"></a>管理员登录到卖方 Insights 门户

1. 单击发送给你的电子邮件中的 URL 登录到卖方 Insights 门户。
2. 输入管理员电子邮件地址和密码。
3. 单击“登录”按钮。
4. 开始管理帐户。 添加用户，设置权限，并编辑用户。
5. 确保用户已获得卖方 Insights 门户的 URL。

    ![登录页][6]

## <a name="user-sign-in-to-the-portal"></a>用户登录到门户

管理员应通过电子邮件通知你帐户已创建，并与你共享卖方 Insights 门户的 [URL 地址](https://reports.azure.com)。

>[!NOTE]
>管理员是将解决方案发布到 Microsoft Azure Marketplace 的帐户所有者。 此帐户的所有者可向其他人提供访问权限。  如果有权以协同管理员身份发布门户，则可以按照以下步骤操作，查找管理员地址：
>1. 登录到发布门户。
>2. 单击“发布者”选项卡。
>3. 打开相关发布者（通常只存在一个）。
>4. 单击左下部的“管理员”。
>5. 要寻找的管理员电子邮件地址标记为“所有者”。


可以使用个人 Microsoft 帐户或使用工作或学校帐户登录到卖方 Insights 门户。 使用哪种帐户取决于管理员在个人资料中设置的帐户。 管理员应建议你要使用哪个帐户。

![帐户的类型][1]



欢迎使用卖方 Insights 门户！

1. 登录后，登录到“摘要”页。
2. 请花几分钟时间阅读隐私策略。
3. 可以通过电子邮件随时联系我们，邮件中附上可能遇到的与卖方 Insights 门户相关的问题。
4. 可以使用页面顶部的标签页导航卖方 Insights 门户。  在本指南中找到页面顶部的标签页，有助于熟悉卖方 Insights 门户。
5. 可随时单击笑脸发送与卖方 Insights 门户相关的反馈。
6. 若要从卖方 Insights 门户中注销，单击右上角的名称。

    ![默认登陆页][2]


## <a name="summary-view"></a>摘要视图


1. 选择“摘要”选项卡。
2. 在此处输入所需时间段的“开始日期”，并单击“应用”。 会显示三个月的数据。
3. 此处显示所选三个月时间段的新订单趋势数据。
4. 此处将按“渠道”和“产品类型”显示所选时间段的新订单。
5. 在此面板中选择一个视图。 “排名靠前的 5 个市场”、“表现最佳的 5 款产品”或“排名靠前的 5 家客户”。
6. 此处会显示付费产品在所选时间段内的布局趋势。 如果只有 BYOL 或第一方产品，则不会显示此面板。

    ![摘要视图][7]

## <a name="orders-and-usage"></a>订单和用途
### <a name="overview"></a>概述

在“面板 A”中，可以选择“订单和用途”选项卡。在此处可以从“报告类型”中选择“订单”或“用途”，并使用滑块选择所需的数据。

在“面板 B”中，可以选择每月数据。 根据在面板 A 的“报告类型”中选择的内容选择“订单”或“用途”。显示的数据将反映所输入的日期，以及在面板 A 中使用滑块应用的筛选器。

在“面板 C”中，可以选择三个月的趋势数据。 根据在面板 A 的“报告类型”中选择的内容选择“订单”或“用途”。显示的数据反映所输入的日期，以及在面板 A 中使用滑块应用的筛选器。

“面板 D”列出各个订阅相关的数据。  可从此面板下载 Excel 或 CSV 格式的数据。

## <a name="user-management"></a>用户管理

此部分中的任务适用于管理员。 使用以下过程添加用户，更改权限，或从卖方 Insights 门户中删除用户。


### <a name="add-user"></a>添加用户

本节演示如何在卖方 Insights 门户中添加新用户或更改其权限。


1. 选择“用户管理”选项卡。
2. 若要添加用户，请选择“添加用户”。
3. 输入与其 Microsoft 帐户或者其工作或学校帐户关联的电子邮件地址。 然后单击“继续”。
4. 管理员可以向用户分配只读的报告访问权限。
5. 输入用户详细信息 -“名字”、“姓氏”和“电子邮件”，并单击“继续”。
6. 选择要向用户分配的权限，并单击“继续”。
7. 确保输入正确的信息，并单击“保存”继续。  现已成功添加用户。

  ![添加用户面板][29]


### <a name="edit-user"></a>编辑用户

1. 选择“用户管理”选项卡，并选择“编辑用户”。
2. 找到要修改其权限的用户。  也可以添加新用户。 单击“添加用户”。
3. 单击要对该用户删除的权限。  也可以删除用户。 单击“删除用户”。
4. 单击“保存”继续。  现已成功编辑该用户的权限。

  ![编辑用户面板][30]

## <a name="help-support-requests"></a>技术支持请求


### <a name="access-the-support-request-format"></a>访问支持请求格式

1. 选择“帮助”选项卡。
2. 选择“支持和文档”，并选择“开始请求”。
3. 也可以从“联系我们”链接提交支持请求。 此链接会出现在应用程序中每一页的底部。
4. 随后将打开一个新选项卡，可在其中提交支持请求。

  ![访问支持表单][31]

### <a name="review-helpful-information"></a>查看有用的信息

1. 若要提交报告相关的支持请求，请从“问题类型”中选择“报告和见解”。
2. 选择一个类别。  选项包括“与报告相关的问题”和“其他”。
3. 随后会显示一些有用的链接，其中可能解答了你咨询的问题。  提交支持请求之前，请先查看这些链接。
4. 如果仍需提交支持请求，请选择“开始请求”。

  ![编辑用户面板][32]


### <a name="enter-information-and-submit-the-request"></a>输入信息并提交请求

1. 输入联系信息、时区、区域和语言首选项，并选择“继续”。
2. 为事件指定一个标题，并提供尽可能多的信息。  还可以上传包含屏幕截图等参考信息的支持文件，并选择“提交”。
3. 随后会收到一条确认消息，指出该支持请求已成功提交。
4. 随时可以查看支持请求的状态。

  ![访问支持表单][33]


## <a name="reset-a-password"></a>重置密码

**帐户和密码验证**

1. *验证所用的帐户是否已注册、密码是否有效。* 如果输入的帐户已在 Azure Active Directory 中注册，则系统会自动提示选择如何使用该帐户登录。 使用工作或学校帐户，或用作个人帐户的 Microsoft 帐户。
2. 转到 [https://reports.azure.com](https://reports.azure.com)，并从登录所用的任何帐户中注销。 也可以在专用窗口中打开该站点以确保没有其他帐户已登录。
3. 输入发布者电子邮件地址，但不要输入密码。 如果已在 Azure Active Directory 中注册该地址，它将重定向到步骤 4。

    >[!NOTE]
    >将注册管理员帐户。 此外，还会注册管理员帐户设置的工作或学校帐户。

4. 选择“工作或学校帐户”。 工作或学校帐户由已注册到 Azure 的公司设置。 

    >[!NOTE]
    >用户可能已使用其组织的电子邮件地址设置了 Microsoft 帐户。 在这种情况下，组织以后会加入 Azure，同一电子邮件地址将注册为工作或学校帐户。 此用户可以有一个 Microsoft 帐户，并使用创建工作或学校帐户时所用的同一密码。 此用户还可以有一个工作或学校帐户以及由其组织使用的单独密码。

5. 在卖方 Insights 门户中，选择“工作或学校帐户”时请确保使用工作或学校帐户密码，而不是 Microsoft 帐户密码。
6. 工作或学校帐户会将用户重定向到发布服务器的页面，进行登录。
7. 在此处验证工作或学校帐户的密码是否有效。 如果密码无效，请转到本文的“密码重置”部分。



**密码重置**

使用以下 Microsoft 和工作或学校帐户重置密码。

>[!NOTE]
>对于管理员帐户设置的工作或学校帐户，如果管理员启用了让你重置自己的密码的功能，你便可以重置自己的密码。 否则，密码重置表单将提供用于联系管理员的选项。

* [Microsoft 帐户](https://account.live.com/ResetPassword.aspx)
![Microsoft 帐户的密码重置][3]

* [工作或学校帐户](https://passwordreset.microsoftonline.com/)
![工作或学校帐户的密码重置][4]
