# <a name="how-to-use-the-microsoft-azure-marketplace-seller-insights-portal"></a>如何使用 Microsoft Azure 应用商店卖方 Insights 门户
##<a name="onboarding"></a>登记

>[!NOTE]
>仅适用于管理员

本部分演示如何首次访问卖方 Insights 门户。

###<a name="overview-of-roles"></a>角色概述

![帐户的类型][5]

卖方 Insights 门户管理员可以添加、编辑和删除用户。

卖方 Insights 门户用户可以查看提供给你的组织的所有数据。

管理员和用户需要使用有效的 Microsoft 帐户 (LiveID http://signup.live.com) 或公司的帐户进行身份验证。 请确保输入的 ID 与受监视的邮箱相关联。

###<a name="invitation-to-activate-your-account"></a>邀请激活你的帐户

理想情况下，在登录到卖方 Insights 门户之前，管理员应识别他们要设置为用户的个人的帐户。 帐户可以是 Microsoft 帐户或公司帐户。

为了激活你的服务，管理员应导航到邀请电子邮件中所列的 URL (https://reports.azure.com)。

>[!NOTE]
>如果管理员电子邮件列为 Microsoft 帐户，并且你尚未创建与从中接收邀请的电子邮件地址关联的 Microsoft 帐户，则在创建 Microsoft 帐户时，需要通过导航到 https://signup.live.command 进行登录，然后利用此电子邮件地址创建该帐户。

###<a name="sign-in-to-the-portal"></a>登录到门户

1. 单击发送给你的电子邮件中的 URL 登录到卖方 Insights 门户。
2. 输入管理员电子邮件地址和密码。
3. 单击“登录”按钮。
4. 开始管理帐户 – 添加用户、设置权限和编辑用户。
5. 请确保为用户提供门户的 URL。

![登录页][6]

##<a name="signing-in"></a>登录

管理员应通过电子邮件通知你帐户已创建，并与你共享卖方 Insights 门户的 URL (https://reports.azure.com)。

>[!NOTE]
>管理员是实际将解决方案发布到 Azure 应用商店的帐户所有者。  此帐户的所有者可向其他人提供访问权限。  如果有权以协同管理员身份发布门户，则可以按照以下步骤来查找管理员地址：
>1. 登录到发布门户
>2. 选择“发布者”选项卡
>3. 打开相关发布者（通常只存在一个）
>4. 单击左下部的“管理员”
>5. 要寻找的管理员电子邮件地址标记为“所有者”。


可以使用个人 Microsoft 帐户或使用公司帐户登录到卖方 Insights 门户，具体取决于管理员在设置你的个人资料时使用了哪个帐户。  管理员应建议你要使用哪个帐户。

![帐户的类型][1]



***欢迎使用卖方 Insights 门户！***

1. 登录后，你将登录到“摘要”页。
2. 请花几分钟时间阅读隐私策略。
3. 可以通过电子邮件随时“联系我们”，邮件中附上你可能遇到的与卖方 Insights 门户相关的问题。
4. 可以使用页面顶部的标签页导航卖方 Insights 门户。  在本指南中找到页面顶部的标签页可帮助你熟悉卖方 Insights 门户。
5. 可随时单击笑脸发送与卖方 Insights 门户相关的反馈。
6. 可随时单击右上角的名称从卖方 Insights 门户中注销。

![默认登陆页][2]


##<a name="summary-view"></a>摘要视图

1. 选择“摘要”选项卡。
2. 在此处输入所需时间段的开始日期，然后单击“应用”。 将显示 3 个月的数据。
3. 此处显示所选 3 个月时间段的新订单趋势数据。
4. 此处将按“渠道”和“产品类型”显示所选时间段的新订单。
5. 从此面板中选择“排名靠前的 5 个市场”、“表现最佳的 5 款产品”或“排名靠前的 5 家客户”视图。
6. 此处将显示付费产品在所选时间段内的布局趋势。 如果你只有 BYOL 或第一方产品，则不会显示此面板。

![摘要视图][7]

##<a name="orders-and-usage"></a>订单和用途
###<a name="overview"></a>概述

A. 在“面板 A”中，可以选择“订单和用途”选项卡。  在此处可以从“报告类型”选项卡中选择“订单”或“用途”，然后使用滑块选择所需的数据。

B. 在“面板 B”中，可以选择每月数据 -“订单”或“用途”，具体取决于在面板 A 的“报告类型”中选择的内容。显示的数据将反映所输入的日期，以及在面板 A 中使用滑块应用的筛选器。

C. 在“面板 C”中，可以选择 3 个月的趋势数据 -“订单”或“用途”，具体取决于在面板 A 的“报告类型”中选择的内容。显示的数据将反映所输入的日期，以及在面板 A 中使用滑块应用的筛选器。

D. *面板 D*：此面板提供各个订阅相关的数据。  可从此面板下载 Excel 或 csv 格式的数据。

##<a name="user-management"></a>用户管理

>[!NOTE]
>仅适用于管理员

本部分演示如何在卖方 Insights 门户中添加新用户或更改其权限。

###<a name="add-user"></a>添加用户

1. 选择“用户管理”选项卡。
2. 若要添加用户，请导航到“用户管理”，然后单击“添加用户”。
3. 若要添加用户，请输入与其 Microsoft 帐户或公司帐户关联的电子邮件地址，然后单击“继续”。
4. 管理员可以向用户分配只读的报告访问权限。
5. 输入用户详细信息 - 名字、姓氏和电子邮件，然后单击“继续”。
6. 单击要向用户分配的权限，然后单击“继续”。
7. 检查输入的信息，然后单击“保存”继续。  现已成功添加用户。

![添加用户面板][29]


###<a name="edit-user"></a>编辑用户

1. 选择“用户管理”选项卡，然后单击“编辑用户”。
2. 找到要修改其权限的用户。  请注意，也可以在此处添加用户。
3. 单击要对该用户删除的权限。  请注意，也可以在此处删除用户。
4. 单击“保存”继续。  现已成功编辑该用户的权限。

![编辑用户面板][30]

##<a name="help"></a>帮助

##<a name="support-requests"></a>支持请求


###<a name="access-the-support-request-format"></a>访问支持请求格式

1. 选择“帮助”选项卡。
2. 选择“支持和文档”及“开始请求”。
3. 也可以通过应用程序中每个屏幕底部的“联系我们”链接提交支持请求。
4. 随后将打开一个新选项卡，可在其中提交支持请求。

![访问支持表单][31]

###<a name="review-helpful-information"></a>查看有用的信息

1. 若要提交报告相关的支持请求，请从“问题类型”中选择“报告和见解”。
2. 选择一个类别。  选项包括“与报告相关的问题”和“其他”。
3. 随后将显示一些有用的链接，其中可能解答了你咨询的问题。  提交支持请求之前，请先查看这些链接。
4. 如果仍需提交支持请求，请选择“开始请求”。

![编辑用户面板][32]


###<a name="fill-out-and-submit-the-request"></a>填写并提交请求

1. 输入你的联系信息、时区、区域和语言首选项，然后选择“继续”。
2. 为事件指定一个标题，并提供尽可能多的信息。  还可以上载包含屏幕截图等参考信息的支持文件，然后选择“提交”。
3. 随后你将收到一条确认消息，指出该支持请求已成功提交。
4. 你随时可以查看支持请求的状态。

![访问支持表单][33]


##<a name="resetting-a-password"></a>重置密码

***帐户和密码验证***

1. ***验证所用的帐户是否已注册、密码是否有效。*** 如果输入的帐户已在 Azure Active Directory 中注册，则系统将自动提示你选择使用帐户登录的方式 - 工作或学校（组织帐户）或个人 LiveID 帐户。
2. 导航到 https://reports.azure.com，并从登录所用的任何帐户中注销，或者在专用窗口中打开该站点以确保没有其他帐户已登录。
3. 输入发布者电子邮件地址，而无需输入密码。 如果已在 Azure Active Directory 中注册该地址，它将重定向到***步骤 4***（将注册管理员帐户，因为任何组织帐户均由管理员帐户设置）。
4. 选择“工作或学校帐户”。 工作或学校帐户（简称“组织帐户”）由已注册到 Azure 的公司设置；个人帐户（简称“LiveID”）帐户是已注册到 Microsoft 的个人帐户。
 * 请注意，可能用户以前已使用其公司电子邮件地址设置 LiveID 帐户，稍后该公司加入到 Azure，同一电子邮件地址已注册为组织帐户。 这意味着，此用户可以有一个 LiveID 帐户以及用于创建该 LiveID 的密码，并可以有一个公司帐户以及由公司使用的单独密码。
5. 在卖方 Insights 门户中，选择“工作或学校帐户”时请确保使用的是组织帐户/密码组合，而不是 LiveID/密码组合。
6. 对于组织帐户，这将重定向到“发布者登录”页。
7. 在此处，验证组织帐户的密码是否有效。 如果密码无效，请执行***密码重置***。

***密码重置***

尝试按照以下链接使用自助服务密码重置向导重置密码。

>[!NOTE]
>注意：对于管理员帐户设置的组织帐户，如果管理员启用了让你重置自己的密码的功能，你将能够重置自己的密码。 否则，密码重置表单将提供用于联系管理员的选项。

****Microsoft 帐户 (Live ID)：https://account.live.com/ResetPassword.aspx**** ![重置 Microsoft 帐户的密码][3]

****工作或学校帐户（组织 ID）：https://passwordreset.microsoftonline.com/**** ![重置工作或学校帐户的密码][4]
