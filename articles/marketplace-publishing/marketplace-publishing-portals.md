# <a name="portals-you-will-need"></a>你将需要的门户
在开始发布产品/服务的过程之前，让我们帮您引入到你将需要的各种门户。 下面是有关中的门户-开发人员中心、 Azure 发布门户和 Azure 门户-不将与其进行交互的顺序的简短摘要。                                                                            

## <a name="developer-center"></a>开发人员中心
[http://dev.windows.com/registration?accountprogram=azure](http://dev.windows.com/registration?accountprogram=azure)

### <a name="description"></a>描述
创建你的 Microsoft 开发人员中心帐户是一次性的任务。 请确保该公司还没有开发人员中心帐户然后再尝试创建一个。 在过程中，我们会收集银行帐户信息、 税务信息和公司地址信息。

> [!NOTE]
> 如果你要发布仅免费产品/服务 （或使你的拥有的许可证提供），我们不需要税务和 bank 的信息。
> 
> 

### <a name="identityaccount-used"></a>使用的标识/帐户
理想情况下，这是分发列表或安全组 (例如，@ azurepublishing*partnercompany*.com)。 分发列表或安全组**必须**注册为 Microsoft 帐户。

> [!TIP]
> 我们建议使用通讯组列表或安全组因为它不再依赖于任何单个，但也可以使用个人帐户。
> 
> 

## <a name="publishing-portal"></a>发布门户
[https://publish.windowsazure.com](https://publish.windowsazure.com)

### <a name="description"></a>描述
这是在门户中使用工作产品/服务，然后将发布 （市场营销、 定价，发布，证书颁发有可能的话，等等。）。

### <a name="identityaccount-used"></a>使用的标识/帐户
上述分发列表或安全组必须使用第一次登录到发布门户。 更高版本，其他用户可以添加为共同管理员。 这是获取映射到开发人员中心注册数据方式。

## <a name="azure-portal"></a>Azure 门户
[https://portal.azure.com](https://portal.azure.com)

### <a name="description"></a>描述
这是你可以在 Azure 应用商店 （适用于 Vm、 解决方案模板和基于 Azure 资源管理器的开发人员服务） 中查看分阶段和已发布产品/服务的门户。

### <a name="identityaccount-used"></a>使用的标识/帐户
正在暂存发布门户某项优惠，而订阅 ID 必须是白名单。 相同的订阅 （没有用户名和与之关联的密码） 需要使用用于登录到此门户来测试过渡产品/服务。
