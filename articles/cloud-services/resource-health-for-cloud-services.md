# <a name="resource-health-check-rhc-support-for-azure-cloud-services-classic"></a>资源运行状况检查 (RHC) 对 Azure 云服务的支持 (经典) 
本文介绍资源运行状况检查 (RHC) 支持 [Microsoft Azure 云服务 (经典) ](https://azure.microsoft.com/services/cloud-services)

云服务[Azure 资源运行状况](https://docs.microsoft.com/azure/service-health/resource-health-overview)可帮助你诊断和获取影响云服务部署、角色 & 角色实例的服务问题。 它报告云服务当前和过去的运行状况（在部署时）、角色 & 角色实例级别。

Azure 状态报告影响广泛的 Azure 客户的问题。 资源运行状况提供资源运行状况的个性化仪表板。 资源运行状况显示由于 Azure 服务问题导致资源不可用的所有时间。 这些数据可方便你了解是否违反 SLA。


## <a name="how-health-is-checked-and-reported"></a>如何检查和报告运行状况？
在部署或角色级别报告资源运行状况。 运行状况检查发生在角色实例级别，聚合状态并将其报告在角色级别。 例如 如果所有角色实例都可用，则角色状态为 "可用"。 同样，我们聚合了所有角色的运行状况状态，并将其报告为部署级别。 例如 如果所有角色都可用，则部署状态将变为可用。 

## <a name="why-i-cannot-see-health-status-for-my-staging-slot-deployment"></a>为什么我看不到过渡槽部署的运行状况状态？
资源运行状况检查仅适用于生产槽部署。 尚不支持过渡槽部署。 

## <a name="does-resource-health-check-also-check-the-health-of-the-application"></a>是否资源运行状况检查是否还检查应用程序的运行状况？
不会，只对角色实例进行运行状况检查，而不监视应用程序的运行状况。 例如 即使3个角色实例中有1个不正常，应用程序仍可以使用。 RHC 不使用 [负载均衡器探测](https://docs.microsoft.com/azure/load-balancer/load-balancer-custom-probe-overview) 或来宾代理探测。 因此，客户应继续使用负载均衡器探测来监视其应用程序的运行状况。 

## <a name="what-are-the-annotations-for-cloud-services"></a>云服务的批注有哪些？
批注是部署或角色的运行状况状态。 根据运行状态、状态更改原因等提供不同的注释。 

## <a name="what-does-it-mean-by-role-instance-being-unavailable"></a>角色实例是 "不可用" 这是什么意思？
这意味着角色实例不会将正常信号发出到平台。 请检查角色实例状态，以了解有关未发出正常信号的原因的详细说明。

## <a name="what-does-it-mean-by-deployment-being-unknown"></a>部署为 "未知" 是什么意思？
"未知" 表示无法确定云服务部署的聚合运行状况。 通常，这表示没有为云服务创建生产部署， (部署是新创建的，并且 Azure 将开始收集运行状况事件) 或平台收集此部署的运行状况事件时出现问题。

## <a name="why-does-role-instance-annotations-mentions-vms-instead-of-role-instances"></a>为什么角色实例批注提到了 Vm 而不是角色实例？
由于角色实例基本上是 Vm，因此会对角色实例重用 vm 的运行状况检查，而 VM 术语用于表示角色实例。

| 设置资源运行状况 | 正在为此资源设置资源运行状况。 资源运行状况监视 Azure 资源，以提供有关当前和过去的事件对其造成影响的详细信息|

| Unhealthy | 云服务部署不正常，因为 {0} {1} 角色实例不可用 |
| 已降级 | 云服务部署已降级，因为 {0} 角色中的 {1} 实例不可用 | 
| 可用且可能受到影响 | 你的云服务部署正在运行，但正在进行的 Azure 服务中断可能会阻止你连接到它。 解决中断后，将还原连接 |

| 未知且可能受到影响 | 我们目前无法确定此云服务部署的运行状况。 这可能是由于正在进行的 Azure 服务中断造成的，这可能会影响此虚拟机，这会在解决中断后自动恢复 |
