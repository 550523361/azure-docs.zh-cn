# <a name="classic-deployment-model"></a>经典部署模型

经典部署模型是较旧一代 Azure 部署模式 land 强制实施全局 vCPU 配额限制的虚拟机和虚拟机规模集。 经典部署模型不不再建议，并且现在已被取代通过 Resource Manager 模型。 若要详细了解这些两种部署模型和资源管理器的优点，请参阅到 Resource Manager 部署模型页。 创建新订阅时，为其分配的默认配额为 Vcpu。 每当新的 VM 是使用经典部署模型部署，在所有区域内的新的和现有 Vcpu 使用率总和不能超过批准适用于经典部署模型的 vCPU 配额。 了解有关 Azure 订阅和服务限制页面上的配额的详细信息。

你可以请求增加 Vcpu 的经典部署模型通过帮助 + 支持边栏选项卡或使用情况 + 配额的限制的门户边栏选项卡。
