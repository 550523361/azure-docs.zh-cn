# <a name="edit-metadata-module"></a>编辑元数据模块

本指南介绍了 Azure 机器学习服务的可视界面 （预览版） 的模块。

使用编辑数据模块更改与数据集中的列相关联的元数据。 使用编辑元数据模块后，将数据集的值和数据类型会发生变化。

典型的元数据更改可能包括：
  
+ 将布尔值或数字列视为分类值。
  
+ 该值指示哪一列包含**类**标记或包含你想要分类或预测的值。
  
+ 将列标记为特征。
  
+ 更改日期/时间值到数字值，反之亦然。
  
+ 重命名列。
  
 任何时候您需要通常修改的列的定义以满足下游模块要求，请使用编辑元数据。 例如，某些模块仅处理特定的数据类型或需要标志的列，如`IsFeature`或`IsCategorical`。  
  
 执行所需的操作后，可以将元数据重置为其原始状态。
  
## <a name="configure-edit-metadata"></a>配置编辑元数据
  
1. 在 Azure 机器学习中，将编辑元数据模块添加到你的试验并连接你想要更新的数据集。 您可以找到在数据集**Data Transformation**中**操作**类别。
  
1. 选择**启动列选择器**，然后选择要使用的列集的列。 您可以选择列单独按名称或索引，或者可以选择一组按类型的列。  
  
1. 选择**数据类型**选项则需要将不同的数据类型分配给所选列。 您可能需要更改某些操作的数据类型。 例如，如果源数据集具有数字作为文本进行处理，您必须更改它们为数值数据类型之前使用数学运算。

    + 支持的数据类型为**字符串**，**整数**， **Double**，**布尔**，并**DateTime**。

    + 如果选择多个列，则必须应用到的元数据更改*所有*所选列。 例如，假设您选择两个或三个数值列。 你可以更改它们全部为字符串数据类型，并在一个操作中重命名它们。 但是，你无法更改为字符串数据类型的一列和另一列从一个浮点数为整数。
  
    + 如果未指定新的数据类型，则列元数据不变。

    + 执行编辑元数据操作后，会发生变化的列类型和值。 通过使用编辑元数据来重置列数据类型，可以随时恢复原始的数据类型。  

    > [!NOTE]
    > 如果更改任何类型的数字**DateTime**类型，不**日期时间格式**字段留空。 当前不能指定目标数据格式。  

1. 选择**分类**选项以指定所选列中的值应视为类别。

    例如，你可能有一列包含数字 0、 1 和 2，但知道数字实际上意味着"Smoker，""非-smoker"和"未知"。 在这种情况下，通过将标记为分类列可确保仅对数据进行分组并不在数值计算中使用的值。
  
1. 使用**字段**选项如果想要更改 Azure 机器学习模型中使用的数据的方式。

    + **功能**：使用此选项将某个列标记为仅对特征列的模块中的功能。 默认情况下，所有列最初被都视为功能。  
  
    + **标签**：使用此选项标记的标签，它是也称为可预测的属性或目标变量。 许多模块要求，且只有一个标签列中数据集。

        在许多情况下，Azure 机器学习可以推断出某一列包含类标签。 通过设置此元数据，可以确保正确标识列。 设置此选项不会更改数据值。 它更改仅某些机器学习算法处理数据的方式。
  
    > [!TIP]
    > 你是否不属于这些类别的数据？ 例如，你的数据集可能包含值，例如不用作变量的唯一标识符。 有时此类 Id 就会导致问题时在模型中使用。
    >
    > 幸运的是，Azure 机器学习会保留所有数据，以便无需从数据集中删除此类列。 当需要对一些特殊的列集执行操作时，只需删除所有其他列暂时使用[选择数据集中的列](select-columns-in-dataset.md)模块。 更高版本可合并列返回到数据集通过使用[中添加列](add-columns.md)模块。  
  
1. 使用下列选项可清除以前的选择并还原为默认值的元数据。  
  
    + **清除特征**:使用此选项删除功能标志。  
  
         最初，将所有列都视为特征。 对于执行数学运算的模块，可能需要使用此选项以防止不被视为变量的数值列。
  
    + **清除标签**:使用此选项可删除**标签**从指定的列的元数据。  
  
    + **清除评分**:使用此选项可删除**分数**从指定的列的元数据。  
  
         您当前不能显式将列标记为在 Azure 机器学习中评分。 但是，某些操作导致在内部被标记为评分的列。 此外，自定义 R 模块可能会输出分数值。

1. 有关**新的列名**，输入所选的列或列的新名称。  
  
    + 列名称可以使用 utf-8 支持的字符编码。 不允许使用空字符串、 null 值或完全由空格组成的名称。  
  
    + 要重命名多个列，以逗号分隔的列表中的列索引的顺序作为输入的名称。  
  
    + 必须重命名所选的所有列。 不能省略或跳过列。  
  
1. 运行试验。
