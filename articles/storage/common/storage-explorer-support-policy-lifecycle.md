# <a name="azure-storage-explorer-support-lifecycle-and-policy"></a>Azure 存储资源管理器支持生命周期和策略

本文档介绍 Azure 存储资源管理器的支持生命周期和策略。

## <a name="update-schedule-and-process"></a>更新计划和进程

Azure 存储资源管理器发布一年四到六次。 Microsoft 还可以绕过此计划来释放关键问题的修补程序。

存储资源管理器每小时检查一次更新，并在更新可用时下载更新。 但需要用户接受才能启动安装过程。 如果用户选择在不同的时间进行更新，则他们可以手动检查更新。 在 Windows 和 Linux 上，用户可以通过从 "**帮助**" 菜单中选择 "**检查更新**" 来检查更新。 在 macOS 上，可以在 "**应用程序" 菜单**下找到**更新**。 用户还可以直接前往[存储资源管理器](https://azure.microsoft.com/features/storage-explorer/)下载页面下载并安装最新版本。

强烈建议始终使用最新版本的存储资源管理器。 如果问题导致无法更新到最新版本的存储资源管理器，请在[GitHub](https://github.com/microsoft/AzureStorageExplorer)上提出问题。

## <a name="support-lifecycle"></a>支持周期

存储资源管理器受[新式生命周期策略](https://support.microsoft.com/help/30881/modern-lifecycle-policy)控制。 预计用户将其安装存储资源管理器保持最新状态。 保持最新状态可确保用户具有最新的功能、性能增强、安全性和服务可靠性。

从版本1.14.1 开始，超过12个月的任何存储资源管理器版本都将被视为不受支持。 在2021年7月14日开始，1.14.1 之前的所有版本均被视为不支持。 不受支持的版本将不再保证完全按设计和预期运行。 有关所有发布的列表、发布日期以及其支持日期的结束时间，请参阅[发布](#releases)。

从版本1.13.0 开始，一旦某个版本大约在一个月内不受支持，就会显示一个应用内警报。 此警报鼓励用户更新到最新版本的存储资源管理器。 版本不受支持后，会在每次启动时显示应用内警报。
