# <a name="how-to-configure-an-application-proxy-application-to-use-pingaccess"></a>如何配置应用程序代理应用程序以使用 PingAccess

我们与 PingAccess 合作，现在使你能够将应用程序代理的优势扩展到使用基于标头的身份验证的应用程序。 如果应用程序没有使用标头，请参阅[单一登录文档](manage-apps/application-proxy-configure-single-sign-on-with-kcd.md)，以了解其他选项的详细信息。

## <a name="overview-of-steps-and-recommended-documents"></a>步骤和建议的文档概述

若要使用 PingAccess 配置应用程序，需要执行四个步骤：

1.  配置应用程序代理连接器

2.  创建 Azure AD 应用程序代理应用程序

3.  下载和配置 PingAccess

4.  在 PingAccess 中配置应用程序

有关这些步骤的详细信息，请参阅[基于标头的单一登录文档](manage-apps/application-proxy-configure-single-sign-on-with-ping-access.md)。
