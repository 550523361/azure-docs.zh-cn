# <a name="azure-ad-domain-services---contact-us"></a>Azure AD 域服务 - 联系我们

## <a name="feedback"></a>反馈
如果您有关于我们的服务或非技术问题的反馈，则可以通过发送电子邮件给我们[：Azure AD 域服务反馈](mailto:aaddsfb@microsoft.com)与我们共享您的请求。 将审核发送到此地址的电子邮件。 产品组的成员**可以**回复请求更多信息。 将不会答复对此电子邮件地址进行技术支持的请求。

## <a name="technical-assistance"></a>技术援助
如果使用托管域时遇到问题，请确认[故障排除指南](troubleshoot.md)中所述的步骤是否能够解决该问题。 如果仍有问题，请登录到 Azure 门户。 打开**Azure AD 域服务**主页，并从 **"支持 + 故障排除**"下的操作窗格中选择 **"新支持请求**"。

![新建支持请求](./media/contact-us/supportRequest.png) 

尽可能准确地填写支持请求的每个部分。 选择服务时，请确保选择**Azure 活动目录域服务（VM - 域控制器）** 以将请求路由到适当的支持专业人员。

> [!IMPORTANT]
> 从服务列表中选择**Azure 活动目录目录、域和对象，** 以便借助 Azure 活动目录进行域验证。
>
> 

## <a name="feature-requests"></a>功能请求
您是否对如何改进 Azure 活动目录域服务有想法？ 我们想听听。 转到 Azure 反馈论坛，导航到[域服务](https://feedback.azure.com/forums/169401-azure-active-directory?category_id=160593)类别并分享您的想法。 产品组的成员会审核这些请求。
