# <a name="ibm-workloads-on-azure"></a>Azure 上的 IBM 工作负荷

许多基于 z/OS 的 IBM 大型机工作负载可以在 Azure 中复制，而不会丢失功能，用户甚至不会注意到其基础系统中的更改。 在 Azure 上重新托管应用程序可为您提供所需的类似大型机的功能，以及云的弹性、可用性和潜在成本节约。

Azure 支持与现有 IBM 大型机环境集成，使您能够迁移有意义的应用项，根据需要运行混合解决方案，并随着时间的推移进行迁移。 尽管您可以完全重写 Azure 的现有基于大型机的程序，但重新托管它们更为常见。 重写会增加迁移项目的成本、复杂性和时间。 通过重新托管，您可以：

- 将应用程序移动到基于云的仿真器。

- 将数据库迁移到基于云的数据库。

- 使用代码转换引擎替换模块和代码。

此外，IBM 软件（包括 WebSphere 和 MQ）现在位于 Azure 应用商店中。 借助 IBM 软件许可证，可以利用 Azure 提供的按需基础结构扩展快速启动虚拟机。

广泛的合作伙伴生态系统可帮助您将 IBM 大型机系统迁移到 Azure。 在开始分阶段部署重写或替换应用程序之前，大多数公司会尽可能采用务实的重用方法。 在[Azure 大型机迁移中心](https://azure.microsoft.com/migration/mainframe/)从合作伙伴处获取更多指导和帮助。
