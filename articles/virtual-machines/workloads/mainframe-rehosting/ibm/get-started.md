# <a name="ibm-workloads-on-azure"></a>在 Azure 上的 IBM 工作负荷

基于 z/OS 的许多 IBM 大型机工作负荷，可以在 Azure 中使用不复制任何中断，而无需用户甚至注意更改到其基础系统中的功能。 重新承载在 Azure 上的应用程序提供所需的良好灵活性，可用性，以及类似于大型机的功能，并可能会在云中节省成本。

Azure 支持集成使用现有的 IBM 大型机环境，使您能够迁移很有用，applicates 运行混合解决方案的需要并随着时间的推移迁移。 尽管您完全可以为 Azure 重写现有的基于大型机的程序，它是更常见的是将它们重新托管。 重写将成本、 复杂性和时间添加到迁移项目。 使用重新承载，可以：

- 将移动到基于云的仿真程序的应用程序。

- 将数据库迁移到基于云的数据库。

- 更换模块和使用代码转换引擎的代码。

此外，IBM 软件，包括 WebSphere 和 MQ，现已在 Azure Marketplace 中。 使用 IBM 软件许可证，你可以利用 Azure 提供的按需的基础结构扩展功能来快速启动虚拟机。

广泛的合作伙伴生态系统，可帮助您将 IBM 大型机系统迁移到 Azure。 大多数都遵循重复使用的编程之法只要有可能在重写或替换应用程序的分阶段部署开始之前。 获取更多指导，以及来自在合作伙伴帮助[Azure 大型机迁移中心](https://azure.microsoft.com/migration/mainframe/)。
