# <a name="azure-white-papers-about-mainframe-topics"></a>有关大型机主题的 Azure 白皮书

获取有关大型机迁移、重新托管和将基于 IBM Z 的系统移动到 Microsoft Azure 的资源。

### <a name="demystifying-mainframe-to-azure-migration"></a>[将大型机与 Azure 迁移解密](https://azure.microsoft.com/resources/demystifying-mainframe-to-azure-migration/)

Azure 提供了一种大型机替代方案，能够提供等效的功能和功能。 本文由 AzureCAT 团队的 Larry Mead 撰写，介绍了主要的 IBM z/OS 大型机组件和 Azure 等效项。 它还提供了一个路线图，用于开始与订阅过时的大型机理念的 IT 决策者进行对话。

### <a name="move-mainframe-compute-and-storage-to-azure"></a>[将大型机计算和存储移动到 Azure](https://azure.microsoft.com/resources/move-mainframe-compute-and-storage-to-azure/)

要在 Microsoft Azure 上运行大型机工作负荷，您需要了解大型机的功能与 Azure 的比较情况。 基于 IBM z14 大型机，AzureCAT 的 Larry Mead 本指南探讨了如何在 Azure 上获取可比较的结果。 Azure 上可大规模扩展的计算和存储资源为运行大型机工作负荷的组织提供了直接的好处。

### <a name="microsoft-azure-government-cloud-for-mainframe-applications"></a>[适用于大型机应用程序的 Microsoft Azure 政府云](https://azure.microsoft.com/resources/microsoft-azure-government-cloud-for-mainframe-applications/)

规划应用程序迁移是为甚至成熟的大型机工作负载增加价值和敏捷性的理想时机。 在此快速指南中，AzureCAT 的 Larry Mead 展示了美国政府机构及其合作伙伴如何使用 Azure 政府进行大型机应用程序，迁移可能不像您想象的那么困难。 Azure 政府功能在更具成本效益和敏捷的环境中提供了大型机的优势。 此外，Azure 政府还获得了运营临时管理局 （P-ATO），以增强 FedRAMP 的高影响力。

### <a name="deploy-ibm-db2-purescale-on-azure"></a>[在 Azure 上部署 IBM DB2 纯标量](https://azure.microsoft.com/resources/deploy-ibm-db2-purescale-on-azure/)

学习我们最近在 Azure 上重新托管其 IBM DB2 环境的企业的经验。 本指南由 AzureCAT 和 DMJ 团队成员编写：拉里·米德、本杰明·吉内贝蒂埃、亚历山德罗·沃扎和乔纳森·弗罗斯特。 它们描述了他们在迁移过程中采取的步骤。 IBM Db2 纯Scale 团队成员对其发现进行了审查。 GitHub 上提供的安装脚本基于团队用于典型中型 OLTP 工作负载的体系结构。

### <a name="install-tmaxsoft-openframe-on-azure"></a>[在 Azure 上安装 TmaxSoft 开放帧](https://azure.microsoft.com/resources/install-tmaxsoft-openframe-on-azure/)

以云规模实现基础架构现代化。 TmaxSoft OpenFrame 可轻松提升现有大型机资产并将其转移到 Azure。 在本文中，Steve Read （AzureCAT） 和 Manoj Aerroju （TmaxSoft） 解释了如何设置适合开发、演示、测试和生产工作负载的 OpenFrame 环境。

### <a name="ibm-mainframe-to-microsoft-azure-reference-architecture"></a>[IBM 大型机到微软 Azure 参考体系结构](https://www.astadia.com/whitepaper/ibm-mainframe-to-microsoft-azure)

本白皮书反映了 Astadia 超过 25 年的大型机平台现代化专业知识。 它们解释了现代化努力的好处和挑战。 本指南概述了 IBM 大型机和 IBM 大型机到 Azure 参考体系结构。 它还介绍了 Astadia 成功方法。

### <a name="deploying-mainframe-applications-to-microsoft-azure"></a>[将大型机应用程序部署到 Microsoft Azure](https://www.microfocus.com/media/white-paper/deploying_mainframe_applications_to_microsoft_azure_wp.pdf)

Micro Focus 的解决方案让您摆脱专有大型机硬件和软件的限制。 在本指南中，Micro Focus 介绍了如何将 IBM 大型机中的 COBOL 和 PL/I 应用程序部署到云中。

### <a name="breathe-new-life-into-mainframes"></a>[为大型机注入新的活力](https://www.infosys.com/services/modernization/breathe-new-life-mainframes.html)

 大型机对于业务维护来说越来越具有挑战性。 Infosys 和 Microsoft 的本白皮书强调了成功大型机迁移的制胜策略。 用例和示例说明了这些选项。
