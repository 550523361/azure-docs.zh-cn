# <a name="security-alerts---a-reference-guide"></a>安全警报 - 参考指南

本文列出了在 Azure 安全中心的威胁保护模块中看到的安全警报。 环境中显示的警报取决于您保护的资源和服务以及自定义配置。

要了解如何响应这些警报，请参阅[在 Azure 安全中心管理和响应安全警报](security-center-managing-and-responding-alerts.md)。

要了解如何导出警报（和建议），请参阅[导出安全警报和建议（预览）。](continuous-export.md)

警报表下方是一个表，用于对这些警报的意图进行分类的 Azure 安全中心终止链。

## <a name="intentions"></a>意图

了解攻击的意图可以帮助您更轻松地调查和报告事件。 Azure 安全中心警报包括"意图"字段，以帮助进行这些努力。

描述网络攻击从侦察到数据外泄过程的一系列步骤通常被称为"杀伤链"。 

安全中心支持的杀灭链意图基于[MITRE&trade; ATT&CK 框架](https://attack.mitre.org/matrices/enterprise)，下表对此进行了说明。

|Intent|说明|
|------|-------|
|**攻击前**|Pre攻击可能是尝试访问特定资源，而不管恶意意图如何，也可以是尝试访问目标系统以在利用之前收集信息的失败尝试。 此步骤通常被检测为尝试，源自网络外部，以扫描目标系统并标识入口点。</br>有关攻击前阶段的更多详细信息，请阅读[MITRE 的页面](https://attack.mitre.org/matrices/pre/)。|
|**初始访问**|初始访问是攻击者设法在受攻击的资源上站稳脚跟的阶段。 此阶段与计算主机和资源（如用户帐户、证书等）相关。在此阶段之后，威胁参与者通常能够控制资源。|
|**持久性**|持久性是对系统的任何访问、操作或配置更改，该更改使威胁参与者在该系统上具有持久性。 威胁参与者通常需要通过系统重新启动、凭据丢失或其他需要远程访问工具才能重新启动或提供备用后门才能重新获得访问权限的中断来保持对系统的访问。|
|**权限升级**|权限升级是允许攻击者在系统或网络上获取更高级别权限的操作的结果。 某些工具或操作需要更高级别的特权才能工作，并且在整个操作的多个点可能是必需的。 具有访问特定系统的权限或执行对手实现其目标所需的特定功能的用户帐户也可能被视为权限的升级。|
|**防御逃生**|防御规避包括攻击者可能用来逃避检测或避免其他防御的技术。 有时，这些操作与其他类别的技术相同（或变化），这些技术具有颠覆特定防御或缓解的附加好处。|
|**凭据访问**|凭据访问表示导致访问或控制企业环境中使用的系统、域或服务凭据的技术。 对手可能会尝试从用户或管理员帐户（本地系统管理员或具有管理员访问权限的域用户）获取合法凭据，以便在网络上使用。 在网络中具有足够的访问权限，攻击者可以创建帐户，以便以后在环境中使用。|
|**发现**|发现包括允许攻击者获得有关系统和内部网络的知识的技术。 当对手获得新系统时，他们必须将自己定位到他们现在所掌握的，以及该系统在入侵期间运营时对其当前目标或总体目标有何好处。 操作系统提供了许多本机工具，有助于在此泄露后的信息收集阶段。|
|**横向运动**|横向移动包括使对手能够访问和控制网络上的远程系统的技术，并且可以但不一定包括在远程系统上执行工具。 横向移动技术允许攻击者从系统收集信息，而无需其他工具，如远程访问工具。 攻击者可以将横向移动用于多种目的，包括远程执行工具、转向其他系统、访问特定信息或文件、访问其他凭据或造成效果。|
|**执行**|执行策略表示导致在本地或远程系统上执行对手控制代码的技术。 这种策略通常与横向移动结合使用，以扩大对网络上的远程系统的访问。|
|**集合**|集合包括用于在渗出之前从目标网络识别和收集信息（如敏感文件）的技术。 此类别还涵盖系统或网络上的攻击者可能查找信息进行渗透的位置。|
|**渗漏**|渗漏是指导致或帮助对手从目标网络中删除文件和信息的技术和属性。 此类别还涵盖系统或网络上的攻击者可能查找信息进行渗透的位置。|
|**命令和控制**|命令和控制策略表示对手如何与目标网络中其控制系统通信。|
|**影响**|影响事件主要尝试直接降低系统、服务或网络的可用性或完整性;包括操纵数据以影响业务或运营流程。 这通常是指勒索软件、污损、数据操作等技术。|
