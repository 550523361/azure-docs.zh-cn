# <a name="threat-detection-for-cloud-native-compute-in-azure-security-center"></a>威胁检测云本机 Azure 安全中心中的计算资源

为云提供程序，安全中心将利用它必须分析内部日志并标识多个目标上的攻击方法的唯一可见性。 本主题介绍了可用于以下 Azure 服务的警报：

* [Azure 应用服务](#app-services)
* [容器](#azure-containers) 

## Azure App Service <a name="app-services"></a>

安全中心利用云来标识针对在 Azure 应用服务，运行客户应用程序的攻击的小数位数。 与 web 应用程序正在几乎任何现代网络中，攻击者探测查找这些和利用的弱点。 之前要路由到特定的环境，在 Azure 中运行的应用程序的请求都通过多个网关，它们是在其中检查和记录。 若要标识攻击，攻击者，并了解将在稍后的新模式，然后使用此数据。

通过利用 Azure 为云提供程序具有的可见性，安全中心将分析应用服务内部日志以确定多个目标上的攻击方法。 例如，大范围扫描和分布式的攻击。 此类攻击通常来自 Ip 一小部分，并且能够提供对进行爬网到类似的终结点搜索易受攻击的页或插件的多个主机上的模式。 这可以使用云检测到，但不能从一台主机的角度来看标识。

安全中心也有权访问基础沙盒和 Vm。 内存取证，以及基础结构可以讲述故事，与新攻击在现实中客户机折中方案到循环。 因此，安全中心可以检测对 web 应用程序长攻击后被利用。

> [!div class="mx-tableFixed"]

|警报|描述|
|---|---|
|**检测到的可疑 WordPress 主题调用**|Azure 应用服务活动日志指示对你的应用服务资源的可能的代码注入活动。<br/> 此可疑活动类似于操作来支持服务器端执行的代码，直接 web 请求来调用操作的主题文件后跟一个 WordPress 主题的活动。 此类型的活动发生在过去攻击活动的一部分通过 WordPress。|
|**从异常检测到的 IP 地址连接到 web 页**|Azure 应用服务活动日志指示连接到敏感的网页从永远不会连接到它之前的源地址。 这可能表示有人正尝试暴力攻击到 Web 应用管理页面。 它也可能是合法用户正在使用的新的 IP 地址的结果。|
|**威胁智能中找到连接到 Azure 应用服务 FTP 接口的 IP**|Azure 应用服务 FTP 日志分析检测到威胁智能源中找到的源地址的连接。 在此连接期间用户访问下面所列的页。|
|**检测到的 web 留下指纹**|Azure 应用服务活动日志指示可能 web 指纹识别对你的应用服务资源的活动。 <br/>检测到此可疑活动都具有一个名为密件大象工具相关联。 该工具指纹 web 服务器并尝试检测已安装的应用程序和其版本。 通常，攻击者使用此工具用于探测的 web 应用程序，以找出安全漏洞。|
|**对检测到的网页上可能易受攻击的可疑访问**|Azure 应用服务活动日志指示已访问似乎是敏感的网页。 <br/>此可疑活动的来源从其访问模式类似于 web 扫描程序的源地址。 此类活动通常是由攻击者以扫描网络以尝试并访问敏感或易受攻击的网页尝试与相关联。|
|**上传文件夹中的 PHP 文件**|Azure 应用服务活动日志指示某些内容已访问到可疑的 PHP 页上传文件夹中。 <br/>这种类型的文件夹通常不包含 PHP 文件。 此类型的文件存在可能表示攻击者利用的任意文件上传漏洞。|
|**尝试在 Windows 应用服务上运行 Linux 命令**|应用服务进程的分析检测到尝试在 Windows 应用服务上运行 Linux 命令。 此操作已运行的 web 应用程序。 此行为通常出现在市场活动，它利用常见的 web 应用程序中的漏洞。|
|**检测到的可疑 PHP 执行**|计算机日志指示可疑 PHP 进程正在运行。 操作包括尝试从命令行使用 PHP 进程中运行操作系统命令或 PHP 代码。 尽管此行为可以是合法的在 web 应用程序也将出现此行为如试图感染网站使用 web shell 的恶意活动。|
|**从临时文件夹的过程执行**|应用服务的进程分析检测到应用程序的临时文件夹中的进程的执行。 虽然此行为可以是合法的此行为还观察到的恶意活动中的 web 应用程序中。|
|**尝试在运行检测到高特权命令**|应用服务进程的分析检测到尝试运行需要较高权限的命令。 在 web 应用程序上下文中运行该命令。 虽然此行为可以是合法的此行为还观察到的恶意活动中的 web 应用程序中。|

> [!NOTE]
> 安全中心威胁检测的应用服务目前不在 Azure 政府版和主权云区域中可用。

详细了解应用服务威胁检测警报访问通过 Azure 安全中心保护应用服务，并查看如何启用监视和保护的应用服务工作负荷。

## 容器 <a name="azure-containers"></a>

安全中心提供基于审核框架的 Linux 计算机上的容器的实时威胁检测。 警报标识多个可疑的 Docker 活动，例如在主机上，运行在 Docker 容器或使用的加密挖掘者采用的安全外壳 (SSH) 服务器的指示特权容器的创建。 可以使用此信息快速补救安全问题，并提高容器的安全性。 除了 Linux 检测安全中心还提供了更具体的容器部署的分析。
