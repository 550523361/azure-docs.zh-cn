# <a name="azure-cosmos-db-documentation-experiment"></a>Azure Cosmos DB 文档试验

Azure Cosmos DB 是一种全球分布式多模型数据库。 由于通过 Azure Cosmos DB 可以执行许多操作（从创建图形到创建文档数据库，或者迁移 MongoDB、Cassandra 或 Azure 表存储中的数据），因此，组织文档可能是一个难题。 因此，我们希望得到帮助。

我们正在测试 Azure Cosmos DB 目录和[登陆页](https://docs.microsoft.com/en-us/azure/cosmos-db/)的组织方式，如下图所示，我们希望得到有关这些建议更改的反馈。

## <a name="to-load-both-views"></a>加载两个视图

1. 要并排打开这两个视图，请从常用的 Web 浏览器导航到[登陆页](https://docs.microsoft.com/en-us/azure/cosmos-db/)。 
2. 使用上面的图像确定看到的是哪个版本：原始版还是试验版。 
3. 打开 inprivate 或 incognito 浏览器窗口，并在新浏览器窗口中加载[登陆页](https://docs.microsoft.com/en-us/azure/cosmos-db/)，然后再次使用这些图像来确定看到的是哪个版本。 

   如果两个视图并未同时出现，请关闭并重新打开新的 inprivate 或 incognito 窗口，然后再次加载[登陆页](https://docs.microsoft.com/en-us/azure/cosmos-db/)。 可能需要多次尝试才能在单独的窗口中加载两个版本。 刷新 incognito 窗口不会加载另一个版本，需要打开新窗口。 

4. 然后，可以调整窗口大小以并排排列，或在视图之间来回切换以进行比较。 

5. 熟悉试验体验的组织和布局后，请单击下方的“试验调查”按钮提供反馈。 请于 2018 年 4 月 4 日午夜前提交表单，以便反馈可供查看并被采纳。
