# <a name="security-recommendations"></a>安全建议

IoT 的 Azure 安全中心扫描 Azure 资源和 IoT 设备，并提供安全建议以减少攻击面。 安全建议是可操作的，旨在帮助客户遵守安全最佳实践。

在本文中，您将找到可在 IoT 中心和/或 IoT 设备上触发的建议列表。

## <a name="recommendations-for-iot-devices"></a>物联网设备建议

设备建议提供改进设备安全状态的见解和建议。 

| 严重性 | “属性”                                                      | 数据源 | 描述                                                                                                                                                                                           |
|----------|-----------------------------------------------------------|-------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 中型   | 打开设备上的端口                                      | 代理       | 在设备上找到一个侦听终结点。                                                                                                                                                        |
| 中型   | 在其中一个链中找到的允许防火墙策略。 | 代理       | 找到允许的防火墙策略（INPUT/输出）。 默认情况下，防火墙策略应拒绝所有流量，并定义规则以允许必要的通信到/从设备进行。                               |
| 中型   | 在输入链中找到允许的防火墙规则     | 代理       | 在防火墙中已发现一条规则，其中包含各种 IP 地址或端口的允许模式。                                                                                    |
| 中型   | 在输出链中找到允许的防火墙规则    | 代理       | 在防火墙中已发现一条规则，其中包含各种 IP 地址或端口的允许模式。                                                                                   |
| 中型   | 操作系统基线验证失败           | 代理       | 设备不符合[CIS Linux基准](https://www.cisecurity.org/cis-benchmarks/)。                                                                                                        |

### <a name="operational-recommendations-for-iot-devices"></a>物联网设备的操作建议

操作建议提供了改进安全代理配置的见解和建议。

| 严重性 | “属性”                                    | 数据源 | 描述                                                                       |
|----------|-----------------------------------------|-------------|-----------------------------------------------------------------------------------|
| 低      | 代理发送未使用的消息          | 代理       | 在过去 24 小时内，10% 或更多的安全消息小于 4 KB。  |
| 低      | 安全双配置不理想 | 代理       | 安全双配置不是最佳配置。                                        |
| 低      | 安全双配置冲突    | 代理       | 在安全双配置中识别了冲突。 |                          |
|

## <a name="recommendations-for-iot-hub"></a>物联网中心的建议

建议警报为改善环境的安全状态的操作提供了见解和建议。  

| 严重性 | “属性”                                                     | 数据源 | 描述                                                                                                                                                                                                             |
|----------|----------------------------------------------------------|-------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 高     | 多个设备使用的相同身份验证凭据 | IoT 中心     | IoT 中心身份验证凭据由多个设备使用。 这可能表示非法设备正在模拟合法设备。 重复的凭据使用会增加恶意参与者模拟设备的风险。 |
| 中型   | 默认 IP 筛选器策略应拒绝                  | IoT 中心     | IP 筛选器配置应具有为允许的流量定义的规则，并且默认情况下应默认拒绝所有其他流量。                                                                                                     |
| 中型   | IP 筛选器规则包括较大的 IP 范围                   | IoT 中心     | 允许 IP 筛选器规则源 IP 范围过大。 过度宽松的规则可能会将 IoT 中心暴露给恶意参与者。                                                                                       |
| 低      | 在 IoT 中心中启用诊断日志                       | IoT 中心     | 启用日志并将其保留长达一年的时间。 保留日志使您能够在发生安全事件或网络受到威胁时重新创建活动跟踪以进行调查。                                       |
|
