# <a name="publishing-guide-by-offer-type"></a>发布指南（按产品/服务类型）

一旦[您决定发布选项](https://docs.microsoft.com/azure/marketplace/determine-your-listing-type)，您就可以选择将用于显示产品/服务的报价类型。

 “产品/服务类型”定义商品结构，其中包括元数据、项目和其他用于在市场中呈现产品/服务的内容**。

在创建套餐之前，必须选择套餐类型。 套餐类型对应于你要发布的解决方案、应用或服务套餐的类型，并取决于与 Microsoft 产品和服务的匹配情况。

可以以不同方式配置单个产品/服务类型，以启用不同的发布选项、操作调用、预配或定价。 套餐类型的发布选项和配置也取决于套餐资格和技术要求。

在创建产品/服务之前，请务必查看店面和产品/服务类型资格要求以及技术方面的发布要求。

## <a name="list-of-offer-types"></a>产品/服务类型列表

下表列出了 Azure 市场产品/服务类型。

| **优惠类型**    | **说明**  |
| :------------------- | :-------------------|
| [**虚拟机**](./marketplace-virtual-machines.md) | 将虚拟设备部署到与客户关联的订阅时，请使用“虚拟机”产品/服务类型。 |
| [**解决方案模板**](./marketplace-solution-templates.md) | 当解决方案需要单个 VM 以外的其他部署和配置自动化时，请使用解决方案模板（也称为 Azure 应用程序）产品类型。 解决方案模板可以使用许多不同类型的 Azure 资源，包括但不限于 VM。  |
| [**托管应用程序**](./marketplace-managed-apps.md) | 需要满足以下条件时，请使用“Azure 应用：托管应用”产品/服务类型： <br> <ul> <li> 使用 VM 为客户部署基于订阅的解决方案，或部署基于 IaaS 的整个解决方案。 </li> <li>你或客户要求解决方案由合作伙伴管理。 </li> <ul> |
| [**SaaS 应用程序**](./marketplace-saas-applications-technical-publishing-guide.md) | 使用 SaaS 应用程序产品/服务类型可让客户以订阅的形式购买基于 SaaS 的技术解决方案。 |
| [**容器产品/服务**](./marketplace-containers.md) | 如果解决方案是预配为基于 Kubernetes 的 Azure 容器服务的 Docker 容器映像，请使用“容器”产品/服务类型。 |
| [**Azure IoT 边缘模块**](./iot-edge-module.md) | Azure IoT Edge 模块是由 IoT Edge 托管的最小计算单位，可以包含 Microsoft 服务（如 Azure 流分析）、第三方服务，或你自己特定于解决方案的代码。 |
| [**咨询服务**](./consulting-services.md) | 咨询服务有助于将客户与支持和扩展其使用 Azure、Dynamics 365 或 Power Suite 服务的服务联系起来。|
| [**Office 365、动态 365 和 Power BI**](./appsource-offer-publishing-guide.md) | 您可以发布基于动态 365、Office 365、Power BI 和电源应用构建或扩展的 AppSource 产品。|
| [**集成的解决方案**](./integrated-solutions-for-publishers.md) | 您可以发布集成的、与行业一致的解决方案，将技术和服务作为单一产品组合。|

有关单一登录要求的信息，请参阅列表选项和产品/服务类型，请参阅[**Azure AD 要求**](./enable-appsource-marketplace-using-azure-ad.md)。
