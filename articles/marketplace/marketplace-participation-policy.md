# <a name="azure-marketplace-participation-policies"></a>Azure 市场参与政策

这些 Microsoft Azure 市场参与政策适用于 Microsoft Azure 市场中的所有发布者和套餐。 这些政策是对“Microsoft 市场发布者协议”中所规定条款和条件的补充。 若要参与 Azure 市场，发布者必须始终遵守本文档中所述和/或所引用的政策。 如果发布者在任何给定时间未能满足所有条款和条件的要求，则 Microsoft 可能从 Azure 市场删除该发布者的套餐。 我们可能会不时更新本文档。

## <a name="base-criteria"></a>基本条件

1. 在 Azure 市场中提供的软件和服务必须至少满足以下条件之一： 

    * 在 Microsoft Azure 上运行：软件或服务的主要功能必须在 Microsoft Azure 上运行。
    * 可以部署到 Microsoft Azure：发布者必须在其套餐的列表信息中说明如何在 Microsoft Azure 上部署该软件或服务。
    * 与 Microsoft Azure 服务集成或扩展 Microsoft Azure 服务：发布者必须在其套餐列表信息中说明该软件或服务与何种 Azure 服务进行了集成或对何种 Azure 服务进行了扩展。 软件或服务如何与 Azure 服务集成或如何扩展该服务。

## <a name="publishing-offers"></a>发布套餐

1. 发布者必须在执行“Microsoft 市场发布者协议”的 60 天内在 Azure 市场中至少发布一个套餐。

1. 发布者必须遵守“市场发布指南”中定义的 Azure 市场上架技术要求，这些要求可在发布门户中进一步确定。

## <a name="offer-listings"></a>套餐列表

1. 发布者必须在其套餐的列表页包括套餐的详细信息，此类信息必须准确且保持最新。 此类信息必须包括（如果适用）：
    * 套餐说明
        * 简短套餐说明
            * SKU 信息
            * 价值主张
        * 建议套餐说明
            * SKU 详细信息
            * 详细价值主张
            * 功能：有关套餐的 3-5 个事实陈述
            * 好处：套餐功能产生的 3-5 个结果
    * 定价模型：套餐必须符合 Azure 市场支持的[定价模型](./marketplace-faq-publisher-guide.md)，如本文档的“定价模型”部分中所述。
    * 客户支持详细信息的链接：发布者必须为 Azure 市场中的套餐提供商业上合理的客户支持。  可将客户支持包含在与套餐相关的用户费用中，也可以将客户支持作为需单独购买的支持套餐。
    * 套餐资源：资源包括但不限于，演示视频、屏幕截图、白皮书、案例研究、客户评价以及关于如何在 Microsoft Azure 上使用发布者的套餐的详细技术文档。
    * 客户退款政策
    * 使用条款
    * 隐私政策

1. 发布者不可在其套餐列表页内将 Azure 客户重定向至 Azure 市场中未提供的软件或服务，或对 Azure 客户追加销售此类软件或服务。 此限制不适用于发布者在 Azure 市场之外销售的支持服务。

1. 发布者不可在 Azure 市场内宣传其套餐在其他云平台上的可用性。

1. Microsoft 保留编辑和修订套餐列表页详细信息以确保质量的权利。 如果 Microsoft 对任何列表页详细信息进行了更改，则 Microsoft 将在发布者的套餐列表页在 Azure 市场中发布之前通知发布者。

| **虚拟机扩展** | VM 代理，可以使用各种选项添加到新的 VM，包括通过 REST API、Azure 门户或 Azure PowerShell cmdlet。 VM 扩展也可在现有 VM 上手动安装，并且可为 Windows Server 或基于 Linux 的 VM 进行配置。 <br> <br> 虚拟机扩展（“**VM 扩展**”）是用于在 Azure 虚拟机中安装软件应用程序或软件应用程序套件的机制。 VM 扩展可以包含软件应用程序。  在虚拟机中安装或由虚拟机执行之后，VM 扩展可以从外部位置下载并安装一个或多个软件应用程序。 在此说明，任何由 VM 扩展安装的软件或其他数据（乃至从外部位置检索而来的软件或数据）均应被视为本协议所称的套餐内容。 你将负责（并且必须向客户提供支持）与你的 VM 扩展套餐关联的任何 VM 扩展处理程序。 |
| **服务** | 完全托管的服务，可供信息工作者、业务分析师、开发人员或 IT 专业人员在自定义应用程序开发或系统管理中使用。 市场支持三种类型的服务： <br> <br> **应用程序服务**提供帮助客户在 Azure 上快速开发云规模应用程序的功能。 <br> <br> 客户必须拥有 Azure 订阅才能购买应用程序服务。 发布者负责计量客户对应用程序服务的使用情况并向 Microsoft 报告使用情况信息，如“Microsoft 市场发布者协议”中所详述。|
| **Web 应用程序** | 应用程序包，可用于在 Azure 网站服务中安装和部署开源或专有网站内容或管理平台。 Web 应用程序必须遵守 Microsoft [Web 应用程序库原则](https://www.iis.net/learn/develop/windows-web-application-gallery/windows-web-application-gallery-principles)。 <br> <br> 在本协议中，“**Web 应用程序**”是一种应用程序包，可以由客户用于在 Azure 网站服务中安装和部署开源或专有网站应用程序。 |
| **目录列表** | 不直接通过市场向 Azure 客户提供，但市场会显示其链接、图标和软件/服务产品列表的套餐。  客户可以定向到发布者的网站，或者可以看到套餐获取方法以及在 Azure 中的用法说明。 “**仅含目录的列表**”是指不直接通过市场向客户提供的套餐。 但是，市场中会显示用于将客户定向到你的网站的链接、图标和/或说明。 此外，会向客户提供套餐获取方法以及在 Azure 中的用法说明。 在此说明，“仅含目录的列表”引用的任何软件或数据均应被视为本协议所称的套餐内容。 |
| **Azure 资源管理器（资源管理器）模板** | 资源管理器模板可以引用多个不同的套餐，包括其他发布者发布的套餐。  它可以让 Azure 客户采用统一且协调的方式部署一个或多个套餐。 <br> <br> “**Azure 资源管理器模板**”是一种数据结构，它引用一个或多个套餐并包含有关套餐的元数据。  该数据结构与列表信息相关联。 资源管理器模板由市场服务用于显示并使客户可以部署特定类别的套餐。 发布者可以在市场中发布引用多个不同套餐（包括其他发布者发布的套餐）的 ARM 模板。 |
| | |

## <a name="offering-suspension-and-removal"></a>套餐暂停和删除

1. Microsoft 保留出于任何原因将某个套餐从 Azure 市场暂停或删除的权利。 Microsoft 可能删除某个套餐的原因包括但不限于：
    * 该套餐已有六个月及以上时间没有任何客户进行设置；
    * 该套餐有较高的付费 SKU 取消率；
    * 该套餐持续收到负面客户反馈；
    * 该套餐持续收到大量支持票证；或
    * 发布者未能遵守“Microsoft 市场发布者协议”、“市场发布指南”或本文档中的条款和条件。

1. 可能由于各种原因决定从市场中删除产品/服务。 产品/服务删除可确保新客户无法再购买或部署该产品/服务，但对现有客户不起作用。 产品/服务终止是终止该服务和/或你与现有客户之间许可协议的过程。 与套餐删除和终止相关的指南和政策受 Microsoft 市场发布者协议约束（请参阅第 7 节）。 可通过记录支持票证申请删除或终止。

## <a name="taxes"></a>税款

1. **Microsoft 汇款国家/地区。**

    1. 以下国家/地区是 Microsoft 汇款国家/地区，详见“Microsoft 市场发布者协议”：欧盟、加拿大、印度、爱尔兰、列支敦斯登、摩纳哥、新西兰、挪威、波多黎各、韩国、瑞士、美国。 澳大利亚、沙特阿拉伯、台湾及阿联酋仅在 Web Direct 活动中属于 Microsoft 汇款国家/地区。

    1. 在爱尔兰，Microsoft 向所有爱尔兰客户收取爱尔兰增值税（除非爱尔兰客户提供有效增值税 56B 证书的详细信息）。 对于欧盟和挪威（位于爱尔兰之外）的商业客户，若提供有效的增值税编号，Microsoft 应用零增值税税率。 如果未提供有效的增值税编号，将根据客户所在国家/地区/管辖权地的适用税率向客户征收增值税。

    1. Microsoft 将向所有位于瑞典和列支敦士登的客户收取瑞士增值税。

1. **ISV 汇款国家/地区。**

    1. ISV 汇款国家/地区包括此政策的第 1 (3) 节中定义的非 Microsoft 汇款国家/地区的所有消费国家/地区。 此外，通过企业渠道进行的与澳大利亚、沙特阿拉伯、台湾和阿联酋有关的销售也应被视为 ISV 汇款销售。 不过，对于台湾地区的发布者，Microsoft 无法确认任何此类台湾地区的发布者是否需要缴纳相应税款。

    1. 如果发布者选择在 ISV 汇款国家/地区提供产品，则此类发布者应单独负责确定其在此类 ISV 汇款国家/地区的纳税义务（如有）。  发布者必须确认，Microsoft 当前可能无法提供发布者向客户提供税务发票所需的交易详细信息。

## <a name="security-events"></a>安全性事件

发布者必须第一时间报告 Azure 市场软件和服务产品相关的可疑安全事件，包括安全事件和安全漏洞。 发布者应使用 [Azure 安全事件支持票证](https://docs.microsoft.com/azure/security/azure-security-event-support-ticket)中所述的流程记录支持票证，并提供所请求的信息。
