# <a name="set-up-your-payout-account-and-tax-forms"></a>设置付款帐户和纳税表单

为了从商业市场的报价销售中收到资金，您需要设置您的支付帐户，并在[合作伙伴中心](https://partner.microsoft.com/dashboard)填写必要的税务表格。

如果您只计划列出免费优惠（并且不打算提供应用内购买或使用 Microsoft 广告），则无需设置支出帐户或填写任何税单。 如果您稍后改变主意，并决定要销售优惠（或附加服务），则可以设置支出帐户并填写税务表格。 在完成您的支出帐户和税务资料之前，您将无法提交任何已支付的优惠或附加服务。

> [!NOTE]
> 在某些市场，出版商只能提交免费优惠。 如果在其中一个市场中注册帐户，则无法设置付款帐户。

设置帐户后，您需要执行两件事，然后才能在商业市场销售要约（或附加项）：：

- [填写税单](#tax-forms)
- [设置付款帐户](#payout-account)

> [!NOTE]
> 有关如何以及何时支付您的报价所赚的钱的详细信息，请参阅[付款](get-paid.md)。

## <a name="tax-forms"></a>税单

### <a name="filling-out-your-tax-forms"></a>填写您的纳税申报表

首先，您需要创建税务配置文件并将其分配给您参与的计划。 您可以通过完成以下步骤为商业市场创建*您的税务配置文件*：

- 指定你的居住地和公民身份所属的国家/地区。
- 填写相应的税单。

您可以在合作伙伴中心以电子方式填写和提交您的纳税申报表;在大多数情况下，您不需要打印和邮寄任何表单。

> [!IMPORTANT]
> 不同的国家和地区有不同的纳税要求。 您必须在纳税中支付的确切金额取决于您销售报价的国家和地区。 请参阅[Microsoft Azure 应用商店发布商协议](https://go.microsoft.com/fwlink/p/?LinkID=699560)，了解 Microsoft 代表您汇出销售和使用税的国家/地区。 在其他国家/地区，根据您注册的位置，您可能需要将销售情况汇出，并将产品/服务销售使用税款直接汇给当地税务机关。 另外，你所获得的应用销售收益可能需要缴纳所得税。 我们强烈建议您联系您所在国家或地区的相关机构，以便最好地帮助您确定适合您的商业市场活动的税务信息。

1. 在[合作伙伴中心](https://partner.microsoft.com/dashboard)中，选择右上角的 **"帐户设置"** 图标，然后选择 **"开发人员设置**"。
2. 在左侧导航菜单中，选择 **"付款和税金**"，然后选择 **"付款和税金分配**"。

3. 选择要为其配置税务信息的程序和卖家 ID 组合。

4. 如果要使用现有税务配置文件，请从下拉列表中选择它。 否则，选择 **"创建新配置文件**"并按"**提交**"。 您将被带到税务档案页面。
5. 单击 **"编辑"** 按钮编辑您的税务信息。
6. 选择相应的单选按钮，如果出现提示，请选择您所在的国家/地区。 此步骤确定将用于在您的帐户上付款的 Microsoft 业务实体。

7. 根据您在步骤 6 中的选择，系统将提示您提供您所在国家/地区所需的税务信息。

> [!NOTE]
> 无论您的居住国还是公民权，您必须填写美国税务表格，才能通过商业市场销售任何优惠或附加服务。 满足某些美国居住要求的发布者必须填写 IRS W-9 表格。 美国以外的其他出版商必须填写 IRS W-8 表格。 可以在完成纳税配置文件时在线填写这些表单。

### <a name="withholding-rates"></a>预缴比率

你在税单中提交的信息将确定相应的预缴比率。 预缴比率仅适用于你在美国的销售；非美国地区的销售不受预缴约束。 预扣率各不相同，但对于大多数在美国境外注册的出版商，违约率为 30%。 如果你的国家和地区与美国签有所得税条约，你可以选择降低此税率。

### <a name="tax-treaty-benefits"></a>免税待遇

如果你在美国境外，你也许能够利用免税待遇。 这些福利因国家/地区而异，并可能允许您减少商业市场扣缴的税款。 你可以通过填写 W-8BEN 表格的第 II 部分来申请免税待遇。 我们建议你咨询所在国家或地区的相关人员，以确定是否可以享受这些优惠。

> [!NOTE]
> 不需要提供美国个人纳税识别号码（或 ITIN）即可从 Microsoft 接收付款或申请免税待遇。

## <a name="payout-account"></a>付款帐户

付款帐户是我们用于为你存入销售收益的银行帐户。 您可以查看您在"配置文件"页上输入的所有付款帐户。

> [!NOTE]
> 在某些市场中，PayPal 可用于付款帐户。 要了解特定市场是否支持 PayPal，请参阅[付款阈值、方法和时间范围](payment-thresholds-methods-timeframes.md)以及下面的[PayPal 信息](#paypal-info)，了解更多详细信息。

### <a name="create-a-payment-profile"></a>创建付款配置文件

1. 在[合作伙伴中心](https://partner.microsoft.com/dashboard)中，选择右上角的 **"设置**"齿轮图标，然后选择 **"开发人员"设置**。
2. 在 *"付款和纳税*"标题下，选择 **"付款"和"税务配置文件分配**"。

    > [!NOTE]
    > 由于这是敏感信息，系统可能提示你再次登录。

3. 选择要配置的付款方式。

4. 选择现有付款配置文件，或单击"**创建新付款配置文件**"以为所选付款方式创建新配置文件。

### <a name="create-a-bank-based-payment-profile"></a>创建基于银行的付款配置文件

如果您选择使用银行帐户接收付款，您将完成以下过程来配置您的银行帐户。

1. 在 *"银行配置文件"* 页上，提供有关您的银行的必要信息。
2. 提供您的银行账户详细信息。

    > [!NOTE]
    > 用于提供帐户信息的字段仅接受字母数字字符。

3. 提供受益人详细信息。
4. 返回 *"配置文件分配"* 页面，选择您希望我们在发放付款时使用的货币。

    > [!WARNING]
    > 确保您的银行接受您选择的支付货币。

5. 您需要为参与的每个计划选择付款配置文件，但您可以对多个程序使用相同的配置文件。

6. 单击"提交"以保存更改。

> [!NOTE]
> Microsoft 可能需要长达 48 小时才能验证您个人资料中的信息。 此过程完成后 *，验证状态*将显示 **"完成"**

为确保您的支出成功，请确保您：

- 在合作伙伴中心为您的支出帐户输入的**账户持有人姓名**必须与您的银行帐户完全相同。 例如，如果银行帐户名称包含中间名，则向“帐户持有者名称”**** 添加中间名。
- 付款会直接从 Microsoft 转移到你的银行帐户，货币单位为美元。
- 在合作伙伴中心输入的拉丁字符的银行信息将转换为西里尔字符。

### <a name="editing-existing-payment-profiles"></a>编辑现有付款配置文件

如果需要进行更改或更正任何不正确的信息，可以编辑现有的付款配置文件。

1. 在[合作伙伴中心](https://partner.microsoft.com/dashboard)中，选择右上角的 **"设置**"齿轮图标，然后选择 **"开发人员"设置**。
2. 在 *"付款和纳税*"标题下方，选择 **"付款"和"纳税档案**"。
3. 您的付款配置文件将连同其状态一起列出。 查找要编辑的配置文件，然后单击最右侧**的编辑**

> [!IMPORTANT]
> 更改付款帐户可能会推迟付款（最多一个付款周期）。 出现此推迟的原因是我们需要验证帐户更改，正如首次设置付款帐户时我们会进行验证。 帐户通过验证后，仍会收到全额款项；当前付款周期应得的任何付款将添加到下一个付款周期。 有关详细信息，请参阅[获得收入](get-paid.md)。

### <a name="paypal-info"></a>PayPal 信息

在选定的国家和地区中，您可以通过输入 PayPal 信息来创建付款帐户。 但是，在选择 PayPal 作为付款帐户选项之前：

- 检查[付款阈值、方法和时间范围](payment-thresholds-methods-timeframes.md)，以确认 PayPal 是否是您所在国家或地区支持的付款方式。
- 查看以下常见问题解答。 根据你的情况，PayPal 可能不是最适合你的付款帐户选项，银行帐户也许是首选。

关于使用 PayPal 作为付款方式的常见问题：

- **我需要有怎样的 PayPal 设置才能接收付款？** 确保您的PayPal帐户不会阻止电子支票付款。 此设置在 PayPal 的“付款接收首选项”页面中管理。 有关详细信息，请参阅 [PayPal 的帐户设置页面](https://go.microsoft.com/fwlink/p/?linkid=513139)。
- **我所在的国家/地区是否受支持？** 请参阅[付款阈值、方法和时间范围](payment-thresholds-methods-timeframes.md)，了解 PayPal 是受支持的付款方式的位置。
- **我的 PayPal 帐户是否必须与合作伙伴中心帐户在同一国家/地区注册？** 不是。 设置 PayPal 帐户时，可以接受默认配置。 您不应该有任何其他国家/地区和货币的问题，除非您阻止了某些货币的付款。 此设置在 PayPal 的“付款接收首选项”页面中管理。
- **我是否必须手动接受 PayPal 付款？** 不是。 PayPal 帐户默认设置为需要用户手动接受付款，这意味着如果你不在 30 天内接受付款，将返回付款。 你可以更改此设置，方法是在 PayPal 的“更多设置”页面中关闭“询问我”。
- **PayPal支持哪些货币？** 有关当前列表，请参阅[PayPal 的支持页面](https://developer.paypal.com/docs/classic/api/currency-codes/#paypal)

### <a name="specific-requirements-for-certain-countriesregions"></a>某些国家/地区的特定要求

在一些国家和地区中，必须遵循付款帐户的其他要求。 如果您是巴基斯坦、俄罗斯或乌克兰的居民，请注意以下要求。

#### <a name="pakistan"></a>巴基斯坦

表单 R 是巴基斯坦银行业法规要求。 它用于指示接收国外资金的目的和原因。 因此，每当你有资格获得来自 Microsoft 的每月付款时，你将需要向银行提交表单 R，然后付款才能发放到你的帐户。 有关如何获取表单 R 的副本的说明，请联系你的当地银行分支机构。

你将需要在你有资格获得付款的每个月都向银行提交表单 R。 例如，如果希望在一年中的每个月都接收付款，你将需要提交表单 R 12 次（每个月一次）。

付款提交到你的银行后，你有 30 天的时间来提交表单 R。 如果未在 30 天内提交，资金将返回到 Microsoft。

#### <a name="russia"></a>俄罗斯

如果您是居住在俄罗斯的出版商，您可能需要向银行提供文件，然后才能将资金存入您的帐户。 当你有资格获得付款时，我们将在电子邮件中为你提供以下文档：

- 验收证书 (AC) - 包含转移到你的帐户的付款金额。
- [Microsoft Azure 应用商店发布商协议](https://go.microsoft.com/fwlink/p/?LinkID=699560)- 需要反签名的发布者协议的已签名副本。

为确保您的支出成功，请确保您：

- 在合作伙伴中心为您的支出帐户输入的**账户持有人姓名**必须与您的银行帐户完全相同。 例如，如果银行帐户名称包含中间名，则向“帐户持有者名称”**** 添加中间名。
- 付款会直接从 Microsoft 转移到您的银行帐户，货币单位为卢比 (RUB)。
- 在合作伙伴中心输入的拉丁字符的银行信息将转换为西里尔字符。
- 付款必须支付到银行帐户而不是银行卡。

#### <a name="ukraine"></a>乌克兰

如果您是居住在乌克兰的出版商，您可能需要向银行提供文件，然后才能将资金存入您的帐户。 当你有资格获得付款时，我们将在电子邮件中为你提供以下文档：

- 验收证书 (AC) - 包含转移到你的帐户的付款金额。
- [Microsoft Azure 应用商店发布商协议](https://go.microsoft.com/fwlink/p/?LinkID=699560)- 需要反签名的发布者协议的已签名副本。
- 修订协议 (AA) - 你的银行可使用此文档帮助识别你的付款资金。

当尝试首次付款时，Microsoft 将提供所有三个文档。 对于任何后续付款，你将仅收到 AC 文档。 保留 ADA 和 AA 文档，以防您需要它们从您的银行接收未来付款。

### <a name="create-a-paypal-payment-profile"></a>创建 PayPal 付款配置文件

如果您选择使用银行帐户接收付款，您将完成以下过程来配置您的银行帐户。

1. 在*PayPal*页面上，提供有关您的PayPal帐户的必要信息。
2. 提供您的PayPal账户详情。

    > [!NOTE]
    > 用于提供帐户信息的字段仅接受字母数字字符。

3. 提供受益人详细信息。
4. 返回 *"配置文件分配"* 页面，选择您希望我们在发放付款时使用的货币。
5. 您需要为参与的每个计划选择付款配置文件，但您可以对多个程序使用相同的配置文件。
6. 单击"提交"以保存更改。
