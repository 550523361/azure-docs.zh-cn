## <a name="access-your-account-settings"></a>访问你的帐户设置

如果尚未这样做，应访问你 （或您组织的管理员）[帐户设置](https://partner.microsoft.com/dashboard/account/management)合作伙伴中心帐户以便为：
- 检查你的公司帐户验证状态
- 确认你的卖方 ID、 MPN ID、 发布者 ID 和联系信息，包括公司审批者和卖方联系人
- 设置公司的财务详细信息，包括税例外，如果相应
- 在合作伙伴中心将使用你的业务帐户的任何人为创建用户帐户

### <a name="open-developer-settings"></a>打开开发人员设置

帐户设置位于右上角你[商业应用商店仪表板](https://partner.microsoft.com/dashboard/commercial-marketplace)在合作伙伴中心。 选择齿轮图标 （靠近仪表板的右上角），然后选择 **开发人员设置**。 

![在合作伙伴中心帐户设置菜单](./media/dashboard-developer-settings.png)

内部**帐户设置**，你将能够查看你：
- **帐户详细信息**:帐户类型和帐户状态
- **发布服务器 Id**:卖方 ID、 用户 ID、 发布者 ID、 Azure AD 租户，等等。
- **联系信息**:发布者显示名称、 卖方联系人姓名、 电子邮件、 电话和地址
- **财务详细信息**:付款帐户、 计税配置文件，并吐钱保存状态
- **设备**:与你的帐户关联的任何测试设备
- **跟踪 Guid**:任何跟踪你的帐户与 Guid 关联

### <a name="account-details"></a>帐户详细信息

在帐户的详细信息部分中，可以看到基本信息，如你**帐户类型**（公司或个人） 和**验证状态**的你的帐户。 在帐户验证过程中，这些设置将显示必需的包括电子邮件验证、 雇佣关系验证和业务验证每个步骤。 此外可以更新你的电子邮件此处并根据需要重新发送验证。 

### <a name="publisher-ids"></a>发布服务器 Id

在发布服务器 Id 部分中，可以看到你**卖方 ID**， **MPN ID**，并**发布者 ID**。 这些值由 Microsoft 的分配用于唯一标识你的开发人员帐户，不能进行编辑。

### <a name="contact-info"></a>联系信息

在联系人信息部分中，可以看到你**发布者显示名称**，**卖方联系信息**（的联系人姓名、 电子邮件、 电话号码和地址公司卖方），和**公司审批者**（名称、 电子邮件和电话号码与颁发机构批准公司决策个人的）。 

### <a name="financial-details"></a>财务详细信息

在财务详细信息部分中，可以提供或更新您的财务信息，如果发布付费的应用、 加载项或服务。 

如果打算仅列出免费产品/服务，不需要将设置付款帐户或任何税务窗体中填写。 如果您改变主意，并决定要通过 Microsoft 销售，可以将设置付款帐户，并在该时间填写税务窗体。 

#### <a name="payout-account"></a>付款帐户

付款帐户是从你的销售将继续发送到的银行帐户。 此银行帐户必须在注册你的合作伙伴中心帐户所在的同一国家/地区中。

若要设置付款帐户，需要**将你的 Microsoft 帐户相关联**:
1. 在中**帐户设置**下**财务详细信息**部分中，选择**将你的 Microsoft 帐户相关联**。 
2. 出现提示时，对使用你的 Microsoft 帐户 (MSA) 进行签名。 此帐户不能已经是与另一个合作伙伴中心帐户相关联。 
3. 若要完成设置付款帐户，完全注销合作伙伴中心，然后重新登录你的 Microsoft 帐户 （而不是你的工作帐户）。 

现在，你的 Microsoft 帐户相关联时，若要添加付款帐户时，您将需要：
- **选择支付方式**:银行帐户或 PayPal
- **添加付款信息**:这可能包括选择帐户类型 （支票或储蓄） 中，输入帐户持有者姓名、 帐号和路由号、 帐单地址、 电话号码或 PayPal 的电子邮件地址。 \* 有关你帐户的付款方式，并了解是否支持你的市场区域中使用 PayPal 的详细信息，请参阅[PayPal 信息](https://docs.microsoft.com/windows/uwp/publish/setting-up-your-payout-account-and-tax-forms#paypal-info)。

> [!IMPORTANT]
> 更改付款帐户可能会推迟付款（最多一个付款周期）。 因为我们需要验证帐户更改，就像我们做的第一次设置付款帐户时，会发生这种延迟。 帐户通过验证后，仍会收到全额款项；当前付款周期应得的任何付款将添加到下一个付款周期。  

#### <a name="tax-profile"></a>计税配置文件

查看您当前的税务配置文件状态，确认正确**实体类型**并**税务证书信息**显示。 选择**编辑**若要更新或完成任何所需窗体。

若要建立税务状态，必须指定国家/地区的居住地和公民身份并完成与您所在国家/地区关联的相应的税务窗体。

无论居住或公民身份的国家/地区，必须填写美国纳税表单销售任何产品/服务通过 Microsoft。 满足某些美国驻留要求的合作伙伴必须填写 IRS W-9 窗体。 美国以外的其他合作伙伴必须填写 IRS W-8 窗体。 你可以填写这些窗体联机完成计税配置文件。

美国个人纳税人标识号 （或 ITIN） 不需要从 Microsoft 收到付款或声明税务条约好处。

您可以完成并提交税务窗体以电子方式在合作伙伴中心;在大多数情况下，不需要打印和邮件的任何窗体。

不同国家/地区和区域具有不同的税率要求。 必须在税款支付的那个取决于国家和地区销售产品/服务的位置。 Microsoft remits 销售和销项税代替您在某些国家/地区。 将列出产品/服务的过程中标识这些国家/地区。 在其他国家/地区，具体取决于你原先注册可能需要用于直接向本地的税务机构销售税务和汇出销售。 此外，销售继续收到可能应为收入纳税。 我们强烈建议您与相关的颁发机构联系您所在国家或区域可以最有效地帮助您确定您的 Microsoft 销售交易记录的右税务信息。

##### <a name="withholding-rates"></a>代扣税费率
在税务窗体中提交的信息确定相应的税务代扣税速率。 预缴速率仅适用于美国; 所做的销售额销售作用于非美国位置不受代扣税。 预缴率不同，但对于大多数开发人员注册在美国以外的默认速率为 30%。 可以减少此速率，如果国家/地区已同意在美国使用税率的所得税条约的选择。

##### <a name="tax-treaty-benefits"></a>税务条约优势
如果您是美国以外，你可能能够利用税务条约优势。 这些权益不同国家/地区到国家/地区，并且可能允许您以减少 Microsoft 预扣税款。 可以通过完成 W 8BEN 窗体的第 II 部分声明税务条约好处。 我们建议您与您所在国家或区域，以确定这些优势是否适用于您相应的资源通信。

[了解更多有关税务详细信息的 Windows 应用程序/游戏开发人员和 Azure Marketplace 发布者](https://docs.microsoft.com/windows/uwp/publish/tax-details-for-paid-apps)。

#### <a name="payout-hold-status"></a>付款保留状态

默认情况下，Microsoft 每月发送付款。 但是，可以选择要付款置于保持状态，这会阻止发送到你的帐户付款。 如果您选择付款置于保持状态，我们会继续录制任何收入，获得并提供中的详细信息您**付款摘要**。 但是，我们不会向你的帐户发送任何付款，直到你删除保留。 

若要将付款置于保持状态，请转到**帐户设置**。 下**财务详细信息**，在**付款保存状态**部分中，切换到滑块**上**。 可以在任何时候，更改您的付款保持状态，但请注意，你的决策将影响下一步的每月付款。 例如，如果你想要保存年 4 月的付款，请确保将您的付款保留状态设置为**上**年 3 月结束之前。

在付款设置后保存状态设置为**上**，所有付款都将保留，直到您切换滑块切换回**关闭**。 当你执行此操作时，就可以包含在下一个每月付款周期 （前提是已满足任何适用的付款阈值）。 例如，如果已经付款处于等待状态，但将需要能够在 6 月，生成付款，请确保切换付款持有状态设置为**关闭**5 月结束之前。

> [!NOTE]
> 你**付款保存状态**选项适用于**所有**通过 Microsoft 合作伙伴中心，包括 Azure Marketplace、 AppSource，Microsoft Store，广告等支付的收入来源。)。 不能选择为每个收入源不同的保留状态。

### <a name="devices"></a>设备

设备管理设置仅适用于 UWP 发布。 [了解详细信息](https://docs.microsoft.com/windows/uwp/publish/manage-account-settings-and-profile#additional-settings-and-info)。

### <a name="tracking-guids"></a>跟踪 Guid

全局唯一标识符 (Guid) 是可用于跟踪 Azure 使用情况的唯一参考编号 （具有 32 个十六进制数字）。 

若要创建用于跟踪的 GUID，应使用 GUID 生成器。 Azure 存储团队已创建 [GUID 生成器窗体](https://aka.ms/StoragePartners)，它将通过电子邮件向你发送格式正确的 GUID，并可在不同的跟踪系统中重复使用。

建议为每个产品/服务和每个产品的分销渠道创建唯一的 GUID。 如果不希望拆分报告，则可以选择对产品的多个分销渠道使用单个 GUID。

如果使用模板部署产品并且产品在 Azure 市场和 GitHub 中都提供，则可以创建并注册 2 个不同的 GUID：

*   Azure 市场中的产品 A
*   GitHub 中的产品 A

报告根据合作伙伴值（Microsoft 合作伙伴 ID）和 GUID 来完成。 此外可以跟踪对齐到产品/服务中的每个计划更高粒度级别的 Guid。

有关详细信息，请参阅[跟踪 Azure 客户使用情况的 Guid 常见问题解答](https://docs.microsoft.com/azure/marketplace/azure-partner-customer-usage-attribution#faq)。



## <a name="multi-user-account-management"></a>多个用户帐户管理

合作伙伴中心利用[Azure Active Directory](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-whatis) (Azure AD) 为多个用户帐户访问和管理。 组织的 Azure AD 注册过程的一部分是自动与你的合作伙伴中心帐户相关联。 

## <a name="manage-users"></a>管理用户

**用户**合作伙伴中心部分 (下**帐户设置**) 让我们使用 Azure AD 管理用户、 组和合作伙伴中心帐户拥有访问权限的 Azure AD 应用程序。 请注意，为了管理用户，您必须登录使用您[工作帐户](./company-work-accounts.md)（关联的 Azure AD 租户）。 若要管理不同的工作帐户中的用户 / 租户中，你将需要注销，然后以用户身份重新登录**Manager**权限的工作帐户 / 租户。 

后登录你的工作帐户 （Azure AD 租户） 时，你可以：
- [添加或删除用户](#add-or-remove-users)
- [更改用户密码](#change-a-user-password)
- [添加或删除组](#add-or-remove-users)
- [添加或删除 Azure AD 应用程序](#add-new-azure-ad-applications)
- [管理 Azure AD 应用程序密钥](#manage-keys-for-an-azure-ad-application)
- [定义用户角色和权限](#define-user-roles-and-permissions)


请注意，所有合作伙伴中心用户 （包括组和 Azure AD 应用程序） 必须在都具有活动的工作帐户[Azure AD 租户](#manage-tenants)在合作伙伴中心帐户与该键相关联。 

### <a name="add-or-remove-users"></a>添加或删除用户

你的帐户必须具有[**管理器级别**](#define-user-roles-and-permissions)权限[工作帐户 （Azure AD 租户）](./company-work-accounts.md)中想要添加或编辑用户。

#### <a name="add-existing-users"></a>将现有用户添加

若要将用户添加到您的合作伙伴中心帐户中已存在于你的公司[工作帐户 （Azure AD 租户）](./company-work-accounts.md):

1. 转到**用户**(下**帐户设置**)，然后选择**将用户添加**。
2. 从显示的列表中选择一个或多个用户。 可以使用搜索框搜索特定用户。
\* 如果选择多个用户将添加到合作伙伴中心帐户，你必须将它们分配相同的角色或自定义权限集。 若要添加多个用户具有不同的角色权限，请为每个角色或自定义权限集重复这些步骤。
3.  在完成选择用户，请单击**添加所选**。
5.  在中**角色**部分中，指定的角色或所选用户的自定义的权限。
6.  选择**保存**。

#### <a name="create-new-users"></a>创建新用户

若要创建全新的用户帐户，您必须具有的帐户[**全局管理员**](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles)权限。 

1. 转到**用户**(下**帐户设置**)，选择**将用户添加**，然后选择**创建新用户**。
1. 为每个新的用户输入名字、 姓氏和用户名。 
1. 如果您希望新的用户在你组织的目录中拥有全局管理员帐户，选中复选框标记为**使此用户的全局管理员在 Azure AD 中，对所有目录资源具有完全控制**。 这将使你的公司中的所有管理功能的 Azure AD 用户完全访问权限。 他们将能够添加和管理组织的工作帐户 （Azure AD 租户） 中的用户不在合作伙伴中心，但除非向帐户授予适当的角色/权限。 
1. 如果选中的框**使此用户的全局管理员**，将需要提供**电子邮件密码恢复**要恢复其密码，如有必要的用户。
1. 在**组成员身份**部分中，选择任何希望新用户加入的组。
1. 在中**角色**部分中，指定的角色或用户的自定义的权限。
1. 选择**保存**。

在合作伙伴中心中创建新的用户还将创建该用户的帐户与你登录到工作帐户 （Azure AD 租户） 中。 对合作伙伴中心中的用户的名称进行更改将在你组织的工作帐户 （Azure AD 租户） 中进行相同的更改。

#### <a name="invite-new-users-by-email"></a>通过电子邮件邀请新用户

若要邀请用户当前不是你公司的工作帐户 （Azure AD 租户） 通过电子邮件的一部分，你必须具有的帐户[**全局管理员**](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles)权限。 

1. 转到**用户**(下**帐户设置**)，选择**将用户添加**，然后选择**通过电子邮件邀请用户**。
2. 输入一个或多个电子邮件地址 （最多 10)，由逗号或分号分隔。
3. 在中**角色**部分中，指定的角色或用户的自定义的权限。
4. 选择**保存**。

你受邀用户将收到电子邮件邀请以加入你的合作伙伴中心帐户。 将你的工作帐户 （Azure AD 租户） 中创建新的来宾用户帐户。 每个用户将需要接受其邀请才能访问你的帐户。

如果你需要重新发送邀请，请访问**用户**页上，找到该邀请的用户列表中，选择其电子邮件地址 (或显示的文本*挂起的邀请*)。 然后，在页面底部，选择**重新发送邀请**。
 

> [!NOTE]
> 如果你的组织使用[目录集成](https://go.microsoft.com/fwlink/p/?LinkID=724033)若要同步的本地目录服务与 Azure AD，你将无法在合作伙伴中心中创建新用户、 组或 Azure AD 应用程序。 你 （或另一个管理员，你的本地目录中） 将需要创建它们直接在本地目录中，你将能够查看并将其添加在合作伙伴中心之前。

#### <a name="remove-a-user"></a>删除用户

若要从你的工作帐户 （Azure AD 租户） 中删除用户，请转到**用户**(下**帐户设置**)，选择用户想要删除在最右侧列中，使用相应的复选框，然后选择**删除**从可用操作。 弹出窗口将出现，以确认你想要删除所选的用户。

#### <a name="change-a-user-password"></a>更改用户密码

如果您的某个用户需要更改其密码，可以实现此本身如果提供**电子邮件密码恢复**时创建的用户帐户。 此外可以通过执行以下步骤来更新用户的密码。 若要更改你的公司工作帐户 （Azure AD 租户） 中的用户的密码，您必须登录使用的帐户上[**全局管理员**](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles)权限。 请注意，这将更改你的 Azure AD 租户中的用户的密码，他们用来访问合作伙伴中心以及密码。

1.  从**用户**页 (下**帐户设置**)，选择你想要编辑的用户帐户的名称。
2.  选择**重置密码**在页面底部的按钮。
3.  将显示一个确认页面，显示用户，包括临时密码的登录信息。 如您将无法访问临时密码，在您离开此页后，则请确保打印或复制此信息并将其提供给用户。


## <a name="manage-groups"></a>管理组

组可以控制多个用户角色和权限的所有内容。

#### <a name="add-an-existing-group"></a>添加现有的组

若要添加的组，它已存在你组织的工作帐户 （Azure AD 租户） 到合作伙伴中心帐户： 

1.  从**用户**页 (下**帐户设置**)，选择**将组添加**。
2.  从显示的列表中选择一个或多个组。 可以使用搜索框来搜索特定组。
如果选择多个组将添加到合作伙伴中心帐户，你必须将它们分配相同的角色或自定义权限集。 若要添加多个具有不同的角色权限组，请为每个角色或自定义权限集重复这些步骤。
3.  在完成选择组，请单击**添加所选**。
4.  在中**角色**部分中，指定的角色或所选的组的自定义的权限。 组的所有成员将都能够访问合作伙伴中心帐户的权限应用于组，而不考虑的角色和权限与其单个帐户相关联。

#### <a name="add-a-new-group"></a>添加新组

## <a name="manage-azure-ad-applications"></a>管理 Azure AD 应用程序

您可以允许应用程序或服务属于公司的 Azure AD 访问合作伙伴中心帐户。 

#### <a name="add-existing-azure-ad-applications"></a>添加现有的 Azure AD 应用程序 

若要添加的应用程序中已存在你公司的 Azure Active Directory: 

1.  从**用户**页 (下**帐户设置**)，选择**将 Azure AD 应用程序添加**。
2.  从显示的列表中选择一个或多个 Azure AD 应用程序。 您可以使用搜索框搜索特定于 Azure AD 应用程序。 如果选择多个 Azure AD 应用程序将添加到合作伙伴中心帐户，你必须将它们分配相同的角色或自定义权限集。 若要添加多个 Azure AD 应用程序具有不同的角色权限，请为每个角色或自定义权限集重复这些步骤。
3.  完成选择 Azure AD 应用程序后，请单击**添加选定项**。
5.  在中**角色**部分中，指定的角色或自定义的权限为所选 Azure AD 应用程序。
6.  选择**保存**。

#### <a name="add-new-azure-ad-applications"></a>添加新的 Azure AD 应用程序 

如果你想要授予合作伙伴中心访问权限的全新的 Azure AD 应用程序的帐户，即可创建一个**用户**部分。 请注意，这将不只是在合作伙伴中心帐户中创建你的公司工作帐户 （Azure AD 租户） 中的新帐户。 如果您主要使用此 Azure AD 应用程序对于合作伙伴中心身份验证，且不需要用户直接访问，可以输入任何有效的地址作为**回复 URL**并**应用程序 ID URI**，只要为这些值不是使用任何其他 Azure AD 应用程序目录中。

1.  从**用户**页 (下**帐户设置**)，选择**将 Azure AD 应用程序添加**。
2.  在下一页上，选择**新建 Azure AD 应用程序**。
3.  输入新的 Azure AD 应用程序的**回复 URL**。 这是用户可以登录并使用 Azure AD 应用程序 （有时也称为应用 URL 或单一登录 URL） 的 URL。 **回复 URL**不能超过 256 个字符，并且必须是唯一在你的目录。
4.  输入新 Azure AD 应用程序的**应用程序 ID URI**。 这是 Azure AD 应用程序时的单一登录请求发送到 Azure AD 呈现的逻辑标识符。 请注意，**应用程序 ID URI**必须是唯一的目录中每个 Azure AD 应用程序。 此 ID 不能超过 256 个字符。 有关应用程序 ID URI 的详细信息，请参阅[将应用程序集成与 Azure Active Directory](https://docs.microsoft.com/azure/active-directory/develop/quickstart-modify-supported-accounts#change-the-application-registration-to-support-different-accounts)。
5.  在中**角色**部分中，指定的角色或 Azure AD 应用程序的自定义的权限。
6.  选择**保存**。

添加或创建一个 Azure AD 应用程序后，您可以返回到**用户**部分，并选择要查看应用程序，包括租户 ID、 客户端 ID、 回复 URL 和应用程序 ID URI 设置的应用程序名称。

#### <a name="remove-an-application"></a>删除应用程序

若要从你的工作帐户 （Azure AD 租户） 中删除应用程序，请转到**用户**(下**帐户设置**)，选择你想要删除在最右侧列中，使用相应的复选框的应用程序然后选择**删除**从可用操作。 弹出窗口将出现，以确认你想要删除所选应用程序。

#### <a name="manage-keys-for-an-azure-ad-application"></a>管理 Azure AD 应用程序密钥

如果 Azure AD 应用程序读取和写入 Microsoft Azure AD 中的数据，它将需要一个密钥。 可以通过编辑其合作伙伴中心中的信息来创建 Azure AD 应用程序的密钥。 此外可以删除不再需要的密钥。

1.  从**用户**页 (下**帐户设置**)，选择 Azure AD 应用程序的名称。 您将看到所有 Azure AD 应用程序，包括对该密钥的创建和它何时到期日期的活动密钥。 
2. 若要删除不再需要的密钥，请选择**删除**。
3.  若要添加新的密钥，请选择**添加新的密钥**。
4.  你将看到屏幕显示**客户端 ID**并**键值**。 如您将无法在您离开此页后再次访问它，则请确保打印或复制此信息。
4.  如果你想要创建多个密钥，请选择**添加另一个密钥**。

## <a name="manage-tenants"></a>管理租户

一个 Azure Active Directory (AD) 租户，也称为在本文档中，"工作帐户"是组织的你的表示形式在 Azure 门户中进行设置，以及可帮助你管理为你内部的 Microsoft 云服务的特定实例和外部用户。 如果你的组织订阅的 Microsoft 云服务，例如 Azure、 Microsoft Intune 或 Office 365 为你建立一个 Azure AD 租户。 

可以设置多个租户使用合作伙伴中心。 具有的任何用户**Manager**合作伙伴中心帐户中的角色将具有添加和删除帐户中的 Azure AD 租户的选项。  

### <a name="add-an-existing-tenant"></a>添加现有的租户

若要将另一个 Azure AD 租户与你的合作伙伴中心帐户相关联：

1.  从**租户**页 (下**帐户设置**)，选择**将另一个 Azure AD 租户相关联**。
2. 输入你想要关联的租户的 Azure AD 凭据。
3. 查看你的 Azure AD 租户的组织和域的名称。 若要完成该关联，请选择**确认**。

如果关联是成功，你将准备好添加和管理帐户中的用户**用户**合作伙伴中心中的部分。

### <a name="create-a-new-tenant"></a>创建新租户

若要使用合作伙伴中心帐户中创建品牌新的 Azure AD 租户：

1.  从**租户**页 (下**帐户设置**)，选择**创建的新 Azure AD 租户**。
2. 输入新的 Azure AD 的目录信息：
    - **域名**：我们将使用你的 Azure AD 域，并使用的唯一名称"。 onmicrosoft.com"。 例如，如果您输入"示例"，你的 Azure AD 域将是"example.onmicrosoft.com"。
    - **联系人电子邮件**:电子邮件地址，其中我们可以与您联系有关你的帐户有必要。
    - **全局管理员用户帐户信息**:第一个名称、 最后一个名称、 用户名和你想要使用新的全局管理员帐户的密码。
3. 选择**创建**以确认新的域和帐户信息。
4. 登录新的 Azure AD 全局管理员用户名和密码以开始[添加和管理用户](#manage-users)。

有关创建新租户在 Azure 门户中，而不是通过合作伙伴中心门户的详细信息，请参阅文章[在 Azure Active Directory 中创建新租户](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-access-create-new-tenant)。

### <a name="remove-a-tenant"></a>删除租户

若要在合作伙伴中心帐户中删除一个租户，查找其名称在**租户**页 (在**帐户设置**)，然后选择**删除**。 系统将提示您确认要删除租户。 一旦您这样做，该租户中的任何用户将能够登录到合作伙伴中心帐户，并将删除任何已配置为这些用户的权限。

当您删除某个租户时，已添加到合作伙伴中心帐户从该租户的所有用户将不再能够登录到帐户。

> [!TIP]
> 如果你当前登录到合作伙伴中心使用同一租户中的帐户，不能删除租户。 若要删除某个租户，你必须登录到合作伙伴中心作为**Manager**与帐户相关联的另一个租户。 如果只有一个与帐户关联的租户，该租户后，可以仅删除使用建立该帐户的 Microsoft 帐户登录。
