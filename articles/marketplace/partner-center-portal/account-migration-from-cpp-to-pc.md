# <a name="account-migration-from-cloud-partner-portal-to-partner-center"></a>从云合作伙伴门户的帐户迁移到合作伙伴中心

产品/服务迁移时通过从云合作伙伴门户 (CPP) 到合作伙伴中心 (PC)，它们被锁定在 CPP 中进行编辑。 此时，需要通过迁移到合作伙伴中心帐户设置。  你的帐户设置，以及产品/服务，可在合作伙伴中心中管理。

## <a name="account-migration-process"></a>帐户迁移过程

产品/服务迁移时通过从 CPP，你的帐户为迁移配置。 
 
如果您具有在 CPP 中对于给定的帐户的所有者角色的用户，在发布者个人资料页上显示黄色的横幅。  需要移到你的帐户设置到合作伙伴中心。 

单击横幅以启动帐户迁移过程。 应输入以下项：

### <a name="work-e-mail-address"></a>**工作电子邮件地址**

在大多数情况下，使用你用于登录到 CPP 的电子邮件地址登录。 在某些情况下，必须使用不同的电子邮件地址：

* Microsoft 帐户：如果 CPP 帐户是一个 Microsoft 帐户，然后您需要输入有效的工作电子邮件地址与租户中，为其注册的 MPN ID 相关联。 

* 租户不匹配：如果你的工作电子邮件地址不属于与 Microsoft 合作伙伴网络 ID 上 CPP 帐户相关联的租户，然后将看到一个错误。 若要移动在出现此错误，请输入与租户相关联的电子邮件地址。 一条错误消息将提供的租户的名称。 

### <a name="sign-up-for-microsoft-partner-network-program"></a>注册 Microsoft 合作伙伴网络计划

如果您 CPP 的帐户不具有 Microsoft 合作伙伴网络 ID，或具有一个无效，必须将注册参加 Microsoft 合作伙伴网络计划激活过程的一部分。

## <a name="account-activation-is-complete"></a>帐户激活已完成

帐户迁移需要发生这种情况对于给定的帐户的一次。 给定的伙伴完成帐户迁移，所有所有者将其发布者个人资料页上都看到此行为：

1. 要查看 Microsoft 合作伙伴网络，你可以在其中管理 Microsoft 合作伙伴的帐户设置中的合作伙伴设置页。 
1. 帐户迁移完成后，发布者个人资料页上黄色的横幅会弹出到属于给定帐户所有者角色在 CPP 中要求他们来管理其帐户设置在合作伙伴中心中的用户。 
1. 帐户设置页在 CPP 中的然后转换为只读模式。
