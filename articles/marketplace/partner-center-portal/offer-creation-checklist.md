# <a name="offer-creation-checklist"></a>产品/服务创建清单

产品/服务创建过程将引导您完成多个页面。 以下是可以提供有关每个页面，包含一个链接，深入了解每个项的详细信息。

系统要求提供，或指定的项是如下所示。 具有默认值提供，可以根据需要更改或某些区域是可选的。 您无需在此处列出的顺序在这些部分上处理。

| **Item**    | **用途**  |
| :---------- | :-------------------|
| [**新产品/服务模式**](#new-offer-modal) | 收集提供标识信息。  |
| [产品/服务安装程序页](#offer-setup-page) | 可以选择在使用主要功能，并选择销售产品/服务通过 Microsoft 的方式。  |
| [属性页](#properties-page) | 定义的类别和行业用于分组产品/服务市场，支持产品/服务和你的应用程序版本的法律协定上。 |
| [产品/服务列表页](#offer-listing-page) | 定义要显示在 marketplace 中包括的产品/服务说明和市场营销资产的产品/服务详细信息。 |
| [预览页](#preview-page) | 为发布产品/服务发布到更广泛的 marketplace 用户实时产品/服务之前定义有限的预览受众。 |
| [产品/服务技术的配置页](#technical-configuration-page)  | 仅当选择销售通过 Microsoft 的产品/服务可用。 定义用于连接到你的产品/服务的技术细节 （URL 路径、 webhook、 租户 ID 和应用程序 ID）。 |
| [**新计划模式**](#plan-identity-modal) | 收集计划的标识信息。  |
| [计划列表页](#plan-listing-page)  | 仅当选择销售通过 Microsoft 的产品/服务可用。 定义用于列出在 marketplace 中的计划的详细信息。  |
| [计划定价和可用性页面](#plan-pricing--availability-page)  | 仅当选择销售通过 Microsoft 的产品/服务可用。  收集业务特征 （定价模型）、 受众和市场的产品/服务的每个计划 （版本） 的可用性。  |
| [测试驱动器列表页](#test-drive-listing-page)  | 仅当你选择提供产品/服务的体验版才可用。 定义用于对列表在 marketplace 中试用的详细信息。  |
| 测试驱动器技术配置页  | 仅当你选择提供产品/服务的体验版才可用。 定义的技术详细信息演示 （或"试用"） 将使客户试用产品/服务提交之前来购买它。  |
| [查看和发布页](#review-and-publish-page)  | 选择你想要发布、 查看每个页的状态并提供对认证团队的说明的更改。  |


## <a name="new-offer-modal"></a>新产品/服务模式 

您将需要提供的第一个部分是信息的一个名称和产品/服务的 ID。 

| **字段名称**    | **说明**   |  
| :---------------- | :-----------| 
| 产品/服务 ID  | 必需，不能在创建后更改。 最多 50 个字符，必须仅包含小写字母、 字母数字字符、 短划线或下划线。 |
| 产品名称  | 必需。 |

## <a name="offer-setup-page"></a>产品/服务安装程序页

产品/服务安装程序页是其中你可以选择使用不同的通道和销售动作，以及声明的关键功能，例如测试驱动器及其对客户使用潜在顾客。 

| **字段名称**    | **说明**   | 
| :---------------- | :-----------|  
| 想要通过 Microsoft 销售？  | 必需。 默认值：是 |
| 你希望如何与产品/服务进行交互的潜在客户列出了？ （行动号召）  | 如果不通过 Microsoft 销售，必需。 默认值：免费试用版，选项："立即获取"、"免费试用"，"与我联系。" |
| 试用 URL  | 选择所需如果"免费试用版"是，在方式客户应与产品/服务列表进行交互。 |
| 优惠 URL  | 如果选择必填"它立即获取"是，在方式客户应与产品/服务列表进行交互 |
| 声道  | 可选。 默认值：未选择加入 CSP （经销商） 通道。  |
| 体验版 | 可选。 默认值：启用没有体验版。  |
| 体验版类型 | 如果启用试用，必需。 默认值：未选择任何项。 选项：Azure 资源管理器、 Dynamics 365 业务中心、 Dynamics 365 for Customer Engagement，对于操作，逻辑应用的 Dynamics 365 的 Power BI。  |
| 潜在客户管理-连接到 CRM 系统 | 需要通过 Microsoft，销售还是列表提供了"与我联系。" 默认值： 没有连接 CRM 系统。 CRM 选项：Azure 表，Azure blob、 Dynamics CRM online，HTTPs 的终结点，Marketo、 Salesforce  |

## <a name="properties-page"></a>“属性”页

属性页是你定义的类别和行业用于分组产品/服务市场，支持产品/服务和你的应用程序版本的法律协定上。 请务必在此页上，提供有关产品/服务的完整且准确详细信息，以便它具有相应地进行显示，并且提供给一组合适的客户。 

| **字段名称**    | **说明**   | 
| :---------------- | :-----------|  
| 类别和子类别 | 所需 1，最大值 3。 默认值：未选择任何项。 |
| 行业和 subindustries | 可选。 最大 2 L1 行业和最大 2 subindustries 内每 L1 行业，默认值：未选择 |
| 应用版本  | 可选。 默认值：无。 |
| 使用标准约定  | 可选。 默认值： 未选中。  | |
| 使用条款  | 如果不选择标准协定所需。  |

## <a name="offer-listing-page"></a>产品/服务列表页

列表页是其中提供的文本和映像的客户会看到在 marketplace 中查看产品/服务的列表时。 

| **字段名称**    | **说明**   |
| :---------------- | :-----------| 
| Name  | 必需，最大 50 个字符。 |
| 摘要  | 所需的最大值超过 100 个字符。 | 
| 描述  | 必需，最大 3000 个字符。 |
| 入门说明  | 必需，最大 3000 个字符。 |
| 入门说明  | 必需，最大 3000 个字符。 |
| 搜索关键字  | 可选但建议，最大的 3 个关键字。 |
| 隐私策略 URL  | 必需。 |
| CSP 计划市场营销材料 URL  | 可选。 |
| 有用的链接标题 + URL  | 可选。 |
| 支持文档标题 + 文件  | 必需的最小值 1，并最大值 3。 必须为 PDF 文件格式。 |
| 屏幕截图  | 必需的最小值 1 屏幕截图和最大 5;四个或更多建议。 必须为 1280 X 720 的 PNG 格式。 |
| 存储徽标 (小、 中、 大、 宽，特大)  | 小型 (48 X 48) 和大型 (216x216) 所需;其他可选的但建议的大小：中型 (90 x 90) 范围内 (255 x 115)，Hero (815 x 290)。 必须为 PNG 格式。 |
| 视频名称 + URL + 缩略图  | 可选但建议，最大 4 个视频。 缩略图必须为 1280 x 720 PNG 格式。 视频必须托管在 YouTube 或 Vimeo。 |
| 联系人 （CSP 计划，工程、 支持）  | 工程和支持联系所需 （名称、 电子邮件和电话号码）;CSP 计划联系人可选但建议这样做。 |
| 支持 URL  | 必需。 |

## <a name="preview-page"></a>预览页

预览页是指定访问群体以有权访问你的产品/服务预览，以验证该产品/服务满足所有要求，它在运行之前的位置。 

| **字段名称**    | **说明**   | 
| :---------------- | :-----------| 
| AAD/MSA 电子邮件 + 说明 | 所需，最小值 1 和最大 10 如果输入最多 20 个手动执行，或者，如果上传 CSV 文件。 |

## <a name="technical-configuration-page"></a>技术配置页 

技术配置页是你在其中指定 Microsoft 用于连接到你的产品/服务的技术详细信息。 如果你决定不通过 Microsoft 销售，此页不是对你可见。

| **字段名称**    | **说明**   |  
| :---------------- | :-----------| 
| 登陆页 URL | 如果通过 Microsoft 销售，必需。 |
| 连接 webhook | 如果通过 Microsoft 销售，必需。 |
| Azure AD 租户 ID | 如果通过 Microsoft 销售，必需。 |
| Azure AD 应用 ID | 如果通过 Microsoft 销售，必需。 |

## <a name="plan-identity-modal"></a>计划标识模式

您需要提供的第一个部分是信息的名称和你的计划的 ID。 如果您已决定不通过 Microsoft 销售，此页不是对你可见。

| **字段名称**    | **说明**   |  
| :---------------- | :-----------| 
| 计划 ID  | 如果通过 Microsoft 销售，必需。 创建后不能更改它。 最多 50 个字符，必须仅包含小写字母、 字母数字字符、 短划线或下划线。 |
| 计划名称  | 如果通过 Microsoft 销售，必需。 必须是唯一的产品/服务中的所有计划。 最多 50 个字符。 |

## <a name="plan-listing-page"></a>计划列表页

列出页计划为在其中提供客户在 marketplace 中查看该计划时看到的文本。 如果你决定不通过 Microsoft 销售，此页不是对你可见。

| **字段名称**    | **说明**   |  
| :---------------- | :-----------| 
| 计划说明   | 如果通过 Microsoft 销售，必需。 最大 500 个字符。 | |

## <a name="plan-pricing--availability-page"></a>计划定价和可用性页

在计划的定价和可用性页是你定义的业务特征、 受众和产品/服务的每个计划 （版本） 的市场可用性。 如果你决定不通过 Microsoft 销售，此页不是对你可见。

| **字段名称**    | **说明**   | 
| :---------------- | :-----------| 
| 市场可用性  | 必需的最小值 1，并最大 141。 |
| 定价模型  | 必需。 默认值：固定的费率。 选项：固定的费率，每个用户。 |
| 最小值和最大座位数  | 可选，仅当基于席位的定价模型选择可用。 |
| 计费术语  | 必需。 默认值：每月。 选项：每月、 年。 |
| 价格  | 每个月，每月费用字词; 如果需要的美元或每年如果年度美元计费所选术语。 |
| 计划的受众  | 可选。 默认值：公共计划。 选项：公共、 私有按租户 ID |
| 受限制的计划受众 （租户 ID 和说明）  | 如果私有计划选择所需。 如果手动输入的最小值 1 和最大 10 租户 Id。 如果 CSV 文件导入的最大 20000。 |
