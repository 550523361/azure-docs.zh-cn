# <a name="saas-offer-creation-checklist"></a>SaaS 产品/服务创建清单

SaaS 产品创建过程将带您浏览多个页面。 以下是您可以在每个页面上提供的详细信息，以及用于了解有关每个项目的详细信息的链接。

下面列出了您需要提供或指定的项目。 某些区域是可选的，或者提供了默认值，您可以根据需要进行更改。 您不必按此处列出的顺序处理这些部分。

| **项目**    | **用途**  |
| :---------- | :-------------------|
| [**新报价模式**](#new-offer-modal) | 收集提供身份信息。  |
| [优惠设置页面](#offer-setup-page) | 允许您选择使用关键功能，并选择如何通过 Microsoft 销售您的产品/服务。  |
| [属性页](#properties-page) | 定义用于在市场对产品/服务进行分组的类别和行业、支持产品/服务的法律合同以及应用版本。 |
| [优惠列表页面](#offer-listing-page) | 定义要在市场上显示的要约详细信息，包括您的报价和营销资产的描述。 |
| [预览页面](#preview-page) | 在将优惠实时发布给更广泛的市场受众之前，定义一个有限的预览受众，以便发布您的优惠。 |
| [提供技术配置页面](#technical-configuration-page)  | 仅当您选择通过 Microsoft 销售产品/服务时，才可用。 定义用于连接到产品/服务的技术详细信息（URL 路径、Webhook、租户 ID 和应用 ID）。 |
| [**新计划模式**](#plan-identity-modal) | 收集计划标识信息。  |
| [计划列表页](#plan-listing-page)  | 仅当您选择通过 Microsoft 销售产品/服务时，才可用。 定义用于在市场上列出计划的详细信息。  |
| [计划定价&可用性页面](#plan-pricing--availability-page)  | 仅当您选择通过 Microsoft 销售产品/服务时，才可用。  收集每个计划（版本）的业务特征（定价模式）、受众和市场可用性。  |
| [试驾列表页面](#test-drive-listing-page)  | 仅在您选择提供产品/服务试驾时才可用。 定义用于在市场中列出试驾的详细信息。  |
| 试驾技术配置页面  | 仅在您选择提供产品/服务试驾时才可用。 定义演示（或"试驾"）的技术细节，使客户能够在承诺购买之前试用您的产品/服务。  |
| [查看和发布页面](#review-and-publish-page)  | 选择要发布的更改，查看每个页面的状态，并向认证团队提供注释。  |


## <a name="new-offer-modal"></a>新报价模式 

您将被要求提供的第一条信息是您的优惠的 ID 和别名。 

| **字段名**    | **说明**   |  
| :---------------- | :-----------| 
| 产品/服务 ID  | 必需，在创建后无法更改。 最大 50 个字符，必须仅包含小写、字母数字字符、破折号或下划线。 |
| 优惠别名  | 必需。 |

## <a name="offer-setup-page"></a>优惠设置页面

产品/服务设置页面是您可以选择进入不同渠道和销售运动，以及声明使用关键功能（如试驾和客户潜在顾客）的地方。 

| **字段名**    | **说明**   | 
| :---------------- | :-----------|  
| 您想通过微软销售吗？  | 必需。 默认值：是 |
| 您希望潜在客户如何与产品/服务列表进行交互？ （行动号召）  | 如果无法通过 Microsoft 销售，则是必需的。 默认：免费试用，选项："立即获取"，"免费试用"，"联系我"。 |
| 试用 URL  | 如果选择"免费试用"，则需要作为客户与产品/服务列表交互的方式。 |
| 提供 URL  | 如果选择"立即获取"，则是必需的，因为客户应与产品/服务列表进行交互 |
| 声道  | 可选。 默认值：未选择进入 CSP（经销商）通道。  |
| 体验版 | 可选。 默认值：未启用试驾。  |
| 体验版类型 | 如果启用了试驱动器，则是必需的。 默认值：未选择。 选项：Azure 资源管理器、面向业务中心的动态 365、客户参与的动态 365、操作动态 365、逻辑应用、Power BI。  |
| 潜在客户管理 - 连接到 CRM 系统 | 如果通过 Microsoft 销售，或者如果上市报价为"与我联系"，则需要这样做。 默认值：未连接 CRM 系统。 CRM 选项：Azure 表、Azure Blob、动态 CRM 联机、HTTP 的终结点、市场、销售队伍  |

## <a name="properties-page"></a>“属性”页

属性页是定义用于在市场对产品/服务进行分组的类别和行业、支持产品/服务的法律合同以及应用版本。 请务必在此页面上提供有关您的优惠的完整和准确的详细信息，以便适当显示并提供给正确的客户。 

| **字段名**    | **说明**   | 
| :---------------- | :-----------|  
| 类别和子类别 | 要求 1 和最大 3。 默认值：未选择。 |
| 工业和子工业 | 可选。 每个 L1 行业中最多 2 个 L1 工业和最多 2 个子行业，默认值：未选择任何 |
| 应用版本  | 可选。 默认值：无。 |
| 使用标准合同  | 可选。 默认值：未选中。  | |
| 使用条款  | 如果未选择标准合同，则为必填项。  |

## <a name="offer-listing-page"></a>优惠列表页

列表页是提供客户在市场中查看产品/服务列表时看到的文本和图像的位置。 

| **字段名**    | **说明**   |
| :---------------- | :-----------| 
| “属性”  | 需要，最多 50 个字符。 |
| 总结  | 需要，最多 100 个字符。 | 
| 描述  | 需要，最多 3000 个字符。 |
| 入门说明  | 需要，最多 3000 个字符。 |
| 入门说明  | 需要，最多 3000 个字符。 |
| 搜索关键字  | 可选、推荐、最多 3 个关键字。 |
| 隐私策略 URL  | 必需。 |
| CSP 计划营销材料 URL  | 可选。 |
| 有用的链接 标题和 URL  | 可选。 |
| 支持文档标题和文件  | 需要，最小 1 和最大 3。 必须为 PDF 文件格式。 |
| 屏幕截图  | 需要，最小 1 屏幕截图和最多 5 个;建议使用四个或更多。 必须为 1280 X 720，采用 PNG 格式。 |
| 商店徽标（小、中、大、宽、英雄）  | 需要小 （48 X 48） 和大 （216 X 216）;其他尺寸可选，但推荐： 中等 （90 x 90）， 宽 （255 x 115）， 英雄 （815 x 290）. 必须采用 PNG 格式。 |
| 视频名称 + URL + 缩略图  | 可选，推荐，最多4个视频。 缩略图必须为 1280 x 720，采用 PNG 格式。 视频必须托管在 YouTube 或 Vimeo 中。 |
| 联系人（CSP 计划、工程、支持）  | 工程和支持联系人（姓名、电子邮件和电话号码）;CSP 计划联系人可选，但建议。 |
| 支持 URL  | 必需。 |

## <a name="preview-page"></a>“预览”页

预览页面是指定受众访问产品/服务预览的位置，以验证产品/服务在上线前是否满足您的所有要求。 

| **字段名**    | **说明**   | 
| :---------------- | :-----------| 
| AAD/MSA 电子邮件 + 描述 | 需要，最小 1 和最多 10（如果手动输入）;如果上传 CSV 文件，则最多为 20。 |

## <a name="technical-configuration-page"></a>技术配置页面 

技术配置页面是指定 Microsoft 用于连接到产品/服务的技术详细信息的位置。 如果您决定不通过 Microsoft 销售，则此页面不可见。

| **字段名**    | **说明**   |  
| :---------------- | :-----------| 
| 登陆页 URL | 通过 Microsoft 销售，则需要。 |
| 连接网钩 | 通过 Microsoft 销售，则需要。 |
| Azure AD 租户 ID | 通过 Microsoft 销售，则需要。 |
| Azure AD 应用 ID | 通过 Microsoft 销售，则需要。 |

## <a name="plan-identity-modal"></a>规划标识模式

要求您提供的第一条信息是计划的名称和 ID。 如果您决定不通过 Microsoft 销售，则此页面不可见。

| **字段名**    | **说明**   |  
| :---------------- | :-----------| 
| 计划 ID  | 通过 Microsoft 销售，则需要。 创建后无法更改它。 最大 50 个字符，必须仅包含小写、字母数字字符、破折号或下划线。 |
| 计划名称  | 通过 Microsoft 销售，则需要。 在优惠中的所有计划中必须是唯一的。 最多 50 个字符。 |

## <a name="plan-listing-page"></a>计划列表页

计划列表页是您提供文本供客户在市场中查看计划时查看的文本的位置。 如果您决定不通过 Microsoft 销售，则此页面不可见。

| **字段名**    | **说明**   |  
| :---------------- | :-----------| 
| 计划描述   | 通过 Microsoft 销售，则需要。 最大 500 个字符。 | |

## <a name="plan-pricing--availability-page"></a>计划定价&可用性页面

计划定价和可用性页面是定义每个计划（版本）的业务特征、受众和市场可用性的位置。 如果您决定不通过 Microsoft 销售，则此页面不可见。

| **字段名**    | **说明**   | 
| :---------------- | :-----------| 
| 市场可用性  | 需要，最小 1 和最大 141。 |
| 定价模型  | 必需。 默认值：统一费率。 选项：每个用户的统一费率。 |
| 最小和最大座位  | 可选，仅在选择基于座位的定价模型时可用。 |
| 计费期限  | 必需。 默认值：每月。 选项：每月，每年。 |
| 价格  | 每月所需美元，如果选择每月计费期限;或每年美元，如果选择年度计费期限。 |
| 规划受众  | 可选。 默认值：公共计划。 选项：公共、按租户 ID 进行 |
| 受限计划受众（租户 ID + 说明）  | 选择私人计划时是必需的。 如果手动输入最小 1 和最多 10 个租户 ID。 如果 CSV 文件导入，则最大 20000。 |

## <a name="test-drive-listing-page"></a>试驾列表页

仅在您选择提供产品/服务试驾时才可用。 定义用于在市场中列出试驾的详细信息。

| **字段名**    | **说明**   | 
| :---------------- | :-----------| 
| 描述  | 必需。 |
| 用户手动名称 + 文件  | 必需，最多 1 个文档必须为 PDF 格式。 |
| 视频名称、URL + 缩略图  | 可选，建议使用。 缩略图必须为 533 x 324（JPGP 或 PNG 格式）。 视频必须托管在 YouTube 或 Vimeo 中。 |
