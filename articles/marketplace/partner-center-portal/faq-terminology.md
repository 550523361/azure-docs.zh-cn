# <a name="commercial-marketplace-analytics-terminology-and-common-questions"></a>商业市场分析术语和常见问题

本文解答有关合作伙伴中心内分析消息的常见问题，并提供分析术语的字典。

## <a name="common-questions"></a>常见问题

**我无法在合作伙伴中心查看分析数据。访问这些页面时会显示以下消息。为什么会这样？**

![你的产品/服务还没有数据](./media/analytics-faq-no-data.png)

为何会收到此消息：

- 你在市场中发布的产品/服务目前还没有任何人购买。 这可能意味着，你的产品/服务虽然已在市场中上线，并且产品显示页面吸引了客户的注意，但是，目前还没有客户采取行动来购买和部署这些产品/服务。
- 你的产品/服务可能正在发布，但还没有上线。 客户只能获取已上线的产品/服务。 若要检查产品/服务的状态，请参阅[分析仪表板](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)中的“概述”。 有关详细信息，请参阅[商业市场分析中的“摘要”仪表板](./summary-dashboard.md)。
- 你的产品/服务可能只是以“与我联系”的形式列出，市场中的客户无法购买这种列表形式的产品/服务。 虽然这些产品/服务能够带来潜在顾客并与你共享，但市场不会为这些产品/服务创建订单，因为客户无法购买。 若要检查产品/服务列表类型，请转到设置页。

**我知道我有分析数据，但出现了以下消息：**

![在给定的日期范围没有数据](./media/analytics-faq-data-range.png)

如果收到此消息，表示你有分析数据，但在所选的日期范围没有数据。 请选择其他日期范围或自定义日期范围，以查看自 2010 年以来的任何数据。 有关详细信息，请参阅[商业市场分析中的“摘要”仪表板](./summary-dashboard.md)的“日期范围”部分。

| Azure 许可证类型：云解决方案提供商 | 客户、订单 | 最终客户通过其云解决方案提供商（充当经销商）购买 Azure 和市场套餐。|
| Azure 许可证类型：Enterprise | 客户、订单 | 最终客户通过直接与 Microsoft 签署的企业协议购买 Azure 和市场套餐。|
| Azure 许可证类型：通过经销商购买产品/服务的企业  | 客户、订单 | 最终客户通过某家经销商购买 Azure 和市场产品/服务，与该经销商合作便于与 Microsoft 达成企业协议。|  |
| Azure 许可证类型：即用即付| 客户、订单 | 最终客户通过直接与 Microsoft 签署的“即用即付”协议购买 Azure 和市场产品/服务。||

| 云实例名称：Azure 政府 | 订单| 特定于以下政府之一的 Microsoft 云：中国、德国或美国。| |

| 客户所在国家/地区 | 客户、订单 | 客户提供的国家/地区名称。 此处的国家/地区可能不同于客户的 Azure 订阅中的国家/地区。|  |

| 客户邮政编码  | 客户| 客户提供的邮政编码。 此处的编码可能不同于客户的 Azure 订阅中提供的邮政编码。| |
| 客户所在的州/省| 客户| 客户提供的州/省（地址）。 此处的州/省可能不同于客户的 Azure 订阅中提供的州/省。| |
| 获取日期| 客户| 客户首次购买你发布的任何产品/服务的日期。| |
| 丢失日期| 客户| 客户最后一次取消以前购买的所有产品/服务中最后一个产品/服务的日期。||
| 是新客户  | 订单| 此值确定客户是否为首次获取你的一个或多个产品/服务的新客户。 如果在“获取日期”的相同日历月份内，则值为“是”。 如果客户在报告的日历月份之前购买了你的任何产品/服务，则值为“否”。 |
| 是预览版 SKU| 订单| 此值告知是否你已将 SKU 标记为“预览版”。 如果已相应地标记了 SKU，则值为“是”，并且只有你授权的 Azure 订阅可以部署和使用此映像。 如果 SKU 未标识为“预览版”，则值为“否”。  |
| 是促销联系人选择加入| 客户| 此值告知是否客户主动从发布者渠道登记为促销联系人。 目前我们不会向客户提供该选项，因此，在整个板块中都指示了“否”。 部署此功能后，我们会相应地做出更新。|

| 市场许可证类型：免费| 订单| 产品/服务配置为供所有用户免费使用。 Microsoft 不会因为客户使用此产品/服务而向客户计费。||

| 产品/服务类型：容器 | 订单| 如果解决方案是预配为基于 Kubernetes 的 Azure 容器服务的 Docker 容器映像，请使用“容器”产品/服务类型。||

| 产品/服务类型：Dynamics 365 for Customer Engagement | 订单| 如果你的解决方案与 Dynamics 365 for Customer Engagement 集成，请使用此产品/服务类型。||
