# <a name="create-a-new-saas-offer"></a>创建新的 SaaS 套餐

要开始创建"软件即服务"（SaaS）优惠，请确保首先[创建合作伙伴中心帐户](./create-account.md)并打开["商业市场"仪表板](https://partner.microsoft.com/dashboard/commercial-marketplace/offers)，并选中 **"概述**"选项卡。

![合作伙伴中心上的商业市场仪表板](./media/new-offer-overview.png)

>[!Note]
> 产品/服务发布后，只有在重新发布后，才会在系统和商店前面更新对合作伙伴中心中提供的优惠进行编辑。 请务必在进行更改后提交要发布的产品/服务。

选择 +**新产品/服务...** 按钮，然后选择 **"软件作为服务"** 菜单项。

如果选择其他产品/服务类型，则可以重定向到较旧的[云合作伙伴门户](https://cloudpartner.azure.com/)。 目前合作伙伴中心的商业市场门户中仅提供 SaaS 和 Dynamics 365 优惠。

![在合作伙伴中心创建产品/服务窗口](./media/new-offer-click.png)

将显示 **"新建产品/服务**"对话框。

![新优惠对话框](./media/new-offer-popup.png)

## <a name="offer-id-and-alias"></a>优惠 ID 和别名

- **优惠 ID**：帐户中每个产品/服务的唯一标识符。 此 ID 将在市场产品/服务的 URL 地址和 Azure 资源管理器模板（如果适用）中向客户可见。 优惠 ID 必须是小写字母数字（包括连字符和下划线，但没有空格）。 **优惠 ID**限制为 50 个字符，在选择"*创建*"后无法更改。  
示例：测试-提供-1
<br>导致 URL：`https://azuremarketplace.microsoft.com/marketplace/../test-offer-1`

- **产品/服务别名**：用于在合作伙伴中心门户中引用产品/服务的名称。 此名称不会在市场中使用，并且与向客户显示的 *"产品/服务名称*"和其他值不同。 选择 "*创建*"后，无法更改此值。

<br>示例：测试优惠 1&#8482;

选择“创建”  。  为此产品/服务创建**产品/服务概述**页面。

## <a name="offer-overview"></a>产品/服务概述

**"产品/服务概述"** 页包括：

- **"发布"状态**显示发布此产品/服务所需的步骤以及每个步骤需要多长时间才能完成的可视化表示形式。 不完整的发布步骤图标将灰显。

- **"产品/服务概述"** 菜单包含用于对此产品/服务执行操作的链接列表。 此操作列表将根据您为产品/服务做出的选择而更改。  
    - 如果要约是草稿 - 删除草稿
    - 如果报价是实时 - 停止出售报价
    - 如果优惠处于预览版 - 上线
    - 如果您尚未完成发布者注销 - 取消发布

## <a name="offer-setup"></a>优惠设置

**"产品/服务设置"** 选项卡询问以下信息。 完成这些字段后选择 **"保存**"。

- **您想通过微软销售吗？** （是/否）
    - **是的**，您希望通过微软销售您的报价，微软代表您托管市场交易;或 
    - **不**，你宁愿通过市场列出您的报价，处理任何独立于微软的货币交易。

### <a name="sell-through-microsoft"></a>通过 Microsoft 进行销售

通过 Microsoft 销售可提供更好的客户发现和获取，允许 Microsoft 代表您托管市场事务，并利用 Microsoft 的全球可用商务功能。

#### <a name="saas-offer-requirements"></a>SaaS 提供要求

为了在合作伙伴中心的商业市场列出软件即服务 （SaaS） 优惠，必须满足以下条件：

- 您的产品/服务必须使用[Azure 活动目录 （Azure AD）](https://azure.microsoft.com/services/active-directory/)进行标识管理和身份验证。
- 您的产品/服务必须使用[SaaS 履行 API](https://docs.microsoft.com/azure/marketplace/partner-center-portal/pc-saas-fulfillment-api-v2)与 Azure 应用商店集成。
- 有关更广泛的要求，请参阅[SaaS 产品发布指南](https://docs.microsoft.com/azure/marketplace/marketplace-saas-applications-technical-publishing-guide)。

#### <a name="saas-pricing-and-billing-options"></a>SaaS 定价和计费选项

在发布者 Azure 订阅中运行 SaaS 解决方案后，客户支付的许可费包括部署软件的基础结构的成本。 Azure 基础结构使用情况将直接管理和计费给合作伙伴。 客户将无法看到实际的基础结构使用费。 发布者应将 Azure 基础结构使用费捆绑到其软件许可证定价中。 

SaaS 根据固定费用、每个用户或使用按流量计费服务的消耗费提供每月或年度计费支持。 微软的商业市场以代理模式运作，即出版商设定价格，微软向客户收费，微软在扣缴代理费的同时向出版商支付收入。

下表显示了用于演示代理模型的成本和支出的示例。

|**你的许可证费用**|**每月 100 美元**|
|:---|:---|
|Azure 使用费用（D1/1 核）|直接向发布者而不是客户收费|
|由 Microsoft 向客户收费|每月 $100.00 （Publisher 必须考虑许可证费中发生的任何或传递的基础设施成本）|

|**Microsoft 收费**|**每月 100 美元**|
|:---|:---|
|Microsoft 将许可证费用的 80% 支付给你 <br>**对于合格的 SaaS 应用，Microsoft 支付 90% 的许可证成本*|80.00 美元/月 <br>*$* 每月 90.00 美元*|

- 在此示例中，Microsoft 向客户收取软件许可证的 100.00 美元，并向客户支付 80.00 美元给发布者。
- 符合**减少市场服务费**条件的合作伙伴将在 2019 年 5 月至 2020 年 6 月期间降低 SaaS 优惠的交易费用。 在这种情况下，Microsoft 会为您的软件许可证支付 100.00 美元，然后向发布者支付 90.00 美元。

> [!NOTE]
> **降低市场服务费**：对于您在我们的商业市场上发布的某些 SaaS 优惠，Microsoft 将将其市场服务费从 20%（如 Microsoft 发布商协议中所述）降至 10%。 为了使您的产品/服务符合条件，您的优惠中至少有一个必须被 Microsoft 指定为 IP 共同销售就绪或 IP 共同销售优先。  每个日历月结束之前必须至少五 （5） 个工作日满足资格，才能获得该月此降低的市场服务费。  减少的市场服务费不适用于 VM、托管应用或通过我们的商业市场提供的任何其他产品。  减少的市场服务费仅适用于 Microsoft 在 2019 年 5 月 1 日至 2020 年 6 月 30 日期间收取的合格许可费用优惠。  在此时间之后，市场服务费将恢复到正常金额。

### <a name="standard-contract-for-the-microsoft-commercial-marketplace"></a>微软商业市场的标准合同

微软提供了标准合同模板。

- **使用微软商业市场的标准合同？**

为了简化客户的采购流程并降低软件供应商的法律复杂性，Microsoft 为 Microsoft 商业市场提供了标准合同，以帮助促进市场交易。 商业市场出版商可以选择根据标准合同提供他们的软件，而不是制定自定义条款和条件，客户只需审查并接受一次。 标准合同可以在这里找到： https://go.microsoft.com/fwlink/?linkid=2041178.

您可以选择使用标准合同，而不是通过选择"为商业市场使用标准合同"复选框来提供您自己的自定义条款和条件。

![使用标准合同复选框](./media/use-standard-contract.png)

> [!NOTE]
> 使用 Microsoft 商业市场的标准合约发布产品/服务后，您将无法使用自己的自定义条款和条件。 这是一个"或"方案。 您要么根据标准合同**或**您自己的条款和条件提供您的解决方案。 如果您想修改标准合同的条款，您可以通过标准合同修订来修改。

#### <a name="standard-contract-amendments"></a>标准合同修订

标准合同修订允许发布商选择标准合同条款，以便简单，并自定义其产品或业务的条款。 客户只需审核并接受 Microsoft 标准合同，就只需查看合同的修订。

商业市场出版商有两种修正案：

- 普遍修改：这些修订适用于所有客户的标准合同。 通用修订显示在采购流程中向产品/服务的每个客户。 客户必须接受标准合同的条款和修改，然后才能使用您的优惠。
- 自定义修订：这些修订是标准合同的特别修订，仅通过 Azure 租户 ID 针对特定客户。 发布者可以选择要定位的租户。 只有租户的客户才会在优惠的采购流程中收到自定义修改条款。  客户必须接受标准合同的条款和修改，然后才能使用您的优惠。

>[!NOTE]
> 这两种类型的修正相互叠加。 以定制修订为目标的客户在购买过程中还将获得标准合同的普遍修订。

**微软商业市场标准合同的通用修正条款**：在此框中输入通用修订条款。 您可以提供每个报价的单一通用修订。 您可以在此框中输入无限数量的字符。 在发现和购买流期间，这些术语在 AppSource、Azure 应用商店和/或 Azure 门户中向客户显示。

**微软商业市场标准合同的自定义修改条款**：从选择**添加自定义修改条款**开始。 每个优惠最多可以提供 10 个自定义修订条款。

- **自定义修改条款**：在自定义修改条款框中输入您的自定义修订条款。 您可以在此框中输入无限数量的字符。 只有您为这些自定义条款指定的租户 ID 的客户才会在 Azure 门户中向产品/服务的采购流中显示自定义修改条款。  
- **租户 ID（** 必需）：每个自定义修订的目标可多达 20 个租户 ID。 如果添加自定义修订，则必须至少提供一个租户 ID。 租户 ID 标识 Azure 中的客户。 您可以向客户询问此 ID，并且他们可以通过导航到portal.azure.com> Azure 活动目录>属性来找到它。 目录 ID 值是租户 ID（例如，50c464d3-4930-494c-963c-1e951d15360e）。 您还可以使用我的[Microsoft Azure 和 Office 365 租户 ID](https://www.whatismytenantid.com)上的域名 URL 查找组织的客户的租户 ID。
- **说明**（可选）：可选地为租户 ID 提供友好说明，帮助您通过修订确定目标客户。

#### <a name="terms-and-conditions"></a>条款和条件

如果要提供自己的自定义条款和条件，可以选择在条款和条件字段中输入它们。 您可以在此字段中输入最多 10，000 个字符的文本。 如果您的条款和条件需要较长的描述，请在此字段中输入单个 URL 链接，其中可以找到您的条款和条件。 它将作为活动链接向客户显示。

客户在试用您的优惠之前，必须接受这些条款。

## <a name="offer-listing"></a>优惠列表

"产品/服务"列表选项卡显示您的优惠可用的语言（和市场），当前英语（美国）是唯一可用的位置。 此外，此页显示特定于语言的列表的状态以及添加该列表的日期/时间。 您需要为每个语言/市场定义市场详细信息（报价名称、说明、搜索词等）。

> [!NOTE]
> 报价列表内容（如产品/服务说明、文档、屏幕截图、使用条款和隐私政策）不需要以英文表示，只要产品/服务说明以短语开头，"此应用程序仅以 [非英语语言]提供。 提供*有用的链接 URL*以提供产品/服务列表内容中所用语言以外的语言提供内容也是可接受的。

### <a name="offer-listings"></a>套餐列表

提供要在市场上显示的详细信息，包括您的报价和营销资产的描述。

- **名称**（必需）：此处定义的名称将显示为您选择的市场上要约列表的标题。 名称会根据以前的 **"新产品/服务**"条目预填充。 该名称可以注册商标。 它不能包含表情符号（除非它们是商标和版权符号），并且必须限制为 50 个字符。
- **摘要**（必需）：提供要在市场列表搜索结果中使用的报价的简短说明。 此字段中最多可输入 100 个字符的文本。
- **说明**（必需）：提供要在市场列表概览中显示的报价说明。 考虑包括价值主张、关键优势、任何类别或行业协会、应用内购买机会、任何必需的披露以及用于了解更多信息的链接。
此字段（包括标记）中，最多可输入 3，000 个字符的文本。 有关其他提示，请参阅文章[编写出色的应用说明](https://docs.microsoft.com/windows/uwp/publish/write-a-great-app-description)。
- **搜索关键字**：最多输入三个搜索关键字，客户可以使用这些关键字在市场上查找您的产品/服务。
- **入门说明**（必需）：说明如何为潜在客户配置并开始使用应用。  此快速入门可以包含指向更详细的联机文档的链接。 此字段中最多可输入 3，000 个字符的文本。

#### <a name="description"></a>**说明**

此字段为必填字段。 要包含在**说明**中的项 ：

* 在前几句描述中清晰描述产品/服务的价值主张。  
* 请牢记，前几句话可能在搜索引擎结果中显示。  
* 不要依赖于特性和功能来销售产品。 而应该强调提供的价值。

> [!IMPORTANT]
> 在预览版中检查您的优惠后，选择 **"上线"，** 以便您的优惠可以实时发布到市场公众受众。

- **定义预览受众：每行添加单个 AAD/MSA 帐户电子邮件以及可选说明。**

手动添加 10 个电子邮件地址，如果上载 CSV 文件，则最多添加 20 个，用于现有 Microsoft 帐户 （MSA） 或 Azure 活动目录帐户，以帮助在实时发布之前验证您的产品/服务。 通过添加这些帐户，您将定义一个受众，该受众将允许在产品/服务发布到市场之前预览访问产品/服务。 如果您的优惠已是实时的，您仍然可以定义预览受众以测试产品/服务的任何更改或更新。

## <a name="technical-configuration"></a>技术配置

"**技术配置**"选项卡定义用于连接到产品/服务的技术详细信息（URL 路径、Webhook、租户 ID 和应用 ID）。 此连接使我们能够为最终客户提供您的报价，如果他们选择购买。 描述收集字段使用情况的图表可用于[SaaS 履行 API 的文档](https://docs.microsoft.com/azure/marketplace/partner-center-portal/pc-saas-fulfillment-api-v2)。

- **着陆页 URL（** 必需）：定义客户从市场获取您的报价后将登录的网站 URL。 此 URL 将是在客户路由到页面时接收令牌的终结点。 该令牌可以使用履行 API 中的解析交换以进行预配详细信息。 这些详细信息和您收集的任何其他详细信息都可以用作您体验中构建的客户交互式网页的一部分，以完成注册并激活其购买。

- **连接 Webhook（** 必需）：对于 Microsoft 需要代表客户发送给您的所有异步事件（例如：SaaS 订阅已无效），我们要求您提供连接 Webhook。 如果还没有 Webhook 系统，最简单的配置是拥有一个 HTTP 终结点逻辑应用，用于侦听发布到它的任何事件，然后适当地处理它们（例如，https：/prod-1westus.logic.azure.com:443/work）。\/ 有关详细信息，请参阅[逻辑应用中使用 HTTP 终结点调用、触发器或嵌套工作流](https://docs.microsoft.com/azure/logic-apps/logic-apps-http-endpoint)。

- **Azure AD 租户 ID（** 必需）：在 Azure 门户中，我们要求您[创建 Azure 活动目录 （AD） 应用](https://docs.microsoft.com/azure/active-directory/develop/howto-create-service-principal-portal)，以便我们可以验证两个服务之间的连接位于经过身份验证的通信后面。 要查找[租户 ID，](https://docs.microsoft.com/azure/active-directory/develop/howto-create-service-principal-portal#get-values-for-signing-in)请转到 Azure 活动目录并选择**属性**，然后查找列出的**目录 ID**号（例如，50c464d3-4930-494c-963c-1e951d15360e）。

- **Azure AD 应用 ID（** 必需）：还需要[应用程序 ID](https://docs.microsoft.com/azure/active-directory/develop/howto-create-service-principal-portal#get-values-for-signing-in)和身份验证密钥。 要获取这些值，请转到 Azure 活动目录并选择**应用注册**，然后查找列出的**应用程序 ID**号（例如，50c464d3-4930-494c-963c-1e951d15360e）。 要查找身份验证密钥，请转到 **"设置"** 并选择 **"密钥**"。 您需要提供说明和持续时间，然后提供数字值。

>[!Note]
>Azure 应用程序 ID 与发布者 ID 相关联，因此请确保在所有产品/服务中使用相同的应用程序 ID。

## <a name="plan-overview"></a>计划概述

"**计划概述**"选项卡使您能够在同一产品/服务范围内提供各种计划选项。 这些计划（有时称为 SKU）在版本、货币化或服务层方面可能有所不同。 您必须至少设置一个计划才能在市场上销售您的报价。

创建后，您将看到计划名称、ID、定价模型、可用性（公共或私有）、当前发布状态以及任何可用操作。

**计划概述**中可用的**操作**因计划的当前状态而异，可能包括：

- 如果计划状态为**草稿**- 删除草稿
- 如果计划状态为 **"实时**- 停止销售计划"或同步私人受众

**创建新计划**（为选择通过 Microsoft 销售的用户至少一个计划）

- **计划 ID：** 为此产品/服务中的每个计划创建唯一的计划 ID。 此 ID 将在产品 URL 和 Azure 资源管理器模板中对客户可见（如果适用）。 仅使用小写、字母数字字符、破折号或下划线。 此计划 ID 最多允许 50 个字符。 选择创建后无法修改 ID。
- **计划名称：** 客户在决定在您的产品/服务中选择哪个计划时，将看到此名称。 为此产品/服务中的每个计划创建唯一的优惠名称。 计划名称用于区分可能是同一产品/服务一部分的软件计划（例如，产品名称：Windows 服务器;计划：Windows 服务器 2016，Windows Server 2019）。

### <a name="plan-listing"></a>计划列表

**"计划列表**"选项卡显示您的计划可用的语言（和市场），当前英语（美国）是唯一可用的位置。 此外，此页显示特定于语言的列表的状态以及添加该列表的日期/时间。 您需要为每个语言/市场定义市场详细信息（报价名称、说明、搜索词等）。

#### <a name="plan-listing-details"></a>计划列表详细信息

选择其中一种计划语言将显示**计划列表**信息，包括**名称**和**说明。**

- **名称**：根据预览 **"新建计划**"条目预填充，并将作为市场上显示的产品/服务"软件计划"的标题显示。
- **描述：** 此说明是一个解释是什么让此软件计划独一无二以及您产品/服务中其他软件计划的任何差异的机会。 可能最多包含 500 个字符。

完成这些字段后选择 **"保存**"。

#### <a name="plan-pricing-and-availability"></a>规划定价和可用性

通过 **"定价和可用性**"选项卡，您可以配置此计划可用的市场、所需的货币化模型、价格和计费期限。 此外，还可以指示是使计划对所有人可见，还是仅对特定客户（私人受众）可见。

##### <a name="enabling-free-trials"></a>启用免费试用

通过商业市场提供 SaaS 功能，使您能够在通过 Microsoft 销售时提供一个月的免费试用版。 对于除计量计划之外的所有计费模型和条款，支持免费试用。 此选项允许客户通过一个月的免费访问获得低门槛进入。  如果您选择在产品/服务范围内为计划启用免费试用，则客户将无法在初始一个月期限结束前转换为付费订阅。  在此期间，购买您的优惠的客户可以试用任何支持的计划，这些计划启用了免费试用，并在它们之间转换。  付费订阅的转换在期限结束时自动完成。

>[!Note]
>如果客户选择在没有免费试用的情况下转换为计划，转换将发生，但免费试用将立即丢失。  此外，一旦客户开始为计划付费，他们就不能再在同一订阅上获得免费试用，即使他们转换为支持免费试用的 SKU 也是如此。

对于优惠中的每个计划，均可配置免费试用版。 导航到每个产品/服务的定价和可用性，然后选中此框以允许为期一个月的试用。

![一个月免费试用复选框](./media/free-trial-enable.png)

>[!Note]
>一旦您的可交易优惠发布免费试用版，则无法为该计划禁用该优惠。 确保此设置对于第一个发布是正确的，以避免重新创建计划。

要获取有关当前参与免费试用的客户订阅的信息，请使用新的 API 属性，该`isFreeTrial`属性将标记为真或假。 有关详细信息，请参阅[SaaS 获取订阅 API](https://docs.microsoft.com/azure/marketplace/partner-center-portal/pc-saas-fulfillment-api-v2#get-subscription)。

>[!Note]
>对于利用市场计量服务的计划，不支持免费试用。

#### <a name="markets"></a>市场

- **编辑市场**（可选）

每个计划必须至少在一个市场可用。 选择您希望提供此计划的任何市场位置的复选框。 包含用于选择"免税"国家/地区的搜索框和按钮，其中 Microsoft 代表您汇出销售和使用税，以帮助。

如果您已为计划设定价格（美元），并添加其他市场位置，则新市场的价格将根据当前汇率计算。 在发布之前，请查看每个市场的价格。 在保存更改后，可以使用"出口价格 （xlsx）"链接查看定价。

#### <a name="pricing"></a>定价

- **定价模式**：基于固定费率或座位

**统一费率：** 以单月或年价格统一价格启用对产品/服务的访问。 这有时称为基于站点的定价。 使用此定价模型，您可以选择定义使用市场计量服务 API 按非标准单位向客户收费的计量计划。  有关计量计费的详细信息，请参阅[使用市场计量服务计量计费。](./saas-metered-billing.md)

**每个用户：** 根据访问优惠或占据席位的用户数量，以价格启用对优惠的访问。 此基于用户的模型使您能够根据价格设置允许的最小和最大用户数。 这样，可以通过配置多个计划来根据用户数量配置不同的价格点。  这些字段是可选的。 如果未选中，则用户数将被解释为没有限制（最小为 1，并且最大为系统可以支持的数量）。 这些字段可能会作为计划更新的一部分进行编辑。

发布后，无法更改计费定价模型选择。 此外，同一报价的所有计划必须共享相同的定价模型。

- **计费期限**：每月或每年

选择客户必须支付所列价格的频率。 必须至少提供一个每月或年价，或者两个选项都可以提供给客户。

- **价格**：每月美元或美元/年

以当地货币（美元 = 美元）设置的价格使用设置期间可用的当前汇率转换为所有选定市场的当地货币。 通过导出定价电子表格并查看每个市场的价格，在发布之前验证这些价格。 如果要在单个市场设置自定义价格，请修改并导入定价电子表格。 您负责验证此定价并拥有这些设置。
*\*您必须首先保存定价更改，以启用定价数据的导出。*

在发布之前，请仔细查看您的价格，因为在计划发布后，可能会更改哪些内容有一些限制：

- 计划发布后，无法更改定价模型。
- 一旦为计划发布了计费期限，以后无法删除它。
- 一旦计划中市场的价格公布，以后就改变不了。

#### <a name="privacy"></a>隐私

- **这是一个私人计划**（可选复选框）

选中此框可使您的计划成为私有的，并且仅对您选择的受限受众可见。 作为私人计划发布后，您可以更新受众或选择使计划可供所有人使用。 一旦计划发布为对每个人都可见，它必须保持对每个人都可见。 （无法再次将计划配置为私有计划）。

- **受限受众（租户 ID）**

分配有权访问此私人计划的访问群体。 使用租户 ID 分配访问权限，并选择包括所分配的每个租户 ID 的说明。 最多可以添加 10 个租户 ID，如果导入 .csv 电子表格文件，最多可以添加 20，000 个客户租户 ID。

租户是组织的表示形式，ID 表示为 GUID（全局唯一标识符，用于标识资源的 128 位整数）。 它是组织或应用开发人员在组织或应用开发人员与 Microsoft 建立关系时（例如，注册 Azure、微软 Intune 或 Microsoft 365 时）接收的 Azure AD 专用实例。 每个 Azure AD 租户都是独特的，独立于其他 Azure AD 租户。 要检查租户，请使用要用于管理应用程序的帐户登录 Azure 门户。 如果你有一个租户，则会自动登录到该租户，并且帐户名的正下方会显示租户名称。 将鼠标指针悬停在 Azure 门户右上角的帐户名上，可以查看你的姓名、电子邮件、目录/租户 ID (GUID) 以及域。 如果帐户与多个租户相关联，则可以选择帐户名打开一个菜单，并在其中切换租户。 每个租户都有自己的唯一租户 ID。 您还可以使用域名 URL 在 ：[https://www.whatismytenantid.com](https://www.whatismytenantid.com)查找组织的租户 ID。

虽然 SaaS 提供使用租户 ID 来定义专用受众，但其他产品/服务类型可能使用 Azure 订阅 ID（也表示为 GUID）。

> [!NOTE]
> 私人观众（或受限观众）与预览观众不同。 在**["预览"](#preview)** 选项卡中，您可以定义预览受众。 在产品/服务在市场上实时发布*之前*，预览受众可以访问您的优惠。 虽然私人受众指定仅适用于特定计划，但预览观众可以查看所有计划（私有或是否），但只能在计划测试和验证的有限预览期间查看。
