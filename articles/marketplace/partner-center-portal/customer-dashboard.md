# <a name="customer-dashboard-in-commercial-marketplace-analytics"></a>商业市场分析中的“客户”仪表板

本文提供有关合作伙伴中心中的“客户”情况仪表板的信息。 此仪表板显示有关客户的信息，包括以图形和可下载的格式显示的增长趋势。

若要访问“客户”仪表板，请在商业市场下打开[分析](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)仪表板 。

>[!NOTE]
> 有关分析术语的详细定义，请参阅[适用于商业市场分析的常见问题和术语](./faq-terminology.md)。

## <a name="customer-dashboard"></a>客户仪表板

“分析”菜单的“客户“仪表板显示已购买产品/服务的客户的数据 。 你可以查看以下各项的图形表示形式：

- [客户汇总](#customer-summary)
- [按地域分类的客户](#customer-by-geography)
- [客户趋势](#customer-trends)
- [按订单量和使用量统计的客户数](#customers-by-orders-and-usage)
- [按计划的客户](#customers-by-plans)
- [按客户类型统计的订单数和使用量](#orders-and-usage-by-customer-type)
- [客户详细信息表](#customer-details-table)
- [客户页筛选器](#customer-page-filters)

合作伙伴中心中客户获取和报告之间的最大延迟为48小时。

### <a name="customer-summary"></a>客户汇总

客户汇总部分显示所选日期范围内所有客户（包括新客户、现有客户和流失客户）的计数。

- 客户总数定义为已购买产品/服务并且至少有一个尚未取消的订单的所有客户的计数。
- 与上个月相比，客户的增长百分比以绿色或下指示器中的数字和向上指示器来表示。
- 增长趋势由条形图表示，将鼠标悬停在图表的列上可显示每个月的值。

有三种客户类型：新客户、现有客户和流失客户。

- 新客户指在选定月份内首次购买一个或多个产品/服务的客户。
- 现有客户指在选定月份之前已购买一个或多个产品/服务的客户。
- 流失客户指已取消以前购买的所有产品/服务的客户。

### <a name="customer-by-geography"></a>按地域统计的客户数

"**按地理位置**排列的客户" 图表显示在所选日期范围内购置的所有客户和客户的计数，这些计数基于客户的国家/地区进行映射。 地图上的由浅到深的颜色代表客户计数从低到高的值。 选择表中的一条记录以放大国家/地区。

热度地图按客户的国家/地区显示客户计数和百分比。 可以移动地图来查看确切位置，并将其放大到特定位置。 此地图具有一个补充网格，可以使用它按地域查看客户的百分比以及新添加到该位置的客户。

### <a name="customer-trends"></a>客户趋势

“客户趋势”图显示了所有客户（包括新客户、现有客户和流失客户）的计数，并显示逐月的增长趋势。

- 折线图表示总体客户增长百分比。
- 月份列表示按新客户、现有客户和流失客户统计的叠加的客户计数。
- 流失客户计数显示在 Y 轴的负方向上。
- 可以选择特定的图例项来显示更详细的视图。 例如，从图例中选择新客户，以只显示新客户。
- 可以使用图表顶部的滑块在 x 轴上向右和向左滚动，并且将焦点置于特定数据点以查看更多详细信息。
- 将鼠标悬停在图表的某一列上，可只显示该月的详细信息。

### <a name="customers-by-orders-and-usage"></a>按订单和使用量分类的客户

“按订单量/使用量统计的客户数”具有三个选项卡：“订单量”、“规范化使用量”和“原始使用量”。 排名靠前客户百分点值沿 x 轴显示，由客户的订单数量确定。 y 轴显示客户的订单计数。 辅助性 y 轴（折线图）显示订单总数的累计百分比。 可以通过将鼠标悬停在折线图的各个点上方来显示详细信息。

例如，请参阅下图以了解规范化使用量：排名前 30% 的客户累计具有 87% 的规范化使用量。 第 30 百分位数的客户仅有 157 万小时的使用量。

### <a name="customers-by-plans"></a>按计划的客户

下面介绍了**按计划/使用情况**图表列出的客户。

1. 排行榜显示按订单计数排名前 50 的客户的详细信息。 选择客户后，客户的详细信息将显示在此排行榜的第 2、3 和 4 部分。
2. 当发布者使用所有者角色登录时，此区域中将显示客户配置文件详细信息。 如果发布者使用参与者角色登录，则此部分中的详细信息将不可用。
3. "**按计划**排列的订单" 环形图显示为计划购买的订单明细。 显示具有最高订单数的前五个计划，而其余订单按 "全部 rest" 分组。
4. "**按计划**显示的座位" 圆环图显示按计划排序的座位的细分。 显示最高座位数最多的5个计划，而其余订单在 "全部 rest" 下分组。

### <a name="orders-and-usage-by-customer-type"></a>按客户类型分类的订单和使用量

“按客户类型统计的订单量/使用量”图显示新客户和现有客户的订单数、规范化使用量和原始使用小时数；根据订单的选择，图中分别显示规范化使用量和原始使用量。

- 新客户指在相同日历月份（y 轴）内第一次购买一个或多个产品/服务或报告了使用量的客户。
- 现有客户指在报告的日历月份之前（在 y 轴上）已购买产品/服务或报告了使用量的客户。

### <a name="customer-details-table"></a>客户详细信息表

“客户详细信息”表显示了排名前 1000 位客户的编号列表（按他们首次购买产品/服务的日期排序）。

- 仅当客户同意时，客户个人信息才可用。 如果使用所有者角色级别的权限登录，只能查看此信息。 了解有关用户角色和权限的详细信息。
- 网格中的每列都可进行排序。
- 如果记录计数小于 1000，则可以将数据提取到 TSV 文件中。
- 如果记录数超过 1000，则导出的数据在接下来的 30 天会异步放置在下载页面中。
- 对表应用筛选器，以便仅显示感兴趣的数据。 按公司名称、客户 ID、Marketplace 订阅 ID、Azure 许可证类型、获取日期、丢失日期、客户电子邮件、客户国家/地区/州/省/市/自治区、客户语言等筛选数据。
- 当受保护的客户购买了某个产品/服务时，“客户详细数据”中的信息将被屏蔽 (************)。
- 客户维度详细信息（如公司名称、客户名称和客户电子邮件）是组织 ID 级别的信息，而不是 Azure 市场或 AppSource 事务级别的信息。

### <a name="customer-page-filters"></a>客户页筛选器

“客户页”筛选器在客户页级别上应用。 可以选择多个筛选器按所选条件以及希望在“详细订单数据”网格/导出结果中查看的数据来呈现图表。 筛选器会应用于为订单页面右上角选择的数据范围提取的数据。
