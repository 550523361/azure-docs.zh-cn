# <a name="power-bi-app-creation-overview"></a>电源 BI 应用创建概述

> [!IMPORTANT]
> 我们将 Power BI 应用产品的管理从云合作伙伴门户迁移到合作伙伴中心。 在迁移产品/服务之前，请按照云合作伙伴门户[Power BI 应用产品/服务](https://docs.microsoft.com/azure/marketplace/cloud-partner-portal/power-bi/cpp-power-bi-offer)中的说明来管理您的优惠。

本文介绍如何将 Power BI 应用程序发布到微软[AppSource](https://appsource.microsoft.com/)。 Power BI 应用包可自定义的内容，包括数据集、报表和仪表板。 然后，您可以使用 AppSource 将应用与其他 Power BI 平台一起使用，执行开发人员允许的调整和自定义，并将其连接到您自己的数据。

## <a name="publishing-benefits"></a>发布的好处

发布到商业市场的好处：

- 使用 Microsoft 品牌推广您的公司。
- AppSource 上可能覆盖超过 1 亿个 Office 365 和 Dynamics 365 用户，并通过 Azure 应用商店覆盖 200，000 多个组织。
- 从这些市场获得高质量的潜在客户。
- 让 Microsoft 现场和电话销售团队推广您的服务。

## <a name="overview"></a>概述

:::image type="content" source="media/power-bi-app-publishing-steps.png" alt-text="发布 Power BI 应用的步骤概述" border="false":::

以下是关键发布步骤：

1. 在 Power BI 中创建应用程序。 您将收到一个包安装链接，这是产品/服务的主要技术资产。 在合作伙伴中心创建产品/服务之前，将测试包发送到预生产。 有关详细信息，请参阅[什么是 Power BI 应用？](https://docs.microsoft.com/power-bi/service-template-apps-overview)
2. 添加营销材料，如官方名称、说明和徽标。
3. 包括产品/服务的法律和支持文档，如使用条款、隐私政策、支持策略和用户帮助。
4. 创建产品/服务：使用合作伙伴中心编辑详细信息，包括产品/服务说明、营销材料、法律信息、支持信息和资产规范。
5. 提交以供发布。
6. 在合作伙伴中心中监视流程，其中 AppSource 载入团队测试、验证和验证您的应用。
7. 认证后，在测试环境中查看应用并发布它。 这将在 AppSource 上列出它（它"上线"）。
8. 在 Power BI 中，将包发送到生产。 有关详细信息，请参阅[管理 Power BI 应用版本](https://docs.microsoft.com/power-bi/service-template-apps-create#manage-the-template-app-release)。

## <a name="requirements"></a>要求

要在商业市场发布，您的 Power BI 应用产品/服务必须满足以下技术和业务需求。

### <a name="technical-requirements"></a>技术要求

您需要的主要技术资产是 Power BI[应用程序](https://go.microsoft.com/fwlink/?linkid=2028636)。 这是主要数据集、报表或仪表板的集合。 它还包括可选的连接服务和嵌入式数据集，以前称为[内容包](https://docs.microsoft.com/power-bi/service-organizational-content-pack-introduction)。 有关开发此类应用的详细信息，请参阅什么是 Power [BI 应用？](https://go.microsoft.com/fwlink/?linkid=2028636)

#### <a name="get-an-installation-web-address"></a>获取安装 Web 地址

您只能在[Power BI](https://powerbi.microsoft.com/)环境中构建 Power BI 应用。

1. 使用 Power [BI Pro 许可证](https://docs.microsoft.com/power-bi/service-admin-purchasing-power-bi-pro)登录。
2. 在 Power BI 中创建和测试应用。
3. 收到应用安装 Web 地址后，将其添加到合作伙伴中心**的技术配置**页面。

在 Power BI 中创建和测试应用后，请保存应用程序安装 Web 地址，因为您需要它才能[创建 Power BI 应用产品/](https://aka.ms/AzureCreatePBIServiceApp)

### <a name="business-requirements"></a>业务要求

业务需求包括程序、合同和法律义务。 必须：

- 成为注册的商业市场发布者。 如果您尚未注册，请按照[《成为商业市场发布者》](https://docs.microsoft.com/azure/marketplace/become-publisher)中的步骤操作。
- 提供符合在 AppSource 上列出的产品标准的内容。 有关详细信息，请参阅在[AppSource 上列出应用？下面是如何](https://appsource.microsoft.com/blogs/have-an-app-to-list-on-appsource-here-s-how)。
- 同意并遵循[微软隐私声明](https://privacy.microsoft.com/privacystatement)。
