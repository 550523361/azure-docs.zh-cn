#<a name="azure-marketplace-consulting-services"></a>Azure Marketplace 顾问服务
Azure Marketplace 支持 Microsoft 的系统集成商 (SI) 合作伙伴提供的顾问服务产品列表，目的是帮助客户与服务互动，支持并拓展 Microsoft Azure 的用法。 

这些顾问服务产品与具体的客户参与度有关，其服务范围和持续时间是固定的，可以定价或免费提供，并且产生的效果有保障。 潜在客户可以顺畅地请求与合作伙伴取得联系。 合作伙伴在 48 小时内就能在其 CRM 系统中收到通知并答复用户。

基于 Microsoft Dynamics 365、Microsoft Office 365 和 Microsoft Power BI 的服务产品可以列在 AppSource 中。 [在此处查看这些服务的详细指导](https://smp-cdn-prod.azureedge.net/documents/Microsoft%20AppSource%20Partner%20Listing%20Guidelines.pdf)。 

##<a name="eligible-services-and-partners"></a>合格的服务与合作伙伴

合作伙伴还需要在其服务领域拥有银级或金级资质。 请参阅下面的合格资质：

|资质  |解决方案领域  |
|---------|---------|
|云平台和基础结构     |   云平台、数据中心      |
|应用程序开发和 ISV     |  应用程序开发、应用程序集成、DevOps       |
|数据管理和分析    |  数据分析、数据平台       |

产品必须属于以下五种服务类型之一。 请注意，可以列出多个产品： 

1. **评估：**评估客户的环境，以确定解决方案的适用性，并提供成本和时间估算。 
2. 简介：使用框架、演示和客户示例介绍解决方案或顾问服务，以吸引客户的兴趣。 
3. **实施：**进行完整安装，以实施完全正常运行的解决方案。 我们建议限制为可在 2 周或更短时间内实施的解决方案。 
4. **概念证明：**进行有限范围的实施，以确定解决方案是否满足客户的要求。 
5. **研讨会：**在客户场地展开的交互式活动，可以涉及到培训、简报、评估，或根据客户数据或环境制作的演示。 

可以面对面地（例如，在客户的现场或者合作伙伴的设施中）或者以虚拟形式（例如，通过电话会议、网络会议、远程实施等）提供顾问服务，但这一点不适用于“简报”。 简报必须面对面提供。

详细了解 [Microsoft 合作伙伴网络的资质](https://partner.microsoft.com/en-US/membership/competencies#simple-tab-content-1)。

##<a name="publishing-process"></a>发布过程
在 Azure Marketplace 中列出顾问服务产品涉及以下操作：

1. **请求：**在[此处](https://azuremarketplace.microsoft.com/en-us/sell/nominate)发送请求，并详细介绍你的组织。 
2. **验证：**我们将评审你的请求，以验证你是否符合条件。 然后，我们会发送有关列出顾问服务产品的说明。 
3. **提交：**提供说明中规定的信息。 所有内容和支持材料都需要以英文提交。 
4. **评审：**我们的营销和编辑团队将会评审提交内容并做出更改，以优化内容的可读性、语法和效果。 
5. **暂行发布：**我们会暂行发布你的提交内容供你审阅。 
6. **发布：**在你和 Microsoft 都对暂行发布的内容感到满意后，便可以在 Azure Marketplace 中推出内容。 内容的推出日期取决于产品中指定的国家/地区。 如果所选国家/地区的目录已在 Azure Marketplace 中推出，则几小时后就会显示内容。 否则，在所选国家/地区的目录推出后，内容就会立即显示。
