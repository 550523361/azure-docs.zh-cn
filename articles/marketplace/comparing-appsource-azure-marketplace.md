# <a name="comparing-microsoft-appsource-and-azure-marketplace"></a>比较微软应用程序源和 Azure 应用商店

微软的商业市场包括[微软应用程序源](https://appsource.microsoft.com)和[Azure市场](https://azuremarketplace.microsoft.com)。 每个网店都提供独特的客户需求和不同的目标受众，因此您可以确保您的解决方案或服务可供合适的客户使用。

## <a name="understanding-the-differences-between-storefronts"></a>了解店面之间的差异

要选择网店，请先确定产品/服务的目标受众。 如果你的目标是多个店面中的受众，可以发布一次产品/服务以在多个店面中销售。

| Azure 市场 | AppSource |
|:--- |:--- |
| IT 专业人员和开发人员 | 业务用户和业务决策者 |

下表介绍了使用店面的好处：

| 优点 | Azure 市场 | AppSource |
|:--- |:--- | :--- |
| **灵活计费** | Azure 应用商店支持免费、试用、BYOL 和交易产品类型。 交易优惠既基于消费，要么基于订阅。 基于消费的优惠按小时使用量计费，并提供初始免费期限。 基于订阅的服务按月或按按座位或固定费率按月计费。| AppSource 支持免费、试用、BYOL 和交易产品类型。 Transact 产品是基于订阅的 SaaS 产品，可以按月计费，也可以按每个座位或固定费率按年计费。 |
|**与其他合作伙伴的连接**  | 目前无法将服务提供商或交付合作伙伴链接到你的产品/服务。 | 可将独立软件供应商、系统集成商和托管服务提供商链接到特定的实现方案。 你能以协作方式销售给新客户。 |
| **自动化** | 目前无法将服务提供商或交付合作伙伴链接到你的产品/服务。 通过附加产品预配，利用自动化服务型软件 (SaaS)。 使用 Azure 应用：解决方案模板发布选项自动实现基于 SaaS 的数据收集和部署方案。 | 可将独立软件供应商、系统集成商和托管服务提供商链接到特定的实现方案。 你能以协作方式销售给新客户。 |
| **多种云类型** | 使用以下类型发布公有云和本地解决方案：<ul> <li>Azure Stack</li> <li>Azure Government</li> <li>区域云，包括中国和德国</li></ul> | 目前不支持 Azure Stack、Azure 政府或区域云。 |
| **向客户介绍上下文内** | 让用户能够在 Azure 门户中对解决方案进行上下文搜索。 使用虚拟机和 Azure 应用：解决方案模板发布选项。 | 通过 Dynamics 365、Power BI 和 Office 365 等 Microsoft 产品的应用内体验来接触更多的客户。 |
