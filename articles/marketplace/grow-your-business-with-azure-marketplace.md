# <a name="grow-your-business-with-azure-marketplace"></a>通过 Azure 市场扩展业务  
为自己的 Azure 市场商品开发出技术资产后，可以遵循市场营销最佳做法，让更多用户关注你的产品/服务并提高产品的使用率。 

>[!NOTE]
>若要了解发布过程和技术要求，请阅读 [Azure 市场和 AppSource 发布指南](https://docs.microsoft.com/azure/marketplace/marketplace-publishers-guide)。

## <a name="how-marketing-helps"></a>市场营销有何作用  
遵循市场营销最佳做法有助于将产品和服务成功地推向市场，也有助于成功实施 Microsoft 共同销售计划， 同时还可以让你从市场获得最大化的业务收益。 Microsoft 合作伙伴网络 (MPN) 是你获取所有与市场相关的营销和编程资源的途径。 

提高产品/服务的质量、在挖掘需求时坚持以客户为中心，让合作伙伴积极参与，这些做法都有助于增加企业的客户数量。 参与这些活动可帮助 Microsoft 强化上市工作效果，并在众多市场店面力推关键解决方案。 

使用以下列表来审查产品/服务的营销做法。  
*   已优化商品信息，以便提高访问量和参与度。  
*   已在网站上创建消息，并创建了独特的登录页面，以增加市场中产品/服务的客户访问量。  
*   已发布体验版或提供其他试用途径，使客户能够体验在 Azure 中实际运行产品的情景。  
*   已计划并创建自己的营销和促销活动，以吸引客户的关注和参与。  
*   已启用按需商机生成，每当客户测试或部署你的产品/服务时，我就会收到客户的姓名和联系信息。  
*   已通过 MPN 了解并连接可以使用的合作伙伴资源。  
    *   有关 MPN 的详细信息，请访问“成员资格概述”页：[partner.microsoft.com/membership](https://partner.microsoft.com/membership)。  

### <a name="create-a-great-listing"></a>创建高质量的商品信息  
市场中的商品信息可能就是你与潜在客户的第一次互动。 你希望客户能够获得深刻的第一印象，以吸引他们继续关注和跟进。 可以采取一些基本的操作来优化客户在市场中的第一印象：

| 印象 | 详细信息 |  
|:--- |:--- |  
| 让客户找到 | 使用客户搜索的关键字和字词来编写产品/服务的说明。  |  
| 直观明了 | 可借助图像和视频向用户展示关键功能和用户体验。 构想可帮助传递价值主张或解答客户首要问题的内容。 |  
| 提供体验 | 客户喜欢在购买前进行试用。 通过演示、试用和体验版吸引潜在顾客，这样有助于提高产品和服务交易量。 试用体验越好，可获得的商机就越多。 Microsoft 发现，体验版潜在客户中有 40%（平均）最终完成了交易。 |  
| 帮助用户了解学习 | 我们Microsoft 主张让概述部分保持简洁明了，但你有足够的空间来链接更多有关产品的资源。 使用该空间以简明扼要的语言介绍产品的功能，说明该产品在哪方面适合客户的需求。 必要时不妨让客户通过其他资料来了解更多信息。 如果这些学习资料或链接能够反映出你的内容市场营销策略，就更好了，这样就可以以一致的方式发展和转化商机。 |  
| 利用评分和评价 | 让客户为你推销产品。 客户口碑和代言可以成为提升销售的最大推动力，这通常是购买者进一步了解产品之前最先接触的内容。 获得多个给力评价，能使产品/服务在搜索结果中脱颖而出，并进入特推专区。 |  

### <a name="promote-your-new-offer"></a>促销新产品/服务  

#### <a name="build-an-effective-marketing-campaign"></a>构建有效的市场营销活动  
市场营销活动是一系列促销活动或市场营销策略，旨在促使目标受众的行为达到预期目标和效果。 设计活动之前，应做以下事情。

##### <a name="know-your-audience"></a>了解受众  
首先，识别购买者和影响者。 针对不同目标群组的策略和行动号召可能会有所不同。 思考这些评估问题：  
*   购买者对购买决策有多少控制力？ 
*   影响者具备多大影响力？ 
*   影响者影响什么？ 
*   他们是否影响预算或解决方案的选择？ 

弄清楚这些问题的答案，有助于制定有关资金投入和分配的决策。

##### <a name="define-where-your-audience-learns"></a>定义受众获取信息的位置  
购买者在访问市场的时候就已经完成了购买旅程的 90%。 购买者通过提前了解解决方案的内容及评估选项，走到了制定决策这一步。 你希望设计一个活动，目的是让购买者和影响者通过这个活动了解产品信息。 

每个行业、纵向市场或类别中的目标受众都是不同的。 你的目标受众是通过在线资源、电子邮件、展览会、社交媒体还是通过受信任的顾问了解产品？ 应根据受众的了解途径来设计活动，并合理分配自己的市场营销资金。 这些策略的组合会成为你最终的活动策略。

#### <a name="create-clear-campaign-goals"></a>创建清晰的活动目标  
应定义市场活动的目标，并创建明晰的 KPI。 可以开展最终目标不同的多个活动。 你和 Microsoft 都想要提高销量。 最终目标是增加收入或增加客户。 但是，可以在购买周期的其他阶段中将市场营销活动与目标挂钩。

例如，如果在市场中新推出了产品，则可能会发现，最好是将精力集中于受众培训和潜在顾客挖掘上。 是否成功实现目标可以通过从市场商品信息产生的潜在顾客数量来衡量。 在这种情况下，营销策略（和登陆页面）应侧重于吸引客户访问市场商品信息。

如果在市场中设置了试用版，并且很清楚自己的产品在被购买前需要一定程度的参与和体验，则可以将试验版的下载次数作为活动目标。 在这种情况下，营销策略的 CTA 会专注于激励市场中的试用行为。 

如果是消费者熟悉的产品或类别，并且已在市场中设置购买功能，则可选择跳过试用步骤，将受众引导至市场中的“立即购买”链接。

等产品/服务发展到更成熟的阶段，活动的侧重点可能变为向客户群体进行追加销售，增加市场中的购买量。 策略应侧重于鼓励客户在市场中“立即购买”。 KPI 可能变为通过市场产生的收入。

不管产品/服务的成熟度以及组织的目标如何，若要最大程度地提高活动有效性，下面这些因素都很关键：

*   关注于该目标
*   根据该目标制定一组综合的营销策略

作为面向 Azure 市场中新发布者提供的支持的一部分，Microsoft 会提供一些免费的市场 GTM 权益。 应谨慎思考如何在活动策略中利用这些优势。 将你的市场活动目标和期望的受众行为告诉我们的营销团队。 Microsoft 可以自定义这些结果，让其服务于你的计划。

智能合作伙伴营销是 Microsoft 合作伙伴网络的一项计划。

#### <a name="invest-in-merchandising"></a>投入促销活动  
发布期间应已想好要创建的产品/服务类型，并能够在 Azure 市场中为产品/服务选择一个类别。 请确保为解决方案选择的类别是准确的，这样它才能准确出现在潜在客户面前。  

在 Azure 市场中，如果选择试用或事务处理商品类型，就有资格成为特色应用。 
*   有关特色产品/服务如何适应面市权益上下文的详细信息，请访问“市场的 GTM 权益”页：[gtm-benefits](./gtm-benefits.md)。  

特色产品/服务是从提供最佳客户体验和利用高质量合作伙伴营销的产品/服务中选出的。 以下条件定义了特色产品/服务。  
*   商品信息清晰。  
*   技术可靠。  
*   客户为平台带来的用量增长。  
*   你在创作高质量营销材料方面的参与度。  

若要最大程度地增加产品/服务被推选为特色应用的机会，应尽力确保市场产品/服务的成功。 可以考虑使用以下方法，确保客户获得极佳的产品/服务体验。  
*   确保营销项目已上传且显示正确。  
*   加入 Microsoft 合作伙伴网络并参与合作伙伴生态系统。  
    *   有关 MPN 的详细信息，请访问“Microsoft 合作伙伴网络”页：[partner.microsoft.com/membership](https://partner.microsoft.com/membership)。
*   通过创建高质量的需求激励活动，在 Azure 市场中激发高质量的访问流量。  
*   确保所有产品/服务已在 Azure 市场中提供。  
*   通过刺激需求，推动 Azure 市场产品/服务的消费，同时提供高品质的客户服务，并及时更新产品/服务。  

## <a name="analytics-and-reporting"></a>分析和报告  
在云合作伙伴门户的“见解”部分，可查看产品/服务的性能概述。 此部分包含以下报告。  
*   订单的摘要快照  
*   使用情况  
*   部署  
*   “见解”主页上的客户趋势  
*   详细订单、用途和客户数据  
*   以每月摘要或六个月趋势视图形式显示的订单和用途  
*   按多个条件划分的使用情况和订单。  

详细报告会显示客户信息，例如公司名称、地理位置甚至邮政编码。 应使用此信息对客户进行比较并对销售人员的信息进行补充。 Microsoft 提供下述有关客户的特定属性。  
*   Reseller   
*   名字  
*   姓氏  
*   Email  
*   公司名称  
*   事务日期  
*   订阅名称  
*   Azure 订阅 ID（仅适用于 PAYG 客户）  
*   云实例名称  
*   订单计数  
*   客户所在国家和地区  
*   客户所在城市  
*   客户通信区域性  
*   客户邮政编码  

对于这些报表中的信息，最佳做法是将其与内部数据进行比较， 并使用这些数据确定营销活动的优先级。  

云合作伙伴门户见解中的“分析”部分提供了一个功能丰富的基于 Power BI 的仪表板。 可以使用此仪表板查看产品/服务详细信息页上的流量。 当前正在推出此仪表板的新功能。  

云合作伙伴门户中还有 Microsoft 活动的相关文档。 可以使用 Microsoft 营销活动文档来设置营销活动，以及在云合作伙伴门户上跟踪营销活动。
