# <a name="saas-application-publishing-prerequisites"></a>SaaS 应用程序发布先决条件

本文介绍在 Azure 市场上发布 SaaS 应用程序（应用）产品/服务的先决条件。  如果尚未这样做，请查看[SaaS 应用程序产品/服务发布指南](../../marketplace-saas-applications-technical-publishing-guide.md)。 


## <a name="publishing-prerequisites"></a>发布先决条件

若要发布新的 SaaS 应用产品/服务，必须满足以下先决条件：
- 具有对[云合作伙伴门户](https://cloudpartner.azure.com/)的访问权限。  云合作伙伴门户提供了基于角色的访问权限，允许多个个人进行协作用于发布产品/服务。 
- 其中一个具有“所有者”角色的个人需同意遵守[使用条款](https://azure.microsoft.com/support/legal/website-terms-of-use/)、[Microsoft 隐私声明](https://www.microsoft.com/privacystatement/default.aspx)以及 [Microsoft Azure 认证计划协议](https://azure.microsoft.com/support/legal/marketplace/certified-program-agreement/)，然后才能以发布者帐户的名义发布产品/服务。
- 所有非英语内容必须附带英语语言版本。 此内容包括 Storefront 文本、文档、屏幕截图、使用条款和隐私策略。 可以提供指向非英语应用程序的有用链接 URL。 
- 准备好元数据以备使用。 下面的列表（非详尽）显示了此元数据的示例：
  - 标题
  - 说明（采用 HTML 格式）
  - 一个徽标图像（采用 PNG 格式），采用以下固定图像大小：40 x 40 像素、90 x 90 像素、115 x 115 像素和 255 x 115 像素。
  - 一个*Terms of Use*并*隐私策略*语句
  - 应用程序文档
  - 支持联系人


## <a name="next-steps"></a>后续步骤

已满足这些先决条件之后，你可以[创建 SaaS 产品/服务](./cpp-create-offer.md)。
