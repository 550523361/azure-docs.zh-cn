# <a name="microsoft-business-applications-independent-software-vendor-isv-connect-program-onboarding-guide"></a>微软业务应用程序独立软件供应商 （ISV） 连接程序载入指南

[业务应用程序 ISV 连接计划](https://partner.microsoft.com/solutions/business-applications/isv-overview)旨在加速独立软件供应商 （ISV） 的成长和整体成功，通过*Dynamics 365 客户参与度和 PowerApps（"* 动态 CE 产品"）或*Dynamics 365 财务和运营*（"动态运营产品"）构建现代、基于云的业务线 （LOB） 解决方案。 下面是如何成为它的一部分。

> [!IMPORTANT]
> 现有的ISV合作伙伴与*Dynamics 365客户参与*，*动态365财务和运营*，和*PowerApps*应用程序应完成以下所有入职步骤之前 **，2019年10月15日**，以避免从[微软AppSource](https://appsource.microsoft.com)中删除其现有的报价。

为了注册并利用业务应用程序 ISV 连接计划的所有[技术、营销和销售支持优势](https://partner.microsoft.com/solutions/business-applications/isv-overview)，我们要求现有的 ISV 合作伙伴（使用*Dynamics CE*和*Ops*提供）完成以下步骤：

- [设置您的 Microsoft 合作伙伴中心帐户](#set-up-your-microsoft-partner-center-account)。
- [接受业务应用程序附录](#accept-the-business-applications-isv-program-addendum)。
- [设置帐单信息](#set-up-your-billing-information)。
- [重新认证您的申请](#recertify-your-application)。

> [!IMPORTANT]
> 在发布对现有产品/服务所做的更改之前，您需要完成上述所有步骤。

## <a name="set-up-your-microsoft-partner-center-account"></a>设置您的 Microsoft 合作伙伴中心帐户

[合作伙伴中心](https://partner.microsoft.com)是管理 ISV 连接计划的帐户设置的地方。

> [!NOTE]
> 此步骤需要给定帐户的云合作伙伴门户中的 *"所有者"* 角色和合作伙伴中心*的所有者*或*经理*角色。

如果您已经发布了 Dynamics CE 或动态操作产品/服务，则最有可能您的云合作伙伴门户 （CPP） 帐户已在合作伙伴中心上迁移和激活。 如果没有，则可以从 CPP 发布者配置文件手动激活合作伙伴中心帐户。

1. 在[云合作伙伴门户](https://cloudpartner.azure.com/)中登录到**发布者配置文件**页面。
2. 如果您看到显示您的帐户设置已迁移到合作伙伴中心的横幅，请继续下一节。 否则，选择提示您激活合作伙伴中心帐户的横幅。

有关详细信息，请参阅[从云合作伙伴门户到合作伙伴中心的帐户迁移](https://docs.microsoft.com/azure/marketplace/partner-center-portal/account-migration-from-cpp-to-pc)。

## <a name="accept-the-business-applications-isv-program-addendum"></a>接受业务应用程序 ISV 计划附录

参加此计划需要您审阅并接受*业务应用程序附录*。

> [!NOTE]
> 此步骤需要合作伙伴中心*的所有者*或*经理*角色，以便您的帐户签署法律协议。 如果您在上一步中[激活合作伙伴中心帐户](#set-up-your-microsoft-partner-center-account)完成了云合作伙伴门户帐户迁移，则您的权限也已迁移到合作伙伴中心。

1. 转到[合作伙伴中心仪表板](https://partner.microsoft.com/dashboard/account/agreements)，然后以提示身份登录合作伙伴中心。

2. 在标题**商业市场**， 找到**微软商业应用程序ISV增编**.

3. 选择版本链接并查看协议。

4. 选择 **"接受协议**"以同意并接受协议。

## <a name="set-up-your-billing-information"></a>设置帐单信息

接下来，为业务应用程序 ISV 连接程序设置帐单信息。

> [!NOTE]
> 此步骤需要合作伙伴中心*的所有者*或*经理*角色才能更新帐单信息。

1. 登录到[合作伙伴中心](https://partner.microsoft.com/dashboard)。

2. 选择 *"设置*（齿轮）"图标并选择 **"开发人员设置**"。

3. 在 *"帐户"* 下，选择 **"帐单配置文件**"。

4. 查看从法人那里填充的主要联系人和帐单信息，并进行任何必要的更新。

5. 供应税 ID/增值税 ID 设置（如果适用）。

6. 选择 **"保存"** 以保存更改。

您可以随时更新帐单配置文件，但以后无法更改*公司名称*和帐单地址*国家/地区*。

## <a name="recertify-your-application"></a>重新认证您的申请

完成上述步骤后，现有的动态 CE 和动态操作产品/服务将在合作伙伴中心进行编辑。 您还可以通过合作伙伴中心创建新优惠。

在业务应用程序 ISV Connect 计划中注册的最后一步是重新认证现有应用程序。

1. [按照以下步骤](https://partner.microsoft.com/solutions/business-applications/isv-publish)将应用程序重新认证给 AppSource。
2. [创建并发布新产品/服务](https://docs.microsoft.com/azure/marketplace/partner-center-portal/create-new-customer-engagement-offer)。 此优惠将是一个全新的报价，不会取代现有的报价。
