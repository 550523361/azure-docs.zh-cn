# <a name="power-bi-apps-prerequisites"></a>Power BI 应用先决条件

本文列出了在将 Power BI 应用套餐发布到 Microsoft AppSource 市场之前必须满足的技术和业务要求。

## <a name="technical-requirements"></a>技术要求

需要为此套餐类型提供的主要技术资产为 [Power BI 应用程序](https://go.microsoft.com/fwlink/?linkid=2028636)。 有关详细信息，请参阅[创建 Power BI 技术资产](./cpp-create-technical-assets.md)。

在 Power BI 中创建并测试服务应用程序后，应保存 Power BI 生成的应用程序安装 URL。 需要使用它来[创建新的 Power BI 应用套餐](./cpp-create-offer.md)。

## <a name="business-requirements"></a>业务要求

业务需求包括在规程、合同和法律方面的义务：

-   你必须是已注册的云市场发布者。 如果未尚未注册，请执行[成为云市场发布者](../../become-publisher.md)一文中的步骤。

    > [!NOTE] 
    > 应当使用相同的 Microsoft Developer Center 注册帐户登录到[云合作伙伴门户](https://cloudpartner.azure.com)。 对于 AppSource 套餐，只应具有一个 Microsoft 帐户。 它不应该特定于个别服务或套餐。
 
-   你需要负责以商业上合理的方式向客户提供技术支持。
-   提供的内容必须满足在 AppSource 中将套餐列入名单的标准。 有关详细信息，请参阅[有需要在 AppSource 上列出的应用？方法如下](https://appsource.microsoft.com/blogs/have-an-app-to-list-on-appsource-here-s-how)。
-   必须遵循 [Microsoft 隐私声明](https://privacy.microsoft.com/privacystatement)。


## <a name="next-steps"></a>后续步骤

满足所有要求后，即可在云合作伙伴门户中[创建 Power BI 套餐](./cpp-create-offer.md)。
