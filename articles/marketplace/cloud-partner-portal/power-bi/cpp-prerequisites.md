# <a name="power-bi-app-offer-prerequisites"></a>Power BI 应用提供先决条件

本文提供了在将 Power BI 应用服务发布到 AppSource marketplace 之前需要满足的技术和业务要求。  如果尚未执行此操作，请查看[Office 365、Dynamics 365、PowerApps 和 Power BI 优惠发布指南](../../appsource-offer-publishing-guide.md)。


## <a name="technical-requirements"></a>技术要求

此产品/服务类型需要的主要技术资产是[Power BI 应用](https://go.microsoft.com/fwlink/?linkid=2028636)。 有关详细信息，请参阅[创建 Power BI 技术资产](./cpp-create-technical-assets.md)。

在 Power BI 中创建并测试了服务应用程序后，应保存 Power BI 生成的应用程序安装 URL。 你需要它来[创建 Power BI 应用服务](./cpp-create-offer.md)。

## <a name="business-requirements"></a>业务要求

业务要求包括过程、合同和法律义务。  必须：

- 是注册的云 marketplace 发布者。 如果未注册，请按照[成为云 Marketplace 发布服务器](https://docs.microsoft.com/azure/marketplace/become-publisher)中的步骤进行操作。

    > [!NOTE] 
    > 使用相同的 Microsoft 开发人员中心注册帐户登录到[云合作伙伴门户](https://cloudpartner.azure.com)。 对于 AppSource 套餐，只应具有一个 Microsoft 帐户。 帐户不应特定于单个服务或产品/服务。
 
- 以商业上合理的方式向客户提供技术支持。
- 提供满足在 AppSource 上列出的产品/服务的条件的内容。 有关详细信息，请参阅[AppSource 上的应用程序列表？下面是操作方法](https://appsource.microsoft.com/blogs/have-an-app-to-list-on-appsource-here-s-how)。
- 遵循[Microsoft 隐私声明](https://privacy.microsoft.com/privacystatement)。


## <a name="next-steps"></a>后续步骤

满足所有要求后，便可以在云合作伙伴门户中[创建 Power BI 应用服务](./cpp-create-offer.md)。
