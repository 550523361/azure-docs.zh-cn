# <a name="power-bi-app-offer-prerequisites"></a>Power BI 应用套餐先决条件

>[!Important]
>从 2020 年 3 月 30 日开始，我们将开始将 Power BI 应用优惠的管理转移到合作伙伴中心。 迁移后，您将在合作伙伴中心创建和管理您的优惠。 按照[Power BI 应用创建概述](https://aka.ms/AzurePBIAppCreationOverview)中的说明进行操作，以管理迁移的优惠。

本文提供了在将 Power BI 应用产品/服务发布到 AppSource 市场之前需要满足的技术性和业务需求。  如果尚未这样做，请查看[Office 365、动态 365、PowerApps 和 Power BI 产品发布指南](../../appsource-offer-publishing-guide.md)。


## <a name="technical-requirements"></a>技术要求

此产品/服务类型需要的主要技术资产是[Power BI 应用程序](https://go.microsoft.com/fwlink/?linkid=2028636)。 有关详细信息，请参阅[创建 Power BI 技术资产](./cpp-create-technical-assets.md)。

在 Power BI 中创建和测试服务应用程序后，应保存 Power BI 生成的应用程序安装 URL。 您需要它来[创建 Power BI 应用产品/](./cpp-create-offer.md)

## <a name="business-requirements"></a>业务要求

业务需求包括程序、合同和法律义务。  必须：

- 成为注册的云市场发布者。 如果您尚未注册，请按照["成为云市场发布者](https://docs.microsoft.com/azure/marketplace/become-publisher)"中的步骤操作。

    > [!NOTE] 
    > 使用相同的 Microsoft 开发人员中心注册帐户登录到[云合作伙伴门户](https://cloudpartner.azure.com)。 对于 AppSource 套餐，只应具有一个 Microsoft 帐户。 帐户不得特定于单个服务或产品。
 
- 以商业上合理的方式向客户提供技术支持。
- 提供符合在 AppSource 上列出的产品标准的内容。 有关详细信息，请参阅在[AppSource 上列出应用？下面是如何](https://appsource.microsoft.com/blogs/have-an-app-to-list-on-appsource-here-s-how)。
- 遵守[微软隐私声明](https://privacy.microsoft.com/privacystatement)。


## <a name="next-steps"></a>后续步骤

满足所有要求后，即可在云合作伙伴门户中[创建 Power BI 应用产品/](./cpp-create-offer.md)
