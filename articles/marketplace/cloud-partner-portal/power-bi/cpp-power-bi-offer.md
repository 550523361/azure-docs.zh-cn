# <a name="power-bi-application-offer"></a>Power BI 应用程序产品/服务

|                  |                                |
|------------------|--------------------------------|
| 本部分介绍如何将 Power BI 应用程序发布到 Microsoft [AppSource 市场](https://appsource.microsoft.com/)。 Power BI 应用程序将 Power BI 内容打包，并使用数据流将报表和仪表板连接到常用数据存储中的数据。 然后，可以通过 AppSource 将应用部署到其他租户。 | ![Power BI 图标](./media/powerbi-icon.png) |


本节划分为以下主要部分：

-   [先决条件](./cpp-prerequisites.md) - 创建或发布 Power BI 应用产品/服务所要满足的技术和业务要求
-   [创建 Power BI 应用产品/服务](./cpp-create-offer.md) - 使用[云合作伙伴门户](https://cloudpartner.azure.com)创建新的 Power BI 应用所需的步骤
-   [发布 Power BI 应用产品/服务](./cpp-publish-offer.md) - 如何提交新产品/服务以发布到 AppSource，以及如何更新现有产品/服务。


## <a name="publishing-steps"></a>发布步骤

下图演示了发布 Power BI 应用产品/服务中的概要步骤。

![Power BI 应用发布步骤](media/publishing-steps.png)

Power BI 应用发布过程

1. 在 Power BI 中创建服务应用程序（这会创建包安装 URL）。  此 URL 表示产品/服务的技术资产。  有关详细信息，请参阅[什么是 Power BI](https://go.microsoft.com/fwlink/?linkid=2028636)。
2. 收集或创建业务资产，包括：
    - 营销材料
    - 法律和支持文档
3.  创建产品/服务 - 使用云合作伙伴门户配置有关产品/服务的详细信息，包括产品/服务说明、营销材料、法律文档、支持信息和资产规格。
4.  认证并发布产品/服务 - 提交后会启动发布过程。
    - AppSource 载入团队测试、验证并认证应用程序。 
    - 通过认证后，应在其测试环境中查看，并将其发布。

完成所有这些步骤后，应用将在 AppSource 中“上线”。

## <a name="next-steps"></a>后续步骤

在开始这些步骤之前，必须满足用于将 Power BI 应用发布到 AppSource 的[要求](./cpp-prerequisites.md)。
