# <a name="how-to-migrate-to-the-new-cloud-partner-portal"></a>如何迁移到新的云合作伙伴门户

迁移到云合作伙伴门户是一个重要步骤。 为了响应你们对发布流程的反馈，我们在门户方面进行了大量投资。 这些投资集中用于提高易用性以及改进发布体验。 

-   支持组织 ID 登录
-   实时字段验证，以便捕获错误
-   简化发布工作流

我们已简化了迁移你的产品/服务的流程，因此，如果你准备获得改进的发布体验，请查看以下迁移步骤。

通过复查以下迁移阶段确保迁移成功。



## <a name="decide-youre-ready-to-migrate"></a>确定你已做好迁移准备

在将产品/服务迁移到云合作伙伴门户之前，请确保完全了解要发生的事情。

**是否要迁移所有产品/服务？**

今天，我们将迁移所有虚拟机产品/服务。 每个虚拟机产品/服务（不管是草稿、暂存的还是已列出的）都将迁移。

其他类型的产品/服务（例如解决方案模板和体验版）目前将继续在旧门户中运行。

**我的实时产品/服务是否仍可继续使用？**

是的，此迁移不会影响实时产品/服务。 我们也已经预填充了发布者帐户信息，因此可以在完成迁移后，立即在新门户中更新产品/服务。 暂存的产品/服务将成为草稿，需要在新门户中发布它们。

**迁移时，是否可以更改产品/服务的定价？**

否，所有现存产品/服务的定价仍需保持不变，因为我们的此次迁移不支持价格变更。

**如果我的产品/服务处于过渡/发布状态，该怎么办？**

如果任何产品/服务目前正在旧门户中发布或暂存，则迁移会失败。 在选取迁移时间时，必须确保产品/服务均处于已发布/已暂存状态。

**基于角色的新访问的工作原理是什么？**

在新门户中，管理员会成为发布配置文件的所有者，所有共同管理员会成为参与者。 如果要编辑发布配置文件中的任何内容，应转到新门户。

管理用户时，如果需要添加某人，让其在旧门户中处理某个产品/服务，则仍请在旧门户中将其添加为共同管理员。 如果需要添加某人，让其在新门户中处理某个产品/服务，则请在新门户中将其添加为参与者。

**进行迁移时，迁移内容是否为最终版本？**

在迁移你的产品/服务时，我们会将旧门户中针对这些产品/服务的所有操作标记为只读的，并且会阻止在旧门户上执行创建操作。 可以继续在之前停下的地方编辑并处理所有迁移的产品/服务，但只能在新门户中进行。

## <a name="submit-a-request-to-migrate"></a>提交迁移请求

认为已做好计划迁移的准备以后，即可在云合作伙伴门户的“了解”页上使用发布者名称和 ID 以提交方式进行迁移。

![发布过程流程图](./media/cloud-partner-portal-how-to-migrate-to-the-new-cloud-partner-portal/migrate.png)

## <a name="confirm-a-date-to-migrate"></a>确认要进行迁移的日期

我们会在每周星期二上午 10 点到下午 1 点 (PST) 迁移产品/服务，并且会提前 24 小时要求发布帐户的所有者确认迁移时间。 如果需要在其他时间进行迁移，以免自己的工作流受影响，我们会与你协商一个最佳时间。

会收到电子邮件通知，告知你迁移已完成，可以继续在新门户中操作了。

## <a name="settle-in-and-enjoy"></a>迁入之后，请尽情享受其中的乐趣！

迁移之后，即可继续在新门户中操作。 从现在开始，产品/服务在旧门户中将处于只读状态，所有未来的创建和编辑操作应在新门户中完成！

**迁移出了问题，该怎么办？**
 
如果迁移失败，我们会还原所做的更改，重新计划一个迁移时间。 迁移完成之后，如果有什么问题，请随时通过反馈工具联系我们。
