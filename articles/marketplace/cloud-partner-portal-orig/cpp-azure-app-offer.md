# <a name="azure-application-offer"></a>Azure 应用程序产品/服务

<table> <tr> <td>本部分介绍了如何将新的托管 Azure 应用程序产品/服务发布到 <a href="https://azuremarketplace.microsoft.com/marketplace/">Microsoft Azure 市场</a>。  每个托管应用程序都包含一个 Azure 资源管理器模板，该模板定义应用程序使用的所有技术资产，该应用程序通常包括一个或多个虚拟机和基于 Azure 或基于 Web 的其他支持服务。 </td> <td><img src="./media/cloud-partner-portal-publish-managed-app/azureapp-icon1.png"  alt="Azure application icon" /></td> </tr> </table>

## <a name="publishing-overview"></a>发布概述

以下视频（[针对 Azure 市场构建解决方案模板和托管应用程序](https://channel9.msdn.com/Events/Build/2018/BRK3603)）首先概述了如何创建 Azure 资源管理器模板来定义 Azure 应用程序解决方案，然后概述了之后如何将应用产品/服务发布到 Azure 市场。

> [!VIDEO https://channel9.msdn.com/Events/Build/2018/BRK3603/player]


## <a name="publishing-process"></a>发布过程

以下文章将指导你完成创建和发布托管 Azure 应用程序的过程：

1. [在 Microsoft 开发中心内注册一个帐户](./cloud-partner-portal-dev-center-accounts-registration.md)以便获批在 Azure 市场中进行发布。
2. [将新的 Azure 应用程序产品/服务](./cloud-partner-portal-managed-app-publish.md)发布到 Azure 市场。
3. [推出你的产品/服务](./cloud-partner-portal-managed-app-offer-go-live.md)！ 


## <a name="next-steps"></a>后续步骤

接下来，你必须验证此产品/服务类型所需的要求并开始发布过程，如[创建托管应用程序](./cloud-partner-portal-managed-app-publish.md)中详述。
