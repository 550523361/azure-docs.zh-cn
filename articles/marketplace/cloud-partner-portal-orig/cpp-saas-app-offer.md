# <a name="azure-saas-application-offer"></a>Azure SaaS 应用程序产品/服务

<table> <tr> <td>本部分介绍了如何将软件即服务 (SaaS) 应用程序发布到 Microsoft <a href="https://azuremarketplace.microsoft.com/marketplace/">Azure 市场</a>或 <a href="https://appsource.microsoft.com">Microsoft AppSource</a>。 当需要将解决方案部署在你自己的 Azure 订阅中并且客户将通过你设计和管理的界面来测试应用程序时，请使用 SaaS 应用程序。 <a href="https://azure.microsoft.com/services/active-directory">Azure Active Directory</a> (AAD) 用于利用你的现有试用环境。 </td> <td><img src="./media/cloud-partner-portal-publish-web-app/saasapp-icon1.png"  alt="Azure SaaS application icon" /></td> </tr> </table>


## <a name="publishing-benefits"></a>发布带来的好处

在 Microsoft 市场中列出你的应用程序的一些好处包括：

- 在 Office 365 和 Dynamics 365 中接触 1 亿 Azure Active Directory 用户。
- 扩展你的销售团队：接触世界各地的业务用户并获得能够吸引最终用户的销售渠道，帮助开发潜在顾客，并与各个行业的新客户进行聊天。

- 获取可操作的见解：我们将针对你的应用在 AppSource 上的表现，什么运行良好，以及如何进一步改进你的销售过程来共享见解。


## <a name="publishing-process"></a>发布过程

若要在 Microsoft Azure 市场或 Microsoft AppSource 中创建并发布你的应用，本部分可引导你完成以下步骤：

1. 确认满足先决条件，包括合作伙伴注册：[开发中心注册](https://dev.windows.com/en-us/registration?accountProgram=Azure)或 [Microsoft 合作伙伴网络注册](https://partner.microsoft.com/)。
2.  定义你的市场产品/服务和计划。
3.  提供市场营销内容和定价。
4.  在[云合作伙伴门户](https://cloudpartner.azure.com)中测试并认证你的产品/服务。
5.  推出你的产品/服务。
