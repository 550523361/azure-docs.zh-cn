<a name="seller-insights-definitions"></a>Seller Insights 定义
=======================

下表提供了 Seller Insights 中使用的许多术语的定义。

|  **术语**                 |  **定义**                                                                                                                              |
|  --------------------------------------------    |  ---------------------------------------------------------------------------------------------------------------------------------           |
| **Azure 许可证类型**                               | 购买 Azure 的客户使用的许可协议类型。  也称为“渠道”。                                                  |
| **Azure 许可证类型：云解决方案提供商**      | 最终客户通过其云解决方案提供商（充当经销商）购买 Azure 和市场套餐。                 |
| **Azure 许可证类型：企业**                   | 最终客户通过直接与 Microsoft 签署的企业协议购买 Azure 和市场套餐。                  |
| **Azure 许可证类型：通过经销商签署的企业协议**  | 最终客户通过经销商采购 Azure 和您的应用商店产品/服务，经销商为与 Microsoft 的企业协议提供便利。     |
| **Azure 许可证类型：即用即付**                | 最终客户通过直接与 Microsoft 签署的"随用付费"协议采购 Azure 和应用商店产品/服务。                |
| **费用金额 (CC)**                              | 向客户收取的金额，以*客户币种*（CC） 计费。                                                                 |
| **费用金额 (PC)**                               | 向客户收取的金额，以*支付货币*（PC） 表示。                                                                      |
| **收费日期**                                      | 计算客户费用的日期（通常在使用期之后）。                                             |
| **云实例名称**                              | 部署 VM 的 Microsoft 云。                                   |
| **云实例名称：Azure 全局**                | 公共全球微软云。                           |
| **云实例名称：Azure 政府**        | 针对政府特定的微软云为以下政府之一：中国，德国或美利坚合众国。                           |
| **收集状态**  | 对于特定费用，计费和收款生命周期的最新状态。  例如，状态可以是正在进行的收集、收集或退款。                  |
| **核心尺寸**  | 产品/服务运行的虚拟机提供的虚拟内核数。               |
| **客户所在城市**  | 客户提供的城市名称。 这可能与客户 Azure 订阅中的城市不同。  |
| **客户通信语言**        | 客户的首选通信语言。                                  |
| **客户公司名称**                  | 客户提供的公司名称。 这可能与客户 Azure 订阅中的城市不同。                                  |
| **客户所在国家/地区**                       | 客户提供的国家/地区名称。  这可能与客户 Azure 订阅中的国家/地区不同。                               |
| **客户货币 (CC)**                 | 客户首选的定价和计费货币。                            |
| **客户电子邮件**                         | 最终客户提供的电子邮件地址。  这可能与客户 Azure 订阅中的电子邮件地址不同。                   |
| **客户名字**                              | 客户提供的第一个名字。  这可能与客户的 Azure 订阅中提供的名称不同。                    |
| **客户姓氏**                               | 客户提供的姓氏。  这可能与客户的 Azure 订阅中提供的名称不同。                    |
| **客户 ID**                               | 分配给客户的唯一标识符。  客户可能具有零个或多个 Azure 应用商店订阅。                    |
| **客户付款类型**                            | 客户使用的付款方式类型。  例如，这可能是卡、发票或电子支票。                             |
| **客户邮政编码**                             | 客户提供的邮政编码。  这可能与客户的 Azure 订阅中提供的邮政编码不同。                                                                 |
| **客户所在的州/省**                                   | 客户提供的状态（地址）。  这可能与客户 Azure 订阅中提供的姓氏不同。                                                                 |
| **购买日期**                                    | 客户购买您发布的任何产品/服务的第一个日期。                 |
| **丢失日期**                                        | 客户取消以前购买的所有优惠中的最后一个日期。                                                         |
| **估计的延伸费用 (CC)**                   | 给定 SKU（以客户货币表示）的使用单位数量的估计扩展费用。 由于舍入或截断错误，此值可能不精确。                            |
| **估计的延伸费用 (PC)**                   | 根据日期使用情况的外汇兑换计算（以发布者的货币）计算给定 SKU 的使用单位数量的估计扩展费用。 由于舍入或截断错误，此值可能不精确。                  |
| **估计的付款金额 (PC)**                            | 根据计算使用日期（以发布者的货币）的外汇兑换计算，对给定 SKU 的使用单位数量的估计付款。 由于舍入或截断错误，此值可能不精确。                        |
| **估计价格 (PC)**                             | 根据日期使用情况上的外汇兑换计算给定 SKU 的使用量单位的估计价格（以发布者的货币表示）。 由于舍入或截断错误，此值可能不精确。                                             |
| **是新客户**                                  | 该值将让您知道，如果是新客户首次消费或首次购买一个或多个应用商店产品/服务（或不）。  如果值为"获取日期"的同一日历月，则该值将为"是"。  如果客户在报告日历月之前购买了您的任何优惠，则该值将为"否"。                                                       |
| **是预览 SKU**                                  | 该值将告知您是否将 SKU 标记为"预览"。 如果 SKU 已相应地标记，并且只有您授权的 Azure 订阅可以部署和使用此映像，则该值将为"是"。  如果 SKU 尚未标识为"预览"，则该值将为"否"。                                          |
| **促销联系人是否选择加入**                   | 该值将让您知道，如果客户主动选择从发布商进行促销联系。 目前我们不会向客户提供该选项，因此，在整个板块中都指示了“否”。 部署此功能后，我们会相应地做出更新。                                          |
| **市场许可证类型**                         | 市场套餐的计费方式。                             |
| **市场许可证类型：通过 Azure 计费**   | Microsoft 是此市场套餐的代理，并代你向客户计费。 （PAYG 信用卡或企业发票）       |
| **市场许可证类型：自带许可** | VM 需要客户提供的许可证密钥才能部署。 Microsoft 不会向使用此市场套餐的客户计费。 |
| **市场许可证类型：免费**                   | 市场套餐配置为供所有用户免费使用。 Microsoft 不会向使用此市场套餐的客户计费。    |
| **市场许可证类型：Microsoft 充当经销商**  | Microsoft 是此市场套餐的经销商。                               |
| **应用商店订阅公司名称**  | 客户在其 Azure 订阅中提供的公司名称。                           |
| **应用商店订阅 ID**            | 与客户用于购买应用商店产品/服务所用的 Azure 订阅关联的唯一标识符。  它以前是 Azure 订阅 GUID。  在某些情况下，如 SAAS 订单，不需要订阅。           |
| **Microsoft 手续费 (CC)**                               | Microsoft 收取的交易手续费，按客户的货币结算。                              |
| **优惠名称**                                | 应用商店产品的名称。                                                             |
| **优惠类型**                                | 微软应用商店产品的类型。  例如，它可以是托管应用程序、虚拟机许可证或容器。                           |
| **订单取消日期**                         | 取消市场订单的日期。                                                       |
| **订单 ID**                                  | 市场服务的客户订单的唯一标识符。  基于虚拟机使用情况的优惠不与订单相关联。                 |
| **订单购买日期**                       | 创建市场订单的日期。                                                        |
| **Order Quantity**                       | 与 Azure 订阅关联的同一 SKU 的订单数量。  对于 SAAS 订单，它表示与此订单实例关联的席位数。                                            |
| **订单状态**                              | 上次刷新数据时市场订单的状态。  例如，订单可以是活动订单或已取消。             |
| **订单状态：活动**                           | 客户已购买订单，尚未取消订单。   |
| **订单状态：已取消**                        | 客户以前购买了订单，随后取消了订单。        |
| **付款金额 (PC)**                             | 客户的付款金额，以首选的付款货币 (PC) 结算。                                       |
| **付款货币 (PC)**                           | 付款时使用的货币。                                                                |
| **付款日期**                                    | 付款请求从 Microsoft 发送到您选择的金融机构的日期。                                                                         |
| **付款状态**                                  | 上次刷新数据时的支出状态。                                   |
| **付款状态：正在进行中**               | 交易尚未做好付款的准备。 （有关详细信息，请参阅收集状态）                             |
| **付款状态：已付**                          | 交易包含在以往付款计算中。 正值表示已付，负值是与逾期未付总金额相差的净额。 |
| **付款状态：即将付款**                   | 交易已做好付款的准备，将包含在下一次应付款计算中。                      |
| **价格 (CC)**                                   | 给定 SKU 的一个使用单位的价格（采用客户的货币）。                                                                  |
| **提供商电子邮件**                                 | 参与 Microsoft 与最终客户之间关系的提供商的电子邮件地址。 如果客户是通过经销商的企业，则经销商将是经销商。  如果涉及云解决方案提供商 （CSP），则为 CSP。                                                                 |
| 提供者名称****                                 | Microsoft 和最终客户之间的关系中涉及的提供商的名称。 如果客户是通过经销商的企业，则经销商将是经销商。  如果涉及云解决方案提供商 （CSP），则为 CSP。                                                                 |
| **资源 URI**                                  | 部署工件的路径。                                            |
| **Sku**                                           | 发布期间定义的 SKU 名称。 一个套餐可能有许多 SKU，但一个 SKU 只能与一个套餐相关联。              |
| **SKU 计费类型**                                 | SKU 的计费方式。                                                                                                             |
| **SKU 计费类型：自带许可证**         | VM 或服务需要客户提供的许可证密钥才能部署。 Microsoft 不会向客户收取使用这些应用商店产品/服务的费用。   |
| **SKU 计费类型：微软作为经销商**          | Microsoft 是此 SKU 的经销商。   |
| **SKU 计费类型：免费**                           | SKU 配置为供所有用户免费使用。 Microsoft 不会向使用此 SKU 的客户计费。                           |
| **SKU 计费类型：付费**                           | Microsoft 是此 SKU 的代理，并代你向客户计费。 （PAYG 信用卡或企业发票）                   |
| **SKU 计费类型：试用**                          | 客户处于套餐的试用期，如果不取消或删除订阅，套餐将转换为付费版。                                         |
| **税额 (CC)**                                  | 在客户帐单中收取的税额，以客户货币 (CC) 结算。                                                               |
| **交易日期**                                 | 付款报告中的交易记录日期。                                                                           |
| **事务类型**                               | 正在报告的交易类型。  例如，这可能是费用、退款或支出调整。                                                                                        |
| **试用结束日期**                                 | 此订单的试用期将要结束或已结束的日期。                                                                           |
| **使用日期**                                       | 客户使用日期。                                                                                                             |
| **使用数量**                                       | 报告的 SKU 客户使用情况。 对于 VM 映像，使用情况记录将表述报告周期内该 VM 大小和 SKU 的使用情况。                                               |
| **使用情况参考**                                       | 针对与付款报表中某个条目关联的给定 SKU，客户使用的一个或多个日期的标识符。                                               |
| **使用类型**                                       | 要度量的使用情况的说明。 （“规范化用量”或“原始用量”）                                                              |
| **使用类型：标准化使用**                | 根据使用时涉及的 VM 核心数规范化的使用小时数：[VM 核心数] x [原始使用小时数]。 指定为“SHAREDCORE”的 VM 使用 1/6（或 0.1666）作为 [VM 核心数] 的乘数。                                        |
| **使用类型：原始用法**                        | 市场套餐的使用小时数。  它以"日历"小时形式呈现。  它是虚拟机运行的时间数。                           |
| **使用情况单位**                                   | 所述使用情况的度量单位。 VM 始终以小时作为度量单位来度量。                               |
| **VM 大小**                                       | 表示与 Azure 套餐相符的虚拟机硬件大小。 示例包括 `Basic_A0`、`Standard_A11`、`Standard_D12` 和 `Standard_G4`。   |
