# <a name="publishing-integrated-solutions"></a>发布-集成解决方案

集成解决方案结合了多个 ISV 点解决方案技术，以及系统集成商（SI）的部署和服务，并将其作为[咨询服务产品](https://docs.microsoft.com/azure/marketplace/consulting-services)/服务在 marketplace 中列出。

这些解决方案的构建旨在满足特定的客户需求或方案，这一点首先了解行业挑战或痛点。 两个或更多同类最佳的技术将通过 SI 进行集成，为客户创建统一且可重复的部署体验，以便快速启动和运行。 系统集成商还为客户提供单一联系点，提供相关的自定义和咨询，并支持解决方案。 其结果是一个集成的解决方案，其中包括技术和服务。

有关[集成解决方案计划](https://aka.ms/AA5qos4)的概述，请参阅此视频。

## <a name="key-benefits-to-integrated-solutions-in-the-marketplace"></a>Marketplace 中集成解决方案的主要优点

* 客户可以找到可通过 marketplace 解决端到端业务挑战的打包解决方案，并通过在前一项集成的技术来加快价值实现。 客户不必单独使用以分段方式部署的多个 ISV 应用。
* Isv 将其 IP 与其他解决方案和服务一起打包在一起，从而增强了专家 SIs 提供的技术价值。
* SIs 创建预定义的预集成解决方案，该解决方案具有一流的技术，可在更多客户帐户中进行缩放、土地和扩展。
* Microsoft 可以利用这些预先集成的行业协调解决方案，更轻松地与客户进行共同销售，并预先排序定价和条款。 

## <a name="eligibility-criteria"></a>资格准则

* 所有参与方伙伴都必须具有 MPN ID
* 参与的 SI 应共同销售就绪
* 参与的 Isv 应为 IP 共同销售 
* 至少一个参与合作伙伴必须具有以下某个领域的金牌资格：

    * 应用程序开发
    * 应用程序集成
    * 应用程序生命周期管理
    * 云平台
    * 数据分析
    * 数据中心
    * 数据平台
    * DevOps

* 解决方案必须包含至少两个合作伙伴组织，将 IP 和服务组合在一起，以获得完整的客户解决方案和体验
* 必须在 Azure、Dynamics 365 和/或 Power Platform 上构建合作伙伴 IP
* 集成解决方案的 IP 只需与 ISV + basic SI 服务的 IP 区分开来
* 所有合作伙伴的目标是创建一个可重复使用的产品/服务，并最终通过 Azure Marketplace 和/或 AppSource 进行交易

## <a name="nomination-process"></a>提名进程

希望参与此计划的合作伙伴应确定他们要使用的其他合作伙伴并完成[集成解决方案提名](https://aka.ms/AA5qicu)表单。 然后，集成解决方案计划团队将查看提名并在10个工作日内跟进，并提供批准和后续步骤。 

## <a name="developing-your-integrated-solution"></a>开发集成解决方案 

批准提名后，请按照以下步骤操作以开发集成解决方案： 

1. 以研讨会格式完成重要的业务和技术讨论，以定义解决方案（范围、目标客户、价值主张、定价、许可、条款和条件、支持方法、市场营销活动、销售流程和活动）以及集成方法（集成点、解决方案体系结构、身份验证、用户界面、监视、部署机制、Api 和统一的数据存储库）。 

    以下资源可用于指导合作伙伴完成此过程：

    1. [业务决策研讨会讨论指南](https://aka.ms/AA5qicx)
    1. [技术决策研讨会讨论指南](https://aka.ms/AA5qid1)
    1. [快速入门视频-业务和技术研讨会](https://aka.ms/AA5qos9)

1. 确定跨合作伙伴项目经理来推动由业务和技术讨论会产生的项目计划和时间线。

1. 完成解决方案的技术集成。

1. 完成解决方案定价并决定在 Azure Marketplace 或 AppSource 上的单一价格点。

1. 完成 Azure Marketplace 或 AppSource 列表的市场宣传资料，包括：

    1. 组合解决方案名称
    2. 列出集成解决方案的描述（[产品/服务列表最佳实践](https://docs.microsoft.com/azure/marketplace/gtm-offer-listing-best-practices)）
    1. 客户1寻呼（[模板](https://aka.ms/AA5s08a)）
    1. 客户演示文稿（[模板](https://aka.ms/AA5s7ql)）
    1. 组合解决方案的徽标 

## <a name="publishing-your-integrated-solution"></a>发布集成解决方案 

技术集成完成并且市场宣传资料准备就绪后，请参阅[Azure Marketplace 咨询服务和 AppSource](https://docs.microsoft.com/azure/marketplace/consulting-services)的发布者指南。 使用此资源，确定是在 Azure Marketplace 还是 AppSource 中发布，具体取决于你想要访问的受众、准备发布项目并完成发布过程。

尽管有五种服务类型可用于咨询服务，但集成解决方案必须是概念证明或完全实现。

## <a name="going-to-market-with-your-integrated-solution"></a>通过集成解决方案进入市场 

一旦你的解决方案位于 Azure Marketplace 或 AppSource 中，你将需要通过市场营销活动推动认知和需求的生成。 查看[项目计划和时间线](https://aka.ms/AA5qiuc)中列出的活动，包括合作伙伴可以在自己的团队中推动的活动，以及另一个活动。 然后，为已发布到 marketplace 中的合作伙伴提供[市场推广权益](https://docs.microsoft.com/azure/marketplace/gtm-your-marketplace-benefits#list-trial-and-consulting-benefits)。 所有集成解决方案产品/服务都有资格获得试用级别的权益。
