# <a name="determine-your-publishing-option"></a>确定发布选项

您为产品/服务选择的发布选项与资格要求和商业市场 GTM 权益直接相关。 更重要的是，选择发布选项和产品/服务类型定义了用户如何与您的商业市场产品/服务进行交互。

要配置产品/服务，您需要了解以下关键的商业市场概念：发布选项、产品类型和配置，以及控制产品/服务在商业市场网店中呈现的方式和位置的操作要求。

![微软商业市场店面和出版选项](./media/marketplace-publishers-guide/storefronts_options_table.png)

在本文中，将学习以下内容：

- 如何确定适合解决方案的店面
- 每个店面中提供哪些发布选项和广告语
- 适用于每个发布选项的具体套餐类型

## <a name="selecting-a-storefront-publishing-option-and-offer-type-for-your-solution"></a>选择适用于解决方案的店面、发布选项和套餐类型

在选择发布选项之前，了解商业市场解决方案、应用和服务的网店资格要求非常重要：

**Microsoft AppSource**应用程序是可内置 Azure 或为"动态 365"、Office 365、Power BI 或 Power Apps 构建的业务线解决方案。 AppSource 咨询服务是专业的服务套餐，可以帮助客户实现 Dynamics 365 和 Power BI 使用入门或提高其使用速度。

**Azure 应用商店**应用程序是为 Azure 构建或构建的技术"构建基块"解决方案，面向 IT 或开发人员受众。 Azure 市场咨询服务是专业的服务套餐，可以帮助客户实现 Azure 使用入门或提高其使用速度。

## <a name="understand-storefront-selection"></a>了解店面选择

每个店面都满足独特的客户需求。 发布优惠的店面将由优惠类型、提供交易功能、类别和/或行业选择确定，以特定受众为目标。 

>[!Note]
>"交叉列表"（仅适用于 SaaS Apps）：当列表或试用产品/服务符合技术和业务用户受众的标准时，您的产品/服务将在两个网店中列出。 详细了解下面的发布选项。

## <a name="choose-a-publishing-option"></a>选择发布选项

可用的发布选项提供了差异化的客户参与度，同时让您获得潜在客户共享[和商业市场优势](https://docs.microsoft.com/azure/marketplace/gtm-your-marketplace-benefits)。  请注意与发布选项相对应的广告语：

| **发布选项**    | **说明**  |
| :------------------- | :-------------------|
| **列表** | 简单列出您的应用程序或服务，使商业市场用户能够通过 **"联系我**行动"请求您与客户联系。 |
| **试验** | 使用商业市场增强可发现性并自动调配解决方案的试用体验，使潜在用户能够在购买前的有限时间内免费使用您的 SaaS、IaaS 或 Microsoft 应用内体验。 用于试用发布选项的操作呼叫是 **"免费试用**版"或 **"测试驱动器**"。 |
| **BYOL** | 使用商业市场增强解决方案的可发现性和自动化配置，并单独完成金融交易。 BYOL 产品类型是本地迁移到云迁移的理想选择。 行动号召就是**立即得到它**。
| **办理** | 交易报价通过商业市场出售。 微软负责计费和收款。 行动号召就是**立即得到它**。|

>[!Note]
>使用“事务处理”发布选项时，在选择套餐类型和创建套餐之前，了解定价、计费、发票和付款注意事项非常重要。 请参阅[市场计费和商业注意事项文章以了解详细信息](./marketplace-commercial-transaction-capabilities-and-considerations.md)。
