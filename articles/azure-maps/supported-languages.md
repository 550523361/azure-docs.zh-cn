## <a name="azure-maps-supported-views"></a>Azure 地图支持的视图

> [!Note]
> 2019 年 8 月 1 日，Azure 地图在以下国家/地区发布：
>  * 阿根廷
>  * 印度
>  * 摩洛哥
>  * 巴基斯坦
>
> 2019 年 8 月 1 日之后，"**视图"** 参数将定义上述新区域/国家/地区的返回的地图内容。 Azure 地图**视图**参数（也称为"用户区域参数"）是一个两个字母的 ISO-3166 国家/地区代码，它将显示该国家/地区的正确地图，指定通过 Azure 地图服务返回哪组具有地缘政治争议的内容，包括地图上显示的边界和标签。 

请确保根据服务正在使用的 REST API 和 SDK 的要求设置**视图**参数。
>  
>
>  **休息 API：**
>  
>  确保已根据需要设置"视图"参数。 视图参数指定通过 Azure 地图服务返回的地缘政治争议内容集。 
>
>  受影响的 Azure 映射 REST 服务：
>    
>    * 获取地图磁贴
>    * 获取地图图像 
>    * 获取搜索模糊
>    * 获取搜索 POI
>    * 获取搜索 POI 类别
>    * 获取搜索附近
>    * 获取搜索地址
>    * 获取结构化搜索地址
>    * 获取反向搜索地址
>    * 获取搜索地址反向交叉街
>    * 在几何结构中发布搜索
>    * 发布搜索地址批量预览
>    * 发布搜索地址反向批处理预览
>    * 沿路线发布搜索
>    * 后搜索模糊批处理预览
>
>    
>  **Sdk：**
>
>  确保已根据需要设置 **"视图"** 参数，并且具有最新版本的 Web SDK 和 Android SDK。 受影响的 SDK：
>
>    * Azure Maps Web SDK
>    * Azure Maps Android SDK

默认情况下，View 参数设置为**Unified**，即使您尚未在请求中定义它。 确定用户的位置。 然后，为该位置正确设置 **"视图"** 参数。 或者，您可以设置"View_Auto"，它将基于请求的 IP 地址返回地图数据。  Azure 地图中的 **"视图"** 参数必须符合适用法律，包括有关地图、图像和其他数据以及通过 Azure 地图有权访问的第三方内容的地图国家/地区的地图的法律。
