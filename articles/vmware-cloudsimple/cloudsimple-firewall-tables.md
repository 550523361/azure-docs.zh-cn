# <a name="firewall-tables-overview"></a>防火墙表概述

防火墙表列出了用于筛选往来私有云资源的网络流量的规则。 您可以将防火墙表应用于 VLAN/子网。 规则控制源网络或 IP 地址与目标网络或 IP 地址之间的网络流量。

## <a name="firewall-rules"></a>防火墙规则

下表描述了防火墙规则中的参数。

| properties | 详细信息 |
| ---------| --------|
| **名称** | 唯一标识防火墙规则及其用途的名称。 |
| **优先** | 数字介于 100 和 4096 之间，其中 100 是最高优先级。 规则按优先顺序处理。 当流量遇到规则匹配时，规则处理将停止。 因此，不会处理优先级较低的规则，这些规则的属性与优先级较高的规则相同。  小心避免冲突的规则。 |
| **状态跟踪** | 跟踪可以是无状态的（私有云、互联网或 VPN）或有状态（公共 IP）。  |
| **协议** | 选项包括"任意"、TCP 或 UDP。 如果需要 ICMP，请使用"任意"。 |
| **方向** | 该规则是应用到入站还是出站流量。 |
| **操作** | 允许或拒绝规则中定义的流量类型。 |
| **源** | IP 地址、无类域间路由 （CIDR） 块 （例如 10.0.0.0/24）或 Any。  指定范围、服务标记或应用程序安全组使您能够创建较少的安全规则。 |
| **源端口** | 网络流量源自的端口。  您可以指定单个端口或端口范围，例如 443 或 8000-8080。 指定范围可以减少创建的安全规则数。 |
| **目的地** | IP 地址、无类域间路由 （CIDR） 块 （例如 10.0.0.0/24）或 Any。  指定范围、服务标记或应用程序安全组使您能够创建较少的安全规则。  |
| **目标端口** | 网络流量流向的端口。  您可以指定单个端口或端口范围，例如 443 或 8000-8080。 指定范围可以减少创建的安全规则数。|

### <a name="stateless"></a>无状态

无状态规则仅查看单个数据包，并根据规则对其进行筛选。  
对于相反方向的流量，可能需要其他规则。  对以下点之间的流量使用无状态规则：

* 私有云子网
* 本地子网和私有云子网
* 来自私有云的互联网流量

### <a name="stateful"></a>有状态

 有状态的规则知道通过它的连接。 将为现有连接创建流记录。 是允许还是拒绝通信取决于流记录的连接状态。  对于公共 IP 地址使用此规则类型来筛选来自 Internet 的流量。

### <a name="default-rules"></a>默认规则

每个防火墙表上都创建了以下默认规则。

|优先度|“属性”|状态跟踪|方向|流量类型|协议|源|Source Port|目标|Destination Port|操作|
|--------|----|--------------|---------|------------|--------|------|-----------|-----------|----------------|------|
|65000|允许全互联网|有状态|出站|公共 IP 或互联网流量|All|Any|Any|Any|Any|Allow|
|65001|拒绝从互联网所有|有状态|入站|公共 IP 或互联网流量|All|Any|Any|Any|Any|拒绝|
|65002|允许所有到内联网|无状态|出站|私有云内部或 VPN 流量|All|Any|Any|Any|Any|Allow|
|65003|允许所有来自内联网|无状态|入站|私有云内部或 VPN 流量|All|Any|Any|Any|Any|Allow|
