# <a name="interpreting-your-scorecard"></a>解释记分卡

记分卡选项卡包含测试的聚合和分析结果。 每个测试都有自己的记分卡。 记分卡提供快速、有意义的衡量结果摘要，为您的网络需求提供数据驱动的结果。 互联网分析器负责分析，使您能够专注于决策。

记分卡选项卡可在 Internet 分析器资源菜单中找到。 


## <a name="filters"></a>筛选器

* ***测试：*** 选择要查看结果的测试 - 每个测试都有自己的记分卡。 测试数据将在有足够的数据完成分析后出现 - 在大多数情况下，应在 24 小时内显示。 
* ***时间段&结束日期：*** 每天生成三个记分卡 - 每个记分卡反映不同的聚合周期 - 前 24 小时（天）、前七天（周）和前 30 天（月）。 使用"结束日期"筛选器选择要查看的时间段的最后一天。 
* ***国家/地区：*** 对于拥有最终用户的每个国家/地区，将生成记分卡。 全局筛选器包含所有最终用户。

## <a name="measurement-count"></a>测量计数

测量次数会影响分析的置信度。 计数越高，结果越准确。 至少，测试应针对每个端点每天至少 100 次测量。 如果测量计数过低，请将 JavaScript 客户端配置为在应用程序中更频繁地执行。 端点 A 和 B 的测量计数应该非常相似，尽管预期差异很小，也可以。 如果存在较大差异，则不应信任结果。

## <a name="percentiles"></a>百评分

延迟（以毫秒为单位）是衡量 Internet 上源和目标之间速度的常用指标。 延迟数据不是正常分布的（即不遵循"贝尔曲线"），因为在使用算术平均值等统计信息时，存在大延迟值的"长尾"，会扭曲结果。 作为替代方法，百分位数提供了一种"无分发"方法来分析数据。 例如，中位数（或第 50 个百分位）汇总了分布的中间部分 - 一半值位于分布之上，一半值位于分布下方。 第 75 个百分位值表示它大于分布中所有值的 75%。 互联网分析仪是指速记中的百分位数，即 P50、P75 和 P95。

互联网分析器百分位数是_示例指标_。 这与真正的_人口指标_形成鲜明对比。 例如，南加州大学和 Microsoft 的学生之间的每日真实人口中位数延迟是当天所有请求的中值延迟值。 实际上，衡量所有请求的值是不切实际的，因此我们假设一个相当大的样本代表真实总体。

出于分析目的，P50（中位数）可用作延迟分布的预期值。 较高的百分位数（如 P95）可用于识别在最坏情况下延迟程度。 如果您有兴趣了解客户延迟，P50 是需要关注的正确指标。 如果您关心的是了解表现最差的客户的性能，那么 P95 应该是重点。 P75 是这两者之间的平衡。


## <a name="deltas"></a>三角洲

增量是端点 A 和 B 的指标值差异。 计算增量以显示 B 优于 A 的好处。 正值表示 B 的性能优于 A，而负值表示 B 的性能较差。 增量可以是绝对的（例如 10 毫秒）或相对 （5%）。

## <a name="confidence-interval"></a>置信区间 

置信区间 （CI） 是一个值范围，其概率包含总体指标（如中位数、P75 或平均值）。 我们遵循使用 95% CI 的常见统计约定。

对于 Internet 分析器，窄置信区间是好的，因为它表明示例指标可能非常接近实际填充指标。 宽置信区间意味着我们的样本指标反映真实总体指标的确定性降低。 改进 CI 的最佳方法是增加测量计数。

## <a name="time-series"></a>时序 

时间序列显示指标如何随时间而变化。 在互联网上，有许多时间因素会影响性能，如交通高峰期、工作日-周末人口差异和节假日。
