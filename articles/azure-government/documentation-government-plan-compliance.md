# <a name="azure-government-compliance"></a>Azure Government 合规性
## <a name="blueprint-customer-responsibilities-matrix--streamlining-ato-processes"></a>Blueprint Customer Responsibilities Matrix：简化 ATO 流程
Azure Blueprint 项目允许客户访问文档以帮助其代理或特定于系统的操作授权 (ATO) 进程。 本文深入地介绍了 Azure Blueprint Customer Responsibilities Matrix (CRM) 和安全工作人员如何利用该文档。

Azure Blueprint CRM 专为实用性设计。 它是电子表格格式，其中仅定义了控制要求：在 Azure 中生成的客户必须实施安全控制。

> [!NOTE]
> 在 Azure Blueprint 的上下文中，“客户”指代在 Azure IaaS/PaaS 环境中直接生成的组织。 Azure 客户可包括代表政府生成的第三方 ISV 或在 Azure 上直接生成的政府部门。
> 
> 

CRM 提供了明确的 <a href="http://nvlpubs.nist.gov/nistpubs/SpecialPublications/NIST.SP.800-53r4.pdf"> NIST SP 800-53 修订 4 </a> 控件，其中包含客户责任。 其中包括客户承担全部责任的控制和客户与 Azure 共同承担责任的控件。 将控制分解为控件子要求的内容以尽可能地明确。

例如，AC-1 要求记录系统请求 ATO 的访问控制策略和流程。 对于此控制，Azure 具有关于用于管理 Azure 基础结构和平台的访问控制机制的内部策略和流程。 客户还必须创建其自身的访问控制策略和流程，以在 Azure 中创建的特定系统中使用。 CRM 文档控制 AC-1a 部分（该部分要求策略和流程中包括特定内容）和 AC-1b 部分（该部分要求客户至少每年查看和更新这些文档一次）。 

若要访问 CRM 及 Azure Blueprint 系统安全计划模板，请发送电子邮件至 AzureBlueprint@microsoft.com。

我们重视有关 CRM 的任何反馈，因为用户的意见对迭代该文档以尽可能高效地简化客户 ATO 流程至关重要。
