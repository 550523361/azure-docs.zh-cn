# <a name="how-to-migrate-to-the-new-cloud-partner-portal"></a>如何迁移到新的云合作伙伴门户

迁移到云合作伙伴门户是一大进步。 我们已了解到你对发布过程的反馈并作了一些大的投入，以便加强使用便捷性并改进整个发布体验。 
*    支持组织 ID 登录
*    实时字段验证，以便捕获错误
*    简化发布工作流

我们已尽力改进产品/服务迁移的便捷性，因此如果你希望获得改进的发布体验，不妨看看你需要做些什么。

## <a name="what-do-you-need-to-do"></a>需要执行的操作
请执行以下步骤，确保迁移成功：

**迁移的各个阶段**
|          |          |
|----------|----------|
|1.  确定你是否已准备就绪 | 在将产品/服务迁移到云合作伙伴门户之前，请确保你完全了解要发生的事情。<br/><br/>**是否要迁移所有产品/服务？**<br/>今天，我们将迁移所有虚拟机产品/服务。 这意味着，你的每个虚拟机产品/服务（不管是草稿、暂存的还是已列出的）都将迁移。 <br/>其他类型的产品/服务（解决方案模板、测试性驱动器等）目前将继续在旧门户中运行。<br/><br/>**我的实时产品/服务是否仍可继续使用？**<br/>是的，此迁移不会影响实时产品/服务。 我们也已经预填充了你的发布者帐户信息，因此你可以在完成迁移以后，立即在新门户中更新产品/服务。 暂存的产品/服务将成为草稿，你需要将其发布到新门户中。<br/><br/>**迁移时，是否可以更改产品/服务的定价？**<br/>否。所有实时产品/服务的定价仍需保持不变，因为我们的此次迁移不支持价格变更。<br/><br/>**如果我的产品/服务处于过渡/发布状态，该怎么办？**<br/>如果你的任何产品/服务目前正在旧门户中发布或暂存，则迁移会失败。 在选取迁移时间时，必须确保产品/服务均处于已发布/已暂存状态。<br/><br/>**基于角色的新访问的工作原理是什么？**<br/>在新门户中，你的管理员会成为发布配置文件的所有者，所有共同管理员会成为参与者。 若要编辑发布配置文件中的内容，应转到新门户。 <br/>管理用户时，如果需要添加某人，让其在旧门户中处理某个产品/服务，则仍请在旧门户中将其添加为共同管理员。 如果需要添加某人，让其在新门户中处理某个产品/服务，则请在新门户中将其添加为参与者。<br/><br/>**进行迁移时，迁移内容是否为最终版本？**<br/>迁移产品/服务时，我们会将你在旧门户中针对这些产品/服务执行的所有操作设置为只读，并且会阻止通过旧门户执行的创建操作。 你可以继续在之前停下的地方编辑并处理所有迁移的产品/服务，但只能在新门户中进行。|
|2.  以提交方式进行迁移 | 你认为已做好计划迁移的准备以后，即可在云合作伙伴门户的“了解”页上使用发布者名称和 ID 以提交方式进行迁移。![发布流程图](./media/cloud-partner-portal-how-to-migrate-to-the-new-cloud-partner-portal/migrate.png)|
|3.  确认日期 | 我们会在每周星期二上午 10 点到下午 1 点 (PST) 迁移产品/服务，并且会提前 24 小时要求发布帐户的所有者确认迁移时间。 如果你需要在其他时间进行迁移，以免自己的工作流受影响，我们会与你协商一个最佳时间。<br/>你会收到电子邮件通知，告知你迁移已完成，你可以继续在新门户中操作了。|
|4.  迁入之后，请尽情享受其中的乐趣！ | 迁移之后，即可继续在新门户中操作。 从现在开始，你的产品/服务在旧门户中将处于只读状态，所有未来的创建和编辑操作应在新门户中完成！ <br/><br/>**迁移出了问题，该怎么办？** 如果迁移失败，我们会还原所做的更改，为你重新计划一个迁移时间。 迁移完成之后，如果你有什么问题，请随时通过反馈工具联系我们。|
