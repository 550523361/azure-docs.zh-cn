# <a name="azure-data-share-preview-concepts"></a>Azure 数据共享预览概念 

Azure 数据共享 Preview 引入了一些新术语与相关的数据共享。 本文介绍了一些常用的可能会看到在整个服务使用的术语。 

## <a name="data-provider"></a>数据提供程序

数据访问接口是其使用者与共享数据的组织。 通常数据提供程序可以是所有者或数据的管理者。 数据提供程序想要共享的各种类型的数据。 数据提供程序可能想要共享的数据的一些示例包括原始数据，例如销售或时序数据的点。 数据提供程序还可能想要共享已包含分析和见解的预处理、 特选数据。 

## <a name="data-consumer"></a>数据使用者 

数据使用者是接收数据的数据提供程序的组织。 数据使用者可能想要加入具有其自己的数据中获得见解的共享的数据。 在某些情况下，数据使用者可能会收到已处理的数据。 

## <a name="data-share"></a>数据共享

数据共享是作为单个实体共享的数据集的组。 数据集可以为多个 Azure 数据源支持的 Azure 数据共享。 目前，Azure 数据共享支持 Azure Blob 存储和 Azure Data Lake Store。 

## <a name="share-subscription"></a>共享订阅 

数据使用者接受来自数据访问接口的数据共享邀请时，会创建共享的订阅。 数据提供程序可以通过导航到查看活动的共享订阅**发送共享**在 Azure 数据共享帐户，并选择**共享订阅**。

数据使用者可以检查它们是否具有活动的共享订阅通过导航到**接收共享**并查看其接收到的共享状态。 

## <a name="snapshot"></a>快照

接受数据共享邀请时，可以通过数据使用者创建快照。 当他们接受邀请后时，可以触发这些事件与之共享的数据的完整的快照。 快照是数据使用者生成快照的时间点上的数据的副本。 

有两种类型的快照的完整和增量。 完整的快照包含的数据共享内的所有数据。 增量快照包含已更新/添加由于上一个快照已触发的所有数据。 

## <a name="snapshot-settings-in-azure-data-share"></a>在 Azure 数据共享的快照设置
 
数据提供程序可以启用的数据共享快照设置。 此设置允许数据使用者发生时接收增量更新。 如果数据提供程序想要接收已共享的数据更新其数据使用者应启用此设置。 

如果数据访问接口启用此设置，可以选择重复周期间隔。 重复间隔可以是每小时或每日。 

数据使用者可以选择要选择加入到此快照计划，以便接收增量更新，其中包括他们首次生成新的快照后已更改的任何数据。 

## <a name="invitation"></a>邀请

数据访问接口可以邀请到其数据共享的多个收件人。 用户可以通过将收件人添加到数据共享这样做。 创建的数据共享后，还可以添加的邀请。 

数据提供程序可以删除邀请后已发送。 请注意，是否数据提供程序删除邀请已被接受后，数据使用者可以仍具有活动的共享订阅。 如果数据提供程序删除邀请，并且尚未接受，数据使用者不能接受它。 

## <a name="recipient"></a>Recipient

收件人是接收到的数据共享邀请的人。 通常情况下，数据访问接口将将收件人添加到他们创建的数据共享。 一旦邀请的接收方接受邀请后，它们会成为数据使用者。
