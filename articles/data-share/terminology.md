# <a name="azure-data-share-concepts"></a>Azure 数据共享概念 

Azure 数据共享引入了一些与数据共享相关的新术语。 本文介绍了一些常用的术语，您可能会看到在整个服务中使用。 

## <a name="data-provider"></a>数据提供程序

数据提供程序是与其使用者共享数据的组织。 通常，数据提供程序可以是数据的所有者或管理员。 数据提供程序希望共享各种类型的数据。 数据提供商可能想要共享的一些数据示例包括原始数据，如销售点或时序数据。 数据提供商可能还希望共享已包含分析和见解的预处理、精心策划的数据。 

## <a name="data-consumer"></a>数据使用者 

数据使用者是从数据提供程序接收数据的组织。 数据使用者可能希望将共享数据与自己的数据联接，以获得见解。 在某些情况下，数据使用者可能正在接收已处理的数据。 

## <a name="data-share"></a>Data Share

数据共享是作为单个实体共享的数据集组。 数据集可以来自 Azure 数据共享支持的众多 Azure 数据源。 目前，Azure 数据共享支持 Azure Blob 存储和 Azure 数据湖存储。 

## <a name="share-subscription"></a>共享订阅 

当数据使用者接受数据提供程序的数据共享邀请时，将创建共享订阅。 数据提供程序可以通过导航到其 Azure 数据共享帐户中的**已发送共享**以及选择 **"共享订阅**"来查看活动共享订阅。

数据使用者可以通过导航到 **"已接收共享"** 并查看其已接收共享的状态来检查他们是否具有活动共享订阅。 

## <a name="snapshot"></a>快照

数据使用者在接受数据共享邀请时，可以创建快照。 当他们接受邀请时，他们可以触发与他们共享的数据的完整快照。 快照是数据使用者生成快照的时间点的数据副本。 

有两种类型的快照 - 完整快照和增量快照。 完整快照包含数据共享中的所有数据。 增量快照包含自上次触发快照以来已更新/添加的所有数据。 

## <a name="snapshot-settings-in-azure-data-share"></a>Azure 数据共享中的快照设置
 
数据提供程序可以为数据共享启用快照设置。 此设置使数据使用者能够在增量更新发生时接收增量更新。 如果数据提供程序希望其数据使用者接收已共享数据的更新，则应启用此设置。 

如果数据提供程序启用此设置，则可以选择定期间隔。 定期间隔可以是每小时或每天。 

数据使用者可以选择加入此快照计划以接收增量更新，其中包括自首次生成新快照以来已更改的任何数据。 

## <a name="invitation"></a>邀请

数据提供程序可以邀请多个收件人访问其数据共享。 他们可以通过将收件人添加到数据共享来执行此操作。 创建数据共享后，也可以添加邀请。 

数据提供程序可以在发送邀请后将其删除（如果尚未被接受）。 如果数据提供程序删除了邀请，但尚未被接受，则数据使用者将无法接受它。 

邀请每天最多可以怨恨五次。 

## <a name="recipient"></a>Recipient

收件人是收到数据共享邀请的人。 通常，数据提供程序会将收件人添加到他们创建的数据共享中。 邀请的接收者接受邀请后，他们将成为数据使用者。
