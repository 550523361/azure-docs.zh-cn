# <a name="advanced-multistage-attack-detection-in-azure-sentinel"></a>Azure 哨兵中的高级多阶段攻击检测


> [!IMPORTANT]
> Azure 哨兵中的某些融合功能当前处于公共预览版中。
> 这些功能在没有服务级别协议的情况下提供，不建议用于生产工作负载。 某些功能可能不受支持或者受限。 有关详细信息，请参阅 [Microsoft Azure 预览版补充使用条款](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)。



通过使用基于机器学习的 Fusion 技术，Azure Sentinel 可以通过将异常行为和在杀伤链的各个阶段观察到的可疑活动相结合，自动检测多阶段攻击。 然后，Azure Sentinel 会生成否则很难捕获的事件。 这些事件包含两个或多个警报或活动。 根据设计，这些事件体积低、保真度高、严重性高。

此检测可针对您的环境进行定制，不仅可降低误报率，还可以检测信息有限或缺失的攻击。

## <a name="configuration-for-advanced-multistage-attack-detection"></a>高级多阶段攻击检测的配置

默认情况下，在 Azure Sentinel 中启用此检测。 要检查状态，或禁用状态，可能是因为您正在使用替代解决方案基于多个警报创建事件，请使用以下说明：

1. 如果尚未这样做，请登录到[Azure 门户](https://portal.azure.com)。

2. 导航到**Azure 哨兵** > **配置** > **分析**

3. 选择**活动规则**，并在**NAME**列中查找**高级多阶段攻击检测**。 检查**状态**列以确认此检测是启用还是禁用。

4. 要更改状态，请选择此条目，并在**高级多阶段攻击检测**边栏选项卡上选择 **"编辑**"。

5. 在 **"规则创建"向导**边栏选项卡上，将自动为您选择状态更改，因此选择 **"下一步：审阅**"，然后**保存**。 

规则模板不适用于高级多阶段攻击检测。

> [!NOTE]
> Azure Sentinel 目前使用 30 天的历史数据来训练机器学习系统。 当数据通过机器学习管道时，始终使用 Microsoft 的密钥进行加密。 但是，如果在 Azure Sentinel 工作区中启用了 CMK，则不会使用[客户托管密钥 （CMK）](customer-managed-keys.md)加密训练数据。 要选择退出融合，请导航到**Azure 哨兵** \> **配置分析** \> **\>活动\>规则高级多阶段攻击检测**，并在 **"状态**"列中选择 **"禁用"。**

## <a name="fusion-using-palo-alto-networks-and-microsoft-defender-atp"></a>融合使用帕洛阿尔托网络和微软后卫ATP

这些方案结合了安全分析师使用的两个基本日志：来自帕洛阿尔托网络的防火墙日志和来自 Microsoft Defender ATP 的端点检测日志。 在下面列出的所有方案中，在涉及外部 IP 地址的端点中检测到可疑活动，然后，从外部 IP 地址回防火墙的异常流量随后检测到。 在 Palo Alto 日志中，Azure Sentinel 侧重于[威胁日志](https://docs.paloaltonetworks.com/pan-os/8-1/pan-os-admin/monitoring/view-and-manage-logs/log-types-and-severity-levels/threat-logs)，当允许威胁时，流量被视为可疑（可疑数据、文件、洪水、数据包、扫描、间谍软件、URL、病毒、漏洞、野火病毒、野火）。

### <a name="sign-in-activity-from-user-with-leaked-credentials-followed-by-anomalous-office-365-activity"></a>来自凭据泄露的用户的登录活动，后跟异常的 Office 365 活动

有七个可能的 Azure Sentinel 事件将用户的登录活动与 Azure AD 标识保护泄漏的凭据警报和 Microsoft 云应用安全生成的异常 Office 365 警报相结合：

- **用户登录事件，其凭据泄露导致 Office 365 邮箱泄露**
    
    此警报表示，在用户的收件箱中设置了按\<*帐户名称*>使用泄露凭据的登录事件，随后在用户的收件箱中设置了可疑的收件箱转发规则。 
    
    这可能表示帐户已泄露，并且邮箱正用于从您的组织中泄露信息。 \<*用户帐户名称*>创建或更新了收件箱转发规则，该规则将所有传入的电子邮件转发到外部\<*地址电子邮件地址*>。 

- **来自具有泄露凭据导致可疑云应用管理活动的用户的登录事件**
    
    此警报表示\<*按帐户名称*登录的事件>使用了泄露的凭据。
    
    接下来，帐户\<*帐户名称*>在单个会话\<中执行>管理活动*的数量*。

- **用户登录事件，凭据泄露导致大量文件删除**
    
    此警报表示\<*按帐户名称*登录的事件>使用了泄露的凭据。
    
    接下来，帐户\<*帐户名称*>单个会话中\<已删除>个唯一文件*的数量*。

- **用户登录事件，凭据泄露导致大量文件下载**
    
    此警报表示\<*按帐户名称*登录的事件>使用了泄露的凭据。
    
    接下来，帐户\<*帐户名称*>在单个会话中\<下载*过*>个唯一文件的数量。

- **来自具有泄露凭据导致 Office 365 模拟的用户的登录事件**
    
    此警报表示\<*按帐户名称*登录的事件>使用了泄露的凭据。 
    
    接下来，帐户\<*帐户名称*>在单个会话中\<模拟超过>不同帐户*数*。

- **用户登录事件，凭据泄露导致大量文件共享**
    
    此警报表示\<*按帐户名称*登录的事件>使用了泄露的凭据。
